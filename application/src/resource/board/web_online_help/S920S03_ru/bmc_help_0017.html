Настройки тревог

Описание функций

На странице Настройки тревог можно настроить:

  • Syslog-уведомления: отправка информации о событиях, регистрируемых в журнале, на сторонний сервер в виде syslog-сообщений.
  • Trap-уведомления: отправка информации по аварийным сигналам, событиям и прерываниям на сторонний сервер в виде trap-сообщений.
  • Уведомления по электронной почте: отправка сообщений по электронной почте на определенные адреса через SMTP-сервер при появлении аварийного сигнала.

    Trap-сообщения — это сообщения, которые отправляет BMC на сторонний сервер без явного запроса. Trap-сообщения используются для передачи информации об аварийных событиях и аварийных сигналах критического, серьезного и незначительного уровней.

Описание параметров

Включение TCP или UDP поставит под угрозу безопасность системы. Выполняйте это действие с осторожностью.

Табл. 1 Syslog-уведомления

Параметр

Описание

Настройки Syslog-уведомлений

Функция Syslog

Отправка уведомлений в виде syslog-сообщений.

По умолчанию данная функция отключена.

Уровень серьезности тревог

Уровни серьезности аварийных сигналов, отправляемых в syslog-сообщениях.

Значения:
  • [NULL]: нет отправленных сообщений об аварийных сигналах и событиях.
  • Срочный: только аварийные сигналы критического уровня.
  • Значительный: аварийные сигналы серьезного и критического уровня.
  • Незначительный: аварийные сигналы незначительного, серьезного и критического уровней.
  • нормальный: события и незначительные, серьезные и критические аварийные сигналы.

Значение по умолчанию — нормальный.

Формат сообщений Syslog

Формат syslog-сообщений.

  • Настроить: к syslog-сообщениям относятся приоритет сообщений, имя продукта, идентификатор syslog-хоста, местоположение устройства и тип журнала.
  • RFC3164: syslog-сообщения соответствуют RFC3164 и содержат приоритет сообщений, метку времени, имя хоста, имя процесса и тип журнала.

Значение по умолчанию — Настроить.

ПРИМЕЧАНИЕ:

Если выбран формат RFC3164, невозможно настроить Источник Syslog-уведомлений и Имя хоста используется по умолчанию.

Источник Syslog-уведомлений

Идентификатор сервера отправки syslog-сообщений.

Значения:

  • Серийный номер платы
  • Идентификатор продукта
  • Имя хоста

Значение по умолчанию — Серийный номер платы.

Протокол передачи

Протокол, используемый для передачи syslog-сообщений между BMC и syslog-сервером.

Значения:

  • TLS: протокол с установлением соединения, который обеспечивает конфиденциальность и целостность передаваемых данных.
  • TCP: протокол с установлением соединения, который передает данные только после установления надежного соединения между отправителем и получателем.
  • UDP: протокол без установления соединения, с помощью которого осуществляется обмен данными между отправителем и получателем.

Значение по умолчанию — TLS.

Режим аутентификации

Режим аутентификации syslog-сертификатов. Данный параметр требует настройки только в том случае, если в качестве Протокола передачи используется TLS.

Значения:

  • Односторонняя: аутентификация только сертификата syslog-сервера.
  • Двусторонняя: аутентификация сертификатов syslog-сервера и клиента.

Значение по умолчанию — Односторонняя.

Корневой сертификат сервера

Сертификат, который используется для проверки сообщений, отправленных с syslog-сервера до установления соединения.

ПРИМЕЧАНИЕ:
  • Поддерживается импорт файла корневого сертификата сервера в формате .crt, .cer или .pem. Размер не может превышать 100 КБ.
  • В целях обеспечения безопасности необходимо периодически обновлять сертификат.

Сертификат клиента

При установке соединения для передачи данных BMC отправляет пакеты, содержащие информацию о локальном сертификате, на Syslog-сервер, который требует информации сертификата для аутентификации клиента Syslog (BMC).

ПРИМЕЧАНИЕ:
  • Поддерживается импорт локальных файлов сертификата в формате .pfx или .p12. Размер не может превышать 1 МБ.
  • В целях обеспечения безопасности необходимо периодически обновлять сертификат.

Данные о сертификате сервера

Информация о загруженном корневом сертификате сервера. Данные о сертификате включают следующую информацию:

  • Организация, выпустившая корневой сертификат.
  • Пользователь, которому был выдан корневой сертификат.
  • Срок действия корневого сертификата.
  • Серийный номер корневого сертификата.
  • Список отозванных сертификатов (CRL).
  • Срок действия CRL.
ПРИМЕЧАНИЕ:
  • CRL указывает статус отзыва сертификатов.
    • Настроено: файл CRL был выгружен; при установке соединения TLS будет выполнена проверка сертификата на предмет отзыва.
    • Не настроено: файл CRL не выгружен.
  • Файл CRL в кодировке Base64 имеет формат *.crl, а размер файла не может превышать 100 КБ.
  • Если срок действия CRL истекает, соответствующая функция аутентификации будет недоступна.

Чтобы задать CRL, нажмите Выгрузка, выберите файл CRL из клиента и введите пароль текущего пользователя. Нажмите далить, чтобы удалить CRL.

ПРИМЕЧАНИЕ:

Удаление CRL может привести к рискам безопасности при использовании сертификатов с истекшим сроком действия.

Сервер Syslog и формат сообщений

Адрес сервера

Адрес syslog-сервера.

Значения: Адрес IPv4 или IPv6, либо имя домена

ПРИМЕЧАНИЕ:
  • Имя домена необходимо указывать, если Протокол передачи задан как TLS. Кроме того, необходимо правильно задать данные DNS на странице Настройки > Настройки Сеть.

  • Кнопка Тест предназначена для проверки отправки syslog-сообщения на указанный syslog-сервер. Если указанный сервер доступен, на экране появится сообщение «Операция выполнена успешно».
    ПРИМЕЧАНИЕ:

    Если параметры в области Настройки Syslog-уведомлений были изменены, перед тестированием сначала нажмите кнопку Сохранить. В противном случае изменение не вступит в силу.

Параметр

Описание

Настройки SMTP

Функция SMTP

Отправка уведомлений по электронной почте через SMTP-сервер.

По умолчанию данная функция отключена.

Адрес сервера SMTP

IPv4-адрес, IPv6-адрес или имя домена SMTP-сервера.

Имя домена должно соответствовать следующим требованиям:
  • Cтрока длиной до 255 символов.
  • Имя домена может состоять только из цифр, букв, дефисов (-) и точек (.).
  • Имя домена не может начинаться с дефиса (-) или точки (.), либо заканчиваться дефисом (-).
  • Имя домена может состоять из не более 63 символов между любыми двумя точками.

Порт SMTP-сервера

Номер порта SMTP-сервера.

Диапазон значений: от 1 до 65535

Значение по умолчанию: 25

Протокол TLS

Включение протокола TLS для передачи данных.

Если TLS отключен, то данные передаются в виде незашифрованного текста.

Значение по умолчанию — Отключено.

ПРИМЕЧАНИЕ:
  • По умолчанию SMTP-сервер поддерживает TLS. В целях обеспечения безопасности рекомендуется включить функцию TLS.
  • После активации TLS в WebUI BMC включите TLS и настройте аутентификацию данных на SMTP-сервере. После включения TLS SMTP-сервер будет получать сообщения электронной почты только от BMC.
  • BMC не поддерживает TLS 1.0 и TLS 1.1.

Проверка сертификата сервера

Функция для включения или отключения проверки сертификата сервера при включении зашифрованной передачи TLS.

Данная функция отключена по умолчанию.

Сертификат сервера

Сертификат сервера используется для аутентификации пакетов, отправленных с сервера SMTP при установлении соединения.

ПРИМЕЧАНИЕ:
  • Формат файла сертификата сервера может быть .crt, .cer или .pem, с максимальным размером 100 КБ.
  • Периодически обновляйте сертификат во избежание рисков безопасности.

Данные о сертификате сервера

Функция отображения информации о выгруженном сертификате сервера, в том числе:

  • Издатель
  • Получатель
  • Срок действия
  • Серийный номер

Данные эл. почты

Функция анонимности

Функция, разрешающая анонимный вход в систему.

Если данная функция включена, то SMTP-сервер передает сообщения об аварийных сигналах по электронной почте без аутентификации имени пользователя и пароля.

Если функция отключена, SMTP-сервер передает сообщения об аварийных сигналах по электронной почте только при условии ввода заданного имени пользователя и пароля. Настройка имени пользователя и пароля осуществляется на SMTP-сервере.

Значение по умолчанию — Включено.

ПРИМЕЧАНИЕ:

По умолчанию SMTP-сервер не поддерживает анонимный вход в систему. В целях обеспечения безопасности не рекомендуется использовать функцию анонимного входа в систему.

Имя пользователя отправителя/Пароль отправителя

Имя пользователя и пароль, которые используются, если функция Анонимный вход отключена.

Имя пользователя и пароль должны совпадать с именем пользователя и паролем, заданными на SMTP-сервере.

Диапазон значений:
  • Имя пользователя: строка длиной от 1 до 64 символов, состоящая из букв, цифр и специальных символов. Данный параметр является обязательным.
  • Пароль: строка длиной от 1 до 50 символов.

Эл. почта отправителя

Адрес электронной почты, с которого отправляются сообщения об аварийных сигналах.

Значение: строка длиной до 255 символов в формате xx@xxx.xx.

Значение может содержать буквы, цифры и специальные символы.

Тема/В теме указаны

Тема сообщения электронной почты.

Значение: строка длиной от 0 до 255 символов, состоящая из букв, цифр и специальных символов. Значение по умолчанию — Server Alert.

Заполните поле Тема и выберите ключевые слова, которые будут содержаться в теме электронного сообщения.

Например, при выборе опций Имя хоста и Серийный номер платы в теме электронного сообщения будет содержаться имя хоста и серийный номер платы.

Уровень серьезности тревог

Уровни серьезности аварийных сигналов, отправляемых через SMTP-сервер.

Значения:
  • [NULL]: нет отправленных сообщений об аварийных сигналах и событиях.
  • Срочный: только аварийные сигналы критического уровня.
  • Значительный: аварийные сигналы серьезного и критического уровня.
  • Незначительный: аварийные сигналы незначительного, серьезного и критического уровней.
  • нормальный: события и незначительные, серьезные и критические аварийные сигналы.

Значение по умолчанию — нормальный.

Адрес электронной почты получателя сообщений. Настройка адресов осуществляется на SMTP-сервере.

Значение: строка длиной до 255 символов в формате xx@xxx.xx.

Табл. 3 Trap-уведомления

Параметр

Описание

Настройки Trap

Функция Trap

Отправка аварийной информации в trap-сообщениях.

Данная функция включена по умолчанию.

Версия Trap

Версия SNMP, используемая для отправки trap-сообщений.

Значения:

  • SNMPv1: первая официальная версия SNMP, определенная в RFC 1157.
  • SNMPv2c: версия SNMPv2, в которую добавлена архитектура управления на уровне сообществ.
  • SNMPv3: версия SNMP, в которую добавлена поддержка безопасности и удаленного конфигурирования.
ПРИМЕЧАНИЕ:
  • Рекомендуется использовать SNMPv3. SNMPv1 и SNMPv2c имеют определенные риски безопасности.
  • Алгоритмы аутентификации и шифрования SNMPv3 можно настроить в меню Пользователи и безопасность > Локальные пользователи в веб-интерфейсе.

Значение по умолчанию: SNMPv3

Пользователь SNMPv3

Имя пользователя SNMPv3. Настраивать данный параметр необходимо только в том случае, если выбрана версия SNMPv3.

По умолчанию пользователем BMC по умолчанию является пользователь SNMPv3.

Режим Trap

Режим отправки информации в trap-сообщениях.

Значения:

  • Подробные данные (рекомендуется): в качестве идентификатора trap-события используется OID узла SNMP, который однозначно соответствует определенному событию. В отличие от режимов OID и Код события данный режим предоставляет более точную информацию.
  • OID: в качестве идентификатора trap-события используется OID узла SNMP.
  • Код события: в качестве идентификатора trap-события используется код события.

Значение по умолчанию: Подробные данные (рекомендуется)

Источник Trap-сообщений

Идентификатор сервера отправки trap-сообщений.

Значения:

  • Серийный номер платы
  • Идентификатор продукта
  • Имя хоста

Значение по умолчанию — Серийный номер платы.

Имя сообщества

Имя сообщества для trap-аутентификации в случае использования SNMPv1 или SNMPv2c.

Диапазон значений зависит от того, включена ли проверка сложности пароля.

  • Если проверка сложности пароля отключена, то в качестве значения используется строка длиной от 1 до 32 символов, состоящая из букв, цифр и специальных символов (за исключением пробелов).
  • Если проверка сложности пароля включена, то должны выполняться следующие условия:
    • Длина имени сообщества должна составлять от 8 до 32 символов.
    • Имя сообщества должно содержать, как минимум, два сочетания из следующих символов: прописные буквы от A до Z, строчные буквы от a до z, цифры от 0 до 9
    • Имя сообщества должно содержать хотя бы один из следующих специальных символов: `~!@#$%^&*()-_=+\|[{}];:'",<.>/?
    • Имя сообщества не должно содержать пробелы.

Подтвердите имя сообщества

Повторный ввод имени сообщества для подтверждения.

Уровень серьезности тревог

Уровни серьезности аварийных сигналов, которые передаются на сторонний сервер в trap-сообщениях.

Значения:
  • [NULL]: нет отправленных сообщений об аварийных сигналах и событиях.
  • Срочный: только аварийные сигналы критического уровня.
  • Значительный: аварийные сигналы серьезного и критического уровня.
  • Незначительный: аварийные сигналы незначительного, серьезного и критического уровней.
  • нормальный: события и незначительные, серьезные и критические аварийные сигналы.

Значение по умолчанию — нормальный.

№

Номер trap-канала для отправки аварийных сигналов. Можно настроить максимум четыре канала.

Адрес Trap-сервера

Адрес сервера для получения trap-сообщений с аварийными сигналами. В качестве адреса сервера можно использовать IPv4-адрес, IPv6-адрес или имя домена.

Имя домена должно соответствовать следующим требованиям:
  • Cтрока длиной до 255 символов.
  • Имя домена может состоять только из цифр, букв, дефисов (-) и точек (.).
  • Имя домена не может начинаться с дефиса (-) или точки (.), либо заканчиваться дефисом (-).
  • Имя домена может состоять из не более 63 символов между любыми двумя точками.

Порт Trap

Номер порта для получения trap-сообщений с аварийными сигналами.

Диапазон значений: от 1 до 65535

Значение по умолчанию: 162

ПРИМЕЧАНИЕ:

Для восстановления для порта Trap значения по умолчанию 162 нажмите кнопку Восстановить настройки по умолчанию.

Передача по основному каналу

Функция, обеспечивающая передачу по основному каналу. После включения данной функции отправка trap-сообщений на trap-сервер может осуществляться через ОС.

По умолчанию данная функция отключена.

ПРИМЕЧАНИЕ:
  • В случае отсутствия связи между BMC и trap-сервером включение основного канала позволяет ОС передавать trap-сообщения на trap-сервер.
  • Использование основного канала возможно только в том случае, если BMA 2.0 работает на базе операционной системы.

Текущий статус

Статус trap-канала.

По умолчанию данная функция отключена.

Разделитель сообщений

Разделитель используется для разделения ключевых слов в trap-сообщениях. Например, «;».

ПРИМЕЧАНИЕ:

Настройте данный параметр только в том случае, если Режим Trap является Код события.

Содержание сообщения

Информация, содержащаяся в trap-сообщениях.

ПРИМЕЧАНИЕ:

Настройте данный параметр только в том случае, если Режим Trap является Код события.

Отображать ключевые слова

Отображение определенных ключевых слов в trap-сообщениях.

ПРИМЕЧАНИЕ:

Настройте данный параметр только в том случае, если Режим Trap является Код события.

Пример формата

Пример формата предоставляется справа от кнопки-флажка в зависимости от того, какие из следующих параметров выбраны: Разделитель сообщений, Содержание сообщения или Отображать ключевые слова.

Операция
  • Кнопка Изменить предназначена для изменения trap-сервера и формата сообщения.
  • Кнопка Тест предназначена для проверки отправки trap-сообщения на указанный trap-сервер. Если указанный trap-сервер доступен, на экране появится сообщение «Операция выполнена успешно».
    ПРИМЕЧАНИЕ:

    Если параметры в области Настройки Trap были изменены, перед тестированием сначала нажмите кнопку Сохранить. В противном случае изменение не вступит в силу.
