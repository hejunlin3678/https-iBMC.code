Управление BMA

Описание функций

Агент управления материнской платой (Baseboard Management Agent; BMA) является программой-агентом управления по внутреннему каналу.

На странице Управление BMA пользователи могут установить BMA в операционной системе сервера через удаленную консоль.

После успешной установки BMA на странице будет отображаться основная информация о BMC, включая номер версии, состояние работы и версию драйвера.

Описание параметров

Табл. 1 Управление BMA

Параметр

Доступная версия BMA

Версия BMA, которая может быть установлена в текущей операционной системе.

Например, Linux 2.1.2.010 указывает, что версия BMA для Linux — 2.1.2.010.

Состояние установщика

Состояние соединения между операционной системой сервера и BMA, которое будет установлено.

  • Не готов (Not ready): операционная система сервера не подключена или соединение не установлено.
  • Готов (Ready): операционная система сервера успешно подключена.

-- указывает, что BMA не установлен или не работает.

Статус работы BMA

Текущий статус ПО BMA.

  • -- указывает, что BMA не установлен или не работает.
  • Статус работы (Running) указывает, что BMA был запущен.
  • Статус работы (Unhealthy) указывает, что BMA установлен и не может работать должным образом.

Установка BMA

  • Только клиент под Linux поддерживает установку BMA.
  • Для подключения драйвера BMA требуются разрешения на VMM.
  • Для запуска удаленной консоли требуются разрешения на удаленное управление.
  • Для установки BMA требуются разрешения на VMM и удаленное управление.
  1. Нажмите Установить BMA.

    BMA устанавливает соединение с операционной системой сервера. Если соединение установлено успешно, отображается страница Описание.

  2. Нажмите Открыть удаленную консоль.

    Удаленная консоль запускается.

  3. На удаленной консоли войдите в операционную систему сервера как администратор, а затем введите имя пользователя и пароль операционной системы.
  4. В списке устройств операционной системы сервера найдите диск драйвера BMA. Если BMA не отображается, подключите диск драйвера BMA и перейдите к 5. Подробную информацию о подключении диска драйвера BMA, см. в разделе Подключение диска драйвера BMA.
  5. Откройте файл README.TXT в операционной системе Linux и запросите информацию о версии ОС Linux, приведенную в Supported Operating Systems in this Release.

    1. Щелкните правой кнопкой мыши в пустой области операционной системы.
    2. В открывшемся меню выберите Open Terminal. CLI операционной системы сервера отображается.
    3. Запустите команды cd Linux и ls и проверьте файл README.TXT в Linux.
      [root@localhost ~]# cd Linux
[root@localhost Linux]# ls
app config drivers install.sh README.TXT script
    4. Запустите команду cat README.TXT.
      [root@localhost ~]# cat README.TXT

      • Если версия установленной операционной системы Linux совпадает с любой из версий операционной системы Linux, перечисленных в Supported Operating Systems in this Release, перейдите к 6.
      • Если версия установленной операционной системы Linux отличается от версий, перечисленных в Supported Operating Systems in this Release, обновите версию операционной системы до любой из версий ОС Linux, перечисленных в файле README.txt, а затем перейдите к 6.

  6. Установите BMA.

    Для получения более подробной информации об установке BMA см. в BMA User Графический интерфейс пользователя (GUI)de последней версии. Путь к установочному файлу BMA зависит от используемой операционной системы.

Подключение диска драйвера BMA

  1. Щелкните правой кнопкой мыши в пустой области операционной системы.
  2. В открывшемся меню выберите Open Terminal. CLI операционной системы сервера отображается.
  3. Запустите команду lsscsi для запроса данных о диске драйвера BMA.

    [root@localhost ~]# lsscsi
[0:0:0:0]    disk    ATA      ST4000NM0033-9ZM SN06  -
[0:0:1:0]    disk    ATA      ST4000NC001-1FS1 CN02  -
[0:0:2:0]    disk    ATA      WDC WD6000F9PZ-3 0R01  /dev/sdc
[0:0:3:0]    disk    ATA      WDC WD6000F9PZ-3 0R01  /dev/sdd
[0:0:4:0]    disk    HGST     HUS726060AL4210  A523  -
[0:0:5:0]    disk    HGST     HUS726060AL4210  A7MH  -
[0:0:6:0]    disk    ATA      SSDSC2BB016T7H   0121  /dev/sde
[0:0:7:0]    disk    HGST     HUS726060AL4210  A523  /dev/sdf
[0:1:0:0]    disk    LSI      Logical Volume   3000  /dev/sdb
[0:1:1:0]    disk    LSI      Logical Volume   3000  /dev/sda
[19:0:0:0]   disk    SERVER   BMA USB Device   225  /dev/sdv

    В выходных данных команды BMA USB Device указывает имя узла диска драйвера BMA, и /dev/sdv — букву диска, назначенную диску драйвера BMA.

  4. Запустите команду mount /dev/sdv /home/file, чтобы смонтировать диск драйвера BMA в каталог /home/file.

    В этом примере /home/file — каталог для хранения смонтированного файла. Создайте каталог на основе фактических требований. Имена папок в каталоге могут содержать цифры, буквы, знаки подчеркивания (_), дефисы (-) и точки (.).

    [root@localhost ~]# mount /dev/sdv /home/file
#

  5. Запустите команду ls /home/file, чтобы проверить, успешно ли смонтирован диск драйвера BMA.

    [root@localhost ~]# ls /home/file
Linux

    Если в выходных данных команды отображается Linux, диск драйвера BMA успешно смонтирован.
