После открытия встроенной удаленной консоли HTML5 отображается сообщение о тайм-ауте настройки доверенного сертификата

Признак неисправности

Признак

Возможные причины

Когда пользователь пытается открыть встроенную удаленную консоль HTML5, отображается сообщение «Сбой открытия KVM из-за тайм-аута настройки доверенного сертификата».

Перед установкой клиентом KVM соединения с сервером осуществляется проверка сертификата SSL. Если сертификат не проходит проверку, соединение со встроенной удаленной консолью HTML5 не может быть установлено.

Решение

  1. В WebUI BMC выберите Конфигурация > Web-службы и проверьте в области Данные о сертификате сервера, не истек ли срок действия сертификата сервера.

    • Если да, выполните 2.
    • Если нет, выполните 3.

  2. Сгенерируйте новый сертификат и замените им сертификат, срок действия которого истек.
  3. Выполните перезапуск BMC.
  4. Проверьте, открывается ли встроенная удаленная консоль HTML5.

    • Если да, то никаких действий больше выполнять не требуется.
    • Если нет, выполните 5.

  5. Обратитесь за помощью в службу технической поддержки.
Исходная тема: Устранение неполадок удаленной виртуальной консоли
