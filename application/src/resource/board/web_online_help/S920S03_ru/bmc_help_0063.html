Не удалось подключиться к системе управления после открытия KVM

Признак неисправности

Признак

Возможные причины

После открытия KVM появляется сообщение «Сбой подключения к системе управления. IP-адрес системы управления: xx.xx.xx.xx.»

Для службы KVM по умолчанию используется порт 2198. Эта ошибка появляется, когда сервисный порт KVM не включен или недоступен.

Решение

  1. В WebUI BMC выберите Настройки > Службы и проверьте, включена ли служба KVM.

    • Если да, выполните 2.
    • Если нет, выполните 3.

  2. Откройте CLI (с помощью команды cmd) и выполните команду telnet, например telnet xx.xx.xx.xx 2198, чтобы проверить достижимость порта, используемого для службы KVM.

    xx.xx.xx.xx – IP-адрес, а 2198 – номер порта KVM по умолчанию. Используйте фактический порт, указанный в настройках, для проверки которых выполнялся 1.

    • Если да, выполните 5.
    • Если нет, выполните 4.

  3. Включите службу KVM и подключите KVM. Затем проверьте, была ли решена проблема.

    • Если да, то никаких действий больше выполнять не требуется.
    • Если нет, выполните 2.

  4. Обратитесь к системному администратору для активации порта, необходимого KVM. После проверки достижимости порта подключите KVM и проверьте, была ли решена проблема.

    • Если да, то никаких действий больше выполнять не требуется.
    • Если нет, выполните 5.

  5. Обратитесь за помощью в службу технической поддержки.
