Certificate Update

Function Description

On the Certificate Update page, you can:

  • Configure settings for the SSL certificate update.
  • Configure the CA server.
  • Customize the certificate subject

Parameter description

Table 1 Certificate update

Parameter

Description

SSL Certificate Update

NOTE:
  • An SSL certificate can be updated only when the system lock is disabled, the user has the security configuration permission, and the CA server is configured.
  • To update the SSL certificate, you need to import both the server root certificate and client certificate. Otherwise, the authentication will fail.

Auto Update

Enables or disables the automatic certificate update function.

After Auto Update is enabled, the SSL certificate is automatically updated at an interval of one hour.

This function is disabled by default.

Certificate Update

You can click Certificate Update to update the SSL certificate.

CA Server Configuration

Server Address

IP address of the CA server.

IPv4 address, IPv6 address, or domain name with the maximum length of 67 characters.

Port

Port number of the CA server.

Value range: 1-65535

Path

Path of the CA server.

A string of 1 to 128 characters. Starts with a slash (/) and consists of digits (0-9), letters (a-z, A-Z), underscores (_), and slashes (/).

Certificate Subject

Country (C)

Country to which a certificate belongs.

A two-letter country code. All English letters are allowed.

State/Province (S/P)

State or province to which a certificate belongs.

A string of 0 to 128 characters. Digits (0-9), letters (a-z, A-Z), hyphens (-), underscores (_), periods (.), and spaces are allowed.

City/Location (L)

City or location to which a certificate belongs.

A string of 0 to 128 characters. Digits (0-9), letters (a-z, A-Z), hyphens (-), underscores (_), periods (.), and spaces are allowed.

Organization Name (O)

Company or organization to which a certificate belongs.

A string of 0 to 64 characters. Digits (0-9), letters (a-z, A-Z), hyphens (-), underscores (_), periods (.), and spaces are allowed.

Organizational Unit (OU)

Organizational unit or department to which a certificate belongs.

A string of 0 to 64 characters. Digits (0-9), letters (a-z, A-Z), hyphens (-), underscores (_), periods (.), and spaces are allowed.

Common Name (CN)

Common name of the certificate.

A string of 1 to 64 characters. Digits (0-9), letters (a-z, A-Z), hyphens (-), underscores (_), periods (.), and spaces are allowed.

Internal Name (IN)

Internal name of the certificate.

A string of 0 to 64 characters. Digits (0-9), letters (a-z, A-Z), hyphens (-), underscores (_), periods (.), and spaces are allowed.

Email (E)

Email address of the certificate.

A string of 0 to 255 characters in compliance with the email address convention.

Server Root Certificate

Server root certificate applied from an official CA. Certificate used to perform TLS authentication on the server certificate packets sent from the CA server before a connection is established.

NOTE:
  • Server root certificates in .crt, .cer, or .pem format are supported. The code of the certificate in *.cer format is Base64. The maximum certificate size is 100 KB. Duplicate certificates are not allowed.
  • For security purposes, periodically update the certificate.

Client Certificate

Client certificate applied from an official CA.

NOTE:
  • Client certificates in *.p12 or *.pfx format are supported. The maximum certificate size is 100 KB.
  • For security purposes, periodically update the certificate.

Certificate Information

Displays information about the uploaded certificate, including:

  • Issued By
  • Issued To
  • Validity Period
  • Serial Number
  • CRL, with either of the following statuses:
    • Configured: The certificate revocation file has been uploaded, and a certificate revocation verification will be performed when a TLS connection is set up.
    • Unconfigured: The certificate revocation file is not uploaded.
NOTE:
  • CRL file must be in *.crl format. The file name cannot be empty. The maximum size of the CRL file is 100 KB.
  • If the CRL expires, the corresponding authentication function will be unavailable.
  • A client certificate does not support the CRL.

Uploading a Certificate

  1. Click Upload next to Server Certificate or Client Certificate.

    A dialog box for local file selection or client certificate upload is displayed.

    • Server root certificates in .cer, .crt, or .pem format are supported. The code of the certificate in *.cer format is Base64. The maximum certificate size is 100 KB. Duplicate certificates are not allowed.
    • Client certificates in *.p12 or *.pfx format are supported. The maximum certificate size is 100 KB.

  2. Select a file in the correct format.
  3. Click OK.

Configuring a CRL

CRL file must be in *.crl format. The file name cannot be empty. The maximum size of the CRL file is 100 KB.

  1. Obtain the CRL file from the CA.
  2. In the Server Certificate Information area, click Upload next to CRL.
  3. Select the CRL file.
  4. Enter the password of the login user and click Save.
