Power

Function Description

On the Power page, you can perform the following operations:

  • View the power supply units (PSUs) of the server.
  • View information about the server power.
  • Enable or disable the power capping function and set the power capping value.
  • View the historical average power and peak power line charts of the last week or day, view the power data obtained at each sampling time, and re-collect power statistics.

    The system obtains server power data at an interval of 10 minutes.

  • Power on, power off, or reset the server OS.
  • Set the power button on the server panel.
  • Set the power restore policy for the server OS.
  • Set the power-on delay for the server.
  • Exercise caution when you set the power capping value. A small power capping value may affect system performance and operations.
  • Before you perform power control on a server, ensure that the operation does not affect services.

System State Upon Power Supply

After the server PSUs are powered on, the OS startup policies can be any of the following:

  • Power On: Automatically starts the OS after the power supply is restored.
  • Remain Off: Keeps the server OS shut down even after the power supply is restored.
  • Restore Previous State: Restores the OS to the state before the power failure.
    • If the server OS is started before the power failure, the server OS starts automatically after the power supply is restored.
    • If the server OS is shut down before the power failure, the server OS is shut down after the power supply is restored.

The default value is Power On.

Power-On Delay

The power-off of the upper-level power supply equipment will cause simultaneous power-on of a large number of servers. The spike peak current will adversely cause great impact on the power supply equipment and may damage the power supply equipment. To prevent this problem, you can set the power-on delay to postpone the servers from powering on for a specified time.

You need to set the power-on delay when the following conditions are met:
  • The server is set to power on after the power supply is restored.
  • Controlled power-on is disabled.

Value:

  • Default: The power-on delay is random up to 2 seconds.
  • Half: The power-on delay probability is 50%. That is, after power-on, there is a 50% probability that the power-on of the server is delayed according to the specified time.

    Value range: 0 to 120, accurate to 0.1 seconds.

  • Fixed: The power-on delay is fixed to the specified time.

    Value range: 0 to 120, accurate to 0.1 seconds.

  • Random: The power-on delay is random up to M seconds.

    Value range: 0 to 120, accurate to 0.1 seconds.

The default value is Default.

Procedure

Table 4 Power operations

Operation

Procedure

Set the PSU work mode.
  1. On the Power Supply Info tab, click Power Supply Settings.
  2. Set the PSU working mode.
  3. (Optional) If the work mode is active/standby, set the active and standby PSUs.
  4. (Optional) Set Hibernate to to enable the hibernation function.
    NOTE:
    • If deep hibernation is enabled, certain PSUs enter the deep hibernation mode and stop outputting power when the server is powered off. When the server is powered on or deep hibernation is disabled, the PSUs in deep hibernation mode start to output power.
    • After deep hibernation is enabled and the server is powered off, if all active PSUs are removed or stop outputting power due to faults, the entire server loses power for about 10 seconds and then the PSUs in deep hibernation mode automatically turn on to output power.
  5. Click Save.

Start the server OS.
  1. On the Power Control page, click the Power On button under Virtual Buttons.
    The following message is displayed:
    Are you sure you want to perform this operation?
  2. Click Yes.

    The server OS startup time varies according to the server configuration. If "Operation Successful" is displayed, the OS starts successfully.

    After the OS starts, System Power changes to On.

Shut down the server OS.
  1. On the Power Control page, click the Power Off button under Virtual Buttons.
    The following message is displayed:
    Are you sure you want to perform this operation?
  2. Click Yes.

    If "Operation Successful" is displayed, the server OS is shut down successfully.

    After the server OS is shut down, System Power changes to Off.

Forcibly shut down the server OS.
  1. On the Power Control page, click the Forced Power Off button under Virtual Buttons.
    The following message is displayed:
    A forced power-off may damage programs or unsaved data of the server. Are you sure you want to continue?
  2. Click Yes.

    If "Operation Successful" is displayed, the server OS is forcibly shut down.

    After the server OS is forcibly shut down, System Power changes to Off.

Forcibly Reset the server OS.
  1. On the Power Control page, click the Forced System Reset button under Virtual Buttons.
    The following message is displayed:
    A forced restart may damage programs or unsaved data of the server. Are you sure you want to continue?
  2. Click Yes.

    If "Operation Successful" is displayed, the server OS is forcibly reset.

Power cycle the server OS.
  1. On the Power Control page, click the Forced Power Cycle button under Virtual Buttons.
    The following message is displayed:
    A forced power cycle may damage programs or unsaved data of the server. Are you sure you want to continue?
  2. Click Yes.

    The time required varies according to the server configuration. If "Operation Successful" is displayed, the server OS is powered off and then on successfully.

    During the power cycle process, System Power changes from On to Off and then to On.

Set the power restore policy.
  1. On the Power Control page, select the power restore policy under System State Upon Power Supply.
  2. Click Save.

    If "Operation Successful" is displayed, the setting is successful.

Set the power-off timeout period.
  1. On the Power Control tab, select the check box of Power-off Timeout.
  2. Click and enter a value.

    The value range varies depending on the server model.

  3. Click to save the setting.

    If Operation Successful is displayed, the power-off timeout period is set successfully.

Check the power-off timeout period.

On the Power Control page, check the value of Timeout Period (s) next to Power-off Timeout under the Virtual Buttons area.

Set the power-on delay.
  1. On the Power Control page, select the power-on delay mode in the Power-On Delay area.
  2. Click and set the delay time.

    The value range of the delay time is 0 to 120 seconds, accurate to 0.1 seconds. The Default mode does not support setting of the delay time.

  3. Click to save the delay time specified.
  4. Click Save.
    If "Operation successful" is displayed, the power-on delay is set successfully.
    NOTE:

    After Save is clicked, the delay time specified in 2 will also apply to the other two modes with delay time configurable.
