Fuseau horaire et NTP

Présentation de la fonction

Sur la page Fuseau horaire et NTP, vous pouvez consulter et configurer :

  • Fuseau horaire
  • Informations NTP

Description du paramètre

Tableau 1 Fuseau horaire et NTP

Parameter

Description

Fuseau horaire

Fuseau horaire de le BMC.

Définissez Région, Fuseau horaire et DST en fonction des besoins réels.

Valeur par défaut : Autres+GMT+activé

REMARQUE:
  • Les informations de fuseau horaire sont automatiquement obtenues si l'option Obtenir automatiquement des informations NTP via DHCPv4 est sélectionnée.
  • Si le DST est activé, dans les fuseaux horaires qui utilisent l'heure d'été, le BMC ajuste automatiquement l'heure une heure en avant lorsque l'heure d'été commence et ajuste l'heure en arrière à l'heure normale lorsque l'heure d'été se termine.
  • Lors de la synchronisation de l'heure sur le système d'exploitation, exécutez la commande hwclock --utc -w. Cette commande peut assurer la cohérence entre l'heure du système d'exploitation et l'heure de le BMC.

NTP

NTP

NTP permet au serveur de BMC synchroniser l'heure avec le serveur NTP.

Il est désactivé par défaut.

Obtenir l'adresse IP
  • Cliquez sur cette option pour autoriser l'attribution automatique d'une adresse IPv4 au serveur NTP.
  • Cliquez sur cette option pour autoriser l'attribution automatique d'une adresse IPv6 au serveur NTP.
  • Cliquez sur cette option pour définir manuellement les serveurs NTP.

La valeur par défaut est Définir manuellement l'adresse IP.

AVIS:
  • Si l'adresse IP du port réseau de gestion de le BMC est automatiquement obtenue, vous pouvez obtenir les informations NTP automatiquement ou manuellement. L'obtention automatique est recommandée.
  • Si l'adresse IP du port réseau de gestion BMC est définie manuellement, les informations NTP doivent également être définies manuellement.
  • Server 1 au Server 3
  • Serveur préféré 1 au Serveur préféré 3 ou Serveur alternatif 1 au Serveur alternatif 3

Les adresses des serveurs NTP.

Valeur : une adresse IPv4 ou IPv6 ou un nom de domaine
REMARQUE:
Le nom de domaine doit répondre aux exigences suivantes :
  • Une chaîne de 0 à 67 caractères.
  • Seulement les chiffres, les lettres, les traits d'union (-) et les points (.) sont autorisés.
  • Ne peut pas commencer par un trait d'union ou un point, ni se terminer par un trait d'union.
  • Un maximum de 63 caractères sont autorisé entre deux points.
L'une ou l'autre des solutions suivantes est proposée.
  • Server 1 au Server 3 : trois serveurs NTP sont disponibles. Les trois serveurs sont efficaces en même temps.
  • Serveur préféré 1 au Serveur préféré 3 ou Serveur alternatif 1 au Serveur alternatif 3 : trois groupes de serveurs NTP sont disponibles. Dans chaque groupe, la gauche est le serveur préféré et la droite est le serveur alternatif. Les serveurs sont sélectionnés selon les règles suivantes :
    • Sélectionner un serveur dans chaque groupe.
    • Si les deux adresses d'un groupe ne sont pas valides, le groupe de serveurs sera rejeté.
    • S'il n'y a qu'une seule adresse valide dans un groupe, sélectionner le serveur ayant l'adresse IP valide.
    • Si les deux adresses d'un groupe sont valides, sélectionner le serveur avec l'adresse IPv6.
    • Si les deux sont des adresses IPv4 ou IPv6, sélectionner le serveur préféré (celui de gauche).
REMARQUE:

Le basculement entre les serveurs NTP actif et de secours dépend de l'intervalle de synchronisation entre le BMC et le serveur NTP (Intervalle d'interrogation minimum ≤ intervalle de synchronisation ≤ Intervalle d'interrogation maximum). Si le BMC ne parvient pas à se synchroniser avec le serveur actif à plusieurs reprises, le serveur de secours devient le serveur NTP.

Intervalle d'interrogation minimum

Intervalle minimal auquel le système BMC synchronise le temps à partir du serveur NTP, c'est-à-dire l'intervalle minimal pour l'interrogation des paquets NTP.

Par exemple, si ce paramètre est défini sur 6, l'intervalle d'interrogation est de 2 à la puissance 6, soit 1 minute et 4 secondes.

Valeur : 3 à 17

La valeur par défaut est 6 (1m4s).

Intervalle d'interrogation maximum

Intervalle maximal auquel le système BMC synchronise le temps à partir du serveur NTP, c'est-à-dire l'intervalle maximal pour l'interrogation des paquets NTP.

Par exemple, si ce paramètre est défini sur 6, l'intervalle d'interrogation est de 2 à la puissance 6, soit 1 minute et 4 secondes.

Valeur : 3 à 17

La valeur par défaut est 10 (17m4s).

Authentification du serveur

Authentification, qui peut être activée ou désactivée, BMC pour la communication entre le serveur et le serveur NTP.

Il est désactivé par défaut.

NTP Group Key

Clé de groupe NTP

La clé privée doit être chargée vers le BMC pour l'authentification d'identité si l'Authentification du serveur est activée.

REMARQUE:
  • Vous pouvez télécharger un générateur de clés (par exemple, ntp-keygen) pour générer des clés privées. Le fichier de clé est au format .keys. La taille ne peut pas dépasser 2 Mo.
  • Pour des raisons de sécurité, mettez périodiquement à jour la clé.

Réglage du fuseau horaire

  1. Sélectionnez une valeur dans Région et Fuseau horaire.

    Pour plus de détails sur les paramètres, voir Tableau 1.

  2. Cliquez sur Enregistrer.

    Si « Opération réussie » s'affiche, le réglage est réussi.

    • Lors de la synchronisation de l'heure sur le système d'exploitation, exécutez la commande hwclock --utc -w. Cette commande peut assurer la cohérence entre l'heure du système d'exploitation et l'heure de le BMC.
    • Après avoir modifié le fuseau horaire, vous devez redémarrer le BMC pour que l'heure système prenne effet pour les courbes de l'utilisation du processeur, de la mémoire, de la bande passante du port réseau, du disque, de l'historique de puissance et de la température d'admission affichées dans l'interface utilisateur Web du le BMC.

Paramètres NTP

  1. Dans la zone Paramètres NTP, définissez les paramètres en fonction des exigences de service.

    Pour plus de détails sur les paramètres, voir Tableau 1.

  2. Cliquez sur Enregistrer.

    Si « Opération réussie » s'affiche, le réglage est réussi.
