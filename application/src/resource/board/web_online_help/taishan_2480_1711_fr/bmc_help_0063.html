Impossible de se connecter au système de gestion après l'ouverture du KVM

Symptôme

Symptôme

Causes possibles

Après que le KVM est ouvert, "échec de la connexion du système de gestion, l'IP du système de gestion est xx.xx.xx.xx."

Le numéro de port par défaut du service KVM est 2198. Cette erreur se produit lorsque le port de service KVM n'est pas activé ou n'est pas disponible.

Solution

  1. Sur l'BMC WebUI, choisissez Services > Services portuaires, et vérifiez si le service KVM est activé.

    • Si oui, passez à 2.
    • Si non, passez à 3.

  2. Ouvrez la CLI (en utilisant la commande cmd) et exécutez la commande telnet, par exemple telnet xx.xx.xx.xx 2198, pour vérifier si le port de service KVM est accessible.

    xx.xx.xx.xx indique l'adresse IP. Et 2198 est le numéro de port KVM par défaut. Utilisez le numéro de port KVM actuel obtenu à 1.

    • Si oui, passez à 5.
    • Si non, passez à 4.

  3. Activez le service KVM et connectez-vous au KVM. Ensuite, vérifiez si le problème est résolu.

    • Si oui, aucune autre action est nécessaire.
    • Si non, passez à 2.

  4. Contactez l'administrateur du réseau pour activer le port requis du KVM. Après avoir vérifié que le port est accessible, connectez-vous au KVM et vérifiez si le problème est résolu.

    • Si oui, aucune autre action est nécessaire.
    • Si non, passez à 5.

  5. Contactez le support technique.
