Kerberos

Présentation de la fonction

Sur la page Kerberos, vous pouvez consulter et configurer les paramètres de base et les informations sur les groupes d'utilisateurs de Kerberos.

Le BMC fournit une interface d'accès pour les utilisateurs Kerberos. L'utilisation d'un compte utilisateur Kerberos pour se connecter à le BMC améliore la sécurité du système. Vous pouvez vous connecter à l'interface Web BMC en tant qu'utilisateur Kerberos.

Description du paramètre

Tableau 1 Kerberos

Paramètre

Description

Kerberos

Paramétrage de la connexion Kerberos.

Il est désactivé par défaut.
REMARQUE:

Lorsqu'un compte Kerberos est utilisé pour se connecter à BMC après l'activation de la fonction Kerberos, le serveur d'authentification garantit les politiques de sécurité (vérification de mot de passe, durée de validité du mot de passe, durée de vie minimum du mot de passe, période inactive, anciens mots de passe interdits et blocage en cas de l'échec de connexion) liées au compte.

Réglage de base

REMARQUE:

Les paramètres marqués d'un astérisque (*) sont obligatoires.

Domaine

Domaine Kerberos.

Valeur : une chaîne de 255 caractères maximum

La valeur peut contenir des chiffres, des lettres (majuscules recommandées) et des caractères spéciaux (y compris les espaces).

Adresse du serveur Kerberos

L'adresse du serveur Kerberos.

Si Kerberos est activé, entrez le nom de domaine complètement qualifié (nom d'hôte.nom de domaine) du serveur Kerberos et configurez DNS sur la page Paramètres BMC > Paramètres réseau.

Format : une adresse IPv4 ou IPv6 ou un nom de domaine

Le nom de domaine doit répondre aux exigences suivantes :
  • Une chaîne de 0 à 255 caractères.
  • Seulement les chiffres, les lettres, les traits d'union (-) et les points (.) sont autorisés.
  • Ne peut pas commencer par un trait d'union ou un point, ni se terminer par un trait d'union.
  • Un maximum de 63 caractères sont autorisé entre deux points.

Port de Kerberos

Numéro de port du service Kerberos.

Plage de valeurs : 1 à 65535

Valeur par défaut : 88

Tableau des clés

Tableau des clés Kerberos à télécharger.

La table des clés doit être au format .keytab .

Il est laissé vide par défaut.

REMARQUE:
  • La taille du tableau des clés ne peut pas être de 0 Ko ou supérieure à 1 Mo.
  • Pour des raisons de sécurité, mettez périodiquement à jour la clé.

Mot de passe de connexion

Mot de passe de l'utilisateur qui utilise actuellement le BMC.

Groupe d'utilisateurs Kerberos

Affiche des informations sur tous les groupes d'utilisateurs Kerberos.

Le BMC prend en charge un maximum de cinq groupes d'utilisateurs Kerberos.

Ajouter un groupe d'utilisateurs

Ajouter un groupe Kerberos.

Nom du groupe

Nom du groupe auquel appartient l'utilisateur Kerberos.

Plage de valeurs : une chaîne de 255 octets (64 à 255 caractères). La longueur spécifique varie avec le nombre d'octets de chaque caractère.

SID

Identificateur de sécurité utilisé pour l'autorisation de Kerberos et des groupes d'utilisateurs. Par exemple, S-1-5-21-310440588-250036847-580389505-500.

Plage de valeurs : une chaîne de 255 octets (64 à 255 caractères), sans espace. La longueur spécifique varie avec le nombre d'octets de chaque caractère.

Rôle

Rôle du groupe d'utilisateurs Kerberos.

  • Administrateur : Les utilisateurs auxquels le rôle Administrateur est attribué peuvent effectuer toutes les opérations.
  • Opérateur : Les utilisateurs auxquels le rôle Opérateur est attribué peuvent effectuer la gestion de base, le contrôle à distance, le support à distance, le contrôle d'alimentation, l'interrogation des informations et configurer leurs propres données.
  • Utilisateur courant : Les utilisateurs auxquels le rôle Utilisateur courant est attribué peuvent demander des informations et configurer leurs propres données.
  • Rôle personnalisé : Les utilisateurs auxquels le rôle Personnalisé 1 au rôle Personnalisé 4 est attribué peuvent effectuer les opérations spécifiées.
REMARQUE:

L'autorisation utilisateur par défaut pour un nouveau groupe d'utilisateurs Kerberos est Utilisateur commun.

Interfaces de connexion

Interfaces par lesquelles les membres du groupe Kerberos peuvent se connecter à BMC.

  • Web

    Par défaut, les nouveaux groupes d'utilisateurs Kerberos peuvent se connecter au système via l'interface utilisateur Web.

  • Redfish

    Par défaut, les nouveaux groupes d'utilisateurs Kerberos peuvent se connecter au système via l'interface utilisateur l'API Redfish.

Dossier de groupe

Nom du dossier utilisateur sur le serveur Kerberos.

Format : « CN=xxx » ou « OU=xxx ».

Lorsqu'il y a plusieurs niveaux de nuds, le nud de niveau supérieur suit le nud de niveau inférieur avec une virgule entre les deux.

Par exemple, si le groupe d'utilisateurs grouptest est dans\testgroups\part1 sur le serveur Kerberos, saisissez OU=part1,OU=testgroups.

REMARQUE:

Pour plus de détails sur la différence entre CN et OU, voir la description du protocole Kerberos.

Dans Windows AD, par exemple, c'est le cas :

  • CN si Type est Container.
  • OU si Type est Organizational Unit.

Plage de valeurs : une chaîne de 255 octets (64 à 255 caractères). La longueur spécifique varie avec le nombre d'octets de chaque caractère.

Règles de connexion

Règles de connexion qui s'appliquent au groupe Kerberos.

Modifie un groupe Kerberos.

Supprimer un groupe Kerberos.

Activation de Kerberos et configuration du serveur de domaine

  1. Définissez Kerberos sur .
  2. Définissez le serveur de domaine. Pour plus de détails sur les paramètres, voir Tableau 1.
  3. Cliquez sur Enregistrer.

Importer une table de clés

La table des clés doit être au format .keytab. La taille du tableau des clés ne peut pas être de 0 Ko ou supérieure à 1 Mo.

  1. Cliquez sur Ajouter le fichier et sélectionnez la table de clés à importer.
  2. Cliquez sur Ouvrir.

    Une fois la table de clés téléchargée, le message "Fichier envoyé avec succès" s'affiche.

  3. Entrez le mot de passe de l'utilisateur actuel.

    L'utilisateur Kerberos qui se connecte à l'aide de l'authentification unique n'a pas besoin de saisir le mot de passe.

  4. Cliquez sur Enregistrer.

Supprimer un groupe d'utilisateurs Kerberos

  1. Dans la zone Groupe d'utilisateurs Kerberos localisez le groupe d'utilisateurs Kerberos à supprimer et cliquez sur Supprimer.

    Une boîte de dialogue Confirmer , vous demandant de saisir le mot de passe de l'utilisateur actuel.

    L'utilisateur Kerberos qui se connecte à l'aide de l'authentification unique n'a pas besoin de saisir le mot de passe.

  2. Entrez le mot de passe de l'utilisateur actuel.
  3. Cliquez sur OK.

Modifier un groupe d'utilisateurs Kerberos

  1. Dans la zone Groupe d'utilisateurs Kerberos localisez le groupe d'utilisateurs Kerberos à modifier et cliquez sur Modifier.
  2. Modifier les paramètres. Pour plus de détails sur les paramètres, voir Tableau 2.
  3. Cliquez sur Enregistrer.
