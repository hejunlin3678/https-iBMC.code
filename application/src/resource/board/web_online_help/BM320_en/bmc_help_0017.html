Alarm Settings

Function Description

On the Alarm Settings page, you can configure:

  • Syslog notifications: enables logs to be sent to a third-party server via syslog messages.
  • Email notifications: enables an email to be sent to the specified mailboxes over a Simple Mail Transfer Protocol (SMTP) server when an alarm is generated.
  • Trap notifications: enables alarms, events, and trap properties to be sent to a third-party server via trap messages.

    Traps are messages that are sent from the BMC to a third-party server without being explicitly requested. Traps are used to report events and critical, major, and minor alarms.

Parameter Description

Enabling TCP or UDP will compromise system security. Proceed with caution.

Parameter

Description

Syslog Notification Settings

Syslog Function

Function for sending notifications through syslog messages.

It is disabled by default.

Alarm Severities

Severities of alarms to be sent through syslog messages.

Values:
  • [NULL]: No alarm or event is sent.
  • Critical: Only critical alarms are reported.
  • Major: Major and critical alarms are reported.
  • Minor: Minor, major, and critical alarms are reported.
  • Informational: Events and minor, major, and critical alarms are reported.

The default value is Informational.

Syslog Message Format
Format of the syslog messages.
  • Custom: The syslog messages contain the message priority, product name, syslog host ID, device location, and log type.
  • RFC3164: The syslog messages comply with RFC3164 and contain the message priority, timestamp, host name, process name, and log type.

The default value is Custom.

NOTE:

If RFC3164 is selected, Syslog Server Identity cannot be set and Host Name is used by default.

Syslog Server Identity

Source of the syslog message.

Values:

  • Board serial number
  • Product asset label
  • Host name

The default value is Board serial number.

Transmission Protocol

Protocol used to transmit syslog messages between the BMC and the syslog server.

Values:

  • TLS: a connection-oriented protocol that ensures confidentiality and integrity of the transmitted data.
  • TCP: a connection-oriented protocol that sends data only after a reliable connection is set up between the sender and the receiver.
  • UDP: a connectionless protocol that sends data without establishing a connection between the sender and the receiver.

The default value is TLS.

Authentication Mode

Mode for authenticating syslog certificates. Set this parameter only when Transmission Protocol is TLS.

Values:

  • One-way: authenticates only the syslog server certificate.
  • Two-way: authenticates certificates of both the syslog server and client.

The default value is One-way.

Server Root Certificate

Certificate used to verify the messages sent from the syslog server before a connection is established.

NOTE:
  • Server root certificate file in the .crt, .cer, or .pem format can be imported. The size cannot exceed 100 KB.
  • For security purposes, periodically update the certificate.

Client Certificate

When establishing a data connection, BMC sends packets that carry local certificate information to the syslog server, which requires the certificate information to authenticate the syslog client (BMC).

NOTE:
  • Client certificate files in the .pfx or .p12 format can be imported. The size cannot exceed 1 MB.
  • For security purposes, periodically update the certificate.

Server Certificate Information

Information about the server root certificate uploaded. The certificate information includes the following:

  • Authority that issued the root certificate.
  • User to which the root certificate was issued.
  • Validity period of the root certificate.
  • Serial number of the root certificate.
  • Certificate revocation list.
  • CRL validity period.
NOTE:
  • The CRL indicates the certificate revocation status.
    • Configured: The certificate revocation file has been uploaded, and a certificate revocation verification will be performed when a TLS connection is set up.
    • Unconfigured: The certificate revocation file is not uploaded.
  • The Base64-encoded certificate revocation file is in *.crl format and cannot exceed 100 KB.
  • If the CRL expires, the corresponding authentication function will be unavailable.

To set the CRL, click Upload, select the certificate revocation file from the client. Click Delete to delete the CRL.

NOTE:

Deleting a CRL may cause security risks of using expired certificates.

Syslog Server and Message Format

No.

Channel for sending syslog messages. A maximum of four channels can be set.

Server Address

Address of the syslog server.

Values: IPv4 address, IPv6 address, or domain name

NOTE:
  • Enter a domain name if Transmission Protocol is TLS. In addition, DNS information must have been correctly configured on BMC Settings > Network Settings.
  • If the IP address of the domain name changes, manually disable the syslog service and then enable it.
  • The domain name must meet the following requirements:
    • Contain 0 to 255 characters.
    • Allow digits, letters, hyphens (-), and dots (.) only.
    • Cannot start with a hyphen or dot, or end with a hyphen.
    • Allow a maximum of 63 characters between any two dots.

Port

Port number of the syslog server.

Value range: 1 to 65535

Log Type

Types of logs reported through syslog messages.

Values:

  • Operation Logs
  • Security Logs
  • Event

One or multiple values can be selected.

All options are selected by default.

Current Status

Setting of syslog notification of the channel, which can be enabled or disabled.

It is disabled by default.

Operation
  • Click Edit to modify the syslog server and message format.
  • Click Test to verify whether a syslog message can be successfully sent to the specified syslog server. If "Operation successful" is displayed, the destination syslog server is reachable.
NOTE:

If the parameters in the Syslog Notification Settings area are modified before you click Test, click Save first. Otherwise, the modification does not take effect.

SMTP Settings

SMTP Function

Function for sending email notifications through the SMTP server.

It is disabled by default.

SMTP Server Address

IPv4 or IPv6 address or domain name of the SMTP server.

The domain name must meet the following requirements:
  • Contain 0 to 255 characters.
  • Allow digits, letters, hyphens (-), and dots (.) only.
  • Cannot start with a hyphen or dot, or end with a hyphen.
  • Allow a maximum of 63 characters between any two dots.

SMTP Server Port

Port number of the SMTP server.

Value range: 1 to 65535

Default value: 25

TLS Enabled

Function for enabling Transport Layer Security (TLS) for data transmission.

If TLS is disabled, data is transmitted in plain text.

The default value is Yes.

NOTE:
  • By default, the SMTP server supports TLS. You are advised to enable the TLS function for security purposes.
  • After enabling TLS on the BMC WebUI, enable TLS and configure identity authentication on the SMTP server. The SMTP server can receive emails from the BMC only after TLS is enabled.
  • BMC does not support TLS 1.0.

Validation Server Certificate

Function for enabling or disabling the validation server certificate upon enabling the TLS encrypted transmission.

This function is disabled by default.

Server Certificate

The server certificate is used to authenticate the packets sent from the SMTP server when establishing connection.

NOTE:
  • The server certificate file format can be .crt, .cer, or .pem, with a maximum size of 100 KB.
  • Update the certificate periodically to avoid security risks.

Server Certificate Information

Function for displaying information about the uploaded server certificate, including:

  • Issued By
  • Issued To
  • Validity Period
  • Serial Number

Email Info

Anonymous Login Allowed

Function for allowing anonymous login.

If anonymous login is allowed, the SMTP server transfers alarm emails without authenticating user name and password.

If anonymous login is not allowed, the SMTP server transfers alarm emails only after the correct user name and password are entered. The user name and password must have been set on the SMTP server.

The default value is No.

NOTE:

By default, the SMTP server does not allow anonymous login. For security purposes, do not use anonymous login.

Sender User Name/Sender Password

User name and password used when Anonymous Login Allowed is set to No.

The user name and password must be the same as the user name and password set on the SMTP server.

Value range:

  • User name: a string of 1 to 64 characters, consisting of letters, digits, and special characters. It cannot be left blank.
  • Password: a string of 1 to 50 characters.

Sender Email

Email address from which alarms are sent.

Value: a string of up to 255 characters in the xx@xxx.xx format.

Letters, digits, and special characters are allowed.

Email Subject/Email Subject Contains

Subject of the email.

Value: a string of 0 to 255 characters, consisting of letters, digits, and special characters. The default value is Server Alert.

Enter the subject in Email Subject, and select the keywords to be contained in the email subject.

For example, if you select Host name and Board serial number, the email subject will contain the host name and board serial number.

Include Alarm Severities

Severities of alarms to be sent through the SMTP server.

Values:
  • [NULL]: No alarm or event is sent.
  • Critical: Only critical alarms are reported.
  • Major: Major and critical alarms are reported.
  • Minor: Minor, major, and critical alarms are reported.
  • Informational: Events and minor, major, and critical alarms are reported.

The default value is Informational.

Recipient Addresses

No.

Channel for sending alarm emails. A maximum of four channels can be set.

Email Address

Email addresses for receiving emails. The addresses must have been set on the SMTP server.

Value: a string of up to 255 characters in the xx@xxx.xx format.

Letters, digits, and special characters are allowed.

Description

Supplementary information about email addresses for receiving emails.

Value: a string of 0 to 255 characters, consisting of letters, digits, and special characters.

Send Email

Function to test whether an email can be successfully sent to the recipient.

It is disabled by default.

Operation
  • Click Edit to modify the recipient address.
  • Click Test to verify whether an email can be successfully sent to the specified mailbox.
NOTE:

If the parameters in the SMTP Settings area are modified before you click Test, click Save first. Otherwise, the modification does not take effect.
Table 3 Trap Notification

Parameter

Description

Trap Settings

Trap Function

Function for sending alarms through trap messages.

It is enabled by default.

Trap Version

SNMP version used for sending traps.

Values:

  • SNMPv1: the first official SNMP version defined in Request for Comments (RFC) 1157.
  • SNMPv2c: a version added community-based management architecture to SNMPv2.
  • SNMPv3: a version added security and remote configuration enhancements to SNMP.
NOTE:
  • SNMPv3 trap is recommended. Exercise caution when using SNMPv1 and SNMPv2c, because they pose security risks.
  • You can set the SNMPv3 authentication and encryption algorithms in User & Security > Local Users on the WebUI.

Default value: SNMPv1

SNMPv3 User

SNMPv3 user name. Set this parameter only when Trap Version is SNMPv3.

By default, the BMC default user is the SNMPv3 user.

Trap Mode

Mode for reporting trap information.

Values:

  • Precise Alarm (recommended): The SNMP node OID that is in one-to-one mapping with the event is used as the ID of a Trap event. Compared with OID and Event Code, this mode provides more accurate information.
  • OID: the OID of an SNMP node is used as the ID of a Trap event.
  • Event Code: The event code is used as the ID of a Trap event.

Default value: Precise Alarm (recommended)

Trap Server Identity

Source of the trap message.

Values:

  • Board serial number
  • Product asset label
  • Host name

The default value is Board serial number.

Community Name

Community string for trap authentication if SNMPv1 or SNMPv2c is used.

The value range varies depending on whether password complexity check is enabled.

  • If password complexity check is disabled, the value is a string of 1 to 32 characters consisting of letters, digits, and special characters (excluding spaces).
  • If password complexity check is enabled, the community name must meet the following requirements:
    • Contain 8 to 32 characters
    • Contain at least two of the following: uppercase letters A to Z, lowercase letters a to z, digits 0 to 9
    • Contain at least one of the following special characters: `~!@#$%^&*()-_=+\|[{}];:'",<.>/?
    • Cannot contain spaces.

Confirm Community Name

Enter the community name again for consistency.

Include Alarm Severities

Severities of alarms to be sent to a third-party server through trap messages.

Values:
  • [NULL]: No alarm or event is sent.
  • Critical: Only critical alarms are reported.
  • Major: Major and critical alarms are reported.
  • Minor: Minor, major, and critical alarms are reported.
  • Informational: Events and minor, major, and critical alarms are reported.

The default value is Informational.

Trap Server and Message Format

No.

Identifies a trap channel for sending alarms. A maximum of four channels can be specified.

Trap Server Address

Server address for receiving alarms sent through trap messages. The server address can be an IPv4 or IPv6 address or a domain name.

The domain name must meet the following requirements:
  • Contain 0 to 255 characters.
  • Allow digits, letters, hyphens (-), and dots (.) only.
  • Cannot start with a hyphen or dot, or end with a hyphen.
  • Allow a maximum of 63 characters between any two dots.

Trap Port

Port number for receiving alarms sent through trap messages.

Values: 1 to 65535

Default value: 162

NOTE:

To restore the trap port number to the default value 162, click Reset.

In-Band Transfer

Function for using the in-band channel. If in-band channel is enabled, trap messages can be transferred to the trap server via the OS.

It is disabled by default.

NOTE:
  • When the BMC fails to communicate with the trap server, enabling the in-band channel allows the OS to transfer trap messages to the trap server.
  • The in-band channel can be enabled only when BMA 2.0 has started on the OS.

Current Status

Current status of the trap channel.

It is disabled by default.

Message Delimiter
Delimiter that separates the keywords in trap messages. For example, ;.
NOTE:

Set this parameter only when Trap Mode is Event Code.

Message Content
Content to be included in trap messages.
NOTE:

Set this parameter only when Trap Mode is Event Code.

Display Keyword in Message
Specifies whether to display the specified keywords in trap messages.
NOTE:

Set this parameter only when Trap Mode is Event Code.

Format Example

An example will be provided on the right of the check box depending on whether Message Delimiter, Select Message Content, or Display Keyword in Message are selected.

Operation
  • Click Edit to modify the trap server and message format.
  • Click Test to verify whether a trap message can be successfully sent to the specified trap server. If "Operation successful" is displayed, the destination trap server is reachable.
NOTE:

If the parameters in the Trap Settings area are modified before you click Test, click Save first. Otherwise, the modification does not take effect.
