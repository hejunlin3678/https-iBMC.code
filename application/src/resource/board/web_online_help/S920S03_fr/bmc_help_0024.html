Utilisateurs locaux

Présentation de la fonction

Sur la page Utilisateurs locaux, vous pouvez consulter et gérer les utilisateurs de le BMC.

Le BMC prend en charge un maximum de 16 utilisateurs. Vous pouvez ajouter, modifier et supprimer des utilisateurs sur la page Utilisateurs locaux .

Description du paramètre

Tableau 1 Utilisateurs locaux

Paramètre

Description

Ajouter un utilisateur

Ajoute un utilisateur local.

ID

Identifie de façon unique un utilisateur du système BMC.

Nom d'utilisateur

Nom d'utilisateur pour se connecter à le BMC.

Rôle

Rôle attribué à l'utilisateur. Le rôle utilisateur spécifie les opérations qui peuvent être effectuées par l'utilisateur.

  • Administrateur: Utilisateur qui peut effectuer toutes les opérations.

    Les permissions de Administrateur ne peut pas être modifiée.

  • Opérateur: Utilisateur qui peut effectuer la gestion de base, la gestion KVM, la gestion VMM, la gestion VMM, le contrôle de l'alimentation, l'interrogation des informations et configurer ses propres mots de passe.

    Les permissions de Opérateur ne peut pas être modifiée.

  • Utilisateur commun: Utilisateur qui peut interroger des informations et configurer ses propres mots de passe.

    Les permissions de Utilisateur commun ne peut pas être modifiée.

  • Rôle personnalisé 1 à 4: Utilisateur qui peut effectuer les opérations spécifiées.
  • Sans accès: Utilisateurs assignés Sans accès Le rôle ne peut effectuer aucune opération.

Interfaces de connexion

Interfaces via lesquelles l'utilisateur peut se connecter à le BMC.

  • SNMP: L'utilisateur peut utiliser un outil SNMP (tel que le navigateur MIB) pour se connecter à le BMC.
  • SSH : L'utilisateur peut utiliser un outil SSH (tel que PuTTY) pour se connecter à la CLI d'BMC.
  • IPMI : L'utilisateur peut utiliser un outil IPMI (tel que IPMItool) pour se connecter à la CLI d'BMC.
  • Local : L'utilisateur peut utiliser le port série du serveur pour se connecter à la CLI d'BMC.
  • SFTP : L'utilisateur peut utiliser un outil SFTP (tel que Xftp) pour se connecter au système de fichiers BMC.
  • Web : L'utilisateur peut utiliser un navigateur Web pour se connecter à l'interface utilisateur Web BMC.
  • Redfish : L'utilisateur peut utiliser un outil Redfish pour se connecter à le BMC.

Opération
  • Cliquez sur Modifier pour modifier l'utilisateur local.
  • Cliquez sur Désactiver pour désactiver l'utilisateur.
  • Cliquez sur Activer pour activer l'utilisateur.
  • Cliquez sur Supprimer pour supprimer l'utilisateur.
    REMARQUE:
    • Tous les utilisateurs locaux, y compris les administrateurs, les opérateurs, les utilisateurs communs et les utilisateurs personnalisés, peuvent être supprimés.
    • Si le BMC possède plusieurs administrateurs activés, les rôles des utilisateurs par défaut peuvent être modifiés. S'il n'y a qu'un seul administrateur activé, cet administrateur ne peut pas être désactivé ou supprimé et le rôle d'administrateur ne peut pas être modifié.
    • Si la Gestion des utilisateurs du système d'exploitation est activée sur la page Utilisateurs et sécurité > Gestion de la sécurité, vous pouvez également ajouter des utilisateurs BMC en envoyant des commandes IPMI standard à partir du système d'exploitation.

Ajout d'utilisateurs

Vous pouvez ajouter un maximum de 15 utilisateurs pour le BMC.

  1. Cliquez sur Ajouter un utilisateur.

    La page permettant d'ajouter un utilisateur est affichée.

  2. Définir les paramètres utilisateur. Pour plus de détails sur les paramètres, voir Tableau 3

    • L'utilisateur avec l'ID 1 est un utilisateur réservé défini dans la norme IPMI. Cet utilisateur n'est pas autorisé à se connecter à le BMC.
    • L'utilisateur avec l'ID 2 est l'utilisateur par défaut.
    Tableau 3 Paramètres pour ajouter un utilisateur

    Paramètre

    Description

    ID utilisateur

    ID de l'utilisateur à ajouter.

    Plage de valeurs : 3 à 17

    Nom d'utilisateur

    Nom de l'utilisateur à ajouter.

    Valeur : une chaîne de 1 à 16 caractères

    Le mot de passe doit répondre aux exigences suivantes :

    • Contient des lettres, des chiffres et des caractères spéciaux (sauf :<>&,'"/\%).
    • Ne peut pas contenir d'espaces ou commencer par #,+ ou -.
    • Ne peut pas être un point (.) ou deux points consécutifs (..).

    Mot de passe

    Mot de passe pour se connecter à le BMC.

    Pour des raisons de sécurité, activez la vérification de la complexité du mot de passe et modifiez périodiquement votre mot de passe.

    REMARQUE:
    • Seuls les administrateurs peuvent activer ou désactiver la vérification de la complexité du mot de passe.
    • Désactiver la vérification de la complexité du mot de passe pourrait poser des risques de sécurité. Activez cette fonction si possible.

    Valeur :

    • Si la vérification de la complexité du mot de passe est désactivée, le mot de passe peut contenir un maximum de 20 caractères, y compris des chiffres, des lettres et des caractères spéciaux. Si le mot de passe contient moins de huit caractères, l'utilisateur ne peut pas utiliser l'interface SNMPv3.
    • Si la vérification de la complexité du mot de passe est activée, le mot de passe doit répondre aux exigences suivantes :
      • Contient de 8 à 20 caractères
      • Contient au moins un espace ou l'un des caractères spéciaux suivants :

        `~!@#$%^&*()-_=+\|[{}];:'",<.>/?

      • Contient au moins deux types des caractères suivants :
        • Lettres majuscules de A à Z
        • Lettres minuscules de a à z
        • Chiffres 0 à 9
      • Ne peut pas être le même que le nom d'utilisateur ou le nom d'utilisateur dans l'ordre inverse.
    • Si la vérification du mot de passe faible est activée, le mot de passe ne peut pas être identique à celui contenu dans le dictionnaire de mots de passe faibles. (Vous pouvez exécuter la commande ipmcset -t user -d weakpwddic -v export pour exporter les mots de passe faibles du dictionnaire de mots de passe faible.)
    REMARQUE:

    Pour des raisons de sécurité, n'utilisez pas de mots de passe composés uniquement de chaînes répétées, telles qu'aa, abababab ou abcdabcd.

    Valeur par défaut : identique au mot de passe de connexion de l'utilisateur.

Il est conseillé de définir le mot de passe de cryptage SNMPv3 à des fins de sécurité. Si vous définissez un mot de passe de cryptage SNMPv3, ce mot de passe ne sera pas synchronisé avec le mot de passe de connexion de l'utilisateur.

Suppression d'un utilisateur

  1. Dans la liste des utilisateurs locaux, localisez l'utilisateur à supprimer et cliquez sur Supprimer.

    Une boîte de dialogue de confirmation s'affiche, vous invitant à saisir le mot de passe de l'utilisateur actuel.

  2. Saisissez le mot de passe de l'utilisateur actuel et cliquez sur Confirmer.

    L'utilisateur est supprimé de la liste des utilisateurs.

Importation d'une clé publique SSH

  • Une fois qu'une clé privée est créée sur un client, importez la clé publique correspondante dans le BMC pour garantir l'accès sécurisé des utilisateurs SSH à le BMC.

  • Chaque utilisateur ne peut importer qu'une seule clé publique. Pour modifier la clé publique, supprimez la clé publique existante, puis importez-en une nouvelle.

  • Les clés publiques peuvent être au format RFC 4716 ou OpenSSH. Le type de clé publique est RSA ou DSA. Une clé RSA contient 2048 ou 4096 bits, et une clé DSA contient 2048 bits.

  • Le fichier de clé publique est au format .pub. La taille ne peut pas dépasser 2 Ko.
  1. Localisez l'utilisateur pour lequel la clé publique SSH doit être importée, et cliquez sur à gauche du nom d'utilisateur.
  2. Cliquez sur Charger à côté de Clé publique SSH.
  3. Sélectionnez le mode d'importation.

    Sélectionnez Fichier de clé publique ou Texte de clé publique.

  4. Cliquez sur Ajouter le fichier pour sélectionner la clé publique.
  5. Entrez le mot de passe de l'utilisateur actuel.
  6. Cliquez sur Enregistrer.
