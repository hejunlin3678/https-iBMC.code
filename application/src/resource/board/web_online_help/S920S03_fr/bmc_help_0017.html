Paramètres d'alarme

Présentation de la fonction

Sur la page Paramètres d'alarme, vous pouvez configurer :

  • Notification Syslog : permet d'envoyer les journaux à un serveur tiers via des messages syslog.
  • Notifications par courriel : permet d'envoyer un courriel aux boîtes aux lettres spécifiées via un serveur SMTP (Simple Mail Transfer Protocol) lorsqu'une alarme est générée.
  • Notification Trap : permet d'envoyer des alarmes, des événements et des propriétés de trap à un serveur tiers via des messages trap.

    Les traps sont des messages envoyés par BMC à un serveur tiers sans être explicitement demandés. Les traps sont utilisés pour signaler les événements et les alarmes critiques, majeures et mineures.

Description du paramètre

L'activation de TCP ou UDP compromet la sécurité du système. Procédez avec prudence.

Tableau 1 Notification Syslog

Paramètre

Description

Notification Syslog

Fonction Syslog

Fonction d'envoi de notifications via les messages syslog.

Il est désactivé par défaut.

Inclure les gravités d'alarme

Gravités des alarmes à envoyer via les messages syslog.

Valeur :
  • [NULL] : Aucune alarme ou événement n'est signalé.
  • Critique : Seules les alarmes critiques sont signalées.
  • Majeur : Les alarmes majeures et critiques sont signalées.
  • Mineur : Les alarmes mineures, majeures et critiques sont signalées.
  • Informationnel : Les événements et les alarmes mineures, majeures et critiques sont signalés.

La valeur par défaut est Informationnel.

Format du message Syslog

Format des messages syslog.

  • Personnalisé : Les messages syslog contiennent la priorité du message, le nom du produit, l'identité du serveur syslog, la localisation du périphérique et le type de journal.

  • RFC3164 : Les messages syslog sont conformes à RFC3164 et contiennent la priorité du message, l'horodatage, le nom d'hôte, le nom du processus et le type de journal.

La valeur par défaut est Personnalisé.

REMARQUE:

Si RFC3164 est sélectionné, Identité du Serveur Syslog ne peut pas être défini et Nom d'hôte est utilisé par défaut.

Identité du serveur Syslog

Source du message syslog.

Valeur :

  • Numéro de série de la carte
  • Étiquette de l'actif du produit
  • Nom d'hôte

La valeur par défaut est Numéro de série de la carte.

Protocole de transmission

Protocole utilisé pour transmettre les messages syslog entre le BMC et le serveur syslog.

Valeur :

  • TLS (Transport Layer Security): un protocole de connexion orientée qui assure la confidentialité et l'intégrité des données transmises.
  • TCP : un protocole de connexion orientée qui envoie des données uniquement après la mise en place d'une connexion fiable entre l'expéditeur et le destinataire.
  • UDP : un protocole sans connexion qui envoie des données sans établir une connexion entre l'expéditeur et le destinataire.

La valeur par défaut est TLS.

Mode d'authentification

Mode d'authentification des certificats syslog. Définissez ce paramètre uniquement lorsque le Protocole de transmission est TLS.

Valeur :

  • Unidirectionnel : authentifie uniquement le certificat du serveur Syslog.
  • Bidirectionnel : authentifie les certificats du serveur syslog et aussi du client.

La valeur par défaut est Unidirectionnel.

Certificat racine du serveur

Certificat utilisé pour vérifier les messages envoyés du serveur syslog avant que la connexion soit établie.

REMARQUE:
  • Le fichier de certificat racine du serveur au format .crt, .cer ou .pem peut être importé. La taille ne peut pas dépasser 100 Ko.
  • Pour des raisons de sécurité, mettez périodiquement à jour le certificat.

Certificat Local

Lors de l'établissement d'une connexion de données, le BMC envoie un paquet portant les informations du certificat local au serveur syslog, qui en a besoin pour authentifier le client syslog (BMC).

REMARQUE:
  • Les fichiers de certificats locaux au format .pfx ou .p12 peuvent être importés. La taille ne peut pas dépasser 1 Mo.
  • Pour des raisons de sécurité, mettez périodiquement à jour le certificat.

Informations sur le certificat du serveur

Informations sur le certificat racine du serveur téléchargé. Les informations du certificat comprennent les éléments suivants :

  • Autorité qui a délivré le certificat racine.
  • Utilisateur auquel le certificat racine a été émis.
  • Période de validité du certificat racine.
  • Numéro de série du certificat racine.
  • Liste des certificats révoqués.
  • Période de validité de la CRL.
REMARQUE:
  • La CRL indique l'état de révocation du certificat.
    • Configuré : le fichier de révocation de certificat a été uploadé et une vérification de révocation de certificat sera effectuée lorsqu'une connexion TLS est établie.
    • Non configuré : le fichier de révocation de certificat n'est pas uploadé.
  • Le fichier de révocation de certificat codé en Base64 est au format *.crl et ne peut pas dépasser 100 Ko.
  • Si la CRL expire, la fonction d'authentification correspondante sera indisponible.

Pour définir la CRL, cliquez sur Charger, sélectionnez le fichier de révocation de certificat à partir du client. Cliquez sur Supprimer pour supprimer la CRL.

REMARQUE:

La suppression d'une CRL peut entraîner des risques de sécurité liés à l'utilisation de certificats expirés.

Serveur Syslog et format de message

No.

Canal d'envoi de messages syslog. Un maximum de quatre canaux peut être réglé.

Adresse du serveur

Adresse du serveur syslog.

Valeur : Adresse IPv4, adresse IPv6 ou nom de domaine

REMARQUE:
  • Saisissez un nom de domaine si le Protocole de transmission est TLS. De plus, les informations DNS doivent avoir été correctement configurées sur Paramètres BMC > Réseau.
  • Si l'adresse IP du nom de domaine change, désactivez manuellement le service syslog et activez-le à nouveau..
  • Le nom de domaine doit répondre aux exigences suivantes :
    • Une chaîne de 0 à 255 caractères.
    • Seulement les chiffres, les lettres, les traits d'union (-) et les points (.) sont autorisés.
    • Ne peut pas commencer par un trait d'union ou un point, ni se terminer par un trait d'union.
    • Un maximum de 63 caractères sont autorisé entre deux points.

Port

Numéro de port du serveur syslog.

Plage de valeurs : 1 à 65535

Type de journal

Types de journaux rapportés via les messages syslog.

Valeur :

  • Journaux des opérations
  • Journaux de sécurité
  • Événement

Une ou plusieurs valeurs peuvent être sélectionnées.

Toutes les options sont sélectionnées par défaut.

État actuel

Paramétrage de la notification syslog du canal, qui peut être activé ou désactivé.

Il est désactivé par défaut.

Opération
  • Cliquez sur Modifier pour modifier le serveur syslog et le format des messages.
  • Cliquez sur Tester pour vérifier si un message syslog peut être envoyé avec succès au serveur syslog spécifié. Si « Opération réussie » s'affiche, le canal est disponible.
    REMARQUE:

    Si les paramètres de Notifications Syslog sont modifiés avant que vous ne cliquiez sur Test, cliquez d'abord sur Enregistrer. Sinon, la modification ne prend pas effet.

Fonction d'envoi de notifications par courrier via le serveur SMTP.

Il est désactivé par défaut.

SMTP Adresse du serveur

Adresse IPv4 ou IPv6 ou nom de domaine du serveur SMTP.

Le nom de domaine doit répondre aux exigences suivantes :

  • Seulement les chiffres, les lettres, les traits d'union (-) et les points (.) sont autorisés.

Numéro de port du serveur SMTP.

Plage de valeur : 1 à 65535

Valeur par défaut : 25

TLS activé

Fonction permettant d'activer le protocole TLS (Transport Layer Security) pour la transmission de données.

Si TLS est désactivé, les données sont transmises en texte clair.

La valeur par défaut est Oui.

REMARQUE:
  • Par défaut, le serveur SMTP prend en charge TLS. Nous vous conseillons d'activer la fonction TLS pour des raisons de sécurité.
  • Après avoir activé TLS sur l'interface utilisateur Web BMC, activez TLS et configurez l'authentification d'identité sur le serveur SMTP. Le serveur SMTP ne peut recevoir des courriels d'BMC qu'une fois TLS activé.
  • Le BMC ne prend pas en charge TLS 1.0 et TLS 1.1.

Certificat du serveur de validation

Fonction permettant d'activer ou de désactiver le certificat du serveur de validation lors de l'activation de la transmission chiffrée TLS.

Cette fonction est désactivée par défaut.

Certificat du serveur

Le certificat du serveur est utilisé pour authentifier les paquets envoyés par le serveur SMTP lors de l'établissement de la connexion.

REMARQUE:
  • Le format du fichier du certificat du serveur peut être .crt, .cer ou .pem, avec une taille maximale de 100 Ko.
  • Mettez le certificat à jour périodiquement pour éviter les risques de sécurité.

Informations sur le certificat du serveur

Fonction permettant d'afficher des informations sur le certificat du serveur téléchargé, y compris :

  • Émis par
  • Émis pour
  • Période de validité
  • Numéro de série

Information e-mail

Connexion anonyme autorisée

Fonction permettant d'autoriser la connexion anonyme.

Si la connexion anonyme est autorisée, le serveur SMTP transfère les courriels d'alarme sans nom d'utilisateur et mot de passe d'authentification.

Si la connexion anonyme n'est pas autorisée, le serveur SMTP ne transfère les e-mails d'alarme qu'après avoir saisi le nom d'utilisateur et le mot de passe corrects. Le nom d'utilisateur et le mot de passe doivent avoir été définis sur le serveur SMTP.

La valeur par défaut est Non.

REMARQUE:

Par défaut, le serveur SMTP n'autorise pas la connexion anonyme. Pour des raisons de sécurité, n'utilisez pas la connexion anonyme.

Nom d'utilisateur de l'expéditeur / Mot de passe de l'expéditeur

Nom d'utilisateur et mot de passe utilisés lorsque l'option Connexion anonyme autorisée est réglée sur Non.

Le nom d'utilisateur et le mot de passe doivent être les mêmes que ceux définis sur le serveur SMTP.

Plage de valeurs :
  • Nom d'utilisateur : chaîne de 1 à 64 caractères, composée de lettres, de chiffres et de caractères spéciaux. Il ne peut pas être laissé en blanc.
  • Mot de passe : une chaîne de 1 à 50 caractères.

Adresse de l'expéditeur

Adresse électronique à partir de laquelle les alarmes sont envoyées.

Valeur : une chaîne de 255 caractères au maximum au format xx@xxx.xx.

Les lettres, les chiffres et les caractères spéciaux sont autorisés.

Objet de l'e-mail / L'objet de l'e-mail contient

Objet de l'e-mail.

Valeur : une chaîne de 0 à 255 caractères, composée de lettres, de chiffres et de caractères spéciaux.

Saisissez l'objet dans Sujet de l'e-mail et sélectionnez les mots-clés à contenir dans l'objet de l'e-mail. La valeur par défaut est Server Alert.

Par exemple, si vous sélectionnez Nom d'hôte et Numéro de série de la carte, l'objet de l'e-mail contiendra le nom d'hôte et le numéro de série de la carte.

Inclure les gravités d'alarme

Gravité des alarmes à envoyer via le serveur SMTP.

Valeur :
  • [NULL] : Aucune alarme ou événement n'est signalé.
  • Critique : Seules les alarmes critiques sont signalées.
  • Majeur : Les alarmes majeures et critiques sont signalées.
  • Mineur : Les alarmes mineures, majeures et critiques sont signalées.
  • Informationnel : Les événements et les alarmes mineures, majeures et critiques sont signalés.

La valeur par défaut est Informationnel.

Adresses des destinataires

N°

Canal pour l'envoi d'e-mails d'alarme. Un maximum de quatre canaux peut être défini.

Adresse e-mail

Adresses de courriel pour recevoir des courriels. Les adresses doivent avoir été définies sur le serveur SMTP.

Valeur : une chaîne de 255 caractères au maximum au format xx@xxx.xx.

Les lettres, les chiffres et les caractères spéciaux sont autorisés.

Description

Informations supplémentaires sur les adresses du courrier pour recevoir des courriers e-mails.

Valeur : une chaîne de 0 à 255 caractères, composée de lettres, de chiffres et de caractères spéciaux.

Envoyer un message

Fonction permettant de tester si un e-mail peut être envoyé avec succès au destinataire.

Il est désactivé par défaut.

Opération
  • Cliquez sur Modifier pour modifier l'adresse du destinataire.
  • Cliquez sur Test pour vérifier si un e-mail peut être envoyé avec succès à la boîte aux lettres spécifiée.
    REMARQUE:

    Si les paramètres de SMTP Fonction sont modifiés avant que vous ne cliquiez sur Test, cliquez d'abord sur Enregistrer. Sinon, la modification ne prend pas effet.

Paramètre

Description

Trap Fonction

Fonction Trap

Fonction d'envoi d'alarmes via les messages traps.

Il est activé par défaut.

Trap Version

Version SNMP utilisée pour l'envoi de traps.

Valeur :

  • SNMPv1 : la première version officielle SNMP définie dans le Request for Comments (RFC) 1157.
  • SNMPv2c : une version a ajouté l'architecture de gestion basée sur la communauté à SNMPv2.
  • SNMPv3 : une version a ajouté des améliorations de sécurité et de configuration à distance à SNMP.
REMARQUE:
  • L'option SNMPv3 trap est recommandé. Soyez prudent lorsque vous utilisez SNMPv1 et SNMPv2c, car ils présentent des risques pour la sécurité.
  • Vous pouvez définir les algorithmes d'authentification et de cryptage SNMPv3 dans Utilisateur et sécurité > Utilisateurs locaux sur l'interface utilisateur Web.

Valeur par défaut : SNMPv3

Utilisateur SNMPv3

Nom d'utilisateur de SNMPv3. Ne définissez ce paramètre que Version de l'interruption est SNMPv3.

Par défaut, l'utilisateur BMC par défaut est l'utilisateur SNMPv3.

Trap Mode

Mode de rapport des informations traps.

Valeur :

  • Alarme précise (recommandée) : L'OID du nud SNMP qui est en mappage un à un avec l'événement est utilisé comme ID d'un événement Trap. Comparé à l'OID et au Code d'événement, ce mode fournit des informations plus précises.
  • OID Mode : l'OID d'un nud SNMP est utilisé comme ID d'un événement Trap.
  • Code d'événement : Le code d'événement est utilisé comme ID d'un événement Trap.

Valeur par défaut : Alarme précise (recommandée)

Identité du serveur d'interruptions

Source du message Trap.

Valeur :

  • Numéro de série de la carte
  • Étiquette de l'actif du produit
  • Nom d'hôte

Valeur par défaut : Numéro de série de la carte

Nom de la communauté

La chaîne de communauté pour l'authentification des traps si SNMPv1 ou SNMPv2c est utilisé.

La plage de valeurs varie selon que la vérification de la complexité du mot de passe est activée ou non.

  • Si la vérification de la complexité du mot de passe est désactivée, la valeur est une chaîne de 1 à 32 caractères, composée de lettres, de chiffres et de caractères spéciaux (espaces non compris).
  • Si la vérification de la complexité du mot de passe est activée, le nom de la communauté doit répondre aux exigences suivantes :
    • Contient de 8 à 32 caractères
    • Contient au moins deux des éléments suivants : lettres majuscules de A à Z, lettres minuscules de a à z, chiffres de 0 à 9.
    • Contient au moins un des caractères spéciaux suivants : `~!@#$%^&*()-_=+\|[{}];:'",<.>/?
    • Ne peut pas contenir d'espaces.

Confirmer le nom de la communauté

Saisissez à nouveau le nom de la communauté par souci de cohérence.

Inclure les gravités d'alarme

Gravités des alarmes à envoyer à un serveur tiers par le biais de messages d'interruption.

Valeur :
  • [NULL] : Aucune alarme ou événement n'est signalé.
  • Critique : Seules les alarmes critiques sont signalées.
  • Majeur : Les alarmes majeures et critiques sont signalées.
  • Mineur : Les alarmes mineures, majeures et critiques sont signalées.
  • Informationnel : Les événements et les alarmes mineures, majeures et critiques sont signalés.

La valeur par défaut est Informationnel.

Serveur d'interruptions et format de message

No.

Il identifie un canal d'interruption pour l'envoi d'alarmes. Un maximum de quatre canaux peut être spécifié.

Trap Adresse du serveur

Adresse du serveur pour la réception des alarmes envoyées via des messages d'interruption. L'adresse du serveur peut être une adresse IPv4 ou IPv6 ou un nom de domaine.

Le nom de domaine doit répondre aux exigences suivantes :
  • Une chaîne de 0 à 255 caractères.
  • Seulement les chiffres, les lettres, les traits d'union (-) et les points (.) sont autorisés.
  • Ne peut pas commencer par un trait d'union ou un point, ni se terminer par un trait d'union.
  • Un maximum de 63 caractères sont autorisé entre deux points.

Trap Port

Numéro de port pour la réception des alarmes envoyées via des messages traps.

Plage de valeurs : 1 à 65535

Valeur par défaut : 162

REMARQUE:

Pour restaurer le numéro de port trap à la valeur par défaut 162, cliquez sur Réinitialiser.

Canal intrabande

Fonction pour l'utilisation du canal intrabande. Si le canal intrabande est activé, les messages traps peuvent être transférés au serveur Trap via le système d'exploitation.

Il est désactivé par défaut.

REMARQUE:
  • Lorsque le BMC ne parvient pas à communiquer avec le serveur Trap, l'activation du canal intrabande permet au système d'exploitation de transférer des messages trap au serveur d'interruption.
  • Le canal intrabande ne peut être activé que lorsque le BMA 2.0 a démarré sur le système d'exploitation.

Statut actuel

État actuel du canal de traps.

Il est désactivé par défaut.

Délimiteur de message

Délimiteur qui sépare les mots-clés dans les messages de traps. Par exemple, ; .

REMARQUE:

Ne réglez ce paramètre que lorsque le mode de trap est Code d'événement.

Contenu du message

Contenu à inclure dans les messages d'interruption.

REMARQUE:

Ne réglez ce paramètre que lorsque le mode de trap est Code d'événement.

Afficher les mots-clés

Indique si les mots-clés spécifiés doivent être affichés dans les messages d'interruption.

REMARQUE:

Ne réglez ce paramètre que lorsque le mode de trap est Code d'événement.

Exemple de cas

En fonction de l'option Délimiteur de message, Contenu du message ou Afficher le mot-clé dans le message soit sélectionnée, un exemple sera fourni à droite de la case à cocher.

Opération
  • Cliquez sur Modifier pour modifier le serveur d'interruption et le format de message.
  • Cliquez sur Test pour vérifier si un message d'interruption peut être envoyé avec succès au serveur d'interruption spécifié. Si « Opération réussie » s'affiche, le canal est disponible.
    REMARQUE:

    Si les paramètres de Trap Fonction (Paramètres Trap) sont modifiés avant que vous ne cliquiez sur Test, cliquez d'abord sur Enregistrer. Sinon, la modification ne prend pas effet.
