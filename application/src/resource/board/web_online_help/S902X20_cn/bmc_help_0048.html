无法启动Java集成远程控制台

问题现象

问题描述

可能原因

无法启动Java集成远程控制台。
  • 没有正确安装JRE。
  • JRE版本与BMC不兼容。

解决方案

  1. 确认客户端JRE已正确安装。

    BMC支持的JRE版本为：AdoptOpenJDK 8 JRE和AdoptOpenJDK 11 JRE。

    • 若JRE版本正确，请联系技术支持定位。
    • 若JRE版本不正确，执行2。

  2. 从AdoptOpenJDK官网下载适配客户端OS的JRE二进制压缩包。
  3. 安装AdoptOpenJDK。

    • 压缩包解压后需要手动配置JAVA_HOME及PATH环境变量。
    • 需要从AdoptOpenJDK官网额外下载IcedTea Web并解压，然后将bin文件夹配置进PATH环境变量。

  4. 登录BMC WebUI，在“首页”的“虚拟控制台”区域，单击“启动虚拟控制台 > Java集成远程控制台(独占)或Java集成远程控制台(共享)”。

    在此过程中，会自动下载.jnlp文件。

  5. 打开.jnlp文件。

    .jnlp文件具有时效性，如果无法使用请参见4重新下载.jnlp文件。

    • 客户端使用Linux命令行或Windows命令行操作时，请切换至.jnlp文件所在目录，运行javaws kvm.jnlp。
    • 客户端使用图形界面操作时，找到下载的.jnlp文件后，右键选择javaws打开。（若右键菜单无javaws，可至IcedTea Web安装目录中的bin目录下查找。）
