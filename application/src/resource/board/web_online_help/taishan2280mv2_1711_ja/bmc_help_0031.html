ポートサービス

機能説明

ポートサービス画面では、BMCでサポートされているポートサービスを変更できます。

  • Web Server（HTTP）/Web Server（HTTPS）ポートが非デフォルトのブラウザポートとして設定されている場合、ChromeまたはFirefoxブラウザではこのポートを介して接続を確立できません。この問題を解決するには、非デフォルトポートの接続を確立できるようにブラウザを設定する必要があります。
  • SSH、HTTPS、RMCP、RMCP+サービスを同時に無効にすると、ネットワークが切断される可能性があります。すべてのサービスが無効になっている場合は、シリアルポートを介してサーバーに接続し、Webサービスを有効にしてください。

これにより、ユーザーがビデオ再生機能を使用できます。この機能の詳細については、ビデオスクリーンショットを参照してください。

5900

VNC（Virtual Network Console）により、ローカルキーボード、ビデオ、マウスを使用してサーバーをリモートで制御できます。

WEB_HTTP

80

HTTPはインターネットの閲覧をサポートし、HTTP（Hypertext Transfer Protocol）ページを解析できます。ブラウザとBMC間の接続を確立するために、HTTPサービスはデフォルトで有効になっています。接続を確立した後、セキュアプロトコルHTTPSが使用されます。

デフォルトで有効になっています。

WEB_HTTPS

443

HTTPSはインターネットの閲覧をサポートし、HTTPS（Hypertext Transfer Protocol over Secure Socket Layer）ページまたはRedfishプロトコルを解析できます。

BMCは最大4人のユーザーの同時HTTPS接続をサポートします。

デフォルトで有効になっています。

IPMI LAN (RMCP)

ポート1（プライマリポート）は623、ポート2（セカンダリポート）は664です。

LAN上のIPMI（Intelligent Platform Management Interface）を表し、RMCP（Remote Management Control Protocol）をサポートします。

IPMI LAN（RMCP）サービスを使用する場合、セキュリティリスクが発生する可能性があります。セキュリティを確保するために、代わりにIPMI LAN（RMCP+）サービスを使用してください。

デフォルトでは無効になっています。

IPMI LAN (RMCP+)

RMCP+とRMCPは同じポートを使用します。

LAN上のIPMIを表し、RMCP+をサポートします。

デフォルトで有効になっています。

NOTE:

RMCP+プロトコルには脆弱性（CVE-2013-4786）があり、セキュリティ上のリスクが存在しています。リスク防止対策を参照してください。

ポートサービスの設定

  1. 編集をクリックします。
  2. サービス要件に基づいてポートサービスを有効または無効にします。

    • :サービスを有効にします。
    • :サービスを無効にします。

  3. これらのサービスのポート番号を設定します。

    ポートを編集した後、新しい設定が有効になるまで約10秒かかります。設定が有効になるまで、重複して編集しないでください。

  4. 保存をクリックします。

リスク防止対策

RMCP +の脆弱性（CVE-2013-4786）に起因するセキュリティリスクを最小限に抑えるために、次の手順を実行します。

  • BMCへのアクセスにIPMIプロトコルを使用しない場合
    • このページでIPMIサービスを無効にします。

      IPMIサービスを無効にすると、他のデバイスがIPMIを使ってBMCにアクセスできなくなります。この設定は、IPMItool、InfoCollect、eSightなどのIPMIベースのツールに影響します。

    • パスワードの複雑度検証機能を有効にして、パスワードの複雑度要件に準拠したパスワードを設定します。
  • BMCへのアクセスにIPMIプロトコルを使用する場合
    • BMC管理ネットワークポートが配置されているネットワークを独立したLANとして設定します。
    • パスワードの複雑度検証機能を有効にして、パスワードの複雑度要件に準拠したパスワードを設定します。
