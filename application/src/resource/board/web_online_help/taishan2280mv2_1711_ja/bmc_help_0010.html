ストレージ管理

機能説明

[ストレージ管理]画面では、サーバーのストレージデバイスを照会・構成できます。

  • RAIDコントローラー、論理ドライブ、物理ドライブの情報は、RAIDカードのアウトオブバンド管理機能に基づいており、BIOSブートが完了した後、またはBMA 2.0がインストールおよび起動された後にのみ表示されます。
  • システムの電源が切断されているか、システムが起動していない場合、ストレージ管理に関する情報は無効です。サーバーの電源が投入され、システムが起動するたびに、BMCはすべての物理ディスクを再度識別します。物理ディスクが再構築されている場合、物理ディスクは識別されません。物理ディスクが識別される前に、物理ディスク情報が無効です。物理ディスクの識別に失敗した場合、該当するセンサー（DISKN）はリンクに異常があることを示すアラームを生成します。
  • すべてのハードディスクを識別し、表示するのにかかる時間は論理ディスクと物理ディスクの数によって異なります。論理ディスクと物理ディスクの数が多いほど、時間がかかります。

NOTE:
  • RAIDコントローラカードがアウトオブバンド管理に対応せず、BMA 2.0がインストールされていない場合、RAIDコントローラカードにマウントされている物理ディスクでは、インターフェースタイプの情報しか表示されません。
  • パススルーディスクの場合は、正常性状態、ロケーション状態とインターフェースタイプのみが表示されます。インターフェースタイプはSAS/SATAとなります。

  • WindowsまたはVMware OSは、NVMeディスクの速度ネゴシエーションに対応しません。そのため、サーバーがWindowsまたはVMware OSを使用している場合、NVMeディスクのネゴシエーション速度はNAです。
  • 残り磨耗率は、SSDの寿命を示します。残り磨耗率の値が大きいほど、ディスクの摩耗具合が小さく、使用寿命が長いです。たとえば、残り磨耗率は100%である場合、ハードディスクに摩耗がありません。
  • BMCはM.2ドライブの位置情報を提供していません。
  • 表示される物理ドライブ情報は、RAIDコントローラカードによって異なります。UIで実際の表示を確認してください。

RAIDコントローラカードのプロパティの照会

この操作を実行する前に、以下の要件を満たす必要があります。

  • RAIDコントローラカードがBMCのアウトオブバンド管理をサポートしているか、またはBMA 2.0がOS上でインストール・実行されています。RAIDコントローラカードがBMCのアウトオブバンド管理に対応するかどうかを確認するには、RAIDコントローラカードのユーザーガイドの技術仕様に関する内容を参照してください。
  • BIOSが起動しました。
  1. [システム] > [ストレージ管理]の順に選択します。
  2. 照会するRAIDコントローラを選択します。

RAIDアレイのプロパティの照会

この操作を実行する前に、以下の要件を満たす必要があります。

  1. [システム] > [ストレージ管理]の順に選択します。
  2. 照会するRAIDアレイを選択します。

ハードディスクのプロパティの照会

この操作を実行する前に、以下の要件を満たす必要があります。

  • ハードディスクはRAIDコントローラカードで管理されています。
  • RAIDコントローラカードがBMCのアウトオブバンド管理をサポートしているか、またはBMA 2.0がOS上でインストール・実行されています。RAIDコントローラカードがBMCのアウトオブバンド管理に対応するかどうかを確認するには、RAIDコントローラカードのユーザーガイドの技術仕様に関する内容を参照してください。

  1. [システム] > [ストレージ管理]の順に選択します。
  2. 照会するハードディスクを選択します。ハードディスクは、RAIDアレイのメンバーディスクまたは独立したハードディスクにすることができます。

RAIDコントローラカードのプロパティの変更

この操作を実行する前に、以下の要件を満たす必要があります。

  • RAIDコントローラカードはBMCのアウトオブバンド管理をサポートします。RAIDコントローラカードがBMCのアウトオブバンド管理に対応するかどうかを確認するには、RAIDコントローラカードのユーザーガイドの技術仕様に関する内容を参照してください。

    コントローラにSMART（Self-Monitoring Analysis and Reporting Technology）エラーを検出した際に、コピーバック操作を実行します。

    JBOD（Just a bunch of disks）では、論理ディスクを構成することなく、コマンドをRAIDコントローラから接続されたハードディスクに直接転送できます。この機能により、上位層のサービスまたは管理ソフトウェアが物理ディスクにアクセスして制御できます。

    [設定復元]をクリックすると、RAIDコントローラのプロパティを復元できます。

    外部構成情報のクリア

    [外部構成情報のクリア]をクリックすると、外部ディスクのRAID構成情報をクリアできます。

    ドライブライトキャッシュポリシー

    RAIDコントローラカード配下のドライブのライトキャッシュポリシーを設定します。

    • UnconfiguredDrive：RAID/Mixedモードでの非RAIDドライブのライトキャッシュポリシーを設定します。
    • HBADrive：HBAモードのドライブでのライトキャッシュポリシーを設定します。

    ライトキャッシュポリシーには次のものがあります。

    • Enabled：ドライブライトキャッシュ機能を有効にします。
    • Disabled：ドライブライトキャッシュ機能を無効にします。
    • Default：ドライブのライトキャッシュ機能を工場出荷時の設定に戻します。

    ノーバッテリーライトキャッシュモード

    スーパーキャパシタが構成されていない場合、またはスーパーキャパシタが完全に充電されている場合に、ライトキャッシュ機能を有効にします。

    リードキャッシュ使用率

    リードキャッシュパーセンテージを設定します。値の範囲は、0～100です。

    整合性検証

    RAIDコントローラカードの整合性検証機能を設定します。

    • 実行状態：整合性検証の実行状態。
    • 自動修復：整合性検証の自動修復機能。
    • 検証レート：整合性検証の速度。
    • 検証間隔：整合性検証がトリガーされる間隔。
    • タスクの待機時間：整合性検証タスクが開始されるまでの待ち時間。

RAIDコントローラカードのログの収集

RAIDカードのログを収集するには、以下の条件を満たす必要があります。

  • RAIDコントローラカードがHi1880/Ariesタイプであること。
  • 帯域外機能が正常に動作していること。
  • BIOSブートが完了したこと。
  1. 上部のナビゲーションメニューから、[システム] > [ストレージ管理]を選択します。
  2. 対象RAIDコントローラカードを選択します。
  3. [ログ収集]をクリックします。

論理ディスクの追加

この操作を実行する前に、以下の要件を満たす必要があります。

  • ハードディスクは、論理ディスクの作成をサポートするRAIDコントローラカードによって管理されます。
  • 論理ディスクとして追加される物理ディスクは、UNCONFIGURED GOOD状態です。
  • RAIDコントローラカードはBMCのアウトオブバンド管理をサポートします。RAIDコントローラカードがBMCのアウトオブバンド管理に対応するかどうかを確認するには、RAIDコントローラカードのユーザーガイドの技術仕様に関する内容を参照してください。
  • RAIDカードの論理ディスク数が上限に達していません。
  • BIOSが起動しました。
  1. [システム] > [ストレージ管理]の順に選択します。
  2. RAIDコントローラを選択します。
  3. [追加]をクリックします。
  4. 表 3にパラメータの説明を示します。

    表 3 パラメータの説明

    カテゴリ

    説明

    名前

    論理ディスクの名称。

    L2キャッシュ

    CacheCadeの有効化/無効化。

    ストライプサイズ

    各物理ディスク上のデータストライプのサイズ。

    読み込みポリシー

    論理ディスクのデータ読み込みポリシー。

    値は以下のとおりです。

    • Read Ahead: RAIDコントローラは、シーケンシャルデータまたは使用予定のデータを先読みし、キャッシュに保存します。
    • No Read Ahead: 先読み機能を無効にします。
    NOTE:

    デフォルトの読み込みポリシーは、実際に使用するRAIDコントローラによって異なります。

    書き込みポリシー

    論理ディスクのデータ書き込みポリシー。

    値は以下のとおりです。

    • Write Through: ディスクがすべてのデータを受信した後、コントローラは、データの送信が完了したことを示すメッセージをホストに送信します。
    • Write Back with BBU: BBUが構成されていないか、または構成されたBBUに障害が発生した場合、RAIDコントローラは自動的にWrite Throughモードに切り替わります。
    • Write Back: コントローラキャッシュがすべてのデータを受信した後、コントローラは、データの送信が完了したことを示すメッセージをホストに送信します。
    NOTE:

    デフォルトの書き込みポリシーは、実際に使用するRAIDコントローラによって異なります。

    IOポリシー

    特殊な論理ディスクからデータを読み込むためのIOポリシー。このポリシーは、キャッシュの先読みに影響しません。

    値は以下のとおりです。

    • Cached IO: すべての読み込み/書き込み要求は、RAIDコントローラのキャッシュによって処理されます。CacheCade 1.1が設定されている場合にのみ、この値を選択します。
    • Direct IO: この値は、読み込みシナリオと書き込みシナリオで異なる意味を持ちます。
      • 読み込みシナリオでは、データは物理ディスクから直接読み込まれます（読み込みポリシーがRead Aheadに設定されている場合、データの読み込み要求はRAIDコントローラのキャッシュによって処理されます）。
      • 書き込みシナリオでは、データ書き込み要求はRAIDコントローラのキャッシュによって処理されます（書き込みポリシー がWrite Throughに設定されている場合、データは物理ディスクに直接書き込まれます）。

    ディスクキャッシュポリシー

    ディスクキャッシュのステータス。値は以下のとおりです。

    • Enable:データをハードディスクに書き込む前に、キャッシュにデータを書き込みます。このオプションにより、データ書き込みのパフォーマンスが向上します。ただし、電源障害に対する保護メカニズムがない場合、データが失われます。
    • Disable:データをキャッシュすることなく、ハードディスクにデータを書き込みます。電源障害が発生した場合、データが失われません。
    • Disk's default:デフォルトのキャッシュポリシーを使用します。

    デフォルト値は[Disk's default]です。

    アクセスポリシー

    論理ディスクのアクセスポリシー。

    値は以下のとおりです。

    • Read Write: 読み込み/書き込み可能です。
    • Read Only: 論理ディスクは読み取り専用です。
    • Blocked: 論理ディスクへのアクセスが拒否されます。

    デフォルト値は[Read Write]です。

    初期化方式

    作成した論理ディスクを初期化するかどうか。

    値は以下のとおりです。

    • No Init: 初期化しません。
    • Quick Init: 論理ディスクの先頭の100MBのスペースを0にクリアします。その後、論理ディスクの状態がOptimalに変わります。
    • Full Init: 論理ディスクを初期化します。初期化が完了する前に、論理ディスクの状態がOptimalになっています。
    • RPI: オフラインでのクイック初期化です。初期化完了後、OSに初期化状態を報告します。
    • OPO: SSDで構成されるRAIDグループの作成時にOPOを有効にするかどうかを設定します。
    • Front: フロントエンドを初期化します。
    • Background: バックグラウンドを初期化します。
    NOTE:

    論理ドライブの初期化状態やデフォルト値は、RAIDコントローラによって異なります。

    デフォルトの読み込みポリシー

    論理ディスクのデータ読み込みポリシー。

    値は以下のとおりです。

    • Read Ahead: RAIDコントローラは、シーケンシャルデータまたは使用予定のデータを先読みし、キャッシュに保存します。
    • No Read Ahead: 先読み機能を無効にします。

    デフォルトの書き込みポリシー

    論理ディスクのデータ書き込みポリシー。

    値は以下のとおりです。

    • Write Through:: ディスクがすべてのデータを受信した後、コントローラは、データの送信が完了したことを示すメッセージをホストに送信します。
    • Write Back with BBU: BBUが構成されていないか、または構成されたBBUに障害が発生した場合、RAIDコントローラは自動的にWrite Throughモードに切り替わります。
    • Write Back: コントローラキャッシュがすべてのデータを受信した後、コントローラは、データの送信が完了したことを示すメッセージをホストに送信します。

    デフォルトのIOポリシー

    特殊な論理ディスクからデータを読み込むためのIOポリシー。このポリシーは、キャッシュの先読みに影響しません。

    値は以下のとおりです。

    • Cached IO: すべての読み込み/書き込み要求は、RAIDコントローラのキャッシュによって処理されます。CacheCade 1.1が設定されている場合にのみ、この値を選択します。
    • Direct IO: この値は、読み込みシナリオと書き込みシナリオで異なる意味を持ちます。
      • 読み込みシナリオでは、データは物理ディスクから直接読み込まれます（読み込みポリシーが Read Aheadに設定されている場合、データの読み込み要求はRAIDコントローラのキャッシュによって処理されます）。
      • 書き込みシナリオでは、データ書き込み要求はRAIDコントローラのキャッシュによって処理されます（ 書き込みポリシーがWrite Throughに設定されている場合、データは物理ディスクに直接書き込まれます）。

    BGI状態

    バックグラウンド初期化を有効にするかどうか。

    アクセスポリシー

    論理ディスクのアクセスポリシー。次のオプションがあります。

    値は以下のとおりです。

    • Read/Write: 読み込み/書き込み可能です。
    • Read Only: 論理ディスクは読み取り専用です。
    • Blocked: 論理ディスクへのアクセスが拒否されます。

    ディスクキャッシュポリシー

    ディスクキャッシュのステータス。値は以下のとおりです。

    • Enable:データをハードディスクに書き込む前に、キャッシュにデータを書き込みます。このオプションにより、データ書き込みのパフォーマンスが向上します。ただし、電源障害に対する保護メカニズムがない場合、データが失われます。
    • Disable:データをキャッシュすることなく、ハードディスクにデータを書き込みます。電源障害が発生した場合、データが失われません。
    • Disk's default:デフォルトのキャッシュポリシーを使用します。

    ブートデバイス

    論理ディスクをブートデバイスとして設定するかどうか。

    SSCDキャッシュ

    CacheCadeディスクをキャッシュとして使用するかどうか。

    ストライプサイズ

    各物理ドライブのデータストリップサイズです。

    高速化方法

    論理ドライブの高速化方法：

    • Controller Cache：リードキャッシュとライトキャッシュの両方を使用します。
    • IO Bypass：データ入出力は、RAIDコントローラカードによってキャッシュされることなく、RAIDグループに直接送信されます。このオプションは、論理ドライブがSSDの場合にのみ有効です。
    • None：高速化は許可されません。

    デフォルト時：None
