性能監視

機能説明

性能監視画面に次の情報が表示されます。

  • 過去1時間のCPU使用率
  • 過去1時間のメモリー使用率
  • ディスクパーティションの容量と使用率
  • すべてのネットワークポートの帯域幅使用率

パラメータの説明

表 1 性能監視

カテゴリ

説明

CPU使用率

実行中のアプリケーションが使用しているCPUリソースの割合。

NOTE:
  • CPU使用率を確認するには、BMA 2.0をインストールし、起動する必要があります。
  • サーバーにBMA 2.0をインストールしていない場合、最新のBMAドキュメントとソフトウェアパッケージを取得し、BMA 2.0をインストールしてください。

メモリー使用率

実行中のアプリケーションが使用しているメモリーリソースの割合。

NOTE:
  • メモリー使用率を確認するには、OSにBMA 2.0をインストールし、起動する必要があります。
  • サーバーにBMA 2.0をインストールしていない場合、最新のBMAドキュメントとソフトウェアパッケージを取得し、BMA 2.0をインストールしてください。

ネットワークポート帯域幅使用率

サーバーNICのすべてのポートが使用しているバンド幅の割合。

NOTE:
  • ネットワークポートの帯域幅の使用率を確認するには、OSにBMA 2.0をインストールして起動する必要があります。
  • サーバーにBMA 2.0をインストールしていない場合、最新のBMAドキュメントとソフトウェアパッケージを取得し、BMA 2.0をインストールしてください。

ディスク使用率

このエリアには、次の情報が表示されます。

  • パーティションスペース全体のうち、使用済みパーティションスペースの割合
  • ディスクパーティション情報
  • 各パーティションの容量と総使用容量
NOTE:
  • ディスク使用率を確認するには、OSにBMA 2.0をインストールして起動する必要があります。
  • サーバーにBMA 2.0をインストールしていない場合、最新のBMAドキュメントとソフトウェアパッケージを取得し、BMA 2.0をインストールしてください。

現在の値

サーバーの現在のCPU、またはメモリーの使用率。

しきい値

CPU、メモリ、ドライブまたはネットワークポートの帯域幅使用量の閾値です。実際の使用量が閾値を超えると、BMCはイベントを報告します。

値の範囲:0～100

をクリックして閾値を設定します。

をクリックして使用率曲線を更新します。

をクリックして、ネットワークポートの帯域幅使用の統計情報をクリアします。

閾値の設定

  1. ターゲットエリアで、をクリックします。

    ダイアログボックスが表示されます。

  2. 表示された値の範囲に基づいて閾値を入力します。
  3. をクリックして設定を保存します。

    閾値の設定後、をクリックして画面上の情報を更新します。
