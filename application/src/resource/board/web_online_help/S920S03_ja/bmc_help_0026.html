Kerberos

機能概要

Kerberos画面では、Kerberosの基本設定とユーザーグループ情報を表示および構成できます。

BMCはKerberosユーザーにアクセスインターフェースを提供しています。Kerberosユーザーアカウントを使用してBMCにログインすることにより、システムのセキュリティを向上できます。KerberosユーザーとしてBMC WebUIにログインできます。

基本設定

NOTE:

アスタリスク(*)付きのパラメータは必須です。

レルム

Kerberosレルム。

値：255桁以内の文字列

値には、数字、英字（大文字推奨）、特殊文字（スペースを含む）を含めることができます。

Kerberosサーバーアドレス

KerberosサーバーのIPアドレス。

Kerberosが有効になっている場合は、KerberosサーバーのFQDN（ホスト名.ドメイン名）を入力し、BMC設定 > ネットワーク設定画面でDNSを設定します。

フォーマット:IPv4アドレス、IPv6アドレス、またはドメイン名

ドメイン名は次の要件を満たす必要があります。
  • 0～255文字が含まれます。

  • 数字、英字、ハイフン（-）、およびピリオド（.）のみが使用できます。

  • ドメイン名の最初にハイフンまたはピリオドを使ったり、ドメイン名の最後にハイフンを使ったりすることはできません。

  • 2つのピリオドの間に最大63文字が使用できます。

Kerberosポート

Kerberosサービスのポート番号。

値の範囲:1～65535

デフォルト値:88

キーリスト

アップロードするKerberosの鍵リスト。

.keytab形式のみがサポートされています。

デフォルトでは空白になっています。

NOTE:
  • 鍵リストのサイズは、0KBまたは1MBを超えることはできません。
  • セキュリティ上の理由から、定期的にキーを更新してください。

ログインパスワード

BMCを現在使用しているユーザーのログインパスワード。

Kerberosユーザーグループ

すべてのKerberosユーザーグループに関する情報が表示されます。

BMCは最大5つのKerberosユーザーグループをサポートしています。

グループの追加

Kerberosグループを追加します。

グループ名

Kerberosユーザーが属するグループの名前。

値の範囲：1～255バイトの文字列です。具体的な長さは各文字のバイト数によって異なります。最大64～255文字です。

SID

Kerberosおよびユーザグループ認証に使用されるセキュリティ識別子。たとえば、S-1-5-21-310440588-250036847-580389505-500。

値の範囲：1～255バイトの文字列で、スペースを含むことはできません。具体的な長さは各文字のバイト数によって異なります。最大64～255文字です。

ロール

Kerberosユーザーグループのロール。

  • [管理者]:[管理者]はすべての操作を実行できるユーザー。
  • [オペレーター]:[オペレーター]は基本管理、リモート管理、リモートメディア、電力制御、情報照会、および自身のパスワード設定の操作権限を持ちます。
  • [一般ユーザー]:[一般ユーザー]は情報照会、自身のパスワード設定の操作権限を持ちます。
  • ［カスタムロール］:［カスタムロール1］～［カスタムロール4］は、指定された操作を実行できます。
    NOTE:

    新規Kerberosユーザーグループのデフォルトユーザー権限は[一般ユーザー]です。

ログインインターフェース

KerberosグループメンバーがBMCにログインする際に使用するインターフェース。

  • Web

    デフォルトでは、新しいKerberosユーザーグループはWebUIログインをサポートしています。

  • Redfish

    デフォルトでは、新しいKerberosユーザーグループはRedfish APIログインをサポートしています。

グループフォルダ

Kerberosサーバーのユーザーフォルダの名前。

フォーマット：CN=xxxまたはOU=xxx

複数レベルのノードがある場合、上位レベルのノードは下位レベルのノードの後に続き、コンマで区切られます。

たとえば、ユーザーグループgrouptestがKerberosサーバーの\testgroups\part1にある場合、OU=part1,OU=testgroupsと入力します。

NOTE:

CNとOUの違いについては、Kerberosプロトコルの説明を参照してください。

たとえば、Windows ADでは、次のようになります。

  • CN（TypeがContainerの場合）
  • OU（TypeがOrganizational Unitの場合）

値の範囲：255バイトの文字列（ここでは64～255文字）。長さは、エンコードのタイプによって異なります。

ログインルール

Kerberosグループに適用されるログインルール。

編集

既存のKerberosグループ設定エリアを表示します。

削除

Kerberosグループを削除します。

Kerberosの有効化とドメインサーバーの設定

  1. Kerberosをに設定します。
  2. ドメインサーバーを設定します。パラメータの詳細については、表 1を参照してください。
  3. [保存]をクリックします。

鍵リストのインポート

.keytab形式のみがサポートされています。鍵リストのサイズは、0KBまたは1MBを超えることはできません。

  1. [ファイルの選択]をクリックして、インポートする鍵リストを選択します。
  2. [開く]をクリックします。

    鍵リストがアップロードされた後、[アップロードに成功したました]と表示されます。

  3. 現在のユーザーパスワードを入力します。

    SSOを使用してログインするKerberosユーザーはパスワードを入力する必要がありません。

  4. [保存]をクリックします。

Kerberosユーザーグループの削除

  1. [Kerberosユーザーグループ]エリアで、削除対象となるKerberosユーザーグループを選択して、削除をクリックします。

    [確認]ダイアログボックスが表示され、現在のユーザーパスワードの入力が求められます。

    SSOを使用してログインするKerberosユーザーはパスワードを入力する必要がありません。

  2. 現在のユーザーパスワードを入力します。
  3. [OK]をクリックします。
