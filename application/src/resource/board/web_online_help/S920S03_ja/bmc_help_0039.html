タームゾーン

BMCのタイムゾーン。

実際の要件に基づいて地域、タイムゾーンとDST設定します。

デフォルト値:その他+UTC+有効

NOTE:
  • DHCPv4自動的に取得を選択した場合、タイムゾーン情報が自動的に取得されます。
  • DSTが有効になっている場合、夏時間（DST）を使用するタイムゾーンでは、毎年DSTが開始すると、BMC時間は自動的に1時間進みます。DSTが終わると、BMC時間は自動的に標準時間に調整します。
  • OSで時刻同期を行う場合、OSの時刻とBMCの時刻が一致するように、「hwclock --utc -w」コマンドを実行してください。

NTP

NTP

NTPを使用することで、BMCサーバーはNTPサーバーの時刻を同期できます。

デフォルトでは無効になっています。

アドレスを自動的に取得する
  • このオプションをクリックすると、IPv4アドレスがNTPサーバーに自動的に割り当てられます。
  • このオプションをクリックすると、IPv6アドレスがNTPサーバに自動的に割り当てられます。
  • このオプションをクリックすると、NTPサーバーを手動で設定できます。

デフォルト値は[IPアドレスの手動設定]です。

注記:
  • BMC管理ネットワークポートのIPアドレスが自動的に取得されている場合は、NTP情報を自動または手動で取得することができます。自動取得をお勧めします。
  • BMC管理ネットワークポートのIPアドレスを手動で設定する場合、NTP情報も手動で設定する必要があります。
  • サーバー1～サーバー3
  • 優先サーバー1～優先サーバー3または代替サーバー1～代替サーバー3

NTPサーバーのアドレス。

値:IPv4/IPv6アドレスまたはドメイン名。
NOTE:
ドメイン名は次の要件を満たす必要があります。
  • 0～67文字が含まれます。

  • 数字、英字、ハイフン（-）、およびピリオド（.）のみが使用できます。

  • ドメイン名の最初にハイフンまたはピリオドを使ったり、ドメイン名の最後にハイフンを使ったりすることはできません。

  • 2つのピリオドの間に最大63文字が使用できます。

次のいずれかのソリューションが提供されます。
  • サーバー1～サーバー3:3つのNTPサーバーが利用可能です。また、3つのサーバーは同時に有効です。
  • 優先サーバー1～優先サーバー3または代替サーバー1～代替サーバー3:3組のNTPサーバーグループが利用可能です。各グループでは、左側が優先サーバー、右側が代替サーバーです。実際に使用する際にサーバーは次の規則に従って選択されます。
    • グループごとに1つのサーバーが選択されます。
    • グループ内の両方のアドレスが全部無効な場合、当該グループは破棄されます。
    • グループ内に有効なアドレスが1つしかない場合、その有効なIPアドレスを持つサーバーが選択されます。
    • グループ内の両方のアドレスが全部有効な場合、IPv6アドレスを持つサーバーが選択されます。
    • 両方ともIPv4またはIPv6アドレスである場合、優先サーバー（左側のサーバー）が選択されます。
NOTE:

アクティブNTPサーバーとスタンバイNTPサーバーのスイッチオーバーは、BMCとNTPサーバーの同期間隔（[最小ポーリング間隔] ≤同期間隔≤ [最大ポーリング間隔]）に依存します。BMCがアクティブサーバーとの同期に複数回失敗した場合は、スタンバイサーバーがNTPサーバーになります。

最小ポーリング間隔

BMCシステムとNTPサーバーとの時刻同期間隔の最小値、すなわち、NTPパケットの最小ポーリング間隔です。

たとえば、このパラメータが6に設定されている場合、ポーリング間隔は2の6乗、つまり1分4秒です。

値の範囲:3～17

デフォルト値は[6（1分4秒）]です。

最大ポーリング間隔

BMCシステムとNTPサーバーとの時刻同期間隔の最大値、すなわち、NTPパケットの最大ポーリング間隔です。

たとえば、このパラメータが6に設定されている場合、ポーリング間隔は2の6乗、つまり1分4秒です。

値の範囲:3～17

デフォルト値は[10（17分4秒）]です。

サーバー認証

BMCサーバーとNTPサーバ間の通信時のID認証を有効または無効にします。

デフォルトでは無効になっています。

NTPグループキー

サーバーID認証が有効になっている場合、ID認証のためにBMCにアップロードする秘密鍵。

NOTE:
  • 鍵生成器（ntp-keygen）をダウンロードして秘密鍵を生成することができます。鍵ファイルの形式は.keysです。そのサイズは2MBを超えることができません。
  • SHA256またはSHA512アルゴリズムで生成された鍵ファイルのみがアップロードできます。
  • セキュリティ上の理由から、定期的に鍵を更新してください。

タイムゾーンの設定

  1. 地域とタイムゾーンの右側にあるドロップダウンリストから値を選択します。

    表 1にパラメータの説明を示します。

  2. 保存をクリックします。

    操作に成功したメッセージが表示された場合、設定に成功しました。

    • OSで時刻同期を行う場合、OSの時刻とBMCの時刻が一致するように、「hwclock --utc -w」コマンドを実行してください。
    • タイムゾーンを変更した後、BMCのWebUIに表示されるCPU使用率、メモリー使用率、ネットワークポート帯域幅使用率、ディスク使用率、電力使用履歴、吸気口温度などのカーブにシステム時間を反映させるには、BMCを再起動する必要があります。

NTP情報の設定

  1. NTP設定エリアで、サービス要件に基づいてパラメータを設定します。

    表 1にパラメータの説明を示します。

  2. 保存をクリックします。

    操作に成功したメッセージが表示された場合、設定に成功しました。
