Управление хранилищем

Описание функций

На странице Управление хранилищем можно посмотреть информацию и настроить устройства хранения сервера.

  • Информацию о RAID-контроллере, логических и физических дисках (интерфейс SAS/SATA) предоставляет функция внешнего управления платой RAID-контроллера, поэтому отображение информации возможно только после загрузки системы или после установки и запуска BMA 2.0.
  • Если сервер выключен, или система не запущена, на странице Управление хранилищем будет отображаться недостоверная информация Каждый раз в процессе включения сервера или запуска системы, BMC осуществляет идентификацию всех физических дисков. После восстановления физический диск не будет идентифицирован. До идентификации физического диска информация о нем является недостоверной. Если физический диск идентифицировать не удастся, соответствующий датчик (DISKN) сгенерирует аварийный сигнал о неисправности канала.
  • Время, необходимое для идентификации и отображения всех дисков, зависит от количества логических и физических дисков. Чем больше дисков, тем больше требуется времени.

Категория

Описание

RAID-контроллер

Данные о контроллере:

  • Имя RAID-контроллера, тип, имя и версия драйвера, версия встроенного ПО, поддержка внешнего управления, состояние работоспособности, поддерживаемые уровни RAID, режим работы, версия NVDATA, размер памяти, интерфейс устройства, адрес SAS, поддерживаемый диапазон размеров полос данных, статус закрепления кэша, сохранение журналов ошибок физических приводов, статус экстренного копирования, экстренное копирование при ошибке SMART и статус JBOD.
  • Имя, статус и состояние работоспособности BBU
ПРИМЕЧАНИЕ:
  • Если плата RAID-контроллера не поддерживает внешнее управление, а ПО BMA 2.0 не установлено, то на экране будет отображаться только имя RAID-контроллера, тип, версия встроенного ПО и поддержка внешнего управления.
  • Информация о поддержке платой RAID-контроллера внешнего управления BMC приводится в разделе Технические характеристики руководства пользователя RAID-контроллера.

Логический диск

Данные о логическом диске:

Имя, статус, уровень и объем RAID, размер полосы данных, статус кэширования SSCD, политика чтения по умолчанию, активная политика чтения, политика записи по умолчанию, активная политика записи, политика ввода-вывода по умолчанию, активная политика ввода-вывода, статус кэша диска, политика доступа, тип инициализации, статус BGI, статус кэша 2 уровня, статус проверки целостности, буква системного диска и сведения о том, является ли данный диск загрузочным.

ПРИМЕЧАНИЕ:
  • Если плата RAID-контроллера не поддерживает внешнее управление, а ПО BMA 2.0 не установлено, то логические диски, управляемые платой RAID-контроллера, на экране отображаться не будут.
  • Информация о поддержке платой RAID-контроллера внешнего управления BMC приводится в разделе Технические характеристики руководства пользователя RAID-контроллера.

Физический диск

Данные о физическом диске:

Производитель, размер памяти, модель, серийный номер, версия и состояние встроенного ПО, тип носителя, тип интерфейса, возможная скорость, согласованная скорость, SAS-адрес(0), SAS-адрес(1), статус питания, температура, статус горячего резервирования, статус восстановления, статус проверки, состояние работоспособности, степень износа, статус определения местоположения, время работы и скорость.

ПРИМЕЧАНИЕ:
  • Если плата RAID-контроллера не поддерживает внешнее управление, а ПО BMA 2.0 не установлено, то на экране под платой RAID-контроллера будут отображаться только типы интерфейсов физических дисков.
  • Для сквозных дисков отображаются только состояние работоспособности, состояние местоположения и тип интерфейса. Тип интерфейса — SAS/SATA.
  • Информация о поддержке платой RAID-контроллера внешнего управления BMC приводится в разделе Технические характеристики руководства пользователя RAID-контроллера.
  • Запрос времени работы поддерживается только для дисков SATA и SAS-дисков Seagate.
  • Операционные системы Windows и VMware не поддерживают функцию согласования скорости дисков NVMe. Таким образом, если на сервере установлена ОС Windows или VMware, параметр Согласованная скорость для дисков NVMe не будет иметь значения.
  • Параметр Степень износа носителя определяет срок службы твердотельного накопителя SSD. Чем больше значение данного параметра, тем диск менее изношен и дольше прослужит. Например, если параметр имеет значение 100%, значит жесткий диск не изношен.
  • BMC не предоставляет информацию о местоположении дисков M.2.

Запрос параметров платы RAID-контроллера

Перед запросом информации убедитесь, что выполняются следующие условия:

  • Плата RAID-контроллера поддерживает внешнее управление BMC, или программное обеспечение BMA 2.0 установлено и работает в операционной системе. Информация о поддержке платой RAID-контроллера внешнего управления BMC приводится в разделе Технические характеристики руководства пользователя RAID-контроллера.
  • Система BIOS запущена.
  1. Выберите Система > Управление хранилищем.
  2. Выберите RAID-контроллер, информацию о котором хотите запросить. Нажмите Настройки.

Запрос параметров RAID-массива

Информация о поддержке платой RAID-контроллера внешнего управления BMC приводится в разделе Технические характеристики руководства пользователя RAID-контроллера.

    Категория

    Режим работы

    Режимы работы, поддерживаемые картой RAID-контроллера, зависят от типа и версии встроенного ПО карты RAID-контроллера. Более подробную информацию см. в руководстве пользователя используемой карты RAID-контроллера.

    Экстренное копирование

    Функция экстренного копирования позволяет дублировать данные одного диска на другой диск.

    Если диск-участник из RAID-массива с поддержкой резервирования выходит из строя, диск горячего резерва автоматически заменяет его и выполняет синхронизацию данных. После установки нового диска выполняется копирование данных с диска горячего резерва на новый диск. После завершения копирования данных диск горячего резерва снова получает статус резервного.

    Экстренное копирование при ошибке SMART

    Копирование данных осуществляется при появлении на диске ошибки SMART (Self-Monitoring Analysis and Reporting Technology).

    JBOD

    Функция JBOD (Just a bunch of disks) позволяет напрямую передавать команды с RAID-контроллера на подключенные жесткие диски, при этом создание логических дисков не требуется. Эта функция обеспечивает службам верхнего уровня или управляющему ПО доступ к физическим дискам и возможность управлять ими.

    Восстановление настроек по умолчанию

    Нажмите Восстановление настроек по умолчанию, чтобы восстановить настройки RAID-контроллера до значений по умолчанию.

    Импорт внешней конфигурации

    Нажмите Импорт внешней конфигурации, чтобы импортировать конфигурацию RAID, хранящуюся на внешних дисках.

Создание логического диска

Перед созданием логического диска убедитесь, что выполняются следующие условия:
  • Жесткие диски находятся под управлением платы RAID-контроллера, поддерживающей функцию создания логических дисков.
  • Физические диски, добавляемые в качестве логических дисков, находятся в состоянии UNCONFIGURED GOOD.
  • Плата RAID-контроллера поддерживает внешнее управление BMC. Информация о поддержке платой RAID-контроллера внешнего управления BMC приводится в разделе Технические характеристики руководства пользователя платы RAID-контроллера.
  • Количество логических дисков платы RAID-контроллера не достигает предельного значения.
  • Система BIOS запущена.
  1. Выберите Система > Управление хранилищем.
  2. Выберите RAID-контроллер.
  3. Нажмите Добавить.

    Категория

    Описание

    Имя

    Имя логического диска.

    Дополнительный кэш

    Включение CacheCade.

    Размер полосы данных

    Размер полосы данных каждого физического диска.

    Значение по умолчанию — 64 KB.

    Политика чтения

    Политика чтения данных с логического диска.

    Значения:

    • Упреждающее чтение: RAID-контроллер заранее считывает последовательно передаваемые данные или данные, которые предположительно будут использоваться, и сохраняет их в кэш-память.
    • Без упреждающего чтения: функция упреждающего чтения данных отключена.

    Значение по умолчанию — Упреждающее чтение.

    Политика записи

    Политика записи данных на логический диск.

    Значения:

    • Сквозная запись: после записи всех данных на диск контроллер отправляет на хост сообщение о том, что передача данных завершена.
    • Обратная запись с BBU: если блок BBU не сконфигурирован или неисправен, RAID-контроллер автоматически переключается в режим сквозной записи.
    • Обратная запись: после записи всех данных в кэш контроллер отправляет на хост сообщение о том, что передача данных завершена.

    Значение по умолчанию — Обратная запись с BBU.

    Политика ввода-вывода

    Политика ввода-вывода для чтения данных с определенных логических дисков. Данная политика не оказывает никакого влияния на предварительное считывание данных из кэша.

    Значения:

    • Кэшированный ввод-вывод: все запросы на чтение и запись обрабатываются в кэше RAID-контроллера. Данное значение используется только при наличии CacheCade 1.1.
    • Прямой ввод-вывод: данное значение несет разную смысловую нагрузку в сценариях чтения и записи.
      • В сценарии чтения Прямой ввод-вывод означает, что данные считываются непосредственно с физических дисков (если параметру Политика чтения установлено значение Упреждающее чтение, то запросы на чтение данных обрабатываются в кэш-памяти RAID-контроллера).
      • В сценарии записи Прямой ввод-вывод означает, что все запросы на запись обрабатываются в кэш-памяти RAID-контроллера (если параметру Политика записи установлено значение Сквозная запись, то данные записываются непосредственно на физические диски).

    Использование кэша

    Возможные значения:

    • Включено: перед записью на жестких диск данные записываются в кэш. Данная опция повышает производительность записи данных. Однако существует риск потери данных, если механизм защиты от сбоев питания не используется.
    • Отключено: данные записываются сразу на жесткий диск без сохранения в кэше. Данные защищены от потери в случае сбоя питания.
    • Диск по умолчанию: политика кэширования, используемая по умолчанию.

    Значение по умолчанию — Диск по умолчанию.

    Политика доступа

    Политика доступа к логическому диску.

    Значения:

    • Чтение-запись: операции чтения и записи разрешены на диске.
    • Только для чтения: логический диск доступен только для чтения.
    • Ограниченный доступ: доступ к логическому диску запрещен.

    Значение по умолчанию — Чтение-запись.

    Состояние инициализации

    Включение инициализации созданного логического диска.

    Значения:

    • No Init: инициализация не выполняется.
    • Quick Init: запись нулей в первые 100 МБ логического диска. После этого статус логического диска меняется на Оптимальный.
    • Full Init: инициализация логического диска. Во время инициализации логический диск сохраняет статус Оптимальный.

    Значение по умолчанию — Без инициализации.

    Уровень

    Уровень RAID логического диска.

    Значение по умолчанию — 0.

    ПРИМЕЧАНИЕ:

    Если уровень RAID — 1, можно выбрать только два физических диска.

    Диски на подгруппу

    Данный параметр необходимо настраивать при использовании уровней RAID 10, 50 или 60.

    Диск

    Диски, которые добавляются в состав логического диска.

    Доступная емкость

    Емкость, которая может быть выделена для логического диска.

    Объем

    Объем логического диска.

  1. Выберите Система > Управление хранилищем.
  2. Удалите логические диски.

    • Чтобы удалить логический диск, найдите этот логический диск и нажмите .
    • Чтобы удалить несколько логических дисков, нажмите Изменить справа от платы RAID-контроллера, выберите логические диски, которые необходимо удалить, и нажмите Удалить.

    Отображается диалоговое окно Подтвердить.

  1. Нажмите Да.

    Имя

    Политика чтения по умолчанию

    Политика чтения данных с логического диска.

    Значения:

    • Упреждающее чтение: RAID-контроллер заранее считывает последовательно передаваемые данные или данные, которые предположительно будут использоваться, и сохраняет их в кэш-память.
    • Без упреждающего чтения: функция упреждающего чтения данных отключена.

    Политика записи по умолчанию

    Политика записи данных на логический диск.

    Значения:

    • Сквозная запись: после записи всех данных на диск контроллер отправляет на хост сообщение о том, что передача данных завершена.
    • Обратная запись с BBU: если блок BBU не сконфигурирован или неисправен, RAID-контроллер автоматически переключается в режим сквозной записи.
    • Обратная запись: после записи всех данных в кэш контроллер отправляет на хост сообщение о том, что передача данных завершена.

    Политика ввода-вывода по умолчанию

    Политика ввода-вывода для чтения данных с определенных логических дисков. Данная политика не оказывает никакого влияния на предварительное считывание данных из кэша.

    Значения:

    • Кэшированный ввод-вывод: все запросы на чтение и запись обрабатываются в кэше RAID-контроллера. Данный значение используется только при наличии CacheCade 1.1.
    • Прямой ввод-вывод: данное значение несет разную смысловую нагрузку в сценариях чтения и записи.
      • В сценарии чтения Прямой ввод-вывод означает, что данные считываются непосредственно с физических дисков (если параметру Политика чтения установлено значение Упреждающее чтение, то запросы на чтение данных обрабатываются в кэш-памяти RAID-контроллера).
      • В сценарии записи Прямой ввод-вывод означает, что все запросы на запись обрабатываются в кэш-памяти RAID-контроллера (если параметру Политика записи установлено значение Сквозная запись, то данные записываются непосредственно на физические диски).

    Значения:

    Использование кэша

    Возможные значения:

    • Включено: перед записью на жестких диск данные записываются в кэш. Данная опция повышает производительность записи данных. Однако существует риск потери данных, если механизм защиты от сбоев питания не используется.
    • Отключено: данные записываются сразу на жесткий диск без сохранения в кэше. Данные защищены от потери в случае сбоя питания.

Изменение параметров диска-участника

Перед изменением параметров убедитесь, что выполняются следующие условия:

  • Жесткие диски находятся под управлением платы RAID-контроллера, поддерживающей функцию создания логических дисков.
  • Плата RAID-контроллера поддерживает внешнее управление BMC. Информация о поддержке платой RAID-контроллера внешнего управления BMC приводится в разделе Технические характеристики руководства пользователя платы RAID-контроллера.
  • Система BIOS запущена.
