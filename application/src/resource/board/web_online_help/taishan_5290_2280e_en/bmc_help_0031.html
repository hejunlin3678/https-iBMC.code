Port Services

Function Description

On the Port Services page, you can view and modify the ports services supported by the BMC.

  • If a Web Server (HTTP)/Web Server (HTTPS) port is configured as a non-default browser port, the Chrome or Firefox browser cannot use the port to establish a connection. To solve this problem, you need to configure the browser to allow connections to be set up over a non-default port.
  • Disabling the SSH, HTTPS, RMCP, and RMCP+ services at the same time may result in network disconnection. If all the services are disabled, you can connect to the server through the serial port and enable the web service.

Parameter Description

Table 1 Port Services

Service

Default Port

Description

SSH

22

The SSH allows a secure channel to be established between a local computer and the server.

The BMC supports a maximum of 10 concurrent SSH connections.

It is enabled by default.

NOTE:

SSH supports encryption algorithms AES128-CTR, AES192-CTR, and AES256-CTR. Use a supported encryption algorithm when logging in to BMC over SSH.

SNMP Agent

161

The SNMP agent translates and transfers requests between management devices and managed devices.

It is enabled by default.

KVM

2198

The KVM allows users to remotely control a server by using the local keyboard, video, and mouse (KVM).

The BMC supports a maximum of two concurrent users.

It is enabled by default.

VMM

8208

The VMM allows a user to use a virtual DVD-ROM drive or floppy disk drive (FDD) to access and control a server.

The BMC supports only one user at a time.

It is enabled by default.

Video

2199

Allows users to use the video playback function. For details about this function, see Video & Screenshot.

The BMC supports only one user at a time.

It is enabled by default.

VNC

5900

The Virtual Network Console (VNC) allows users to remotely control a server by using the local keyboard, video, and mouse.

The BMC supports a maximum of five concurrent users.

It is disabled by default.

WEB_HTTP

80

The HTTP supports Internet browsing and translates Hypertext Transfer Protocol (HTTP) pages. The Web Server (HTTP) service is enabled by default to establish a connection between the browser and BMC. After the connection is set up, the secure protocol HTTPS is used.

It is enabled by default.

WEB_HTTPS

443

The HTTPS supports Internet browsing and translates Hypertext Transfer Protocol over Secure Socket Layer (HTTPS) pages or Redfish Protocol.

The BMC supports a maximum of four concurrent HTTPS connections.

It is enabled by default.

IPMI LAN (RMCP)

623 for port 1 (primary port) and 664 for port 2 (secondary port).

Stands for Intelligent Platform Management Interface (IPMI) over LAN, and supports the Remote Management Control Protocol (RMCP).

Using the IPMI LAN (RMCP) service may pose security risks. For security purposes, use the IPMI LAN (RMCP+) service instead.

It is disabled by default.

IPMI LAN (RMCP+)

RMCP+ and RMCP use the same port.

Stands for Intelligent Platform Management Interface (IPMI) over LAN and supports RMCP+.

It is enabled by default.

NOTE:

The RMCP+ protocol has security vulnerabilities (CVE-2013-4786), and using RMCP+ poses security risks. Refer to Risk Prevention Measures.

Setting Port Services

  1. Click Edit.
  2. Enable or disable the port services based on service requirements.

    • : enables a service.
    • : disables a service.

  3. Set port numbers for these services.

    After editing the port, it takes about ten seconds for the new configuration to take effect. Do not edit the configuration repeatedly before it takes effect.

  4. Click Save.

Risk Prevention Measures

Do as follows to minimize the security risks caused by the vulnerability (CVE-2013-4786) of RMCP+:

  • If you do not use IPMI protocol to access the BMC:
    • Disable the IPMI service on this page.

      After IPMI is disabled, other devices cannot use IPMI to access the BMC. This setting affects the IPMI-based tools, such as IPMItool, InfoCollect, and eSight.

    • Enable password complexity check and set passwords complying with the password complexity requirements.
  • If you need to use IPMI protocol to access the BMC:
    • Set the network where the BMC management network port is located as an independent LAN.
    • Enable password complexity check and set passwords complying with the password complexity requirements.
