BMA Management

Function Description

The Baseboard Management Agent (BMA) is an in-band management agent software.

On the BMA Management page, you can install the BMA on the server operating system through the Remote Console.

After the BMA is successfully installed, the BMA basic information, including the version number, running status, and driver version, will be displayed on the page.

Parameter Description

Table 1 BMA Management

Parameter

Description

Available BMA Version

Version of the BMA that can be installed for the current operating system.

For example, Linux 1.10 indicates that the BMA version for Linux is 1.10.

Installer Status

Status of the connection between the server operating system and the BMA to be installed.

  • Not ready: The server operating system is not connected or the connection has failed.
  • Ready: The server operating system is successfully connected.

BMA Status

BMA Version

Version of the BMA installed on the server OS.

-- indicates that the BMA is not installed or is not running.

BMA Running Status

Running status of the BMA software.

  • -- indicates that the BMA is not installed or is not running.
  • Running indicates that the BMA has been started.
  • Unhealthy indicates that the BMA is installed and cannot run properly.

BMA Driver Version

BMA driver version.

-- indicates that the BMA is not installed or is not running.

Installing the BMA

  • Only the client running Linux supports installation of the BMA.
  • To mount the BMA drive, you must have the remote media permission.
  • To start the Remote Console, you must have the remote control permission.
  • To install the BMA, you must have the remote media and remote control permissions.
  1. Click Install BMA

    The BMA sets up a connection with the operating system of the server. If the connection is set up successful, the Description page is displayed.

  2. Click Open Remote Console.

    The Remote Console starts.

  3. On the Remote Console, log in to the operating system of the server as an administrator, and then enter the user name and password of the operating system.
  4. In the device list of the server operating system, locate the driver disk BMA.

    If BMA is not displayed, mount the BMA driver disk and go to 5. For details about how to mount the BMA driver disk, see Mounting the BMA Driver Disk.

  5. Open the README.TXT file on the Linux operating system and query the Linux operating system version information listed in Supported Operating Systems in this Release.

    1. Right-click in the blank area of the operating system.
    2. Choose Open Terminal from the menu displayed. The server operating system CLI is displayed.
    3. Run the cd Linux and ls commands and check the README.TXT file on Linux.
      [root@localhost ~]# cd Linux
[root@localhost Linux]# ls
app config drivers install.sh README.TXT script
    4. Run cat README.TXT.
      [root@localhost ~]# cat README.TXT
BMA on-board installation package
Version 2.1.3.020

********************************************************************************
Installation
********************************************************************************

* On the Linux operating systems, execute "sh install.sh -s" from the "Linux" directory to install BMA silently and execute "sh install.sh -u" to upgrade BMA.

For more information on installation instructions, includingsilent installation options, see the "BMA 2.0 User Guide".

********************************************************************************

      • If the version of the Linux operating system installed is the same as any of the Linux operating system versions listed in Supported Operating Systems in this Release, go to 6.
      • If the version of the Linux operating system installed is different from the values listed in Supported Operating Systems in this Release, update the operating system version to any of the Linux operating system versions listed in README.txt, and then go to 6.

  6. Install the BMA.

    For details about how to install the BMA, see the BMA User Guide of the latest version.

Mounting the BMA Driver Disk

  1. Right-click in the blank area of the operating system.
  2. Choose Open Terminal from the menu displayed. The server operating system CLI is displayed.
  3. Run the lsscsi command to query the properties of the BMA driver disk.

    [root@localhost ~]# lsscsi
[0:0:0:0]    disk    ATA      ST4000NM0033-9ZM SN06  -
[0:0:1:0]    disk    ATA      ST4000NC001-1FS1 CN02  -
[0:0:2:0]    disk    ATA      WDC WD6000F9PZ-3 0R01  /dev/sdc
[0:0:3:0]    disk    ATA      WDC WD6000F9PZ-3 0R01  /dev/sdd
[0:0:4:0]    disk    HGST     HUS726060AL4210  A523  -
[0:0:5:0]    disk    HGST     HUS726060AL4210  A7MH  -
[0:0:6:0]    disk    ATA      SSDSC2BB016T7H   0121  /dev/sde
[0:0:7:0]    disk    HGST     HUS726060AL4210  A523  /dev/sdf
[0:1:0:0]    disk    LSI      Logical Volume   3000  /dev/sdb
[0:1:1:0]    disk    LSI      Logical Volume   3000  /dev/sda
[19:0:0:0]   disk    SERVER   BMA USB Device   225  /dev/sdv

    In the command out, BMA USB Device indicates the node name of the BMA driver disk, and /dev/sdv indicates the drive letter assigned to the BMA driver disk.

  4. Run mount /dev/sdv /home/file to mount the BMA driver disk to the /home/file directory.

    In this example, /home/file is the directory for storing the file mounted. Create a directory based on actual requirements. The names of the folders in the directory can contain digits, letters, underscores (_), hyphens (-), and dots (.).
    [root@localhost ~]# mount /dev/sdv /home/file
#

  5. Run the ls /home/file command to check whether the BMA driver disk is successfully mounted.

    [root@localhost ~]# ls /home/file
Linux

    If Linux is displayed in the command output, the BMA driver disk is successfully mounted.
