ログインについて

機能概要

ログイン画面は、BMC WebUIへのアクセスを提供します。

  • 最大4人のユーザーがBMC WebUIに同時にログインできます。
  • デフォルトでは、システムのタイムアウト期間は5分です。WebUIで5分以内に何の操作も実行しないと、ユーザーはWebUIから自動的にログアウトされます。
  • 不正なパスワードの連続試行回数が指定された値に達すると、システムはユーザーアカウントをロックします。ロック時間が経過した後、ユーザーはシステムにログインできます。
  • システムのセキュリティを確保するために、初回ログインした後に、初期パスワードを変更し、パスワードを定期的に変更してください。
  • ネットワークの状態により、リソースの取得に失敗し、BMC WebUIが正しく表示されない場合があります。ブラウザを更新して、もう一度BMC WebUIにログインしてください。

説明

ユーザー名

BMC WebUIにログインするユーザー名。

  • がドメインローカルのBMCの場合、ユーザー名の最大長は16文字です。
  • がドメインローカルのBMCではない場合、ユーザー名の最大長は255文字です。

ログイン時は次の点に注意してください。

  • ローカルユーザーとしてBMCにログインする場合は、ドメインはローカルのBMCまたは自動マッチングにすることができます。
  • LDAP方式でBMCにログインする場合は、以下の2種類のフォーマットのユーザー名をサポートします。
    • LDAPユーザー名（[自動マッチング]を選択するか、ドメインのドメイン名を入力）
    • LDAPユーザー名@ドメイン名（[自動マッチング]を選択するか、ドメインのドメイン名を入力）
  • Kerberos方式でBMCにログインする場合は、以下の2種類のフォーマットのユーザー名をサポートします。
    • Kerberosユーザー名（[自動マッチング]を選択するか、ドメインのドメイン名を入力）
    • Kerberosユーザー名@ドメイン名（[自動マッチング]を選択するか、ドメインのドメイン名を入力）ドメイン名の文字は大文字でなければなりません。
  • BMCはKerberosユーザー名またはKerberosユーザー名@ドメイン名を使用してSSOをサポートしています。

パスワード

BMC WebUIにログインするためのパスワード。

NOTE:

BMC WebUIにログインするLDAPまたはKerberosユーザーのパスワードの最大長は255文字です。

ローカルユーザーとしてBMC画面にログインする

  1. BMC WebUIログインページで、画面の言語を選択します。
  2. ユーザー名とパスワードを入力します。
  3. プルダウンリストから、ドメインローカルのBMCまたは自動マッチングを選択します。
  4. ログインをクリックします。

    ログインに成功した場合、ホーム画面が表示され、右上隅にユーザー名が表示されます。

LDAPユーザーとしてBMC画面にログインする

ログインする前に、次の設定が要件を満たしていることを確認してください。

  • ドメインコントローラがネットワーク上に存在し、ドメインコントローラ上にユーザードメインとLDAPユーザーが作成されました。

ドメインコントローラー、ユーザードメイン、およびLDAPユーザーの作成方法については、ドメインコントローラーの関連ドキュメントを参照してください。BMCはLDAPユーザーのアクセス機能のみを提供します。

  • LDAPはBMC WebUIのユーザー&セキュリティ > LDAPで有効になっており、ユーザードメインとLDAPユーザーが設定されています。
  1. BMC WebUIログインページで、画面の言語を選択します。
  2. LDAPユーザー名とパスワードを入力します。

    • LDAP方式で BMCにログインする場合は、以下の2種類のフォーマットのユーザー名をサポートします。
      • LDAPユーザー名

        LDAPユーザー名を入力した場合、ドメインは自動マッチングまたは指定されたドメインにすることができます。

      • LDAPユーザー名@ドメイン名

        LDAPユーザー名@ドメイン名を入力した場合、ドメインは自動マッチングまたは指定されたドメインにすることができます。

    • BMC WebUIにログインするLDAPユーザーのパスワードの最大長は255文字です。

  1. ドメインドロップダウンリストからLDAPユーザードメインを選択します。

    ドメインドロップダウンリストには、次のオプションがあります。

    • ローカルのBMC:ローカルユーザーとしてログインするときに、このオプションを選択します。BMCはローカルユーザーリストからユーザーアカウントを自動的に検索します。
    • 構成されたドメインサーバー:LDAPユーザーとしてログインするときに、ドメインサーバーを選択します。BMCはドメインサーバーからユーザーアカウントを自動的に検索します。
    • 自動マッチング:このオプションを選択すると、BMCはローカルユーザーリストからユーザーを検索します。一致しているユーザーがいない場合は、ドメインドロップダウンリストの順に沿ってドメインサーバーからBMCを順次検索します。

  2. ログインをクリックします。

    ログインに成功した場合、情報画面が表示され、右上隅にユーザー名が表示されます。

KerberosユーザーとしてBMC画面にログインする

Kerberosの実行環境:

  • クライアントは、Windows 10 64ビットに対応しています。
  • Kerberosサーバーは、Windows Server 2012 R2 64ビットとWindows Server 2016 64ビットに対応しています。

次の2種類のログインモードをサポートしています。

  • Kerberosドメインアカウントによるログイン
  • SSO

ログインする前に、次の設定が要件を満たしていることを確認してください。

  • KerberosはBMC WebUIのユーザー&セキュリティ > Kerberosで有効になっており、Kerberos機能とユーザーグループの構成が完了しました。
  • Kerberosユーザーグループとユーザー名がKerberosサーバー上に作成されており、ユーザーがKerberosユーザーグループに追加されました。このユーザーは、クライアントのOSにログインするユーザーです。

Kerberosドメインアカウントを使用してBMC WebUIにログインするには、次の手順に従ってください。

  1. BMC WebUIログインページで、画面の言語を選択します。
  2. Kerberosユーザー名とパスワードを入力します。詳細については、表 1を参照してください。
  1. ドメインのドロップダウンリストから、Kerberosユーザードメイン（たとえば、ADMIN.COM(KRB)または自動マッチング）を選択します。
  2. [ログイン]をクリックします。

    ログインに成功すると、[ホーム]ページが表示されます。

  1. SSOを使用してログインするには、次の手順に従ってください。
  2. Kerberosサーバーで設定されたKerberosユーザー名とパスワードを使用し、クライアントOSにログインします。
  3. ブラウザを開き、アドレスボックスにBMC. FQDN（たとえば、https://ホスト名.ドメイン名）を入力してEnterを押します。

    BMCログインページが表示されます。

  4. [シングルサインオン]をクリックします。

    ログインに成功すると、[ホーム]ページが表示されます。
