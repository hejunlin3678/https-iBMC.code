アラーム&イベント

機能説明

アラーム&イベント画面で、次の操作を実行できます。

  • サーバーのクリアされていないすべてのアクティブなアラームを照会します。
  • システムイベントを表示、検索したり、すべてのシステムイベントをダウンロード、クリアしたりします。

パラメータの説明

表 1 アラーム&イベント

パラメータ

説明

番号

イベントのシーケンシャル番号。

重大度

アラームの重大度。

  • :クリティカルアラームが発生している場合、サーバーの電源がオフになり、システムサービスが中断される可能性があります。対応措置を直ちに講じる必要があります。
  • :メジャーアラームはシステムに大きな影響を与えます。システムの通常動作に影響を与え、またはサービス中断を引き起こす可能性があります。
  • :マイナーアラームはシステムに軽微な影響を与えますが、より深刻なアラームを防止するために、早急に対処する必要があります。
  • :システムの通常実行中にイベントが発生しました。

オブジェクトタイプ

アラームが発生したコンポーネントのタイプ。

ステータス

システムイベントの状態。

設定範囲

  • Asserted:システムイベントが発生したことを示します。
  • Deasserted:システムイベントが復元されたことを示します。

イベントコード

アラームを一意に識別します。

発生時間

アラームが発生した日時。

イベントの詳細

アラームに関する詳細な情報。

推奨処理

アラームをクリアするための推奨方法。

システムイベントの検索

  1. [保守＆診断] > [アラーム&イベント]を選択してシステムイベントタブを選択します。
  2. [検索条件]をクリックします。

    フィルタ条件を設定する画面が表示されます。

  3. フィルタ条件を設定します。パラメータの詳細については、表 2を参照してください。

    表 2 検索条件

    パラメータ

    説明

    アラーム重大度

    システムイベントのレベル。

    設定範囲:すべて、クリティカル、メジャー、マイナー、または正常

    オブジェクトタイプ

    システムイベントが発生したコンポーネントのタイプ。

    値:サーバーモデルによって異なります。

    発生時間

    システムイベントが発生した日時。

    設定範囲

    • すべて
    • 本日
    • 過去7日間
    • 過去30日間
    • カスタマイズム
    NOTE:

    カスタマイズムを選択する場合、開始日と終了日を指定する必要があります。

    照会入力

    カスタマイズされたフィルタ条件。

    テキストボックスに、次の内容を入力できます。

    • イベント内容
    • 0xが付くかまたは付かない完全なイベントコード

  4. [照会]をクリックします。

    フィルタ条件を満たすイベントが表示されます。

すべてのシステムイベントのクリア

クリアされたシステムイベントは復元できませんので、慎重にご操作ください。

  1. [[保守＆診断] > [アラーム&イベント]]を選択して[システムイベント]タブを選択します。
  2. ページの右上隅にあるをクリックします。

    すべてのシステムイベントがクリアされます。

システムイベントのダウンロード

  1. [[保守＆診断] > [アラーム&イベント]]を選択して[システムイベント]タブを選択します。
  2. ページの右上隅にあるをクリックします。

    ダウンロードしたファイルは自動的にローカルPCのデフォルトディレクトリに保存されます。
