アラーム設定

機能概要

アラーム設定画面では、次の操作を実行できます。

  • Syslog通知:Syslogメッセージの形式でサードパーティのサーバーにログを送信できます。
  • メール通知:アラームが発生したときに、SMTP（Simple Mail Transfer Protocol）サーバーを介して指定されたメールボックスにEメールを送信できます。
  • Trap通知:アラーム情報、イベント情報、およびトラッププロパティをトラップメッセージの形式でサードパーティのサーバーに送信できます。

    トラップは、BMCからサードパーティのサーバーに向けて能動的に送信される通知のことです。クリティカルアラーム、メジャーアラーム、マイナアラームおよびイベントの通知に使用されます。

パラメータの説明

TCPまたはUDPを有効にすると、システムセキュリティが損なわれます。注意して続行してください。

パラメータ

説明

Syslog通知

Syslog機能

Syslogメッセージの形式で通知を送信する機能。

デフォルトでは無効になっています。

アラーム重大度

Syslog方式でサードパーティのサーバーに送信するイベント情報のレベル。

設定範囲:
  • [NULL]:アラームやイベントを送信しない。
  • クリティカル:クリティカルアラームアラームのみを送信する。
  • メジャー:メジャーおよびクリティカルアラームアラームのみを送信する。
  • マイナー:マイナー、メジャー、クリティカルアラームを送信する。
  • 正常:イベントおよびマイナー、メジャー、およびクリティカルアラームを送信する。

デフォルト値は[正常]です。

Syslogメッセージ形式

Syslogメッセージの形式。

  • カスタマイズ:Syslogメッセージには、メッセージ優先度、製品名、SyslogホストID、デバイス位置、およびログタイプが含まれます。

  • RFC3164:SyslogメッセージはRFC 3164に準拠しており、メッセージ優先度、タイムスタンプ、ホスト名、プロセス名、およびログタイプが含まれます。

デフォルト値は[カスタマイズ]です。

NOTE:

RFC3164が選択されている場合、[SyslogサーバーID]はデフォルトで[ホスト名]になり、設定することができません。

Syslogサーバー識別情報

Syslogメッセージの送信元。

設定範囲:

  • ボードシリアル番号
  • 製品資産ラベル
  • ホスト名

デフォルト値は[ボードシリアル番号]です。

伝送プロトコル

BMCシステムとSyslogサーバー間でSyslogメッセージを転送する場合に使用される伝送プロトコル。

設定範囲:

  • TLS:接続ベースのプロトコルであるため、データ転送の秘密性とデータの完全性が確保されます。
  • TCP:接続ベースのプロトコルであるため、データを送受信する前に、送信先と受信先の間で必ず信頼できる接続を確立しなければなりません。
  • UDP:接続ベースのプロトコルではないため、データを送受信する前に送信先と受信先の間で接続を確立する必要はなく、直接正式なデータを転送します。

デフォルト値は[TLS]です。

認証モード

Syslog証明書を認証するためのモード。伝送プロトコルがTLSである場合にのみ、このパラメータを設定します。

設定範囲:

  • 片方向:Syslogサーバー側の証明書のみを認証します。
  • 二方向:Syslogサーバー側とクライアント側の双方の証明書を認証します。

デフォルト値は[片方向]です。

サーバールート証明書

接続が確立される前にSyslogサーバーから送信されたメッセージを検証するために使用される証明書。

NOTE:
  • .crt、.cer、または.pem形式のサーバールート証明書ファイルをインポートできます。そのサイズは100KBを超えることができません。
  • セキュリティ上の理由から、定期的に証明書を更新してください。

ローカル証明書

データ接続を確立する際に、BMCはローカル証明書情報を含むパケットを syslogサーバーに送信します。syslogサーバーはsyslogクライアント(BMC)を認証するためにその証明書情報が必要です。

NOTE:
  • .pfxまたは.p12形式のローカル証明書ファイルをインポートできます。そのサイズは1MBを超えることができません。
  • セキュリティ上の理由から、定期的に証明書を更新してください。

サーバー証明書情報

アップロードされたサーバールート証明書に関する情報。証明書情報には、次のものが含まれます。

  • ルート証明書を発行した権限。
  • ルート証明書が発行されたユーザー。
  • ルート証明書の有効期間。
  • ルート証明書のシリアル番号。
  • ルート証明書の失効リスト。
  • CRL有効期限
NOTE:
  • CRLは、証明書の失効ステータスを示します。
    • 設定済み：証明書失効ファイルがアップロードされており、TLS接続がセットアップされると、証明書失効の検証が実行されます。
    • 未設定：証明書失効ファイルはアップロードされませんでした。
  • Base64でエンコードされた証明書失効ファイルは*.crl形式であり、100 KBを超えることはできません。
  • CRL有効期限が切れると、対応する認証機能は使用できなくなります。

CRLを設定するには、[アップロード]をクリックし、クライアントから証明書失効ファイルを選択します。 [削除]をクリックして、CRLを削除します。

NOTE:

CRLを削除すると、期限切れの証明書を使用するセキュリティリスクが発生する場合があります。

Syslogサーバーとメッセージ形式

番号

Syslogメッセージを送信するチャネル。最大4つのチャネルを設定できます。

サーバーアドレス

SyslogサーバーのIPアドレス。

設定範囲:IPv4アドレス、IPv6アドレス、またはドメイン名

NOTE:
  • 伝送プロトコルがTLSの場合、ドメイン名を入力します。さらに、BMC設定 > ネットワーク構成設定ページでDNS情報を正しく設定する必要があります。
  • ドメイン名は次の要件を満たす必要があります。
    • 0～255文字が含まれます。

    • 数字、英字、ハイフン（-）、およびピリオド（.）のみが使用できます。

    • ドメイン名の最初にハイフンまたはピリオドを使ったり、ドメイン名の最後にハイフンを使ったりすることはできません。

    • 2つのピリオドの間に最大63文字が使用できます。

ポート

Syslogサーバーのポート番号。

設定範囲:1～65535

ログタイプ

Syslogメッセージで送信するログの種類。

設定範囲:

  • 操作ログ
  • セキュリティログ
  • イベントログ

1つまたは複数のオプションを選択できます。

デフォルトでは、すべてのオプションが選択されています。

現在の状態

あるチャンネルの有効化状態を設定します。

デフォルトでは無効になっています。

操作
  • [編集]をクリックして、Syslogサーバーとメッセージ形式を変更します。
  • [テスト]をクリックして、指定したSyslogサーバーにSyslogメッセージを正常に送信できるかどうかを確認します。[操作に成功しました。]が表示されると、宛先Syslogサーバーに到達可能です。
    NOTE:

    [Syslog通知設定]エリアにあるパラメータを変更した場合、[テスト]をクリックする前に、まず[保存]をクリックしてください。変更が有効にならない可能性があります。

電子メール情報

送信者の非表示

匿名ログインを許可する機能。

匿名ログインが許可されている場合、SMTPサーバーはユーザー名とパスワードを認証することなくアラームメールを転送します。

匿名ログインが許可されていない場合、SMTPサーバーは正しいユーザー名とパスワードが入力された後にのみアラームメールを転送します。ユーザー名とパスワードは、SMTPサーバーで設定されている必要があります。

デフォルト値は[no]です。

NOTE:

デフォルトでは、SMTPサーバーは匿名ログインを許可しません。セキュリティ上の理由で、匿名ログインを使用しないでください。

送信者のユーザー名とパスワード

匿名ログイン許可がいいえに設定されている場合、ユーザー名とパスワードを使用します。

ユーザー名とパスワードは、SMTPサーバーで設定されているユーザー名とパスワードと同じにする必要があります。

設定範囲:
  • ユーザー名:1～64文字。数字、英字、特殊文字を使用できます。空白にすることはできません。
  • パスワード:1～50文字。

送信者の電子メールアドレス

アラームが送信された電子メールアドレス。

設定範囲:xx@xxx.xx形式の最大255文字。

英字、数字、および特殊文字を使用できます。

メール件名/件名の追加情報

電子メールのタイトル。

設定範囲:0～255文字。数字、英字、特殊文字を使用できます。

メール件名に電子メールの件名を入力し、電子メールの件名に含まれるキーワードを選択します。デフォルト値は[Server Alert]です。

たとえば、ホスト名とボードシリアル番号を選択した場合、電子メールの件名にはホスト名とボードのシリアル番号が含まれます。

アラーム重大度

SMTPサーバーから発送されたアラーム情報のレベル。

設定範囲:
  • [NULL]:アラームやイベントを送信しない。
  • クリティカル:クリティカルアラームアラームのみを送信する。
  • メジャー:メジャーおよびクリティカルアラームアラームのみを送信する。
  • マイナー:マイナー、メジャー、クリティカルアラームを送信する。
  • 正常:イベントおよびマイナー、メジャー、およびクリティカルアラームを送信する。

デフォルト値は[正常]です。

番号

メールアドレス

メール受信用メールアドレス。必ず、SMTPサーバーに登録されているアドレスでなければなりません。

Trap設定

Trap機能

トラップメッセージの形式でアラームを送信する機能。

デフォルトで有効になっています。

Trapバージョン

トラップの送信に使用されるSNMPバージョン。

設定範囲:

  • SNMPv1:SNMP TrapプロトコルのV1バージョンは簡易ネットワーク管理プロトコルの正式なバージョン1で、RFC（Request For Comments）1157で定義されています。
  • SNMPv2c:V2CバージョンはV2の改良版です。SNMP TrapプロトコルのV2Cバージョンはコミュニティストリングベース（Community-Based）管理フレームワークで、RFC 1901で定義されている実験的プロトコルです。
  • SNMPv3:SNMPプロトコルのV3バージョンはRFC 3411-RFC 3418で定義されており、主にセキュリティとリモート設定が強化されています。
NOTE:
  • SNMPv3トラップが推奨されます。SNMPv1およびSNMPv2cを使用する際に、セキュリティリスクが発生する可能性があるので、注意してください。
  • SNMPv3の認証アルゴリズムおよび暗号化アルゴリズムは、WebUIの[ユーザー&セキュリティ] > [ローカルユーザー]で設定できます。

デフォルト値:SNMPv1

SNMPv3ユーザー

Trapバージョンを[SNMPv3]に選択している場合、プロトコルが必要なユーザー名を同時設定する必要があります。

デフォルトでは、BMCのデフォルトユーザーはSNMPv3ユーザーです。

Trapモード

トラップ情報をレポートするモード。

設定範囲:

  • 高精度アラーム(推奨):ベントと1対1対応するSNMPノードのOIDがTrapイベントの識別子として使用されます。OIDおよびイベントコードに比べて、より正確なロケーション情報を提供できます。
  • OIDモード:SNMPノードのOIDがTrapイベントの識別子として使用されます。
  • イベントコード:トラップイベントの識別子として使用されます。

デフォルト値:高精度アラーム（推奨）

Trapサーバー識別情報

トラップメッセージの送信元。

設定範囲:

  • ボードシリアル番号
  • 製品資産ラベル
  • ホスト名

デフォルト値は[ボードシリアル番号]です。

コミュニティ名

SNMPに使用されるコミュニティ名。コミュニティ名とは、トラップ方式を認証するのに必要なパスワードです。トラップバージョンをSNMPv1またはSNMPv2cに設定している場合のみ、コミュニティ名を設定できます。

値の範囲は、パスワード複雑度チェックが有効になっているかどうかによって異なります。

  • パスワード複雑度チェックが無効になっている場合、値は文字、数字、特殊文字（スペースを除く）からなる1～32文字です。
  • パスワードチェックが有効になっている場合、コミュニティ名は次の要件を満たす必要があります。
    • 8～32文字が含まれます。
    • 以下の文字のうち少なくとも2種類を含む必要があります。アルファベットの大文字（A～Z）、数字（0～9）
    • 少なくとも以下の特殊文字を含んでいなければなりません。`~!@#$%^&*()-_=+\|[{}];:'",<.>/?
    • スペースを使用できません。

コミュニティ名の確認

SNMPに使用されるコミュニティ名。ここに入力する内容がコミュニティ名と同じでなければなりません。

アラーム重大度

トラップメッセージの形式でサードパーティのサーバーに送信されるアラームのレベル。

設定範囲:
  • [NULL]:アラームやイベントを送信しない。
  • クリティカル:クリティカルアラームアラームのみを送信する。
  • メジャー:メジャーおよびクリティカルアラームアラームのみを送信する。
  • マイナー:マイナー、メジャー、クリティカルアラームを送信する。
  • 正常:イベントおよびマイナー、メジャー、およびクリティカルアラームを送信する。

デフォルト値は[正常]です。
