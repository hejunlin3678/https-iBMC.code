ローカルユーザー

機能概要

[ローカルユーザー]画面で、BMCのユーザーを確認・管理できます。

BMCは最大16人のユーザーをサポートします。[ローカルユーザー]画面でユーザーの追加、設定および削除操作を実行できます。

ログインインターフェース

ユーザーがBMCにログインする際に使用するインターフェース。

  • [SNMP]:ユーザーはSNMPツール（MIB Browserなど）を使用してBMCにログインできます。
  • [SSH:ユーザーは、SSHツール（PuTTYなど）を使用してBMCコマンドラインにログインできます。
  • [IPMI]:ユーザーはIPMIツール(IPMItoolなど)を使用してBMCコマンドラインにログインできます。
  • [Local]:ユーザーはサーバーのシリアルポートを使用してBMC CLIにログインすることができます。
  • [SFTP]:ユーザーはSFTPツール(Xftpなど)を使用してBMCファイルシステムにログインできます。
  • [Web]:ユーザーはWebブラウザを使用してBMCにログインできます。
  • [Redfish]:ユーザーはRedfishツールを使用してBMCにログインできます。

操作
  • 編集をクリックしてローカルユーザーを変更します。
  • 無効をクリックして、ユーザーを無効にします。
  • 有効をクリックして、ユーザーを有効にします。
  • 削除をクリックしてユーザーを削除します。
    NOTE:
    • 管理者、オペレーター、一般ユーザー、カスタムユーザーを含め、すべてのローカルユーザーを削除できます。
    • BMCに有効になっている管理者が複数いる場合、デフォルトユーザーのロールを変更できます。有効になっている管理者が1人のみの場合、この管理者を削除したり、無効にしたりすることができず、管理者のロールを変更することもできません。
    • ユーザー&セキュリティ > セキュリティ管理画面でOS ユーザーが有効になっている場合、OSから標準IPMIコマンドを送信することでBMCユーザーを追加することもできます。

ユーザーの追加

BMCでは最大15人のユーザーを追加できます。

  1. [ユーザーの追加]をクリックします。

    ユーザー追加の画面が表示されます。

  2. ユーザーのパラメータを設定します。パラメータの詳細については、表 3を参照してください。

    • IDが1のユーザーはIPMI規格で定義されている予約ユーザーです。このユーザーはBMCにログインできます。
    • IDが2のユーザーはデフォルトユーザーです。
    表 3 ユーザーを追加するためのパラメータ

    パラメータ

    説明

    ユーザーID

    追加するユーザーのID。

    設定範囲:3～17。

    ユーザー名

    新規作成するユーザー名。

    設定範囲:1～16文字。

    パスワードは、次の複雑さの要件を満たす必要があります。

    • 文字、数字、特殊文字が含まれます（:<>&,'"/\%を除く）。

    • スペースを含めたり、[#]、[+]、または[-]で始めたりすることはできません。

    • ピリオド(.)または2つの連続したピリオド(..)は使用できません。

    パスワード

    BMC.にログインするためのパスワードです。

    セキュリティ上の理由から、パスワード強度チェックを有効にして、定期的にパスワードを変更してください。

    NOTE:
    • 管理者のみがパスワード強度チェックを有効／無効にすることができます。
    • パスワード強度チェックを無効にすることにはセキュリティ上のリスクがあります。可能であれば、有効にしてください。

    設定範囲:

    • パスワード強度チェック機能を無効にした場合、パスワードには20文字以内の数字、英字、特殊文字を使用できます。 パスワードが8文字未満の場合、ユーザーはSNMPv3インターフェースを使用することができません。
    • パスワードチェックが有効になっている場合、パスワードは次の要件を満たす必要があります。
      • 8～20文字が含まれます。
      • 少なくとも1つのスペースまたは次のいずれかの特殊文字を含む必要があります。

        `~!@#$%^&*()-_=+\|[{}];:'",<.>/?

      • 次のうち、少なくとも2種類の文字を使用する必要があります。
        • 大文字（A～Z）
        • 小文字（a～z）
        • 数字（0～9）
        • ユーザー名またはユーザー名の逆スペルと異なる必要があります。
      • 弱いパスワードチェックが有効になっている場合、パスワードは弱いパスワード辞書に含まれているパスワードと同じにはできません。（ipmcset -t user -d weakpwddic -v exportコマンドを実行することで、弱いパスワード辞書から弱いパスワードをエクスポートできます。)
      NOTE:
      • セキュリティのため、aa、abababab、abcdabcdなどの繰り返し文字列のみで構成されるパスワードは使用しないでください。

    通信のためにSNMPを使用しているユーザに対して、通信セキュリティを確保するために、暗号化パスワードを設定できます。暗号化パスワードは、ローカルユーザーのパスワードルールに準拠する必要があります。

    デフォルト値：ユーザーログインパスワードと同じです。

    NOTE:
    • このパラメーターを設定しない場合、SNMPv3暗号化パスワードはユーザーログインパスワードと同期されます。セキュリティのために、SNMPv3暗号化パスワードを設定することをお勧めします。SNMPv3暗号化パスワードを設定した場合、このパスワードはユーザーのログインパスワードと同期されません。

    SNMPv3認証アルゴリズム

    値は以下のとおりです。

    • MD5
    • SHA
    • SHA256
    • SHA384
    • SHA512
    NOTE:

    SNMPv3暗号化アルゴリズム

    値は以下のとおりです。

    • AES
    • AES256

ユーザーの削除

  1. ローカルユーザーリストで、削除対象となるユーザーを選択して、[削除]をクリックします。

    ユーザーがユーザーリストから削除されます。

SSH公開鍵のインポート

  • クライアントで秘密鍵を生成した後、BMC側にこれと関連した公開鍵をインポートする必要があります。これにより、ユーザーがBMCシステムにSSHでログインする場合、セキュリティと一意性を確保できます。
  • 各ユーザーがインポートできる公開鍵は1つだけです。公開鍵を変更するには、既存の公開鍵を削除してから、新しい公開鍵をインポートします。
  • サポートされている公開鍵の形式はRFC 4716とOpenSSHです。公開鍵のタイプがRSAまたはDSAです。公開鍵のタイプがRSAである場合、2048ビットと4096ビットの長さをサポートします。公開鍵のタイプがDSAである場合、2048ビットの長さをサポートします。
  • 公開鍵ファイルの形式は.pubです。そのサイズは2KBを超えることができません。
  1. SSH公開鍵をインポートする対象ユーザーを特定し、そのユーザー名の左にあるをクリックします。
  2. SSH公開鍵領域からアップロードをクリックします。

  3. インポートモードを選択します。

    公開鍵ファイルまたは公開鍵テキストを選択します。

  4. ファイルの選択をクリックして公開鍵を選択します。
  5. 現在のユーザーパスワードを入力します。
  6. 保存をクリックします。
