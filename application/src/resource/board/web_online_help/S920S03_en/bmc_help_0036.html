SNMP

Function Description

The Simple Network Management Protocol (SNMP) consists of a set of network management standards. It contains an application layer protocol, a database model, and a group of resource objects. This protocol helps a network management system to detect devices connected to the network.

On the SNMP page, you can query and configure Simple Network Management Protocol (SNMP) service settings.

The BMC supports the following SNMP versions:

  • SNMPv1: The first official SNMP version defined in Requests for Comments (RFC) 1157. Using SNMPv1 may pose security risks. For security purposes, use SNMPv3.
  • SNMPv2: An experimental protocol defined in RFC 1901 and adopts a community-based management architecture. Using SNMPv2 may pose security risks. For security purposes, use SNMPv3.
  • SNMPv3: The third official SNMP version, which enhances security and remote configuration capabilities on the basis of earlier versions.

NOTE:
  • SNMPv1 and SNMPv2 are disabled by default.
  • If SNMPv1/SNMPv2 is enabled, change the SNMP community name upon the first login, and change it periodically.

Long Password

Setting of the long password function, which can be enabled or disabled.

Enable this function to enforce a minimum of 16 characters for community names.

Default value: Enabled

Read-Only Community

Read-only community name.

Value:

  • If password check is disabled, the community names must meet the following requirements:
    • If the long password feature is enabled, the community name is a string of 16 to 32 characters without spaces.
    • If the long password feature is disabled, the community name is a string of 1 to 32 characters without spaces.
  • If password check is enabled, the community names must meet the following requirements:
    • Length:
      • A string of 16 to 32 characters if the long password feature is enabled.
      • A string of 8 to 32 characters without spaces if the long password feature is disabled.
    • Contain at least one of the following special characters:

      `~!@#$%^&*()-_=+\|[{}];:'",<.>/?

    • Contain at least two types of the following characters:
      • Uppercase letters A to Z
      • Lowercase letters a to z
      • Digits 0 to 9
    • Cannot contain spaces.
  • If weak password check is enabled, the password cannot be the same as the passwords contained in the weak password dictionary. You can run the ipmcset -t user -d weakpwddic -v export <filepath | file_URL> command to export the weak passwords from the weak password dictionary.

Confirm Read-Only Community

Read-only community name re-entered for confirmation.

Delete Read-Only Community

Deletes the configured read-only community.

NOTE:
  • Read-Only Community cannot be configured while Delete Read-Only Community is selected.
  • It is recommended that you delete the configured read-only community when SNMPv1 and SNMPv2 are not in use.
