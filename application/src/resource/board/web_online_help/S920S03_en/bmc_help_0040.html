Firmware Upgrade

Function Description

On the Firmware Upgrade page, you can perform the following operations:

  • View firmware version information of the server.
  • Restart the BMC.
  • Switch over services to the available BMC image.
  • Upgrade the server firmware through the out-of-band channel.
  • Upgrade the server firmware through the in-band channel.
The BMC has the following partition images:
  • Primary partition image: the BMC partition image in working.
  • Secondary partition image: the backup of the primary partition image. When the primary partition image is faulty, the secondary partition image changes to the primary state, and the original primary partition images changes to the secondary state and synchronizes the image version with the primary one. When the BMC is upgraded, the primary and secondary partition images are upgraded at the same time.
  • Available partition image: the reserved BMC partition image. You can click the Switch to Available Image button to make the available partition image as the primary one. Meanwhile, the original primary partition image changes to the secondary state and synchronizes data with the current primary partition image, and the original secondary one automatically changes to the available state.
  • During the upgrade process, do not power off the server or restart the BMC.
  • After upgrading the BMC firmware, restart the BMC for the new version to take effect. However, you do not need to restart the server. Therefore, the services running on the server will not be affected.
  • You do not need to restart the server after the PSU firmware is upgraded.
  • After the BMC or SD card controller firmware is upgraded, the BMC automatically restarts for the upgrade to take effect.
  • Before upgrading the BIOS or CPLD, stop the services running on the server. This prevents service interruption when the server is restarted.
  • If the BIOS or CPLD is upgraded while the system is powered on, power cycle or restart the system for the BIOS upgrade to take effect, or power off the system for the CPLD upgrade to take effect.
  • If the BIOS or CPLD is upgraded while the system is powered off, power on the system for the upgrade to take effect.
  • Before upgrading the firmware through the in-band channel, stop the services running on the server to prevent service interruption during automatic restart of the server.

Viewing Firmware Versions

  1. On the menu bar, choose BMC Settings.
  2. In the navigation tree, choose Firmware Upgrade.

    The Version Info page is displayed on the right, showing the server firmware name and version information.

Upgrading the Firmware (Out-of-Band Channel)

The BMC is used as an example in the following operations that are also applicable to the upgrade of other firmware.

  1. Choose Firmware Update > Out-of-Band Channel.
  2. Click Select File and select the upgrade file.

    • The upgrade file is in the .hpm format. The size cannot exceed 90 MB.
    • The upgrade file name allows digits, English letters, spaces, brackets (), dots (.), underscores (_), and hyphens (-).

  3. (Optional) Select Restart the BMC immediately after the upgrade for the upgraded firmware to take effect.

    This option applies only to the BMC upgrade.

  4. Click Upgrade.

    The following information is displayed:

    Are you sure you want to perform this operation?

  5. Click Yes.

    The upgrade starts.

    After the upgrade is successful, the BMC automatically restarts and the login page is displayed.

    It takes a few minutes for the BMC to restart.

Upgrading the Firmware (In-Band Channel)

The following operations apply to the firmware (such as the NIC, RAID controller card, and hard drive firmware) that can be upgraded through Smart Provisioning. Before the upgrade, download the firmware upgrade package and signature file.

  1. Choose Firmware Update > In-Band Channel.
  2. Click Select File next to Upgrade File and Signature File to select and upload the upgrade file and signature file.

    • The upgrade file is in .zip format and cannot exceed 499 MB.
    • The signature file is in .asc (pgp) or .p7s (cms) format and cannot exceed 90 MB.

  3. Click Upload.

    The following information is displayed:

    Are you sure you want to go to the Smart Provisioning (SP) management page and set the OS to boot from the SP next time?
    • If you need to upload other firmware upgrade packages, click Cancel and repeat 2 and 3 to upload the firmware upgrade package and signature file.
    • If you need to delete the uploaded firmware upgrade packages, click Cancel and then click Clear.

  4. Click OK.

    The Smart Provisioning Management page is displayed.

  5. Enable Boot OS from SP and click Confirm in the displayed dialog box.
  6. Restart the server and upgrade the firmware on Smart Provisioning.

Performing an BMC Image Switchover

Perform an image switchover for the BMC firmware only when necessary. The image switchover is optional during the upgrade.

  1. On the Version Info page, click Switch to Available Image.

    The following information is displayed:
    Are you sure you want to switch over to the BMC available partition image?

  2. Click Yes.

    If the switchover is successful, the BMC automatically restarts and the login page is displayed.

    It takes a few minutes for the BMC to restart.

Restarting the BMC

Restart the BMC as required. This operation is not mandatory during the upgrade.

  1. On Version Info, click Restart BMC.

    The following information is displayed:

    Are you sure you want to restart the BMC?

  2. Click Yes.

    The BMC restarts and the login page is displayed.

    It takes a few minutes for the BMC to restart.
