Gestion de le BMA

Présentation de la fonction

Baseboard Management Agent (BMA) est un logiciel d'agent de gestion intrabande.

Sur la page Gestion de le BMA, vous pouvez installer le BMA sur le système d'exploitation du serveur via le Console à distance.

Une fois le BMA installé avec succès, les informations de base de le BMA, y compris le numéro de version, l'état de fonctionnement et la version du pilote, sont affichées sur la page.

Description du paramètre

Tableau 1 Gestion de le BMA

Paramètre

Description

Version de le BMA disponible

Version de le BMA qui peut être installé sur le système d'exploitation actuel.

Par exemple, Linux 1.10 indique que la version BMA pour Linux est 1.10.

État de l'installateur

État de la connexion entre le système d'exploitation du serveur et le BMA à installer.

  • Non prêt : le système d'exploitation du serveur n'est pas connecté ou la connexion a échoué.
  • Prêt : le système d'exploitation du serveur est connecté avec succès.

État de le BMA

Version de le BMA

Version de le BMA installé sur le système d'exploitation du serveur.

-- indique que le BMA n'est pas installé ou ne fonctionne pas.

État de l'exécution BMA

État de fonctionnement du logiciel BMA.

  • -- indique que le BMA n'est pas installé ou ne fonctionne pas.
  • Running indique que le BMA a été démarré.
  • Unhealthy indique que le BMA est installé et ne peut pas fonctionner correctement.

Version du pilote de le BMA

Version du pilote BMA.

-- indique que le BMA n'est pas installé ou ne fonctionne pas.

Installation de le BMA

  • Seul le client fonctionnant sous Linux prend en charge l'installation de le BMA.
  • Pour monter le lecteur BMA, vous devez disposer de l'autorisation de média distant.
  • Pour démarrer la console à distance, vous devez disposer de l'autorisation de contrôle à distance.
  • Pour installer le BMA, vous devez disposer des autorisations de média distant et de contrôle à distance.
  1. Cliquez sur Installer le BMA.

    Le BMA établit une connexion avec le système d'exploitation du serveur. Si la connexion est établie avec succès, la page Description s'affiche.

  2. Cliquez sur Ouvrir la console à distance.

    La console à distance démarre.

  3. Sur le Console à distance, connectez-vous au système d'exploitation du serveur en tant qu'administrateur, puis entrez le nom d'utilisateur et le mot de passe du système d'exploitation.
  1. Sur la liste des dispositifs du système d'exploitation du serveur, localisez le disque du pilote BMA. Si le BMA n'est pas affiché, montez le disque du pilote BMA et allez à 5. Pour plus d'informations sur le montage du disque du pilote BMA, voir Montage du disque du pilote BMA.
  2. Ouvrez le fichier README.TXT sur le système d'exploitation Linux et interrogez les informations de version du système d'exploitation Linux énumérées dans Supported Operating Systems in this Release.

    1. Cliquez avec le bouton droit de la souris dans la zone vide du système d'exploitation.
    2. Choisissez Open Terminal dans le menu affiché. La CLI du système d'exploitation du serveur est affichée.
    3. Exécutez les commandes cd Linux et ls et vérifiez le fichier README.TXT sous Linux.
      [root@localhost ~]# cd Linux
[root@localhost Linux]# ls
app config drivers install.sh README.TXT script
    4. Exécutez cat README.TXT.
      [root@localhost ~]# cat README.TXT
BMA on-board installation package
Version 2.1.3.020

********************************************************************************
Installation
********************************************************************************

* On the Linux operating systems, execute "sh install.sh -s" from the "Linux" directory to install BMA silently and execute "sh install.sh -u" to upgrade BMA.

For more information on installation instructions, includingsilent installation options, see the "BMA 2.0 User Guide".

********************************************************************************
Supported Operating Systems in this Release
********************************************************************************

* EulerOS 2.0 SP8
* CentOS 7.6 on aarch64
--More details on limitations and supported Operating Systems can be located in the "BMA 2.0 User Guide".
      • Si la version du système d'exploitation Linux installée est identique à l'une des versions du système d'exploitation Linux énumérées dans Supported Operating Systems in this Release, allez à 6.
      • Si la version du système d'exploitation Linux installée diffère des valeurs énumérées dans Supported Operating Systems in this Release, mettez à jour la version du système d'exploitation vers l'une des versions du système d'exploitation Linux énumérées dans README.txt, puis allez à 6.

  3. Installez le BMA.

    Pour plus d'informations sur l'installation d'BMA, voir la dernière version de le BMA User Guide. Le chemin de fichier d'installation de le BMA varie en fonction du système d'exploitation utilisé.

    Chemin de fichier d'installation

    Linux

    Linux/install.sh

Montage du disque du pilote BMA

  1. Cliquez avec le bouton droit dans la zone vide du système d'exploitation.
  2. Choisissez Open Terminal dans le menu affiché. La CLI du système d'exploitation du serveur est affichée.
  3. Exécutez la commande lsscsi pour interroger les propriétés du disque du pilote BMA.

    [root@localhost ~]# lsscsi 
[0:0:0:0]    disk    ATA      ST4000NM0033-9ZM SN06  - 
[0:0:1:0]    disk    ATA      ST4000NC001-1FS1 CN02  - 
[0:0:2:0]    disk    ATA      WDC WD6000F9PZ-3 0R01  /dev/sdc 
[0:0:3:0]    disk    ATA      WDC WD6000F9PZ-3 0R01  /dev/sdd 
[0:0:4:0]    disk    HGST     HUS726060AL4210  A523  - 
[0:0:5:0]    disk    HGST     HUS726060AL4210  A7MH  - 
[0:0:6:0]    disk    ATA      SSDSC2BB016T7H   0121  /dev/sde 
[0:0:7:0]    disk    HGST     HUS726060AL4210  A523  /dev/sdf 
[0:1:0:0]    disk    LSI      Logical Volume   3000  /dev/sdb 
[0:1:1:0]    disk    LSI      Logical Volume   3000  /dev/sda 
[19:0:0:0]   disk    SERVER   BMA USB Device   225  /dev/sdv

    Dans la sortie de commande, BMA USB Device indique le nom du nœud du disque du pilote BMA et /dev/sdv indique la lettre de lecteur attribuée au disque du pilote BMA.

  4. Exécutez mount /dev/sdv /home/file pour monter le disque du pilote BMA au répertoire /home/file.

    Dans cet exemple, /home/file est le répertoire pour stocker les fichiers montés. Créez un répertoire selon les exigences réelles. Le nom des dossiers du répertoire peuvent contenir des chiffres, des lettres, des traits d'union (_), des tirets (-) et des points (.).

    [root@localhost ~]# mount /dev/sdv /home/file 
#

  5. Exécutez la commande ls /home/file pour vérifier si le disque du pilote BMA est correctement monté.

    [root@localhost ~]# ls /home/file 
Linux

    Si Linux s'affiche dans la sortie de commande, le disque du pilote BMA est correctement monté.
