Журналы BMC

Описание функций

  • На странице Журнал операций можно просматривать и загружать записи журнала, регистрируемые во время работы системы, включая информацию о запуске системы, изменении ее состояния и настройках, выполненных пользователями в BMC.

    BMC предоставляет 200 КБ памяти для хранения до 2000 записей об операциях.

    При превышении 200 КБ произойдет автоматическое сжатие записей журнала. После создания нового сжатого пакета старый пакет удалится автоматически.

    Журнал операций регистрирует операции включения и выключения системы, а также сброс настроек. Данные записи означают, что BMC успешно инициировал данные операции, однако это не означает, что операции были успешно выполнены на оборудовании.

  • На странице Журнал работы можно посмотреть информацию RAS сервера.

    BMC предоставляет 200 КБ памяти для хранения до 2000 записей журнала работы.

    При превышении 200 КБ произойдет автоматическое сжатие журнала. После создания нового сжатого пакета старый пакет удалится автоматически.

  • На странице Журнал безопасности доступны следующие операции:
    • Просмотр записей об операциях входа и выхода пользователей из BMC через SSH, а также об операциях настройки параметров.
    • Просмотр записей об операциях запроса и настройки, выполненных посредством SNMP.
    • Загрузка записей журнала безопасности.

    BMC предоставляет 200 КБ памяти для хранения до 2000 записей журнала безопасности.

    При превышении 200 КБ произойдет автоматическое сжатие журнала. После создания нового сжатого пакета старый пакет удалится автоматически.

Описание параметров

Табл. 1 Журнал операций

Параметр

Описание

№

Идентификатор операции. Первой в списке отображается операция, выполненная последней.

Интерфейс

Интерфейс, с которого была выполнена операция.

Пользователь

Пользователь, выполнивший операцию.

Параметр Пользователь может иметь значение «–» (имя пользователя не отображается) в одном из следующих случаев:

  • Кнопка UID или кнопка питания нажаты.
  • В качестве интерфейса используется SNMPv1 или SNMPv2c.
  • В качестве интерфейса используется IPMI, и IP-адрес имеет значение HOST (журнал регистрирует получение IPMI-сообщения, отправленного служебной системой).
  • IP-адрес и пароль пользователя по умолчанию BMC были изменены с помощью переключателя.
  • Компонент был заменен резервным в «горячем» режиме.

IP-адрес

IP-адрес, с которого была выполнена операция.

  • Значение HOST означает, что операция была инициирована служебной системой.
  • IP-адрес в формате X.X.X.X указывает на то, что операция выполняется клиентом, получившим доступ к BMC.
  • Значение 127.0.0.1 означает, что операция была инициирована локальным хостом в одном из следующих случаев:
    • Кнопка UID, кнопка райзер-карты или кнопка питания нажаты.
    • Доступ к BMC осуществляется через локальный последовательный порт.
    • IP-адрес и пароль пользователя по умолчанию BMC были изменены с помощью переключателя.
    • Компонент был заменен резервным в «горячем» режиме.

Время возникновения

Время выполнения операции.

Подробная информация

Подробная информация об операции.

Если после обновления BMC через WebUI, CLI или IPMI происходит его перезагрузка, то операция регистрируется в следующем формате:

  • Интерфейс: –
  • Пользователь: –
  • IP-адрес: 127.0.0.1
  • Подробная информация: Reset BMC caused by upgrade successfully
