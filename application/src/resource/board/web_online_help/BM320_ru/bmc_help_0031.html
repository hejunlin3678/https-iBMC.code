Службы портов

Описание функций

На странице Службы портов можно посмотреть и настроить службы портов, поддерживаемые BMC.

  • Если порт HTTP/HTTPS не настроен в качестве порта браузера, используемого по умолчанию, браузеры Chrome или Firefox не смогут его использовать для установления соединения. Поэтому необходимо настроить браузер таким образом, чтобы он мог устанавливать соединения через порты, не используемые по умолчанию.
  • Одновременное отключение служб SSH, HTTPS, RMCP и RMCP+ может стать причиной сбоя сетевого соединения. Если все веб-службы отключены, то для того чтобы их включить, пользователь может подключиться к серверу через последовательный порт.

Описание

SSH

22

SSH обеспечивает установление безопасного канала между локальным компьютером и сервером.

BMC поддерживает максимум десять одновременных соединений SSH.

Данная функция включена по умолчанию.

ПРИМЕЧАНИЕ:

SSH поддерживает такие алгоритмы шифрования, как AES128-CTR, AES192-CTRи AES256-CTR. При входе в BMC через SSH используйте один из поддерживаемых алгоритмов шифрования.

SNMP Agent

161

SNMP Agent обеспечивает преобразование и передачу запросов между управляющими и управляемыми устройствами.

Данная функция включена по умолчанию.

KVM

2198

KVM обеспечивает пользователям возможность дистанционно управлять сервером при помощи локальных клавиатуры, монитора и мыши (KVM).

BMC поддерживает одновременное подключение максимум двух пользователей.

Данная функция включена по умолчанию.

VMM

8208

VMM предоставляет пользователям возможность использовать виртуальный DVD-дисковод или дисковод флоппи-дисков (FDD) для управления и доступа к серверу.

BMC поддерживает одновременное подключение только одного пользователя.

Данная функция включена по умолчанию.

Video

2199

Служба предоставляет пользователям возможность использовать функцию воспроизведения видео. Более подробная информация приводится в разделе Видео и скриншоты.

BMC поддерживает одновременное подключение только одного пользователя.

Данная функция включена по умолчанию.

VNC

5900

VNC позволяет пользователям удаленно управлять сервером при помощи локальной клавиатуры, видео и мыши (KVM).

BMC поддерживает максимум пять одновременных пользователей.

По умолчанию данная функция отключена.

WEB_HTTP

80

HTTP обеспечивает возможность Интернет-браузинга и преобразования страниц HTTP (Hypertext Transfer Protocol). Для обеспечения связи между браузером и BMC служба Web Server (HTTP) включена по умолчанию. После установления соединения используется безопасный протокол HTTPS.

Данная функция включена по умолчанию.

WEB_HTTPS

443

HTTPS обеспечивает возможность Интернет-браузинга и преобразования страниц HTTPS (Hypertext Transfer Protocol over Secure Socket Layer) или протокола Redfish.

BMC поддерживает максимум четыре одновременных соединения HTTPS.

Данная функция включена по умолчанию.

IPMI LAN (RMCP)

623 для порта 1 (основной порт) и 664 для порта 2 (дополнительный порт)

Служба обеспечивает интеллектуальный интерфейс управления платформой (IPMI) по LAN с поддержкой протокола RMCP.

Использование службы IPMI LAN (RMCP) может представлять угрозу сетевой безопасности. В целях повышения безопасности рекомендуется использовать службу IPMI LAN (RMCP+).

По умолчанию данная функция отключена.

IPMI LAN (RMCP+)

Службы RMCP+ и RMCP используют один и тот же порт.

Служба обеспечивает интеллектуальный интерфейс управления платформой (IPMI) по LAN с поддержкой протокола RMCP+.

Данная функция включена по умолчанию.

ПРИМЕЧАНИЕ:

Использование протокола RMCP+ создает риски в плане безопасности, так как он имеет уязвимости (CVE-2013-4786). См. Профилактика рисков.

Настройка служб портов

  1. Нажмите Изменить.
  2. Включите или отключите службы портов, исходя из фактических требований.

    • : включение службы.
    • : отключение службы.

  3. Настройте номера портов для этих служб.

    После изменения порта требуется около десяти секунд, чтобы новая конфигурация вступила в силу. Не изменяйте конфигурацию повторно до ее вступления в силу.

  4. Нажмите Сохранить.

Профилактика рисков

Чтобы минимизировать риски безопасности, связанные с уязвимостью (CVE-2013-4786) RMCP+, выполните следующие действия:

  • Если протокол IPMI не используется для доступа к BMC:
    • Отключите службу IPMI на этой странице.

      После отключения IPMI другие устройства не смогут использовать IPMI для доступа к BMC. Кроме того, после отключения данной службы станут недоступными инструменты, работающие на основе IPMI, такие как IPMItool, InfoCollect и eSight.

    • Включите проверку сложности пароля и установите пароли, соответствующие требованиям сложности.
  • Если протокол IPMI необходимо использовать для доступа к BMC:
    • Настройте сеть, в которой находится сетевой порт управления BMC, в качестве независимой LAN.
    • Включите проверку сложности пароля и установите пароли, соответствующие требованиям сложности.
