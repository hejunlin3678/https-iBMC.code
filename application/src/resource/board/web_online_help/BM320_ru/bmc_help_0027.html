Двухфакторная аутентификация

Описание функций

Двухфакторная аутентификация — улучшенный метод контроля доступа. Пользователям предоставляется доступ только при условии ввода правильного сертификата клиента и пароля. В отличие от аутентификации только по паролю аккаунта данный метод обеспечивает более эффективную защиту.

Для установления безопасного соединения между клиентом и WebUI BMC необходимо загрузить в BMC корневой сертификат и сертификат клиента, полученные из центра сертификации.

Параметр

Описание

Двухфакторная аутентификация

Функция, предоставляющая пользователям доступ к WebUI BMC только при условии ввода правильного сертификата клиента и пароля.

По умолчанию данная функция отключена.

ПРИМЕЧАНИЕ:
  • Перед включением двухфакторной аутентификации необходимо импортировать действительный корневой сертификат и сертификат клиента. Убедитесь, что существует хотя бы один набор действительных сертификатов. В противном случае при последующих попытках входа может произойти сбой аутентификации.
  • После включения двухфакторной аутентификации произойдет автоматическое отключение службы SSH, и запустить ее вручную будет невозможно.
  • После того, как двухфакторная аутентификация включена, а затем отключена, сервис SSH не будет активирован автоматически. Если вам нужно использовать сервис SSH, включите его вручную.

Проверка интерактивного статуса сертификата (OCSP)
УВЕДОМЛЕНИЕ:
  • Проверка выполняется с помощью OCSP. Перед включением проверки протокола состояния сетевого сертификата убедитесь в исправности соединения между BMC и сервером OCSP. В противном случае веб-сервис может быть недоступен.
  • Убедитесь, что в корневой сертификат была записана информация об адресе OCSP. В противном случае может произойти сбой двухфакторной аутентификации после включения проверки протокола состояния сетевого сертификата.

Проверка протокола состояния сетевого сертификата (OCSP) позволяет проверить действительность сертификата клиента во время аутентификации. Если сертификат клиента окажется недействительным, то пользователь не сможет войти на WebUI BMC.

По умолчанию данная функция отключена.

Проверка списка отозванных сертификатов (CRL)

Проверка списка отозванных сертификатов (CRL) позволяет проверить, был ли отозван сертификат во время аутентификации. После включения данной функции система проверяет, был ли отозван текущий сертификат клиента во время входа в WebUI BMC. Если сертификат был отозван, происходит сбой аутентификации при осуществлении входа в WebUI BMC.

По умолчанию данная функция отключена.

ПРИМЕЧАНИЕ:

Убедитесь, что CRL был импортирован. В противном случае может произойти сбой двухфакторной аутентификации после включения проверки списка отозванных сертификатов.

Корневой сертификат

Список корневых сертификатов, которые были загружены в BMC, и информация о них, включая имя эмитента, пользователя, срок действия, CRL и срок действия CRL каждого корневого сертификата.

BMC поддерживает максимум 16 корневых сертификатов.

ПРИМЕЧАНИЕ:
  • Поддерживается выгрузка корневых сертификатов с кодировкой Base64. Корневые сертификаты могут быть в форматах *.cer, *.crt и *.pem. Размер не может превышать 1 МБ.
  • Статус отзыва сертификата.
    • Настроено: файл CRL был выгружен; при установке соединения TLS будет выполнена проверка сертификата на предмет отзыва.
    • Не настроено: файл CRL не выгружен.
  • Если срок действия CRL истекает, соответствующая функция аутентификации будет недоступна.

Сертификат клиента

Список сертификатов клиента, которые были загружены в BMC, и информация о них, включая имя пользователя, роль, статус и время отзыва сертификата, а также отпечаток сертификата клиента (хеш-значение файла сертификата клиента).

BMC поддерживает сертификаты клиентов максимум 16 пользователей.

ПРИМЕЧАНИЕ:
  • Поддерживается выгрузка сертификатов клиента (сертификата открытого ключа) с кодировкой Base64. Сертификаты клиента могут быть в форматах *.cer, *.crt и *.pem. Размер не может превышать 1 МБ.
  • Статус отзыва сертификата.
    • Отозвано
    • Не отозвано

ключение двухфакторной аутентификации и выгрузка сертификатов в BMC

  • Перед выполнением данной операции необходимо подать заявку на получение корневого сертификата и сертификата клиента (включая сертификаты открытого и закрытого ключей) в официальный CA.
    • Сертификаты закрытых ключей имеют формат .pem, .p12 или .pdx. Более подробную информацию об операциях см. в описании операций CA.
    • В целях обеспечения безопасности необходимо периодически обновлять сертификат.
  • Поддерживается выгрузка корневых сертификатов и сертификатов клиента (сертификата открытого ключа) с кодировкой Base64. Корневые сертификаты и сертификаты клиента могут быть в форматах *.cer, *.crt и *.pem. Размер не может превышать 1 МБ.
  1. Нажмите Выгрузка сертификата.

    Выберите сертификат для выгрузки. Выгрузите корневой сертификат на странице Корневой сертификат. Выгрузите сертификат открытого ключа клиента для указанного пользователя на странице Сертификат клиента.

  2. Нажмите Открыть.

    Отображается "Операция выполнена успешно".

  3. Установите переключатель Двухфакторная аутентификация в положение.

Выгрузка CRL

Файл отзыва сертификатов имеет кодировку Base64 и формат *.crl. Размер файла не может превышать 100 КБ.

  1. Получите файл CRL в центре сертификации (CA).
  2. Найдите целевой сертификат и нажмите Выгрузить в столбце CRL.
  3. Выберите файл CRL.

Включение аутентификации сертификата

После выгрузки сертификатов для включения аутентификации пользователей, которые осуществляют вход в WebUI BMC, выполните следующие действия.

  1. Откройте браузер на клиенте, например Google Chrome 81.0.4044.138.

    Действия могут отличаться в зависимости от типа и версии браузера.

  2. Нажмите в правом верхнем углу и выберите Настройки, затем Конфиденциальность и безопасность.
  3. Нажмите Управление сертификатами.
  4. Импортируйте сертификат закрытого ключа клиента.

    Если требуется ввести пароль, введите пароль, который установлен при подаче заявки на получение сертификата.

  5. В адресной строке браузера введите адрес страницы входа в BMC.
  6. Выберите сертификат клиента, следуя инструкциям.

    После этого вход в WebUI BMC будет выполнен успешно.

Удаление корневого сертификата

Корневые сертификаты могут быть удалены только в случае отключения функции двухфакторной аутентификации.

  1. На вкладке Корневой сертификат нажмите Удалить рядом с сертификатом, который хотите удалить.

    Откроется диалоговое окно подтверждения.

  2. Нажмите Да.

Удаление сертификата клиента

  1. На вкладке Сертификат клиента нажмите Удалить рядом с пользователем, сертификат клиента которого хотите удалить.

    Откроется диалоговое окно подтверждения.

  2. Нажмите Да.

Удаление CRL

Удаление CRL может привести к рискам безопасности при использовании сертификатов с истекшим сроком действия.

  1. Найдите целевой сертификат и нажмите Удалить CRL в столбце CRL.

    Откроется диалоговое окно подтверждения.

  2. Нажмите Да.
