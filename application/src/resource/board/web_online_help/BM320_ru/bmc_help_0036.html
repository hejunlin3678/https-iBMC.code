SNMP

Описание функций

SNMP (Simple Network Management Protocol) — стандартный протокол для управления устройствами в сети. Он состоит из набора стандартов для сетевого управления, включая протокол прикладного уровня, модель баз данных и набор ресурсных объектов. Этот протокол позволяет системе управления сетью обнаруживать устройства, подключенные к сети.

На странице SNMP можно выполнить запрос данных и настроить параметры протокола управления сетью.

BMC поддерживает следующие версии SNMP:

  • SNMPv1: первая официальная версия SNMP, определенная в RFC 1157. SNMPv1 имеет ряд уязвимостей, из-за которых безопасность сети может оказаться под угрозой. В целях обеспечения безопасности рекомендуется использовать SNMPv3.
  • SNMPv2: экспериментальный протокол, определенный в RFC 1901, который использует архитектуру управления на основе сообществ. SNMPv2 имеет ряд уязвимостей, из-за которых безопасность сети может оказаться под угрозой. В целях обеспечения безопасности рекомендуется использовать SNMPv3.
  • SNMPv3: третья официальная версия SNMP с улучшенными возможностями безопасности и удаленным конфигурированием по сравнению с более ранними версиями.

Версия SNMP

Варианты версий SNMP.

SNMPv1/SNMPv2

ПРИМЕЧАНИЕ:
  • SNMPv1 и SNMPv2 по умолчанию выключены.
  • При выборе опции SNMPv1 или SNMPv2 рекомендуется сменить имя сообщества SNMP после первого входа в систему и периодически менять его.

Длинный пароль

Включение или отключение функции длинного пароля.

Включите данную функцию, чтобы увеличить длину имени сообщества, как минимум, до 16 символов.

Значение по умолчанию: Включить

Сообщество только для чтения

Имя сообщества только для чтения.

Значения:

  • При отключенной функции проверки пароля должны выполняться следующие условия:
    • Если функция длинного пароля включена, имя сообщества должно состоять из 16–32 символов без пробелов.
    • Если функция длинного пароля отключена, имя сообщества должно состоять из 1–32 символов без пробелов.
  • При включенной функции проверки пароля имена сообществ должны соответствовать следующим требованиям:
    • Длина:
      • Строка из 16–32 символов, если включена функция длинного пароля.
      • Строка из 8–32 символов без пробелов, если функция длинного пароля отключена.
    • Имя сообщества должно содержать хотя бы один из следующих специальных символов:

      `~!@#$%^&*()-_=+\|[{}];:'",<.>/?

    • Имя сообщества должно включать символы, как минимум, двух видов:
      • Прописные буквы: от A до Z
      • Строчные буквы: от a до z
      • Цифры: от 0 до 9
    • Имя сообщества не должно содержать пробелы.
  • Если включена проверка слабого пароля, то пароль не может совпадать с паролями, занесенными в справочник слабых паролей (для экспорта слабых паролей из справочника используется команда ipmcset -t user -d weakpwddic -v export <filepath | file_URL>).
