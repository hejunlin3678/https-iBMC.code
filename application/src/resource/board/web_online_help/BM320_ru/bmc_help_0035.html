VNC

Описание функций

Виртуальная сетевая консоль (VNC, Virtual Network Console) предназначена для предоставления доступа к серверу и выполнения различных операций на сервере в режиме реального времени с помощью клавиатуры, мыши и монитора локального ПК (клиента).

На странице VNC можно посмотреть и настроить параметры VNC.

Описание параметров

Табл. 1 VNC

Параметр

Описание

VNC

Статус службы VNC.

Данная функция включена по умолчанию.

Порт

Номер порта для службы VNC.

Значения: от 1 до 65535

Значение по умолчанию: 5900

Срок тайм-аута (мин)

Максимальное время ожидания (в минутах) после последней операции, выполненной в интерфейсе VNC. Если в течение этого срока не будет выполнена ни одна операция, произойдет автоматическое отключение интерфейса VNC.

Диапазон значений: от 0 до 480

Значение 0 означает, что время ожидания не ограничено.

Значение по умолчанию: 60

Раскладка клавиатуры

Раскладка клавиатуры ОС, управляемой VNC.

Значения:

  • Японская
  • Английская (США)
  • Немецкая

Значение по умолчанию: Японская

Пароль VNC

Пароль для входа в интерфейс VNC.

Значения:

  • Если проверка сложности пароля отключена, то в качестве пароля VNC может использоваться строка длиной от 1 до 8 символов, состоящую из цифр, букв и специальных символов.
  • Если проверка сложности пароля включена, то должны выполняться следующие условия:
    • Пароль должен состоять из 8 символов.
    • Пароль должен соответствовать следующим требованиям сложности:
    • Пароль должен содержать, как минимум, один пробел или один из следующих специальных символов:

      `~!@#$%^&*()-_=+\|[{}];:'",<.>/?

    • Пароль должен включать символы, как минимум, двух видов:
      • Прописные буквы: от A до Z
      • Строчные буквы: от a до z
      • Цифры: от 0 до 9

Подтвердить пароль VNC

Повторный ввод пароля для подтверждения.

Срок действия пароля (дн.)

Период времени, в течение которой действует пароль VNC.

Правила входа

Правила входа, применяемые к пользователям VNC.

К пользователям VNC применимы правила входа, настроенные для локальных пользователей.

Шифрование SSL

Включение и отключение шифрования SSL.

В целях обеспечения безопасности рекомендуется включить данную функцию. Если шифрование SSL отключено, клиент VNC будет инициировать процесс удаленного кадрового буфера (RFB, Remote Frame Buffer).

Данная функция включена по умолчанию.

ПРИМЕЧАНИЕ:

Если шифрование SSL включено, только клиенты VNC с включенной функцией шифрования SSL смогут подключаться к ОС сервера. Если клиент VNC не поддерживает функцию шифрования SSL, то для ее реализации используется туннелирование SSL.

Максимальное количество сеансов

Максимальное количество пользователей, которым разрешено получить доступ к интерфейсу VNC.

Параметр имеет фиксированное значение 5.

Активные сеансы

Количество пользователей, одновременно получивших доступ к интерфейсу VNC.
