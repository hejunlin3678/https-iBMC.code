Виртуальная консоль

Описание функций

Служба KVM (Keyboard, Video, Mouse) используется для дистанционного управления рабочим столом сервера в режиме реального времени. После включения KVM пользователь может выполнять различные операции на сервере с помощью клавиатуры, монитора и мыши локального компьютера (клиента).

На странице Виртуальная консоль можно посмотреть и настроить параметры KVM.

Описание параметров

Табл. 1 Виртуальная консоль

Параметр

Описание

KVM

Статус службы KVM.

Данная функция включена по умолчанию.

Порт

Номер порта для службы KVM.

Значения: от 1 до 65535

Значение по умолчанию: 2198

Срок тайм-аута (мин)

Максимальное время ожидания (в минутах) после последней операции на удаленной консоли (включая операции чтения данных на виртуальном CD-диске). Если в течение этого срока не будет выполнена ни одна операция, произойдет автоматическое отключение удаленной консоли.

Диапазон значений: от 0 до 480

Значение 0 означает, что время ожидания не ограничено.

Значение по умолчанию: 60

Параметр является обязательным для настройки.

Локальная KVM-консоль

Включение и отключение локальной KVM-консоли.

Данная функция включена по умолчанию.

  • Если функция включена, то к рабочему столу сервера можно получить доступ с помощью локальной KVM-консоли или удаленной виртуальной консоли.
  • Если функция отключена, доступ к рабочему столу сервера можно получить только с помощью удаленной виртуальной консоли.

Постоянно подключенные клавиатура и мышь

Включение и отключение постоянно подключенных клавиатуры и мыши.

Данная функция включена по умолчанию.

  • После включения функции виртуальная клавиатура BMC и мышь будут всегда подключены к USB-контроллеру UHCI BMC.
  • Если функция отключена, то виртуальная клавиатура BMC и мышь будут динамически подключаться к контроллеру UHCI BMC только при запуске приложения удаленной консоли и подключении к BMC. Данная функция позволяет снизить энергопотребление в режиме ожидания ОС сервера и при отсутствии подключения мыши и виртуальной клавиатуры USB.

Автоматическая блокировка ОС

Настройка автоматической блокировки ОС.

По умолчанию она отключена.

  • Если она включена, ОС будет автоматически заблокирована при выходе из системы последнего пользователя.
  • Если она отключена, функция автоматической блокировки ОС не поддерживается.
ПРИМЕЧАНИЕ:

Этот параметр вступает в силу только после запуска ОС. В интерфейсе BIOS перед выходом из удаленной виртуальной консоли необходимо выйти из BIOS вручную.

Режим блокировки

Режим блокировки ОС.

Эта опция доступна только при включенной автоматической блокировке ОС.

  • Настройка: использование режима, настроенного пользователем.
  • Windows: использование режима блокировки Windows.

Значение по умолчанию — Настройка.

Сочетание клавиш

Сочетание клавиш используется для блокировки ОС.

Необходимо установить этот параметр, когда Режим блокировки является Настройка.

Сочетание клавиш может состоять из цифр (от 0 до 9), букв (от a до z), некоторых специальных символов и функциональных клавиш.

ПРИМЕЧАНИЕ:

Поддерживаемые специальные символы:

`-=/*+.[];'\,

Максимальное количество сеансов

Максимальное количество пользователей, которым разрешено использовать удаленную консоль для подключения к системе сервера. Параметр имеет фиксированное значение 2.

Активные сеансы

Количество пользователей, одновременно подключенных к серверу через удаленную консоль.
