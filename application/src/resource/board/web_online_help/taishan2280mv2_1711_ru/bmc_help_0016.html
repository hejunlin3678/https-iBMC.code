Тревоги и журнал событий

Описание функций

На странице Тревоги и журнал событий доступны следующие операции:

  • Запрос данных по активным аварийным сигналам сервера.
  • Просмотр и поиск событий системы, а также загрузка и удаление всех системных событий.

Описание параметров

Табл. 1 Текущие тревоги и системные события

Параметр

Описание

№

Порядковый номер события.

Уровень серьезности

Уровень серьезности сбоя.

  • : критический сбой, в результате которого может произойти отключение сервера и прерывание работы всех системных служб. При появлении критических аварийных сигналов необходимо немедленно предпринять меры по локализации и устранению неисправностей.
  • : серьезный сбой, который оказывает значительное влияние на работу системы. Неисправность такого уровня может привести к системному сбою или прерыванию работы служб.
  • : незначительный сбой, который не оказывает особого влияния на работу системы. Однако необходимо как можно скорее предпринять меры по их устранению, чтобы предотвратить более серьезную неисправность.
  • : события, которые происходят во время работы системы.

Поиск системных событий

  1. Выберите Обслуживание > Тревоги и журнал событий, а затем перейдите на вкладку Системные события.
  2. Нажмите Критерий поиска.

    Откроется страница, на которой можно настроить критерии поиска.

  3. Введите критерий поиска. Более подробная информация о параметрах приводится в Табл. 2.

    Табл. 2 Критерий поиска

    Параметр

    Описание

    Уровень серьезности тревог

    Уровень серьезности системного события.

    Значения: Все, Срочный, Значительный, Незначительный или нормальный

    Тип объекта

    Тип компонента, для которого генерируется системное событие.

    Значение параметра может отличаться для разных моделей сервера.

    Время возникновения

    Дата и время возникновения системного события.

    Значения:

    • ВСЕ
    • Сегодня
    • Последние 7 дней
    • Последние 30 дней
    • Настроить
    ПРИМЕЧАНИЕ:

    При выборе Настроить необходимо указать начальную и конечную даты для интервала поиска.

    Другое

    Настройка критерия поиска.

    Настроить можно следующие параметры:

    • Описание события
    • Код события с указанием или без 0x.

  4. Нажмите Запрос.

    На экране отобразится информация о системных событиях, соответствующих критерию поиска.

Удаление всех системных событий

После удаления системные события восстановить нельзя. Будьте внимательны при удалении системных событий.

  1. Выберите Обслуживание > Тревоги и журнал событий, а затем перейдите на вкладку Системные события.
  2. Нажмите кнопку в правом верхнем углу страницы.

    Все системные события будут удалены.

Загрузка системных событий

  1. Выберите Обслуживание > Тревоги и журнал событий, а затем перейдите на вкладку Системные события.
  2. Нажмите кнопку в правом верхнем углу страницы.

    Загруженный файл автоматически сохраняется в каталог по умолчанию на локальном ПК.
