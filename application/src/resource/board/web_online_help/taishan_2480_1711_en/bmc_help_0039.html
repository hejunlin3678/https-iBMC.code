Time Zone & NTP

Function Description

On the Time Zone & NTP page, you can:

  • View and set time zone information.
  • View and set NTP information

Parameter Description

Table 1 Time Zone & NTP

Parameter

Description

Time Zone

Time zone of the BMC

Set Region, Time Zone, and DST based on actual requirements.

Default value: Others+UTC+enabled

NOTE:
  • The time zone information is automatically obtained if Automatically obtain NTP information through DHCPv4 is selected.
  • If the DST is enabled, in the time zones that use daylight saving time (DST), the BMC automatically adjusts the time one hour forward when the DST starts and adjusts the time backward to standard time when the DST ends.
  • When performing time synchronization on the OS, run the hwclock --utc -w command. This command can ensure consistency between the OS time and the BMC time.

NTP

NTP

NTP allows the server to synchronize time with the BMC NTP server.

It is disabled by default.

Automatically obtain address
  • Automatically obtain IPv4 address: Click this option to allow IPv4 address to be automatically allocated for the NTP server.
  • Automatically obtain IPv6 address: Click this option to allow IPv6 address to be automatically allocated for the NTP server.
  • Manually set IP address: Click this option to manually set the NTP servers.

The default value is Manually set IP address.

NOTICE:
  • If the IP address of the BMC management network port is automatically obtained, you can obtain the NTP information automatically or manually. Automatic obtaining is recommended.
  • If the IP address of the BMC management network port is set manually, the NTP information must be set manually.
  • Server 1 to Server 3
  • Preferred Server 1 to Preferred Server 3 or Alternate Server 1 to Alternate Server 3

Addresses of NTP servers.

Value: an IPv4 or IPv6 address or a domain name

NOTE:
The domain name must meet the following requirements:
  • Contain 0 to 67 characters.
  • Allow digits, letters, hyphens (-), and dots (.) only.
  • Cannot start with a hyphen or dot, or end with a hyphen.
  • Allow a maximum of 63 characters between any two dots.
Either of the following solutions are provided. For details, see the GUI.
  • Server 1 to Server 3: Three NTP servers are available. The three servers are effective at the same time.
  • Preferred Server 1 to Preferred Server 3 or Alternate Server 1 to Alternate Server 3: Three NTP server groups are available. In each group, the left is the preferred server, and the right is the alternate server. The servers are selected according to the following rules:
    • Select one server from each group.
    • If both addresses in a group are invalid, the server group will be discarded.
    • If there is only one valid address in a group, select the server with the valid address.
    • If both addresses in a group are valid, select the server with the IPv6 address. If both of them are IPv4 or IPv6 addresses, select the preferred server (left one).
NOTE:

The switchover between the active and standby NTP servers depends on the synchronization interval between the BMC and the NTP server (Minimum Polling Interval ≤ synchronization interval ≤ Maximum Polling Interval). If the BMC fails to synchronize with the active server for multiple times, the standby server becomes the NTP server.

Minimum Polling Interval

Minimum interval at which the BMC system synchronizes time from the NTP server, that is, the minimum interval for polling NTP packets.

For example, if this parameter is set to 6, the polling interval is 2 to the power of 6, that is, 1 minute and 4 seconds.

Value: 3 to 17

The default value is 6(1 minute and 4 seconds).

Maximum Polling Interval

Maximum interval at which the BMC system synchronizes time from the NTP server, that is, the maximum interval for polling NTP packets.

For example, if this parameter is set to 6, the polling interval is 2 to the power of 6, that is, 1 minute and 4 seconds.

Value: 3 to 17

The default value is 10(17 minute and 4 seconds).

Server Authentication

Authentication, which can be enabled or disabled, for communication between the BMC server and the NTP server.

It is disabled by default.

NTP Group Key

Private key to be uploaded to the BMC for identity authentication if Server Authentication is enabled.

NOTE:
  • You can download a key generator (for example, ntp-keygen) to generate private keys. The key file is in the .keys format. The size cannot exceed 2 MB.
  • For security purposes, periodically update the key.

Setting the Time Zone

  1. Select a value from Region and Time Zone.

    For details about the parameters, see Table 1.

  2. Click Save.

    If "Operation Successful" is displayed, the setting is successful.

    • When performing time synchronization on the OS, run the hwclock --utc -w command. This command can ensure consistency between the OS time and the BMC time.
    • After changing the time zone, you need to restart the BMC to have the system time take effect for the CPU usage, memory usage, network port bandwidth usage, drive usage, power history, and air inlet temperature curves displayed on the BMC WebUI.

Setting NTP Information

  1. In NTP Settings area, set parameters based on service requirements.

    For details about the parameters, see Table 1.

  2. Click Save.

    If "Operation Successful" is displayed, the setting is successful.
