VNC

Présentation de la fonction

Vous pouvez utiliser la Console réseau virtuelle (VNC) pour accéder à un serveur et gérer le serveur en temps réel en utilisant le clavier, la souris et le moniteur d'un PC local (client).

Sur la page VNC, vous pouvez consulter et configurer les paramètres VNC.

Description du paramètre

Tableau 1 VNC

Paramètre

Description

VNC

Statut du service VNC.

Il est désactivé par défaut.

Port

Numéro de port pour le service VNC.

Valeur : 1 à 65535

Valeur par défaut : 5900

Dépassement du délai (min)

Temps d'inactivité maximum (en minutes) après la dernière opération sur l'interface VNC. Si aucune opération n'est effectuée dans le délai spécifié, le système se déconnecte automatiquement de l'interface VNC.

Plage de valeurs : 0 à 480

La valeur 0 indique une durée illimitée.

Valeur par défaut : 60

Disposition du clavier

Disposition du clavier du système d'exploitation contrôlée par le VNC.

Valeur :

  • Japonais
  • Anglais (États-Unis)
  • Allemand

Valeur par défaut : Japonais

Mot de passe VNC

Mot de passe pour se connecter à l'interface VNC.

Valeur :

  • Si la vérification de la complexité du mot de passe est désactivée, le mot de passe VNC est une chaîne de 1 à 8 caractères, y compris des chiffres, des lettres et des caractères spéciaux.
  • Si la vérification de la complexité du mot de passe est activée, le mot de passe VNC doit répondre aux exigences suivantes :
    • Contient 8 caractères.
    • Répondre aux exigences de complexité suivantes :
    • Contient au moins un espace ou l'un des caractères spéciaux suivants :

      `~!@#$%^&*()-_=+\|[{}];:'",<.>/?

    • Contient au moins deux des éléments suivants :
      • Lettres majuscules de A à Z
      • Lettres minuscules de a à z
      • Chiffres 0 à 9

Confirmer le mot de passe VNC

Saisissez à nouveau le nom de la communauté par souci de cohérence.

Validité du mot de passe (jours)

Durée de validité du mot de passe VNC.

Règles de connexion

Règles de connexion appliquées aux utilisateurs VNC.

Les règles de connexion configurées pour les utilisateurs locaux s'appliquent également aux utilisateurs VNC.

Cryptage SSL

Fonction permettant d'activer ou de désactiver le cryptage SSL.

Activez cette fonction pour des raisons de sécurité. Si le cryptage SSL est désactivé, le client VNC démarre le processus RFB (Remote Frame Buffer).

Il est activé par défaut.

REMARQUE:

Si le cryptage SSL est activé, seuls les clients VNC avec le cryptage SSL activé peuvent se connecter au système d'exploitation du serveur. Si le client VNC ne fournit pas l'option de cryptage SSL, utilisez une application de tunneling SSL pour implémenter le cryptage SSL.

Nombre maximum de sessions

Nombre maximum d'utilisateurs autorisés à accéder à l'interface VNC.

Ce paramètre a une valeur fixe de 5.

Sessions actives

Nombre d'utilisateurs qui accèdent simultanément à l'interface VNC.
