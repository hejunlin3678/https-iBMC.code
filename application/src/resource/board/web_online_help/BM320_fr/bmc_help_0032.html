Services Web

Présentation de la fonction

La page Services Web vous permet d'effectuer les opérations suivantes :

  • Voir et définir les paramètres de base du service Web
  • Voir le certificat SSL utilisé.
  • Personnaliser et importer des certificats SSL.

Le certificat SSL configure un canal de sécurité SSL via HTTPS entre le navigateur Web sur le client et le serveur Web pour transmettre des données cryptées entre le client et le serveur et empêcher la divulgation des données. SSL garantit la sécurité des informations transmises et sert à vérifier l'authenticité du site Web auquel vous souhaitez accéder. Les serveurs vous permettent de remplacer les certificats SSL. Pour des raisons de sécurité, remplacez le certificat et les clés d'origine par votre certificat et votre paire de clés publiques et privées personnalisés, et mettez à jour rapidement le certificat.

  • Le certificat SSL peut être un seul certificat SSL ou une chaîne de certificats de moins de 10 niveaux.
  • Le fichier de certificat d'importation prise en charge est au format .crt, .cer, .pem, .pfx et .p12., et le fichier de certificat au format .crt, .cer ou .pem ne doit pas dépasser 1 Mo et celui au format .pfx ou .p12 ne doit pas dépasser 100 Ko.
  • Le MD5 et le SHA1 sont des algorithmes de signature faibles, qui présente des risques de sécurité. Le BMC ne prend pas en charge l'import des certificats MD5 et SHA1.

Paramètres de base

HTTP

Prend en charge la navigation sur Internet et traduit les pages HTTP (Hypertext Transfer Protocol). Le service Serveur Web (HTTP) est activé par défaut pour établir une connexion entre le navigateur et le BMC. Une fois la connexion établie, le protocole sécurisé HTTPS est utilisé.

Valeur par défaut : 80.

REMARQUE:

Si le service HTTP est désactivé, il ne peut pas être automatiquement commuté sur le service HTTP après avoir saisis http:l'adresse IP du port réseau de gestion BMC dans la zone adresse du navigateur.

HTTPS

Un serveur fournissant un service de navigation en ligne peut analyser le protocole de transfert hypertexte （HTTPS, Protocole de transfert hypertexte sur couche de socket sécurisée）et le protocole Redfish.

Valeur par défaut : 443.

REMARQUE:

Si le service HTTP est désactivé, vous ne pouvez pas vous connecter à l'interface utilisateur Web BMC à l'aide d'un navigateur.

Port

Numéro de port utilisé pour un service.

Plage de valeurs : 1 à 65535

Dépassement du délai (min)

Période d'inactivité maximale (en minutes) après laquelle l'utilisateur sera déconnecté de l'interface utilisateur Web BMC.

Plage de valeurs : 5 à 480

Valeur par défaut : 5

Mode de la session
Mode dans lequel un compte utilisateur peut être utilisé pour se connecter à l'interface utilisateur Web BMC
  • Partagé : Chaque compte utilisateur peut être utilisé pour se connecter à l'interface utilisateur Web BMC à partir de jusqu'à quatre clients en même temps.
  • Exclusif : Chaque compte utilisateur peut être utilisé pour se connecter à l'interface utilisateur Web BMC à partir d'un client à tout moment.

La valeur par défaut est Exclusif.

Certificat SSL

Émis par

Informations sur l'émetteur du certificat SSL, notamment :

  • CN : Nom d'utilisateur.
  • OU : Département de l'utilisateur.
  • O : société ou organisation de l'utilisateur.
  • L : ville de l'utilisateur.
  • S : province ou état de l'utilisateur.
  • C : pays de l'utilisateur.

Émis pour

Informations sur l'utilisateur (utilisateur actuel BMC) du certificat SSL. Les contenus dans la domaine Émis par sont les mêmes que ceux dans la domaine Émis pour.

REMARQUE:

Définissez CN sur le nom de domaine complet (FQDN) du serveur, c'est-à-dire nom d'hôte.nom de domaine.

Période de validité

Période de validité du certificat SSL.

Numéro de série

Numéro de série du certificat SSL, utilisé pour l'identification et la migration du certificat.

Personnalisation des informations du certificat SSL et importation d'un certificat SSL

  • Effectuez cette opération lorsque vous souhaitez demander un certificat SSL.
  • Pour des raisons de sécurité, mettez périodiquement à jour le certificat.
  1. Dans la zone Certificat SSL, cliquez sur Personnalisé.
  2. Sélectionnez Générer un CSR, saisissez les informations de demande de CSR (certificate signing request), puis cliquez sur Générer.
  3. Envoyez le fichier CSR exporté à l'autorité de certification SSL et demandez un certificat SSL.

    Après avoir obtenu le certificat SSL formel, enregistrez-le sur le client.

  4. Sélectionnez Importer le certificat du serveur dans la boîte de dialogue Personnaliser le certificat.
  5. Sélectionnez le certificat SSL officiel du client local.

    • Le fichier de certificat d'importation prise en charge est au format .crt, .cer, .pem, .pfx et .p12., et le fichier de certificat au format .crt, .cer ou .pem ne doit pas dépasser 1 Mo et celui au format .pfx ou .p12 ne doit pas dépasser 100 Ko.
    • Le MD5 et le SHA1 sont des algorithmes de signature faibles, qui présente des risques de sécurité. Le BMC ne prend pas en charge l'import des certificats MD5 et SHA1.

  6. Cliquez sur Confirmer.
  7. Dans la zone d'édition Mot de passe du certificat, saisissez le mot de passe du certificat.
  8. Cliquez sur Confirmer.

    Le certificat prend effet immédiatement après son importation.

    Le fichier CSR généré sur mesure a une correspondance un à un avec le certificat de serveur appliqué à partir de l'organisation de l'autorité de certification. Ne générez pas de nouveau fichier CSR avant d'importer le certificat de serveur. Sinon, le fichier CSR d'origine est écrasé par le nouveau fichier CSR et ne peut être récupéré. Vous devez utiliser le nouveau fichier CSR pour demander un nouveau certificat de serveur auprès de l'organisation de l'autorité de certification.

  9. Connectez-vous à nouveau à l'interface Web BMC.

Importation d'un certificat SSL

  • Effectuez cette opération uniquement lorsqu'un certificat SSL est disponible sur le client.
  • Pour des raisons de sécurité, utilisez un algorithme de chiffrement sécurisé, par exemple RSA2048, pour chiffrer le certificat SSL personnalisé.
  • Pour des raisons de sécurité, mettez périodiquement à jour le certificat.
  1. Dans la page Certificat SSL, cliquez sur Personnalisé.

    La page de personnalisation des informations du certificat SSL s'affiche.

  2. Sélectionnez Importer le certificat du serveur.
  3. Sélectionnez le fichier de certificat SSL à importer.

    • Le fichier de certificat d'importation prise en charge est au format .crt, .cer, .pem, .pfx et .p12., et le fichier de certificat au format .crt, .cer ou .pem ne doit pas dépasser 1 Mo et celui au format .pfx ou .p12 ne doit pas dépasser 100 Ko.
    • Le MD5 et le SHA1 sont des algorithmes de signature faibles, qui présente des risques de sécurité. Le BMC ne prend pas en charge l'import des certificats MD5 et SHA1.

  4. Cliquez sur Confirmer.
  5. Dans la zone d'édition Mot de passe du certificat, saisissez le mot de passe du certificat.
  6. Cliquez sur Confirmer.

    Le certificat prend effet immédiatement après son importation.

    Si la taille du fichier à télécharger dépasse 1 Mo, un message indiquant un échec de la demande de page s'affiche. Vous pouvez actualiser la page pour résoudre ce problème.

  7. Connectez-vous à nouveau à l'interface Web BMC.
