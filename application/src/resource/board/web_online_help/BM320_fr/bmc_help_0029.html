Gestion de la sécurité

Présentation de la fonction

La page Gestion de la sécurité vous permet d'effectuer les opérations suivantes :

  • Voir et configurer les règles de renforcement de la sécurité utilisateur pour les utilisateurs BMC.
  • Voir et gérer les permissions des utilisateurs BMC locaux.

Description du paramètre

Tableau 1 Sécurité

Parameter

Description

Mode de verrouillage du système

Mode de verrouillage du système BMC, qui peut être activé ou désactivé.

Cette fonction est désactivée par défaut.

Une fois cette fonction activée, toute tentative de modification des configurations suivantes du système sera bloquée :

  • Importation de configuration
  • Gestion du BMA
  • Paramètre de l'option de démarrage dans le BIOS
  • Mise à jour du micrologiciel : redémarrage, basculement d'image et mise à jour
  • Paramètres communs : gestion du stockage, configuration du réseau, informations sur le produit, gestion de SmartProvisioning, seuil de surveillance, langue, fuseau horaire et NTP
  • Contrôle de l'alimentation : réglage de l'alimentation, re-collection et suppression de statistiques de l'alimentation, et réglage intelligent de la vitesse du ventilateur
  • Paramètres de base pour l'utilisateur : gestion des licences et paramètres SNMP
  • Paramètres de base du service Web et de la fonction VNC
  • Paramètres de base de la console virtuelle et des médias virtuels
  • Paramètres de sécurité de base (à l'exclusion du mode de verrouillage du système)
  • Paramètres de base du débogage et du diagnostic : journaux système, capture d'écran et enregistrement vidéo, signalisation des alarmes
REMARQUE:

Cette fonction n'est disponible que lorsqu'une licence de niveau avancé est appliquée. Seul l'administrateur a l'autorisation de configurer cette fonction.

Gestion des utilisateurs du système d'exploitation

Fonction de gestion des utilisateurs sur le système de service.

Si cette fonction est désactivée, les commandes IPMI envoyées par le système de service pour la gestion des utilisateurs, telles que l'ajout ou la suppression d'utilisateurs, la définition des autorisations et des mots de passe, deviennent invalides.

Il est activé par défaut.

  • S'il est activé, le côté service peut gérer les utilisateurs BMC.
  • S'il est désactivé, le côté service ne peut pas gérer les utilisateurs BMC.

Pour des raisons de sécurité, définissez ce paramètre sur Désactiver.

Vérification du mot de passe

La vérification du mot de passe permet de vérifier si les mots de passe répondent aux exigences de complexité. Il est activé par défaut.

La fonction de vérification du mot de passe est activée par défaut. La vérification comprend :
  • Le contrôle de la complexité des mots de passe des utilisateurs locaux, des noms de communauté Trap, des noms de communauté SNMPv1/v2c, des mots de passe de cryptage SNMPv3 et des mots de passe VNC.
  • Le contrôle de la longueur minimale des mots de passe des utilisateurs locaux et des mots de passe de cryptage SNMPv3.
REMARQUE:
  • Pour des raisons de sécurité, activez la vérification du mot de passe.
  • Si la vérification du mot de passe faible est activée, le mot de passe ne peut pas être identique à celui contenu dans le dictionnaire de mots de passe faibles. (Vous pouvez exécuter la commande ipmcset -t user -d weakpwddic -v export pour exporter les mots de passe faibles du dictionnaire de mots de passe faible.)

Authentification par mot de passe SSH

L'authentification par mot de passe SSH permet aux utilisateurs de se connecter à le BMC via SSH à l'aide du mot de passe ou de la clé publique.

  • Désactiver : permet aux utilisateurs de se connecter via SSH en utilisant uniquement des clés publiques.
  • Activer : permet aux utilisateurs de se connecter via SSH en utilisant des mots de passe ou des clés publiques.

Il est active par défaut.

Empêcher le DNS rebinding

Permet aux utilisateurs d'activer ou désactiver la fonction d'empêcher le DNS rebinding.

Après avoir activé cette fonction, vous ne pouvez accéder à le BMC qu'en utilisant le nom d'hôte, le FQDN ou l'adresse IP configurés sur le BMC.

Cette fonction est désactivée par défaut.

Versions TLS

Version du protocole TLS utilisée pour assurer la sécurité et l'intégrité des données pendant la communication entre deux applications.

Le TLS peut être activé pour assurer une connexion sécurisée entre un navigateur Web et un serveur Web.

  • TLS 1.2 ou version ultérieure : TLS 1.2 ou 1.3 est pris en charge.
  • TLS 1.3 uniquement : Seul TLS 1.3 est pris en charge.

La valeur par défaut est LS 1.2 ou version ultérieure.

REMARQUE:
  • Si seul TLS 1.3 est activé, AdoptOpenJDK 11 doit être utilisé pour accéder à la Console distante intégrée Java.
  • Cette option est valable uniquement pour les services Web et Redfish de l'BMC.

Validité du mot de passe (jours)

Période de validité (en jours) d'un mot de passe utilisateur.

Plage de valeurs : 0 à 365

La valeur 0 indique que le mot de passe n'expire jamais.

Valeur par défaut: 0

REMARQUE:

Pour des raisons de sécurité, définissez une période de validité du mot de passe appropriée et modifiez le mot de passe périodiquement.

Longueur minimale du mot de passe

Longueur minimale du mot de passe de l'utilisateur local ou du mot de passe de cryptage SNMPv3.

Ce paramètre n'est valable que lorsque la fonction de vérification du mot de passe est activée.

Plage de valeur : 8 à 20

Valeur par défaut : 8

Âge minimum du mot de passe (jours)

Durée minimale (en jours) pendant laquelle le mot de passe doit être utilisé. Le mot de passe ne peut pas être modifié pendant cette période.

Plage de valeurs : 0 à 365

La valeur 0 indique que les mots de passe n'ont pas d'âge minimum.

Valeur par défaut: 0

REMARQUE:

L'âge minimum du mot de passe doit être au moins dix jours avant la date d'expiration du mot de passe.

  • Si l'expiration du mot de passe (jours) est inférieure ou égale à 10, l'âge minimal du mot de passe (jours) ne peut être que 0.
  • Si l'âge minimum du mot de passe (jours) est égal ou supérieur à 355, l'expiration du mot de passe (jours) ne peut être que 0.

Limite de temps inactive (jours)

Période d'inactivité maximale (en jours) après laquelle le compte utilisateur sera désactivé.

Valeur :

  • 0
  • 30 à 365

La valeur 0 indique une durée illimitée, c'est-à-dire que les comptes d'utilisateurs inactifs ne seront jamais désactivés.

Valeur par défaut: 0

Utilisateur de connexion d'urgence

Nom d'utilisateur pour se connecter à le BMC en cas d'urgence.

Cet utilisateur n'est soumis à aucune règle de connexion ou interface de connexion, et le mot de passe de cet utilisateur n'expirera jamais.

La valeur par défaut est NULL.

REMARQUE:
  • Seul un administrateur peut être défini en tant qu'utilisateur de connexion d'urgence.
  • Seuls les administrateurs peuvent voir les utilisateurs de connexion d'urgence.

Mots de passe précédents non autorisés

Nombre de mots de passe précédents qui ne peuvent pas être réutilisés comme nouveau mot de passe.

Plage de valeurs : 0 à 5

La valeur 0 indique que tous les mots de passe précédents sont autorisés.

Valeur par défaut: 5

Politique de verrouillage des utilisateurs

Nombre maximum de tentatives de connexion invalides consécutives et durée du verrouillage du compte..

  • Le nombre maximum de tentatives de connexion non valides consécutives autorisées est un nombre entier compris entre 1 et 6 ou Illimité (verrouillage du compte désactivé), et la valeur par défaut est 5.
  • La durée de verrouillage du compte (en minutes) est un entier compris entre 1 et 30, et la valeur par défaut est 5.

Une fois qu'un compte utilisateur est verrouillé, l'utilisateur ne peut tenter de se connecter qu'après l'expiration de la durée de verrouillage du compte.

REMARQUE:
  • Pour des raisons de sécurité, activez la fonction de verrouillage du compte.
  • Pour déverrouiller un compte utilisateur en cas d'urgence, exécutez la commande de déverrouillage sur le CLI. Pour plus de détails, consultez le Guide de l'utilisateur BMC du serveur.

Notification d'expiration du certificat (jours)

Nombre de jours à l'avance, les utilisateurs sont informés que les certificats BMC sont sur le point d'expirer. Par exemple, s'il est réglé sur 7, une alarme sera signalée lorsque le délai d'expiration du certificat est inférieur ou égal à 7 jours.

Plage de valeurs : 7 à 180

Valeur par défaut : 90
Tableau 2 Règles de connexion

Paramètre

Description

Nom

Nom d'une règle.

Temps

Période durant laquelle les utilisateurs sont autorisés à se connecter.

La valeur peut être dans l'un des formats suivants :

  • AAAA-MM-JJ :

    Exemple de valeur : 2013-08-30 au 2013-12-30

  • HH:MM

    Exemple de valeur : 08:30 à 20:30

  • AAAA-MM-JJ HH:MM

    Exemple de valeur : 2013-08-30 08:30 au 2013-12-30 20:30

REMARQUE:
  • L'année de début et l'année de fin doivent être entre 1970 et 2050.
  • Les heures de début et de fin d'une règle de connexion doivent être au même format.

IP

Adresse IP ou plage d'adresses IP autorisées pour la connexion. La valeur peut être dans l'un des formats suivants :

  • Adresse IPv4 (xxx.xxx.xxx.xxx): indique une adresse IP.
  • IPv4 / masque de sous-réseau (xxx.xxx.xxx.xxx/masque) : indique un segment d'adresse IP.

MAC

Adresse MAC ou plage d'adresses MAC autorisée pour la connexion. La valeur peut être dans l'un des formats suivants :

  • Xx:xx:xx:xx:xx:xx : indique une adresse MAC.
  • xx:xx:xx : indique un segment d'adresse MAC.

Statut

État actuel du règles de connexion.

Il est désactivé par défaut.

Opération

Cliquez sur Modifier. Les champs Temps, IP, MAC et Statut de la règle de connexion sont modifiables.

Configuration des règles de connexion

Le BMC prend en charge jusqu'à trois règles de connexion. Les utilisateurs qui respectent l'une des trois règles peuvent se connecter à le BMC.

Une règle de connexion est effective pour les utilisateurs locaux, les groupes LDAP, les services SNMPv3 ou les interfaces de CLP (ssh), KVM_VMM, RMCP et Redfish uniquement si les conditions suivantes sont remplies :

  • La règle de connexion est configurée et activée dans la zone Règles de connexion.
  • La règle de connexion est sélectionnée dans la zone de configuration.
  • Si une règle de connexion est vide et que le statut de la règle est activé et sauvegardé, il n'y a pas de restriction de connexion.
  • Si la zone de texte de la règle de connexion est vide, il n'y a pas de restriction de règle de connexion.
  1. Dans la zone Règles de connexion, cliquez sur Modifier dans la colonne Opération.
  2. Définissez le statut de la règle sur .
  3. Définissez les règles de connexion. Pour plus de détails sur les paramètres, voir Tableau 2.
  4. Cliquez sur Enregister.

Configuration des rôles personnalisés

Les autorisations d'opération des rôles par défaut (administrateur, opérateur et utilisateur commun) ne peuvent pas être modifiées, mais l'administrateur peut définir les autorisations d'opération pour les rôles personnalisés.

  1. Dans la liste de fonctions, sélectionnez les modules pour les rôles personnalisés.

    Le Tableau 3 décrit les autorisations.

  2. Cliquez sur Sauvegarder.

    Dans la boîte de dialogue affichée, entrez le mot de passe de l'utilisateur et cliquez sur OK.

Définition de la bannière de sécurité de connexion

  1. Dans la page à onglet Bannière de sécurité de connexion, définissez Bannière de sécurité sur .
  2. Saisissez les informations de sécurité dans la zone de texte de la bannière de sécurité.
  3. Cliquez sur Enregister.

Restauration de la bannière de sécurité de connexion par défaut

  1. Dans la page à onglet Bannière de sécurité de connexion, définissez Bannière de sécurité sur .
  2. Cliquez sur Restaurer les valeurs par défaut.
  3. Cliquez sur Enregister.
