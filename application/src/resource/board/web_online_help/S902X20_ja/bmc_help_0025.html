LDAP

機能概要

LDAP画面では、Lightweight Directory Access Protocol（LDAP）ユーザー情報を表示および設定できます。

BMCシステムにはLDAPユーザーのアクセス機能が定義されています。ドメインコントローラにあるドメインユーザー、グループドメイン、ユーザードメインに属するLDAPユーザー名およびそのパスワードを使用してBMCシステムにログインすることにより、システムのセキュリティを高めることができます。LDAPユーザーはBMC Web画面にログインしたり、SSH方式でBMC CLIにログインしたりすることができます。

LDAPサーバーでは、DisplayNameとCNは一致する必要があります。

BMCは最大6台のドメインサーバーを同時にサポートします。

BMC WebUIへのログイン時に、ドメインサーバーを手動で指定するか、自動的に検索することができます。BMC CLIへのログイン時に、ドメインサーバーが自動的に検索されます。

BMCはWindows Active Directory (AD)、Linux OpenLDAPとFreeIPAをサポートします。

基本設定

LDAPサーバーアドレス

LDAPサーバーのIPアドレス。

フォーマット:IPv4アドレス、IPv6アドレス、またはドメイン名

証明書の検証が有効になっている場合、このパラメータをLDAPサーバーのFQDN（ホスト名.ドメイン名）に設定し、ネットワーク設定でDNSアドレス情報を設定します。

NOTE:
ドメイン名は次の要件を満たす必要があります。
  • 0～255文字が含まれます。

  • 数字、英字、ハイフン（-）、およびピリオド（.）のみが使用できます。

  • ドメイン名の最初にハイフンまたはピリオドを使ったり、ドメイン名の最後にハイフンを使ったりすることはできません。

  • 2つのピリオドの間に最大63文字が使用できます。

LDAPSポート

LDAPサービスのポート番号。

設定範囲:1～65535の整数

デフォルト値:636

NOTE:

BMCは暗号化転送を利用するLDAPサービスのみをサポートします。LDAPサーバーでは関連する構成を行う必要があります。

ドメイン

ドメインコントローラに定義されているLDAPユーザーが属するユーザードメイン。

ドメイン名は次の要件を満たす必要があります。
  • 0～255文字が含まれます。

  • 数字、英字、ハイフン（-）、およびピリオド（.）のみが使用できます。

  • ドメイン名の最初にハイフンまたはピリオドを使ったり、ドメイン名の最後にハイフンを使ったりすることはできません。

  • 2つのピリオドの間に最大63文字が使用できます。

バインドDN

LDAPプロキシユーザーの識別名（DN）。

たとえば、CN=username,OU=company,DC=domain,DC=com。LDAPサーバーで設定されているDNと同じにする必要があります。

値の範囲：255バイトの文字列（ここでは64～255文字）。長さは、エンコードのタイプによって異なります。

バインドパスワード

LDAPプロキシーユーザーの認証パスワード。

値の範囲：数字、英字、特殊文字を含む1～20文字です。

ユーザーフォルダ

BMCにログインできるLDAPユーザーのLDAPサーバーのディレクトリ。

フォーマット：CN=xxxまたはOU=xxx

複数レベルのノードがある場合、上位レベルのノードは下位レベルのノードの後に続き、コンマで区切られます。

たとえば、ユーザーinfotestがLDAPサーバーの\testusers\part1にある場合、OU=part1、OU=testusersと入力します。

NOTE:

CNとOUの違いについては、LDAPプロトコルの説明を参照してください。

たとえば、Windows ADでは、次のようになります。

  • CN（TypeがContainerの場合）
  • OU（TypeがOrganizational Unitの場合）

値の範囲：255バイトの文字列（ここでは64～255文字）。長さは、エンコードのタイプによって異なります。

LDAP証明書の検証

LDAPサーバーの証明書検証。有効または無効にすることができます。

セキュリティ上の理由で、証明書の検証を有効にすることをお勧めします。

証明書の検証が有効になっている場合、LDAP CA証明書をインポートし、LDAPサーバーにAD、DNS、およびCA証明書の発行元をインストールしてCA証明書をLDAPサーバーとBMCにインポートする必要があります。

デフォルト値:無効

証明書の検証レベル

LDAP証明書検証のレベル。

  • Demand:クライアント証明書が正しくないか、利用できない場合は、BMCへのアクセスを拒否します。セキュリティ上の理由で、デフォルトオプション（Demand）を使用することをお勧めします。
  • Allow:クライアント証明書が正しくないか、利用できない場合でも、BMCへのアクセスを許可します。

デフォルト値:Demand

LDAP証明書

ファイルは、cer、pem、cert、またはcrt形式である必要があります。

NOTE:
  • アップロードされるファイルが100Mを超えると、ページの請求に失敗します。ページを更新すれば、復元できます。
  • 最大10レベルの証明書チェーンがサポートされています。

証明書情報

証明書情報を示します。

証明書チェーンである場合、表示される証明書情報の順序はサーバー証明書 > 中間証明書 > ルート証明書です。

ロール

LDAPグループに割り当てられたロール。

  • [管理者]:[管理者]はすべての操作を実行できるユーザー。
  • [オペレーター]:[オペレーター]は基本管理、リモート管理、リモートメディア、電力制御、情報照会、および自身のパスワード設定の操作権限を持ちます。
  • [一般ユーザー]:[一般ユーザー]は情報照会、自身のパスワード設定の操作権限を持ちます。
  • ［カスタムロール］:［カスタムロール1］～［カスタムロール4］は、指定された操作を実行できます。
NOTE:

新規LDAPユーザーグループのデフォルトユーザー権限は[一般ユーザー]です。

ログインインターフェース

  • [Redfish]:ユーザーはRedfishツールを使用してBMCにログインできます。
NOTE:

新規LDAPユーザーグループはすべてのログインインターフェースをサポートしています。

  3. 証明書失効ファイルを選択します。
  4. 現在ログインしているユーザーのパスワードを入力し、[保存]をクリックします。

LDAPグループの削除

  1. LDAPグループエリアで、削除をクリックします。

    削除確認ダイアログボックスが表示され、その画面に現在のユーザーパスワードの入力が求められます。

  2. 現在のユーザーパスワードを入力します。
  3. [はい]をクリックします。

    操作成功を示すメッセージが表示されます。

LDAPグループの変更

  1. LDAPグループエリアで、LDAPグループの右側にある編集をクリックします。
  2. 現在のユーザーパスワードを入力し、LDAPグループパラメータを変更します。パラメータの詳細については、表 1を参照してください。
  3. 保存をクリックします。

    LDAPグループ情報を変更したり、LDAPグループを削除したりする場合、KVMにログインしたユーザーはシステムから自動的にログアウトされません。これらのKVMユーザーをログアウトするには、[オンラインユーザー]ページに移動してログアウトを行う必要があります。
