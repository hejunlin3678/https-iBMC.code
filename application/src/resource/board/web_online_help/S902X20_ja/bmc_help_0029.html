セキュリティ管理

機能概要

セキュリティ管理画面で、次の操作を実行できます。

  • BMCユーザーのユーザーセキュリティ強化ルールの表示と設定を行います。
  • ローカルBMCユーザー権限の表示と管理を行います。

説明

システムロックモード

BMCシステムロックモード。有効または無効に設定できます。

この機能は、デフォルトでは無効になっています。

この機能を有効にすると、次のようなシステム構成を変更しようとする試みはブロッグされます。

  • 構成のインポート
  • BMA管理
  • BIOSでのブートオプション設定
  • ファームウェアのバージョンアップ：再起動、イメージの切り替え、アップデート
  • 共通設定：ストレージ管理、ネットワーク構成、製品情報、SmartProvisioning管理、監視閾値、言語、タイムゾーン、NTP
  • 電源制御：電源設定、電源統計の再収集とクリア、ファン回転速度のインテリジェント調整
  • 基本ユーザー設定：ライセンス管理、SNMP設定
  • ウェブサービスとVNC機能の基本設定
  • 仮想コンソールと仮想メディアの基本設定
  • 基本セキュリティ設定（システムロックモードを除く）
  • デバッグと診断の基本設定：システムログ、スクリーンショットとビデオ録画、アラーム報告
NOTE:

この機能は、上級ライセンスが適用されている場合にのみ利用可能です。管理者のみがこの機能を設定する権限を持っています。

OSユーザー管理

サービスシステムのユーザー管理機能。

この機能を無効にした場合、サービスシステムから送信されるユーザー管理のためのIPMIコマンド（ユーザーの追加または削除、権限設定、パスワード設定など）は無効となります。

デフォルトで有効になっています。

  • 有効になっている場合、サービス側でBMCユーザーを管理できます。
  • 無効になっている場合、サービス側でBMCユーザーを管理できません。

セキュリティ上の理由で、このパラメータを無効に設定することをお勧めします。

パスワードチェック

パスワードチェックにより、パスワードが複雑度要件を満たしているかどうかを検証します。デフォルトで有効になっています。

パスワードチェック機能はデフォルトで有効になっています。以下のチェックが含まれます。
  • ローカルユーザーパスワード、トラップ用のコミュニティ名、SNMPv1/v2cコミュニティ名、SNMPv3暗号化パスワード、VNCパスワードの強度チェック。
  • ローカルユーザーパスワードとSNMPv3暗号化パスワードの最小文字数チェック。
NOTE:
  • セキュリティ上の理由で、パスワードチェックを有効にすることをお勧めします。
  • 弱いパスワード辞書認証機能を有効にする場合は、パスワードは弱いパスワード辞書に含まれているパスワードと同じにはできません。（弱パスワード辞書から弱いパスワードをエクスポートするには、ipmcset -t user -d weakpwddic -v exportコマンドを実行します。）

SSHパスワード認証

SSHパスワード認証により、ユーザーはパスワードまたは公開鍵を使用して、SSH経由で

BMCにログインできます。

  • 無効:ユーザーは、公開鍵のみを使用してSSH経由でログインできます。
  • 有効:ユーザーは、パスワードまたは公開鍵を使用してSSH経由でログインできます。

デフォルトで有効になっています。

DNSリバインディング攻撃防止

DNSリバインディング攻撃防止機能の有効化または無効化を行います。

この機能を有効にした場合、BMCに設定されているホスト名、FQDN、IPアドレスによってのみBMCにアクセスすることができます。

デフォルトでは無効になっています。

TLSバージョン

2つのアプリケーション間の通信中にデータのセキュリティと整合性を確保するために使用されます。

TLSを有効にすると、WebブラウザとWebサーバー間の安全な接続を確保できます。

  • TLS 1.2以上：TLS 1.2または1.3がサポートされています。
  • TLS 1.3のみ：TLS 1.3のみがサポートされています。

デフォルト値は[TLS 1.2以上]です。

NOTE:
  • TLS 1.3のみが有効になっている場合、Java統合リモートコンソールにアクセスするにはAdoptOpenJDK 11が必要です。
  • このオプションはBMC WebサービスおよびRedfishサービスでのみ有効です。

パスワード有効期限（日）

ユーザーパスワード有効期限（日）。

設定範囲:0～365

0は、パスワードが無期限になることを示します。

デフォルト値：0

NOTE:

セキュリティ上の理由により、適切なパスワード有効期間を設定し、定期的にパスワードを変更してください。

パスワードの最小長さ

ローカルユーザーパスワードまたはSNMPv3暗号化パスワードの最小文字数を示します。

このパラメータはパスワードチェック機能が有効になっている場合にのみ有効です。

値の範囲：8～20

デフォルト値：8

パスワード変更禁止期間（日）

パスワードの最小有効期間（日）。この期間中にパスワードを変更できません。

設定範囲:0～365

0は、パスワードの最小使用期間が無期限になることを示します。

デフォルト値：0

NOTE:

パスワードの最小使用期間は、パスワード有効期限より10日以上短くする必要があります。

  • パスワード有効期限（日）が10以下の場合、パスワード変更禁止期間（日）が0のみにできます。
  • パスワード変更禁止期間（日）が355以上の場合、パスワード有効期限（日）が0のみにできます。

非アクティブ期限(日)

ユーザーアカウントが無効になるまでの最大アイドル期間（日）。

設定範囲:

  • 0
  • 30～365

0は、時間無制限を示します。つまり、アイドル状態のユーザーアカウントが無効になることはありません。

デフォルト値：0

非常時ユーザー

緊急時にSMTPサーバーにログインするユーザー名。

このユーザーは、ログインルールやログインインターフェースによる制限を受けず、そのパスワードは期限切れになりません。

デフォルト値は[NULL]です。

NOTE:
  • 緊急ログインユーザーとして設定できるのは管理者のみです。
  • 緊急ログインユーザーは管理者のみで閲覧可能です。

過去パスワードの再利用制限

新規パスワードとして再利用できない過去パスワードの利用回数。

設定範囲:0～5

0は、過去のすべてのパスワードが許可されていることを示します。

デフォルト値：5

ユーザーロックアウトポリシー

連続した無効なログイン試行の最大回数とアカウントロック期間。

  • 連続した無効なログイン試行の最大回数は、1～6の範囲内の整数または無制限（アカウントロック機能を無効）です。デフォルトの値は５です。
  • アカウントのロック期間（分数）は、1～30の範囲内の整数です。デフォルトの値は５です。

ユーザーアカウントがロックされた後、ユーザーはアカウントロック期間が満了した後にのみログインを試みることができます。

NOTE:
  • セキュリティ上の理由で、アカウントロック機能を有効にすることをお勧めします。
  • 緊急時にユーザーアカウントをロック解除するには、CLIでunlockコマンドを実行します。詳細については、サーバーのBMCユーザーガイドを参照してください。

証明書有効期間通知（日）

BMC証明書の有効期限が切れるまでの日数を事前にユーザーに通知します。例えば、7と設定している場合、証明書の有効期限が7日間以下になると、アラームが通知されます。

値の範囲：7～180

デフォルト値：90
表 2 ログインルール

パラメータ

説明

名称

ルールの名前。

期間

ログインが許可されている期間。

値は、以下のいずれかの形式にすることができます。

  • YYYY-MM-DD:

    例:2013-08-30～2013-12-30

  • HH:MM:

    例:08:30～20:30

  • YYYY-MM-DD HH:MM:

    例:2013-08-30 08:30～2013-12-30 20:30

NOTE:
  • 時間は1970年から2050年までの間に限定されています。
  • ログインルールの開始時間と終了時間は同じ形式である必要があります。

IP

ログインが許可されているIPアドレスまたはIPアドレス範囲。値は、以下のいずれかの形式にすることができます。

  • IPv4（xxx.xxx.xxx.xxx）アドレス:IPアドレスを示します。
  • IPv4/サブネットマスク（xxx.xxx.xxx.xxx/mask）:IPアドレスセグメントを示します。

MAC

ログインに許可されているMACアドレスまたはMACアドレス範囲。値は、以下のいずれかの形式にすることができます。

  • xx:xx:xx:xx:xx:xx:MACアドレスを示します。
  • xx:xx:xx:MACアドレスセグメントを示します。

ステータス

ログインルールの現在の状態。

デフォルトでは無効になっています。

操作

編集をクリックします。ログインルールの時間、IP、MAC、ステータスのフィールドが編集可能です。

ユーザーとパスワードの設定。

ユーザー管理には、次の項目が含まれます。

自身のパスワードを設定し、SSH公開鍵、SNMPv3暗号化パスワード、SNMPv3暗号化アルゴリズム、認証アルゴリズムを管理します。

編集をクリックします。基本管理、リモートコントロール、VMM、セキュリティ管理、電源管理、診断、およびユーザーの設定変更のフィールドが編集可能です。。

セキュリティ強化機能の有効化

  1. 必要に応じて、サーバーのパスワードチェック、SSHパスワード認証機能、パスワード有効期間、非アクティブな制限期間、緊急ログインユーザー、過去のパスワードの使用禁止、ログイン失敗によるロックなどのセキュリティ強化機能を設定します。パラメータの詳細については、表 1を参照してください。
  2. 保存をクリックします。

ログインルールの設定

BMCは最大3つのログインルールを同時にサポートします。3つのルールのいずれかを満たす場合、BMCにログインできます。

ログインルールは、次の2つの条件を満たす場合にのみローカルユーザー、LDAPグループ、SNMPv3サービスまたはCLP（ssh）インターフェース、KVM_VMM、RMCP、およびRedfishインターフェースに対して有効です。

  • ログインルールが設定され、ログインルールエリアで有効になっています。
  • 設定エリアでログインルールが選択されています。
  • 空欄であるログインルールのステータスを有効にして保存した場合、ログインの制限はありません。
  • ログインルールのテキストボックスが空欄である場合、ログインルールの制限はありません。
  1. ログインルールエリアで、操作欄にある編集をクリックします。
  2. ルール状態を に設定します。
  3. ログインルールを設定します。パラメータの詳細については、表 2を参照してください。
  4. 保存をクリックします。

カスタムロールの設定

デフォルトロール（[管理者]、[オペレーター]、[一般ユーザー]）の操作権限は変更できませんが、管理者はカスタムロールの操作権限を設定できます。

  1. 機能リストからカスタムロールのモジュールを選択します。

    表 3には権限を示します。

  2. [保存]をクリックします。

    ダイアログボックスが表示されたら、現在のユーザーパスワードを入力し、[OK]をクリックします。

ログインセキュリティバナーの設定

  1. ログインセキュリティバナータブページで、セキュリティバナーをに設定します。
  2. セキュリティバナーのテキストボックスにセキュリティ情報を入力します。
  3. 保存をクリックします。

デフォルトのログインセキュリティメッセージの復元

  1. ログインセキュリティバナータブページで、セキュリティバナーをに設定します。
  2. 初期値に戻すをクリックします。
  3. 保存をクリックします。
