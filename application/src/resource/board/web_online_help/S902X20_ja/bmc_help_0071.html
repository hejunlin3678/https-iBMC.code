証明書の更新

機能概要

[証明書の更新]ページで、次の操作を実行できます。

  • SSL証明書の更新の設定
  • CAサーバーの設定
  • 証明書のサブジェクトのカスタマイズ

パラメータの説明

表 1 証明書の更新

パラメータ

説明

SSL証明書の更新

NOTE:
  • SSL証明書は、システムロックが無効で、ユーザーにセキュリティ設定権限があり、CAサーバーが設定されている場合にのみ更新できます。
  • SSL証明書を更新するには、サーバルート証明書とクライアント証明書の両方をインポートする必要があります。 そうしないと、認証は失敗します。

自動更新

証明書の自動更新機能を有効または無効に設定します。

[自動更新]を有効にすると、SSL証明書は1時間間隔で自動的に更新されます。

この機能は、デフォルトでは無効になっています。

証明書の更新

[証明書の更新]をクリックして、SSL証明書を更新できます。

CAサーバーの設定

サーバーアドレス

CAサーバーのIPアドレス。

最大67文字のIPv4アドレス、IPv6アドレス、またはドメイン名。

ポート

CAサーバーのポート番号。

値の範囲：1～65535

パス

CAサーバーのパス。

1～128文字の文字列。スラッシュ(/)で始まり、数字(0～9)、文字(a～z、A～Z)、アンダースコア(_),、およびスラッシュ(/).で構成されます。

証明書のサブジェクト

国（C）

証明書が属する国。

2文字の国コード。すべての英字が使用可能です。

都道府県（S）

証明書が属する都道府県。

0～128文字の文字列。数字（0～9）、英文字（a～z、A～Z）、ハイフン（-）、アンダースコア（_）、ピリオド（.）、およびスペースを使用できます。

都市/位置（L）

証明書が属する位置（市区町村）。

0～128文字の文字列。数字（0～9）、英文字（a～z、A～Z）、ハイフン（-）、アンダースコア（_）、ピリオド（.）、およびスペースを使用できます。

会社（O）[l(1]

証明書が属する会社または組織。

0～64文字の文字列。数字（0～9）、英文字（a～z、A～Z）、ハイフン（-）、アンダースコア（_）、ピリオド（.）、およびスペースを使用できます。

部署（OU）

証明書が属する組織単位または部署。

0～64文字の文字列。数字（0～9）、英文字（a～z、A～Z）、ハイフン（-）、アンダースコア（_）、ピリオド（.）、およびスペースを使用できます。

一般名（CN）[l(2]

証明書の一般名。

1～64文字の文字列。数字（0～9）、英文字（a～z、A～Z）、ハイフン（-）、アンダースコア（_）、ピリオド（.）、およびスペースを使用できます。

内部名（IN）[l(3]

証明書の内部名。

0～64文字の文字列。数字（0～9）、英文字（a～z、A～Z）、ハイフン（-）、アンダースコア（_）、ピリオド（.）、およびスペースを使用できます。

電子メール（E）[l(4]

証明書の電子メールアドレス。

電子メールアドレスの規約に準拠した0〜255文字の文字列。

サーバールート証明書

公式CAから適用されたサーバールート証明書。接続が確立される前にCAサーバから送信されたサーバ証明書パケットでTLS認証を実行するために使用される証明書。

NOTE:
  • .crt、.cer、または.pem形式のサーバールート証明書がサポートされます。*.cer形式の証明書のコードは、Base64です。証明書サイズは最大100KBです。証明書の重複は許可されません。
  • セキュリティ上の理由から、証明書を定期的に更新してください。

クライアント証明書

公式CAから適用されたクライアント証明書。

NOTE:
  • *.p12または*.pfx形式のクライアント証明書がサポートされます。証明書サイズは最大100KBです。
  • セキュリティ上の理由から、証明書を定期的に更新してください。

証明書情報

アップロードされた証明書に関する情報。次の情報が含まれます。

  • [発行元]
  • [使用者]
  • [有効期間]
  • [シリアル番号]
  • 次のいずれかの状態を持つ[CRL]
    • [設定済み]：証明書失効ファイルがアップロードされており、TLS接続確立時に証明書失効検証が実行されます。
    • [未設定]：証明書失効ファイルはアップロードされません。
NOTE:
  • CRLファイルは*.crl形式である必要があります。ファイル名は空白にできません。CRLファイルのサイズは最大100KBです。
  • CRLが期限切れになると、対応する認証機能が使用できなくなります。
  • クライアント証明書はCRLをサポートしていません。

証明書のアップロード

  1. サーバー証明書またはクライアント証明書の横にある[アップロード]をクリックします。

    ローカルファイル選択用またはクライアント証明書アップロード用のダイアログボックスが表示されます。

    • .cer、.crt、または.pem形式のサーバールート証明書がサポートされます。*.cer形式の証明書のコードは、Base64です。証明書サイズは最大100KBです。証明書の重複は許可されません。
    • *.p12または*.pfx形式のクライアント証明書がサポートされます。証明書サイズは最大100KBです。

  2. 正しい形式のファイルを選択します。
  3. [OK]をクリックします。

CRLの設定

CRLファイルは*.crl形式である必要があります。ファイル名は空白にできません。CRLファイルのサイズは最大です。

  1. CAからCRLファイルを取得します。
  2. サーバー証明書情報エリアで、[CRL]の横にある[アップロード]をクリックします。
  3. CRLファイルを選択します。
  4. ログインユーザーのパスワードを入力し、[保存]をクリックします。
