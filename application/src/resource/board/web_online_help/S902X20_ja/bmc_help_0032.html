Webサービス

機能説明

Webサービス画面では、以下の操作を実行できます。

  • Webサービスの基本設定を照会・設定します。
  • 使用中のSSL証明書を照会します。
  • SSL証明書をカスタマイズまたはインポートします。

SSL証明書は、クライアント上のWebブラウザとWebサーバー間にHTTPSを介したSSLセキュリティチャネルを構築することで、クライアントとサーバー間のデータ暗号化送信を実現し、データ漏洩を防ぎます。SSLは、送信された情報のセキュリティを保証し、アクセスするWebサイトの信頼性を確認するために使用されます。SSL証明書の置き換えがサポートされます。セキュリティのために、元の証明書と鍵をカスタマイズした証明書と公開/秘密鍵ペアに置き換えて証明書を適時に更新してください。

  • SSL証明書は、単一のSSL証明書またはレベル10未満の証明書チェーンにすることができます。
  • 証明書は.crt、.cer、.pem、.pfx、または.p12形式である必要があります。.crt、.cer、または.pem形式の証明書は1MBを超えることができません。.pfxまたは.p12形式の証明書ファイルは100KBを超えることができません。
  • MD5およびSHA1を使用する場合、セキュリティリスクが発生する可能性があります。BMCは、脆弱な署名アルゴリズム（MD5およびSHA1など）のインポートをサポートしていません。

基本設定

HTTP

インターネットの閲覧とHTTP（Hypertext Transfer Protocol）ページの解析をサポートします。ブラウザとBMC間の接続を確立するために、Webサーバー（HTTP）サービスはデフォルトで有効になっています。接続を確立すると、セキュアプロトコルHTTPSが使用されます。

デフォルト値は[80]です。

NOTE:

HTTPサービスが無効になっている場合、ブラウザのアドレスボックスにhttp:BMC management network port IP addressを入力した後、HTTPサービスに自動的に切り替えることはできません。

HTTPS

インターネットの閲覧とHTTPS（Hypertext Transfer Protocol over Secure Socket Layer）ページとRedfishの解析をサポートします。

デフォルト値は[443]です。

NOTE:

HTTPSサービスが無効になっている場合、ブラウザでBMC WebUIにログインできません。

サーバーの証明書情報のカスタマイズとインポート

  • この操作は、SSL証明書を申請する際に実行します。
  • セキュリティ上の理由から、定期的に証明書を更新してください。
  1. SSL証明書エリアで、カスタマイズをクリックします。
  2. [CSRの生成]を選択し、CSR（certificate signing request）情報を入力して生成をクリックします。
  3. 生成したCSRをSSL証明書の認証機関に送信してSSL証明書を申請します。

    正式のSSL証明書を取得した後、クライアントに[生成]します。

  4. 証明書ダイアログボックスで[サーバー証明書のインポート]を選択します。
  5. ローカルクライアントから正式のSSL証明書を選択します。

    • 証明書は.crt、.cer、.pem、.pfx、または.p12形式である必要があります。.crt、.cer、または.pem形式の証明書は1MBを超えることができません。.pfxまたは.p12形式の証明書ファイルは100KBを超えることができません。
    • MD5およびSHA1を使用する場合、セキュリティリスクが発生する可能性があります。BMCは、脆弱な署名アルゴリズム（MD5およびSHA1など）のインポートをサポートしていません。

  6. [OK]をクリックします。
  7. [証明書のパスワード]に証明書パスワードを入力します。
  8. [OK]をクリックします。

    インポートした後、証明書はすぐに有効になります。

    カスタマイズで生成した各CSRファイルは、CA機関に申請したサーバー証明書に対応するものです。サーバー証明書をインポートする前に新しいCSRファイルを生成しないでください。新しいCSRファイルを生成すると、元のCSRファイルは新しいCSRファイルによって上書きされ、復元できない可能性があります。この場合、新しいCSRファイルを使用してCA機関に新しいサーバー証明書を申請する必要があります。

  9. 再度BMC WebUIにログインします。

SSL証明書のインポート

  • この操作は、クライアントに使用可能なSSL証明書がある場合に実行します。
  • セキュリティ上のために、RSA2048などの安全な暗号化アルゴリズムを使用してカスタマイズしたSSL証明書を暗号化してください。
  • セキュリティ上の理由から、定期的に証明書を更新してください。
  1. SSL証明書画面で、カストマイズをクリックします。

    SSL証明書情報のカスタマイズ画面が表示されます。

  2. [サーバー証明書のインポート]を選択します。
  3. インポートするSSL証明書を選択します。

    • 証明書は.crt、.cer、.pem、.pfx、または.p12形式である必要があります。.crt、.cer、または.pem形式の証明書は1MBを超えることができません。.pfxまたは.p12形式の証明書ファイルは100KBを超えることができません。
    • MD5およびSHA1を使用する場合、セキュリティリスクが発生する可能性があります。BMCは、脆弱な署名アルゴリズム（MD5およびSHA1など）のインポートをサポートしていません。

  4. [OK]をクリックします。
  5. [証明書のパスワード]に証明書パスワードを入力します。
  6. [OK]をクリックします。

    インポートした後、証明書はすぐに有効になります。

    アップロードするファイルのサイズが100MBを超えると、ページ要求が失敗したことを示すメッセージが表示されます。この問題を解決するには、ページを更新してください。

  7. 再度BMC WebUIにログインします。
