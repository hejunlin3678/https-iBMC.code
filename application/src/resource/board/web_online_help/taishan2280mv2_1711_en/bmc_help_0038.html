Network Settings

Function Description

On the Network Settings page, you can view and configure the following settings of the BMC:

  • Host name
  • Network port
  • IP version and addresses
  • DNS info
  • VLAN
  • LLDP

Changing the IP address of the management network port will cause network disconnection. Change the IP address only when necessary.

Parameter Description

Table 1 Network Settings

Parameter

Description

BMC Host Name

BMC host name.

Value: a string of 1 to 64 characters

The value can contain letters, digits, and hyphens (-), but cannot start or end with a hyphen.

Mode

Type of the server management network port, that is, the BMC network port.

Default value: Fixed

Fixed

If you select this option, you must also specify the BMC network port.

  • Dedicated Port: dedicated BMC port (that is, the Mgmt port of the server).
  • External PCIe port: a service network port on a PCIe card (that is, a PCIe card that supports NC-SI and is connected with an NC-SI cable).

The default value is Dedicated Port.

Auto

If you select this option, the BMC automatically selects the BMC network port based on port status.

selection. If multiple ports are available, the BMC selects a port based on the following priority:

Dedicated Port > PCIe port (ports 1 to 2 or ports 1 to 4)

NOTE:

If a network port is selected as the BMC network port, will be displayed behind the network port.

NCSI Mode

NCSI network port switching mode when Mode is set to Auto.

The default value is Manual switch.

Auto failover

The system automatically switches to the external network port with a higher priority based on the port connection status. Based on the automatic selection rule, the BMC selects the external network port in the following sequence:

port 1 > port 2 or port 1 > port 2 > port 3 > port 4

Manual switch

Manually switch to the external network port with a higher priority.

BMC Network Port
  • If Mode is set to Fixed, specify a management network port.
  • If Mode is set to Auto, select the network ports for auto-selection.

Network Protocols

IP versions that can be enabled:

  • IPv4: allows only IPv4. If this option is selected, only IPv4 addresses can be configured.
  • IPv6: allows only IPv6. If this option is selected, only IPv6 addresses can be configured.
  • IPv4/IPv6: allows IPv4 and IPv6. If this option is selected, IPv4 and IPv6 addresses can be configured.

Default value: IPv4/IPv6

IPv4
  • Automatically obtain IP address: click this option to allow an IPv4 address to be automatically allocated for the BMC network port.
  • Manually set IP address: click this option to manually set an IPv4 address for the BMC network port.

The IPv4 address information includes IP Address, Subnet Mask, Default Gateway, and MAC Address.

The default value is Manually set IP address.

NOTE:
  • MAC Address specifies the physical address of a network interface card (NIC).
  • If you do not use the default gateway, set the gateway to any IP address in the network segment of the static and floating IP addresses.

IPv6
  • Automatically obtain IP address: click this option to allow an IPv6 address to be automatically allocated for the BMC network port.
  • Manually set IP address: click this option to manually set an IPv6 address for the BMC network port.

The IPv6 address information includes IP Address, Prefix, Default Gateway, Link-Local Address and IP Address2.

The default value is Manually set IP address.

NOTE:
  • Link-Local Address is used for local link communication.
  • IP Address2 supports a maximum of fifteen IPv6 addresses when stateless address autoconfiguration (SLAAC) is used.
  • If you do not use the default gateway, set the gateway to any IP address in the network segment of the static and floating IP addresses.

DNS
  • Automatically obtain DNS IPv4 address: click this option to allow an IPv4 address to be allocated for the DNS server.
  • Automatically obtain DNS IPv6 address: click this option to allow an IPv6 address to be allocated for the DNS server.
  • Manually set address: click this option to manually set the DNS information. The DNS address information includes Domain, Preferred Server, and Alternate Server.

The default value is Manually set address.

NOTICE:
  • If the IP address of the BMC management network port is automatically obtained, you can obtain the DNS information automatically or manually. Automatic obtaining is recommended.
  • If the IP address of the BMC management network port is set manually, the DNS information must be set manually.

Domain: domain name for the server.

The domain name must meet the following requirements:
  • Contain 0 to 67 characters.
  • Allow digits, letters, hyphens (-), and dots (.) only.
  • Cannot start with a hyphen or dot, or end with a hyphen.
  • Allow a maximum of 63 characters between any two dots.

Preferred Server: IP address of the preferred DNS server.

Value: an IPv4 or IPv6 address or leave it empty.
  • Alternate Server 1: IP address of the first alternate DNS server.
  • Alternate Server 2: IP address of the second alternate DNS server.

Value: an IPv4 or IPv6 address or leave it empty.

VLAN

Setting of the VLAN to which the NC-SI network port belongs.

Default value: Disable

NOTE:
  • You are advised to enable VLAN and set VLAN IDs to implement isolation between the service network and management network.
  • If Dedicated Port is selected as the BMC management network port, the VLAN configuration is invalid. If any other value except Dedicated Port is selected as the BMC management network port, the VLAN configuration is valid.

VLAN ID

VLAN to which the NC-SI network port belongs.

Value range: an integer from 1 to 4094

NOTE:

After the VLAN ID is saved, the setting takes effect several seconds later.

LLDP

NOTE:

LLDP is supported only when the dedicated network port is used as the BMC management network port.

LLDP

After LLDP is enabled, the BMC can send device MAC address to the directly connected devices in standard packets, facilitating the NMS to query and determine the link communication status.

Default value: Disable

Work Mode

The work mode must be Transmit.

The BMC only supports transmission of LLDP packets and cannot receive LLDP packets.

Transmit Delay (s)

In the current work mode, an LLDP packet is sent to notify the neighbor devices when the BMC local configuration (management network port is changed or the network cable of the management network port is connected or disconnected) is changed.

The transmit delay (in seconds) is used to prevent a large number of LLDP packets from being sent due to frequent changes of the local configuration. If the BMC local configuration is changed during the delay period, the system resets the counter. A next LLDP packet will be sent only when the delay has expired.

Value: 1 to 8192

Default value: 2

Transmit Interval (s)

Interval (in seconds) at which the LLDP packets are sent if the local configuration information is not changed.

Value: 5 to 32768

Default value: 30

Hold Time Multiplier

If the neighbor node does not receive an LLDP packet from the BMC within the specified period (transmit interval Ã— hold time multiplier), the neighbor node automatically clears the previously reserved packets.

Value: 2 to 10

Default value: 4
