Kerberos

Function Description

On the Kerberos page, you can view and configure Kerberos basic settings and user group information.

The BMC provides an access interface for Kerberos users. Using a Kerberos user account to log in to the BMC improves system security. You can log in to the BMC WebUI as a Kerberos user.

Parameter Description

Table 1 Kerberos

Parameter

Description

Kerberos

Setting of the Kerberos login.

It is disabled by default.

NOTE:

The security policies (password complexity check, password validity period, minimum password age, previous passwords disallowed, and inactive timelimit, and user lockout policy) configured on the authentication server apply to the Kerberos users attempting to log in to the BMC.

Basic Settings

NOTE:

Parameters with asterisks (*) are mandatory.

Realm

Kerberos realm.

Value: a string of up to 255 characters

The value can contain digits, letters (uppercase letters recommended), and special characters (including spaces).

Kerberos Server Address

Address of the Kerberos server.

If Kerberos is enabled, enter the FQDN (Host name.Domain name) of the Kerberos server, and configure DNS on the BMC Settings > Network Settings page.

Format: an IPv4 or IPv6 address or a domain name

The domain name must meet the following requirements:
  • Contain 0 to 255 characters.
  • Allow digits, letters, hyphens (-), and dots (.) only.
  • Cannot start with a hyphen or dot, or end with a hyphen.
  • Allow a maximum of 63 characters between any two dots.

Kerberos Port

Port number of the Kerberos service.

Value range: 1 to 65535

Default value: 88

Key Table

Kerberos key table to be uploaded.

The key table must be in the .keytab format.

It is left blank by default.

NOTE:
  • The key table size cannot be 0 KB or greater than 1 MB.
  • For security purposes, periodically update the key.

Login Password

Password of the user for logging in to the BMC.

Kerberos User Group

Displays information about all Kerberos user groups.

The BMC supports a maximum of five Kerberos user groups.

Add Group

Adds a Kerberos group.

Group Name

Name of the group to which the Kerberos user belongs.

Value range: a string of 1 to 255 bytes. The specific length varies with the number of bytes of each character. The maximum length ranges from 64 to 255 characters.

SID

Security identifier used for Kerberos and user group authorization. For example, S-1-5-21-310440588-250036847-580389505-500.

Value range: a string of 1 to 255 bytes, without any space. The specific length varies with the number of bytes of each character. The maximum length ranges from 64 to 255 characters.

Role

Role of the Kerberos user group.

  • Administrator: Users assigned the Administrator role can perform all operations.
  • Operator: Users with the Operator role can perform basic management, remote control, remote media, power control, query information, and configure their own data.
  • Common User: Users assigned with the Common User role can query information and configure their own data.
  • Custom Role: Users assigned Custom Role 1 to Custom Role 4 can perform the specified operations.
NOTE:

Common User is the default user permission for a new Kerberos user group.

Login Interfaces

Interfaces through which the Kerberos group members can log in to the BMC.

  • Web

    By default, new Kerberos user groups can log in to the system through the WebUI.

  • Redfish

    By default, new Kerberos user groups can log in to the system through the Redfish API.

Group Folder

Name of the user folder on the Kerberos server.

Format: "CN=xxx" or "OU=xxx"

When there are multiple levels of nodes, the upper-level node follows the lower-level node with a comma in between.

For example, if the user group grouptest is in \testgroups\part1 on the Kerberos server, enter OU=part1,OU=testgroups.

NOTE:

For details about the difference between CN and OU, see the description of the Kerberos protocol.

In Windows AD, for example, it is:

  • CN if Type is Container.
  • OU if Type is Organizational Unit.

Value range: a string of 255 bytes (64 to 255 characters). The specific length varies with the number of bytes of each character.

Login Rules

Login rules that apply to the Kerberos group.

Edit

Displays the region for configuring an existing Kerberos group.

Delete

Modifies a Kerberos group.

Enabling Kerberos and Configuring the Domain Server

  1. Set Kerberos to .
  2. Set the domain server. For details about the parameters, see Table 1.
  3. Click Save.

Importing a Key Table

The key table must be in the .keytab format. The key table size cannot be 0 KB or greater than 1 MB.

  1. Click Select File and select the key table to be imported.
  2. Click Open.

    After the key table is uploaded, "File uploaded successfully" is displayed.

  3. Enter the password of the current user.

    The Kerberos user who logs in using SSO does not need to enter the password.

  4. Click Save.

Adding a Kerberos User Group

A maximum of five Kerberos user groups can be added to the BMC system.

  1. In the Kerberos User Group area, click Add Group.

Deleting a Kerberos User Group

  1. In the Kerberos User Group area, locate the Kerberos user group to be deleted and click Delete.

    A Confirm dialog box is displayed, asking you to enter the current user password.

    The Kerberos user who logs in using SSO does not need to enter the password.

  2. Enter the current user password.
  3. Click OK.

Modifying a Kerberos User Group

  1. In the Kerberos User Group area, locate the Kerberos user group to be modified and click Edit.

    The Modify User Group page is displayed.

  2. Modify parameters. For details about the parameters, see Table 2.
  3. Click Save.
