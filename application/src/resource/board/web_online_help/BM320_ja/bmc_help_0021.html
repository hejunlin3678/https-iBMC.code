BMCログ

機能説明

  • 操作ログ画面で、システム運用中に記録されたログ（たとえば、システム起動、状態遷移、BMCのユーザーにより行われた構成などの情報）を表示およびダウンロードできます。

    BMCは200KBの容量を備えており、最大2000件の操作ログ記録を保存できます。

    操作ログが200KBに達すると、自動的に圧縮されます。新しい圧縮パッケージが生成されると、古い圧縮パッケージが自動的に削除されます。

    電源オン、電源オフ、リセット操作に成功した操作ログは、BMCが動作を正常にトリガーしたことを示していますが、必ずしもこれらの操作がハードウェア上で正常に実行されたことを意味するわけではありません。

  • 実行ログ画面で、サーバーのRASログを表示できます。

    BMCは200KBの容量を備えており、最大2000件の実行中ログエントリを保存できます。

    ログが200KBに達すると、自動的に圧縮されます。新しい圧縮パッケージが生成されると、古い圧縮パッケージが自動的に削除されます。

  • セキュリティログ画面で、次の操作を実行できます。
    • シリアルポート、SSHポートを介したBMCのログインとログアウト、および設定操作に関するログを表示します。
    • SNMPで実行された照会および設定操作に関するログを表示します。
    • セキュリティログをダウンロードします。

    BMCは200KBの容量を備えており、最大2000件のセキュリティログエントリを保存できます。

    ログが200KBに達すると、自動的に圧縮されます。新しい圧縮パッケージが生成されると、古い圧縮パッケージが自動的に削除されます。

パラメータの説明

表 1 操作ログ

パラメータ

説明

番号

操作順序。最新の操作が最初にリストされています。

インターフェース

操作が実行されたインターフェース。

ユーザー

操作を実行したユーザー。

次のいずれかのシナリオで、ユーザーは、N/A（ユーザー名は表示されません）として表示されます。

  • UIDボタンまたは電源ボタンが押されています。
  • インターフェースはSNMPv1またはSNMPv2cです。
  • インターフェースはIPMI、IPアドレスはHOST。（ログには、サービスシステムから送信されたIPMIメッセージが記録されています。)
  • ジャンパを使用して、デフォルトユーザーのBMC IPアドレスとパスワードを変更しました。
  • コンポーネントがホットスワップされました。

IPアドレス

操作の実行に使用されたターミナルのIPアドレス。

  • HOSTは、操作がサービスシステムによってトリガーされたことを示します。
  • IPアドレスはX.X.X.Xと表示されている場合、BMCに接続しているクライアントが操作を実行することを示します。
  • 次のいずれかのシナリオで、127.0.0.1は、操作がローカルホストによってトリガーされたことを示します。
    • UIDボタン、メモリーライザーボタン、または電源ボタンが押されています。
    • BMCはローカルのシリアルポートを介してアクセスします。
    • ジャンパを使用して、デフォルトユーザーのBMC IPアドレスとパスワードを変更しました。
    • コンポーネントがホットスワップされました。

発生時間

操作が実行された時間。

詳細情報

操作に関する詳細情報。

BMC WebUI、CLIまたはIPMIを使用してアップグレードした後、BMCが再起動した場合、操作が次の形式で記録されます。

  • インターフェース:N/A
  • ユーザー:N/A
  • IPアドレス:127.0.0.1
  • 詳細情報:Reset BMC caused by upgrade successfully

ダウンロード

をクリックします。操作ログファイルは、ローカルPCのデフォルトディレクトリに自動的にダウンロードされます。

備考:ユーザーおよびIPアドレスを解析できない場合は、unknownとして表示されます。
