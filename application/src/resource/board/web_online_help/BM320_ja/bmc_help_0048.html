Java統合リモートコンソールを開けない

故障現象

症状説明

考えられる原因

Java統合リモートコンソールを開けません。
  • JREが正しくインストールされていません。
  • JREバージョンがBMCと互換性がありません。

解決策

  1. JREが正しくインストールされているかを確認します。

    BMCは、AdoptOpenJDK 8 JREおよびadptopenjdk 11 JREをサポートしています。

    • JREバージョンが正しい場合、テクニカルサポートまでお問い合わせください。
    • JREバージョンが正しくない場合、2に移動します。

  2. 公式AdoptOpenJDKサイトから、クライアントOSに適合するJREバイナリパッケージをダウンロードします。
  3. AdoptOpenJDKをインストールします。

    • パッケージを解凍した後でJAVA_HOMEおよびPATH環境変数を手動で設定します。
    • AdoptOpenJDK公式サイトからIcedTea Webをダウンロードして解凍し、PATH環境変数にビンフォルダを設定します。

  4. BMC WebUIにログインして、[ホーム]画面の[仮想コンソール]エリアで、[開始] > [Java統合リモートコンソール（専用）]または[Java統合リモートコンソール（共有）]を選択します。

    この処理中に、.jnlpファイルが自動的にダウンロードされます。

  5. .jnlpファイルを開きます。

    .jnlpファイルには時間的制限があります。無効になった場合は、4を参照してもう一度ダウンロードしてください。

    • クライアントがLinuxまたはWindows CLIを使用している場合は、.jnlpがあるディレクトリに切り替えてjavaws kvm.jnlpを実行します。
    • クライアントがGUIを使用している場合は、.jnlpファイルを選択し、ファイルを右クリックしてjavawsを選択します。javawsがショートカットメニューで使用できない場合は、IcedTea Webのインストールディレクトリにあるbinフォルダに移動します。
