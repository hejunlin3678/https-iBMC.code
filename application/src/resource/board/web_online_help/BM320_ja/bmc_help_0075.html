TPCM

機能概要

[[TPCM]]ページでは、現在のサーバーにおけるファームウェアの測定情報の表示と、外部にある信頼できる管理センターのサーバーの構成を実行できます。

構成の編集

TSBが有効で、かつシステムがロックされていない場合、管理者のみが構成を編集できます。

  1. [[構成]]をクリックします。
  2. 表示された[構成]ウィンドウで、構成を編集します。

    • [[測定]]の右にあるボタンをクリックします。[測定]を有効または無効にします。
      • デフォルト値は有効です。
      • 信頼性測定機能を無効にすると、ファームウェアへの悪意のある改ざんを即座に検知できなくなります。
      • 測定機能を有効にした後、ファームウェアのバージョンアップ時に、ファームウェアの基準値を更新する必要があります。
    • [[制御]]の右にあるボタンをクリックします。[制御]を有効または無効にします。
      • デフォルト値はディセーブルです。
      • 制御機能を無効にすると、ファームウェアが悪意的に改ざんされた場合に即座に停止できなくなるため、システムのセキュリティリスクが発生します。
      • [制御]は、[測定]が有効な場合にのみ編集可能です。
      • 制御機能を有効にした後、ファームウェアが測定に失敗した場合、ファームウェアの起動がブロックされます。
    • [[信頼できるサーバーアドレス]]テキストボックスにアドレスを入力します。信頼できるサーバーのアドレスを編集します。

      输入格式：IPv4地址、IPv6地址或域名。

      ドメイン名は次の要件を満たす必要があります。
      • 0～255文字が含まれます。

      • 数字、英字、ハイフン（-）、およびピリオド（.）のみが使用できます。

      • ドメイン名の最初にハイフンまたはピリオドを使ったり、ドメイン名の最後にハイフンを使ったりすることはできません。

      • 2つのピリオドの間に最大63文字が使用できます。

  3. [[OK]]をクリックします。

    確認ダイアログボックスが表示されます。

  4. ログインパスワードを入力し、[OK]をクリックします。

基準値の更新

  • TSBが有効でシステムロックが無効になっている場合、管理者のみが基準値を更新できます。
  • ファームウェアがアップグレードされた場合にのみ、基準値を更新してください。それ以外の場合は、基準値を変更しないでください。
  1. [更新]をクリックします。

    [更新]ダイアログボックスが表示されます。
    • BIOSファームウェアがBMCを通じてアップグレードされていない場合は、BIOSファームウェアのアップグレードが完了した後、表示される実際の測定値が最新のファームウェア測定値を反映するように、まずOSの電源を入れ、BMCをリセットしてから、基準値を更新してください。
    • 他のファームウェアの基準値を更新するには、ファームウェアのアップグレード後にOSをリセットし、表示される実際の測定値が最新のファームウェア測定情報であることを確認してください。次に、基準値を更新します。

  2. （オプション）テキストボックスの[基準値]を変更します。

    基準値は、次の要素で構成されます。

    • 64文字。
    • 数字（0～9）と英字（a～f、A～F）。

  3. [保存]をクリックします。

    確認ダイアログボックスが表示されます。

  4. 現在のユーザーのログインパスワードを入力し、[OK]をクリックします。

    • BIOSファームウェアの基準値が正常に更新されました。実際の測定値が基準値と一致するように、BMCを再度リセットして基準値を有効にしてください。
    • 他のファームウェアの基準値が更新された後、OSをリセットして基準値を有効にし、実際の測定値が基準値に一致するようにします。
