HTML5統合リモートコンソールを開くときに「信頼証明書の設定がタイムアウトしました」が表示される

故障現象

症状説明

考えられる原因

ユーザーがHTML5統合リモートコンソールを開くするとき、「信頼証明書の設定がタイムアウトしましたので、KVMを有効にできません。」というメッセージが表示されます。

KVMクライアント（HTML5統合リモートコンソール）とサーバー間の接続を確立する前に、SSL証明書を確認する必要があります。検証に失敗した場合、接続は確立できません。

解決策

  1. BMC WebUIで、[サービス] > [Webサービス]の順に選択します。サーバー証明書情報エリアで、サーバー証明書の有効期限が切れているかを確認します。

    • はいの場合は、2に移動します。
    • いいえの場合は、3に移動します。

  2. 新しい証明書を生成して元の証明書を置き換えます。
  3. BMCを再起動します。
  4. HTML5統合リモートコンソールを開くことができるかどうかを確認します。

    • はいの場合は、これ以上のアクションは必要ありません。
    • いいえの場合は、5に移動します。

  5. テクニカルサポートまでお問い合わせください。
