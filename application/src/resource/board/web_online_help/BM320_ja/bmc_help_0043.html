ライセンス管理

機能説明

インセンス管理画面では、ライセンス認証によりBMCアドバンス機能を使用できます。

ユーザーは、ライセンスの有効期間内にのみ、BMCアドバンス機能を使用できます。ライセンスがない場合、スタンバイBMCエディションのデフォルト機能のみ使用できます。

アドバンスドエディションは標準エディションよりもっと多くのアドバンスド特性を提供します。例えば:

  • RedfishによってOSをデプロイします。
  • Redfishによってインテリジェント診断のオリジナルデータを収集します。
  • ハードウェアセキュリティエンジン（SEC、Security Engine）、高性能RSAエンジン（HPRE、High Performance RSA Engine）、RAID DIFエンジン（RDE、RAID DIF Engine）、およびZIPエンジンを含むKunpengアクセラレーションエンジンを有効にします。
  • システムロックモードを有効にする。

デバイスのESN

メインボードSNから生成され、ライセンスの申請に使用されるデバイスシリアル番号（equipment serial number、ESN）。

インストール

ライセンスをインストールします。

NOTE:
  • ライセンスの形式は.xmlです。そのサイズは1MBを超えることができません。
  • 失効したライセンスをインストールしないでください。

削除

ライセンスを削除します。

エクスポート

インストールされているライセンスをエクスポートします。

必要に応じてライセンスをエクスポートしてバックアップできます。

失効

ライセンスを失効させます。

ライセンスが期限切れになり、猶予期間に入った後、この画面から失効コードを取得できます。たとえば、コンポーネントを交換する前に、失効ボタンをクリックして失効コードを取得する必要があります。失効コードを使用して新しいライセンスを申請し、新しいライセンスを新しいコンポーネントにインストールしてください。

NOTE:

失効したライセンスをインストールしないでください。

ライセンス状態

ライセンス状態。値は以下のとおりです。

  • Normal: 商用ライセンスがインストールされています。ライセンスの有効期限が切れておらず、許可されているすべての機能が正常に動作しています。
  • Commissioning: コミッショニングライセンスがインストールされています。ライセンスの有効期限が切れておらず、許可されているすべての機能が正常に動作しています。
  • Grace Period: 商用またはコミッショニングライセンスがインストールされています。ライセンスの有効期限が切れています。猶予期間の満了後にすべての許可された機能は使用できません。
  • Default: 商用またはコミッショニングライセンスがインストールされており、ライセンスの猶予期間が切れています。

失効コード

ライセンスがインストールされている際に失効ボタンをクリックして生成したコード。このコードを使用することで新しいライセンスを申請できます。

ライセンス情報

ライセンス情報には、次のものが含まれます。

  • ライセンスSN
  • ライセンスタイプ
  • 次の2種類のライセンスが使用できます。
    • 商用: 契約に基づいてお客様に対して発行された正式なライセンス。許可されているすべての機能の有効期限は同じです。有効期限はPermanentまたは特定の時間に指定できます。ライセンスが期限切れになると、猶予期間は60日になります。
    • トライアル: 新機能のトライアル、システムコミッショニング、ブランド展示に使用される一時ライセンス。有効期間は、実際の状況によって異なります。ライセンスが期限切れになると、猶予期間は60日になります。
  • 有効期限: ライセンスの終了日。永久または特定の時間に指定できます。
  • ライセンスレベル
    • 標準エディション（デフォルト）: 標準BMCエディションで提供されている機能のライセンス。ユーザーはライセンスを購入する必要はありません。
    • アドバンスドエディション: BMCアドバンスドエディションで提供されている機能のライセンス。ユーザーはライセンスを購入した後にのみ、これらの機能を使用できます。
  • ライセンスの猶予期間: ライセンスの有効期限が切れた後でもBMCが使用できる日数。60に指定されています。

拡張機能

ライセンスの拡張機能を表示します。

各機能の番号、機能名称、機能状態、現在状態、有効期限が含まれます。
