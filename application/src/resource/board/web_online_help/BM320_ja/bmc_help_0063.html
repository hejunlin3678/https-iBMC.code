KVMを開いた後、管理システムに接続できない

故障現象

症状説明

考えられる原因

KVMを開いた後、「管理システムへの接続に失敗しました。管理システムのIP：parameter0」のメッセージが表示されました。

KVMサービスのデフォルトポートが2198です。KVMサービスポートが有効になっていないか、使用できない場合にこの問題が発生します。

解決策

  1. BMC WebUIから[サービス] > [ポートサービス]を選択して、KVMサービスが有効になっているかどうかを確認します。

    • はい => 2
    • いいえ => 3

  2. cmdコマンドでCLIを開き、telnetコマンド（例：telnet xx.xx.xx.xx 2198）を実行して、KVMサービスポートにアクセスできるかどうかを確認します。

    xx.xx.xxはIPアドレスです。2198はデフォルトのKVMポート番号です。1で取得した実際のKVMポート番号を使用してください。

    • はい => 5
    • いいえ => 4

  3. KVMサービスを有効にして、改めてKVMにアクセスします。問題が解決されたかどうかを確認します。

    • はい =>処理終了
    • いいえ => 4

  4. ネットワーク管理者に連絡して、KVMに必要なポートを有効にします。ポートにアクセスできることを確認した上、改めてKVMにアクセスして、問題が解決されたかどうかを確認します。

    • はい =>処理終了
    • いいえ => 5

  5. 技術サポートのエンジニアにご連絡ください。
