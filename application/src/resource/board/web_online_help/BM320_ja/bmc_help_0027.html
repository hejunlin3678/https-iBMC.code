二要素認証

機能概要

二要素認証は、クライアント証明書とパスワードの両方が正しい場合にのみユーザーアクセスを許可します。従来のアカウントパスワードだけでの認証に比べて、セキュリティが向上します。

CAによって発行されたルート証明書とクライアント証明書をBMCにアップロードすることで、クライアントとBMC WebUI間の安全な接続を実装できます。

説明

二要素認証

二要素の認証機能を有効にするかどうかを設定します。この機能が有効になっている場合、クライアントからBMC WebUIにログインする際にユーザー名とパスワードを入力せず、証明書を使用してログインします。

デフォルトでは無効になっています。

NOTE:
  • 二要素認証を有効にする前に、有効なルート証明書とクライアント証明書をインポートする必要があります。また、有効な証明書が少なくとも1セット存在していることを確認してください。そうしないと、後のログイン時に認証失敗が発生する可能性があります。
  • 二要素認証が有効になっている場合、SSHサービスは自動的に無効になり、手動では有効にできません。、
  • 二要素認証を有効にしてから再度無効にすると、SSHサービスは自動的に有効になりません。SSHサービスを使用する必要がある場合は、手動で有効にしてください。

OCSPチェック

OCSPチェックにより、認証中にクライアント証明書の有効性を検証します。クライアント証明書が無効な場合、ユーザーはBMC WebUIにログインできません。

デフォルトでは無効になっています。

NOTE:
  • OCSPチェックでは、OCSP（Online Certificate Status Protocol）が使用されます。OCSPチェックを有効にする前に、BMCとOCSPサーバー間の通信が正常であることを確認してください。通信に異常がある場合、Webサービスが利用できなくなる可能性があります。
  • OCSPアドレス情報がルート証明書に書き込まれていることを確認してください。書き込まれていない場合、OCSPチェックが有効にすると、二要素認証が失敗する可能性があります。

CRLチェック

証明書失効リスト（CRL）チェックは、認証中に証明書が失効したかどうかを確認します。この機能を有効にすると、システムは、BMC WebUIへのログイン中に現在のクライアント証明書が失効したかどうかを確認します。証明書が失効した場合、認証は失敗し、BMC WebUIにログインすることはできません。

デフォルトでは無効になっています。

NOTE:

CRLがインポートされていることを確認してください。インポートされていない場合、CRLチェックを有効にすると、二要素認証が失敗する可能性があります。

ルート証明書

BMCに存在しているルート証明書のリスト、また、各ルート証明書の発行元、利用者、有効期限、CRLおよびCRL有効期限が表示されます。

BMCは最大16個のルート証明書をサポートします。

NOTE:
  • Base64エンコードのルート証明書をアップロードすることができます。有効なルート証明書の形式には、*.cer、*.crt、*.pemがあります。サイズは1 MBを超えることはできません。
  • 証明書失効ステータス。
    • 設定済み：証明書失効ファイルがアップロードされており、TLS接続がセットアップされると、証明書失効の検証が実行されます。
    • 未設定：証明書失効ファイルはアップロードされませんでした。
  • CRL有効期限が切れると、対応する認証機能は使用できなくなります。

クライアント証明書

BMCに存在しているクライアント証明書リスト、また、各クライアント証明書に関連付けられているユーザー名、ロール、ルート証明書のアップロード状態、失効状態、失効時間、およびクライアント証明書フィンガープリント（つまり、クライアント証明書ファイルのハッシュ値）が表示されます。

BMCは最大16人のユーザーに対応するクライアント証明書をサポートします。

NOTE:
  • Base64エンコードのクライアント証明書（公開鍵証明書）をアップロードすることができます。有効なクライアント証明書の形式には、*.cer、*.crt、*.pemがあります。サイズは1 MBを超えることはできません。
  • 証明書失効ステータス。
    • 失効
    • 有効

二要素認証の有効化とBMCへの証明書のアップロード

  • 操作を行う前に、信頼できるCA証明書発行機関にルート証明書とクライアント証明書を申請してください。
    • 秘密鍵証明書は.pem、.p12、または.pdx形式になります。操作の詳細については、認証局の操作説明を参照してください。
    • セキュリティ上の理由から、定期的に証明書を更新してください。
  • Base64エンコードのルート証明書とクライアント証明書（公開鍵証明書）をアップロードすることができます。有効なルート証明書とクライアント証明書の形式には、*.cer、*.crt、*.pemがあります。サイズは1 MBを超えることはできません。
  1. [証明書のアップロード]をクリックします。

    アップロードする証明書を選択します。

  2. [開く]をクリックします。
  3. [二要素認証]をに設定します。

CRLのアップロード

Base64でエンコードされた証明書失効ファイルは* .crl形式であり、100 KBを超えることはできません。

  1. CAから証明書失効ファイルを取得します。
  2. ターゲット証明書を見つけて、[CRL]列にあるを[証明書のアップロード]クリックします。
  3. 証明書失効ファイルを選択します。

証明書認証の有効化

証明書をアップロードした後、次の操作を実行して、BMC WebUIにログインしようとするユーザーの証明書認証を有効にしてください。

  1. クライアントで、ブラウザ（Google Chromeなど）を開きます。
  2. 右上隅のをクリックして[設定]を選択します。
  3. [設定]ウィンドウで、[HTTPS/SSL]の下にある[証明書の管理]をクリックします。
  4. クライアント証明書をインポートします。
  5. ブラウザのアドレスボックスにBMCログインアドレスを入力します。
  6. プロンプトに従ってクライアント証明書を選択します。

    BMC WebUIへのログインに成功しました。

ルート証明書の削除

ルート証明書は、二要素認証機能が無効になっている場合にのみ削除できます。

  1. [ルート証明書]タブページで、削除対象となるルート証明書の横にある削除をクリックします。

    確認ダイアログボックスが表示されます。

  2. [はい]をクリックします。

クライアント証明書の削除

  1. [クライアント証明書]画面で、クライアント証明書が削除されるユーザーの横にある削除をクリックします。

    確認ダイアログボックスが表示されます。

  2. [はい]をクリックします。

CRLの削除

CRLを削除すると、期限切れの証明書を使用するセキュリティリスクが発生する場合があります。

  1. ターゲット証明書を見つけて、[CRL]列にあるを[CRLを削除]クリックします。

    確認ダイアログボックスが表示されます。

  2. [はい]をクリックします。
