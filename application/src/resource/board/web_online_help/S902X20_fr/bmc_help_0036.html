SNMP

Présentation de la fonction

Le protocole SNMP (Simple Network Management Protocol) consiste en un ensemble de normes de gestion de réseau. Il contient un protocole de couche d'application, un modèle de base de données et un groupe d'objets ressources. Ce protocole aide un système de gestion de réseau à détecter les périphériques connectés au réseau.

Sur la page SNMP, vous pouvez interroger et configurer les paramètres de service SNMP (Simple Network Management Protocol).

L'BMC prend en charge les versions SNMP suivantes :

  • SNMPv1 : La première version officielle de SNMP définie dans les Requests for Comments (RFC) 1157. L'utilisation de SNMPv1 peut poser des risques de sécurité. Pour des raisons de sécurité, utilisez SNMPv3.
  • SNMPv2 : Un protocole expérimental défini dans la RFC 1901 et qui adopte une architecture de gestion basée sur la communauté. L'utilisation de SNMPv2 peut poser des risques de sécurité. Pour des raisons de sécurité, utilisez SNMPv3.
  • SNMPv3 : La troisième version officielle de SNMP, qui améliore les capacités de sécurité et de configuration à distance sur la base des versions précédentes.

Mot de passe long

Réglage de la fonction de mot de passe long, qui peut être activée ou désactivée.

Activez cette fonction pour imposer un minimum de 16 caractères pour les noms de communauté.

Valeur par défaut : Activer

Communauté en lecture seule

Nom de la communauté en lecture seule

Valeur :
  • Si la vérification du mot de passe est désactivée, les noms de communauté doivent répondre aux exigences suivantes :
    • Si la fonction de mot de passe long est activée, le nom de la communauté est une chaîne de 16 à 32 caractères sans espaces.
    • Si la fonction de mot de passe long est désactivée, le nom de la communauté est une chaîne de 1 à 32 caractères sans espaces.
  • Si la vérification du mot de passe est activée, les noms de communauté doivent répondre aux exigences suivantes :
    • Longueur :
      • Une chaîne de 16 à 32 caractères si la fonction de mot de passe long est activée.
      • Une chaîne de 8 à 32 caractères sans espaces si la fonction de mot de passe long est désactivée.
    • Contient au moins un des caractères spéciaux suivants :

      `~!@#$%^&*()-_=+\|[{}];:'",<.>/?

    • Contient au moins deux types des caractères suivants :
      • Lettres majuscules de A à Z
      • Lettres minuscules de a à z
      • Chiffres 0 à 9
    • Ne peut pas contenir d'espaces.
  • Si la vérification du mot de passe faible est activée, le mot de passe ne peut pas être identique à celui contenu dans le dictionnaire de mots de passe faibles. (Vous pouvez exécuter la commande ipmcset -t user -d weakpwddic -v export <filepath | file_URL> pour exporter les mots de passe faibles à partir du dictionnaire de mots de passe faibles.)
