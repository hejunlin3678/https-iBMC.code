Mise à jour du certificat SSL

Présentation de la fonction

Sur la page Mise à jour du certificat, vous pouvez :

  • Configurer les paramètres de la mise à jour du certificat SSL.
  • Configurer le serveur CA.
  • Personnaliser le sujet du certificat.

Description du paramètre

Tableau 1 Mise à jour du certificat

Paramètre

Description

Mise à jour du certificat SSL

REMARQUE:
  • Un certificat SSL ne peut être mis à jour que lorsque le verrouillage du système est désactivé, que l'utilisateur dispose de l'autorisation de configuration de la sécurité et que le serveur CA est configuré.
  • Pour mettre à jour le certificat SSL, vous devez importer à la fois le certificat racine du serveur et le certificat client. Sinon, l'authentification échouera.

Mise à jour automatique

Active ou désactive la fonction de mise à jour automatique du certificat.

Une fois l'option Mise à jour automatique activée, le certificat SSL est automatiquement mis à jour à un intervalle d'une heure.

Cette fonction est désactivée par défaut.

Mise à jour du certificat

Vous pouvez cliquer sur Mise à jour du certificat pour mettre à jour le certificat SSL.

Configuration du serveur CA

Adresse du serveur

Adresse IP du serveur CA.

Adresse IPv4, adresse IPv6 ou nom de domaine d'une longueur maximale de 67 caractères.

Port

Numéro de port du serveur CA.

Plage de valeurs : 1 à 65535

Chemin

Chemin du serveur CA.

Une chaîne comportant de 1 à 128 caractères. Commence par une barre oblique (/) et se compose de chiffres (0-9), de lettres (a-z, A-Z), de (_), de soulignement et de barres obliques (/).

Sujet du certificat

Pays (P)

Pays auquel appartient le certificat.

Un code de pays à deux lettres. Toutes les lettres anglaises sont autorisées.

État/Province (É/P)

État ou province auquel appartient un certificat.

Une chaîne comportant de 0 à 128 caractères. Les chiffres (0-9), les lettres (a-z, A-Z), les tirets (-), les traits de soulignement (_), les points (.) et les espaces sont autorisés.

Ville/Localisation (L)

Ville ou localisation à laquelle appartient un certificat.

Une chaîne comportant de 0 à 128 caractères. Les chiffres (0-9), les lettres (a-z, A-Z), les tirets (-), les traits de soulignement (_), les points (.) et les espaces sont autorisés.

Nom de l'organisation (O)

Entreprise ou organisation à laquelle appartient un certificat.

Une chaîne comportant de 0 à 64 caractères. Les chiffres (0-9), les lettres (a-z, A-Z), les tirets (-), les traits de soulignement (_), les points (.) et les espaces sont autorisés.

Unité organisationnelle (UO)

Unité organisationnelle ou service à la quelle/auquel appartient un certificat.

Une chaîne comportant de 0 à 64 caractères. Les chiffres (0-9), les lettres (a-z, A-Z), les tirets (-), les traits de soulignement (_), les points (.) et les espaces sont autorisés.

Nom Commun (NC)

Nom commun du certificat.

Une chaîne comportant de 1 à 64 caractères. Les chiffres (0-9), les lettres (a-z, A-Z), les tirets (-), les traits de soulignement (_), les points (.) et les espaces sont autorisés.

Nom interne (NI)

Nom interne du certificat.

Une chaîne comportant de 0 à 64 caractères. Les chiffres (0-9), les lettres (a-z, A-Z), les tirets (-), les traits de soulignement (_), les points (.) et les espaces sont autorisés.

E-mail (E)

Adresse e-mail du certificat.

Une chaîne de 0 à 255 caractères conforme à la convention des adresses électroniques.

Certificat racine du serveur

Certificat racine du serveur appliqué à partir d'une autorité de certification officielle. Certificat utilisé pour effectuer l'authentification TLS sur les paquets de certificats de serveur envoyés par le serveur CA avant qu'une connexion ne soit établie.

REMARQUE:
  • Les certificats racine du serveur au format .crt, .cer ou .pem sont pris en charge. Le code du certificat au format *.cer est Base64. La taille maximale du certificat est de 100 Ko. Les certificats en double ne sont pas autorisés.
  • Pour des raisons de sécurité, mettez périodiquement à jour le certificat.

Certificat client

Certificat client appliqué à partir d'une autorité de certification officielle.

REMARQUE:
  • Les certificats client au format *.p12 ou *.pfx sont pris en charge. La taille maximale du certificat est de 100 Ko.
  • Pour des raisons de sécurité, mettez périodiquement à jour le certificat.

Informations sur le certificat

Affiche des informations sur le certificat téléchargé, y compris :

  • Émis par
  • Émis pour
  • Période de validité
  • Numéro de série
  • CRL, avec l'un des statuts suivants :
    • Configuré : le fichier de révocation de certificats a été chargé et une vérification de la révocation de certificats sera effectuée lors de l'établissement d'une connexion TLS.
    • Non configuré : le fichier de révocation du certificat n'est pas téléchargé.
REMARQUE:
  • Le fichier CRL être au format *.crl. Le nom de fichier ne peut pas être vide. La taille maximale du fichier CRL est de 100 Ko.
  • Si la CRL expire, la fonction d'authentification correspondante sera indisponible.
  • Le certificat du client ne prend pas en charge la CRL.

Chargement d'un certificat

  1. Cliquez sur Charger à côté de Certificat du serveur ou Certificat client.

    Une boîte de dialogue pour la sélection de fichier local ou le chargement de certificat client s'affiche.

    • Les certificats racine du serveur au format .cer, .crt, ou .pem sont pris en charge. Le code du certificat au format *.cer est Base64. La taille maximale du certificat est de 100 Ko. Les certificats en double ne sont pas autorisés.
    • Les certificats client au format *.p12 ou *.pfx sont pris en charge. La taille maximale du certificat est de 100 Ko.

  2. Sélectionnez un fichier au format correct.
  3. Cliquez sur OK.

Configuration d'une CRL

Le fichier CRL être au format *.crl. Le nom de fichier ne peut pas être vide. La taille maximale du fichier CRL est de 100 Ko.

  1. Obtenez le fichier auprès de l'autorité de certification officielle
  2. Dans la zone Informations sur le certificat, cliquez sur Charger à côté de CRL.
  3. Sélectionnez le fichier CRL.
  4. Entrez le mot de passe de l'utilisateur de connexion et cliquez sur Enregistrer.

Supprimer d'une CRL

La suppression d'une CRL peut entraîner des risques de sécurité liés à l'utilisation de certificats expirés.

  1. Localisez le certificat cible et cliquez sur Supprimer la CRL dans la colonne CRL.

    Une boîte de dialogue de confirmation s'affiche.

  2. Cliquez sur Oui.
