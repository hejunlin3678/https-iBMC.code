Réseau

Présentation de la fonction

Sur la page Paramètres réseau, vous pouvez consulter et configurer les paramètres suivants de le BMC :

  • Nom de l'hôte
  • Port réseau
  • Version et adresses IP
  • Infos DNS
  • VLAN
  • LLDP

La modification de l'adresse IP du port réseau de gestion entraîne la déconnexion du réseau. Modifiez l'adresse IP uniquement si nécessaire.

Description du paramètre

Tableau 1 Réseau

Parameter

Description

Nom de l'hôte BMC

Nom de l'hôte BMC.

Valeur : une chaîne de 1 à 64 caractères

La valeur peut contenir des lettres, des chiffres et des tirets (-), mais ne peut pas commencer ou se terminer par un tiret.

Mode

Type du port réseau de gestion du serveur, c'est-à-dire le port réseau BMC.

Valeur par défaut : Paramètres fixes

Paramètres fixes

Si vous sélectionnez cette option, vous devez également spécifier le port réseau BMC.

Valeur :

  • Port dédié : port BMC dédié (c'est-à-dire le port de la gestion du serveur).
  • Port PCIe externe : port d'une carte PCIe (c'est-à-dire une carte PCIe prenant en charge l'interface de bande latérale du contrôleur réseau (NC-SI) et connectée avec un câble de l'interface de bande latérale du contrôleur réseau (NC-SI).
  • Port OCP : un port réseau de service sur un OCP1.
  • Port OCP2 : un port réseau de service sur un OCP2.

La valeur par défaut est Port dédié.

Sélection automatique

Si vous sélectionnez cette option, le BMC sélectionne automatiquement le port BMC en fonction de l'état du port.

Vous devez également sélectionner les ports impliqués dans la sélection automatique. Si plusieurs ports sont disponibles, le BMC sélectionne un port en fonction de la priorité suivante:

Port dédié > port PCIe (ports 1 à 2 ou ports 1 à 4) > port OCP (ports 1 à 2 ou ports 1 à 4) > port OCP2 (ports 1 à 2 ou ports 1 à 4)

Les ports OCP2 et PCIe s'excluent mutuellement. Si la carte d'interface réseau PCIe est connectée à un câble de l'interface de bande latérale du contrôleur réseau (NC-SI), les ports de la carte d'interface réseau PCIe peuvent être utilisés, mais ceux de ports OCP2 ne peuvent pas être utilisés pour accéder à le BMC. Si la carte d'interface réseau PCIe n'est pas connectée à un câble de l'interface de bande latérale du contrôleur réseau (NC-SI), les ports OCP2 peuvent être utilisés, mais ceux de la carte d'interface réseau PCIe ne peuvent pas être utilisés pour accéder à le BMC.

REMARQUE:
  • Seule le port réseau sur la carte PCIe connectée avec des câbles de l'interface de bande latérale du contrôleur réseau (NC-SI) peut être utilisée en tant que port BMC.
  • Si un port PCIe, un port OCP ou un port OCP2 est sélectionné manuellement ou automatiquement, le même port physique sert de port de gestion et de port réseau de service. Pour des raisons de sécurité, si le mode Paramètres fixes ou Sélection automatique implique le PCIe, OCP ou OCP2, configurez le VLAN pour le port réseau de gestion.
  • Si un port réseau est sélectionné en tant que port réseau de gestion BMC, s'affichera derrière le port réseau.

Mode NCSI

Mode de commutation de port réseau NCSI lorsque le paramètre Mode est réglé sur Sélection automatique.

La valeur par défaut est Commutation manuelle.

Basculement automatique

Le système bascule automatiquement vers le port réseau externe ayant une priorité plus élevée en fonction de l'état de connexion du port. En fonction de la règle de sélection automatique, le BMC sélectionne le port réseau externe de gestion dans l'ordre suivant :

port 1 > port 2 ou port 1 > port 2 > port 3 > port 4.

Commutation manuelle

Basculer manuellement vers le port réseau externe ayant une priorité plus élevée.

Port réseau BMC
  • Si Mode est défini sur Paramètres fixes, spécifiez un port réseau de gestion.
  • Si Mode est défini sur Sélection automatique, sélectionnez les ports réseau pour la négociation automatique.

Protocoles réseau

Versions IP qui peuvent être activées :

  • IPv4 : autorise uniquement IPv4. Si cette option est sélectionnée, seules les adresses IPv4 peuvent être configurées.
  • IPv6 : autorise uniquement IPv6. Si cette option est sélectionnée, seules les adresses IPv6 peuvent être configurées.
  • IPv4 / IPv6 : autorise IPv4 et IPv6. Si cette option est sélectionnée, les adresses IPv4 et IPv6 peuvent être configurées.

Valeur par défaut : IPv4/IPv6

IPv4
  • Obtenir automatiquement une adresse IP : cliquez sur cette option pour autoriser l'attribution automatique d'une adresse IPv4 au port réseau BMC.
  • Définir manuellement une adresse IP : cliquez sur cette option pour définir manuellement une adresse IPv4 pour le port réseau BMC.

    Les informations d'adresse IPv4, y compris : Adresse IP, Masque de sous-réseau, Passerelle par défaut et Adresse MAC.

La valeur par défaut est Définir manuellement une adresse IP.

REMARQUE:
  • Adresse MAC spécifie l'adresse physique d'une carte d'interface réseau (NIC).
  • Si vous n'utilisez pas la passerelle par défaut, définissez la passerelle pour toute adresse IP dans le segment de réseau des adresses IP statiques et flottantes.

IPv6
  • Obtenir automatiquement une adresse IP : cliquez sur cette option pour autoriser l'attribution automatique d'une adresse IPv6 au port réseau BMC.
  • Définir manuellement une adresse IP : cliquez sur cette option pour définir manuellement une adresse IPv6 pour le port réseau BMC.

    Les informations d'adresse IPv6, y compris : Adresse IP, Préfixe, Passerelle par défaut, Adresse de lien local et Adresse IP2.

La valeur par défaut est Définir manuellement une adresse IP.

REMARQUE:
  • Une Adresse de lien local est utilisée pour la communication locale de lien.
  • Adresse IP2 prend en charge un maximum de quinze adresses IPv6 lorsque la configuration automatique des adresses sans état (SLAAC) est utilisée.
  • Si vous n'utilisez pas la passerelle par défaut, définissez la passerelle pour toute adresse IP dans le segment de réseau des adresses IP statiques et flottantes.

DNS
  • Obtenir automatiquement une adresse IPv4 DNS : cliquez sur cette option pour autoriser l'attribution d'une adresse IPv4 au serveur du système de noms de domaine.
  • Obtenir automatiquement une adresse IPv6 DNS : cliquez sur cette option pour autoriser l'attribution d'une adresse IPv6 au serveur du système de noms de domaine.
  • Définir manuellement une adresse : cliquez sur cette option pour définir manuellement les informations du système de noms de domaine. Les informations d'adresse du système de noms de domaine, y compris : Domaine, Serveur préféré et Serveur alternatif.

La valeur par défaut est Définir manuellement une adresse.

AVIS:
  • Si l'adresse IP du port réseau de gestion de le BMC est obtenue automatiquement, vous pouvez obtenir les informations DNS automatiquement ou manuellement. L'obtention automatique est recommandée.
  • Si l'adresse IP du port réseau de gestion BMC est définie manuellement, les informations DNS doivent également être définies manuellement.

Domaine : nom de domaine pour le serveur.

Le nom de domaine doit répondre aux exigences suivantes :
  • Une chaîne de 0 à 67 caractères.
  • Seulement les chiffres, les lettres, les traits d'union (-) et les points (.) sont autorisés.
  • Ne peut pas commencer par un trait d'union ou un point, ni se terminer par un trait d'union.
  • Un maximum de 63 caractères sont autorisé entre deux points.

Serveur préféré : Adresse IP du serveur préféré du système de noms de domaine.

La valeur peut être une adresse IPv4, une adresse IPv6 ou vide.
  • Serveur alternatif 1 : adresse IP du premier serveur DNS alternatif.
  • Serveur alternatif 2 : adresse IP du deuxième serveur DNS alternatif.

La valeur peut être une adresse IPv4, une adresse IPv6 ou vide.

VLAN

Configuration Vlan du port réseau NC-SI.

Valeur par défaut : Désactiver

REMARQUE:
  • Nous vous conseillons d'activer le VLAN et de configurer les ID VLAN pour mettre en uvre l'isolation entre le réseau de service et le réseau de gestion.
  • Si Port dédié est sélectionné en tant que port réseau de gestion BMC, la configuration du VLAN est invalide. Si une autre valeur que Port dédié est sélectionnée en tant que port réseau de gestion BMC, la configuration du VLAN est valide.

VLAN ID

VLAN auquel appartient le port réseau NC-SI.

Plage de valeur : un entier de 1 à 4094

REMARQUE:

Une fois l'ID VLAN enregistré, il faut quelques secondes pour que ces paramètres prennent effet.

LLDP

REMARQUE:

LLDP n'est pris en charge que lorsque le port réseau dédié est utilisé en tant que port réseau de gestion BMC.

LLDP

Une fois que LLDP est activé, le BMC peut envoyer l'adresse MAC du dispositif aux dispositifs directement connectés en paquets standard, ce qui permet au NMS d'interroger et de déterminer l'état de la communication par liaison.

Valeur par défaut : Désactiver

Mode de fonctionnement

Le mode de travail doit être Émettre.

Le BMC ne prend en charge que la transmission de paquets LLDP et ne peut pas recevoir de paquets LLDP.

Délai de transmission (s)

Dans le mode de travail actuel, un paquet LLDP sera envoyé pour avertir les dispositifs voisins lorsque la configuration locale d'BMC (le port réseau de gestion est modifié ou le câble réseau du port réseau de gestion connecté ou déconnecté) est modifié.

Un délai de transmission (en secondes) est utilisé pour empêcher l'envoi d'un grand nombre de paquets LLDP en raison de modifications fréquentes de la configuration locale. Si la configuration locale d'BMC est modifiée pendant le délai, le système réinitialisera le compteur. Un prochain paquet LLDP ne sera envoyé que lorsque le délai aura expiré.

Valeur : 1 à 8192

Valeur par défaut : 2

Intervalle de transmission (s)

Intervalle (en secondes) auquel les paquets LLDP sont envoyés si les informations de configuration locales ne sont pas modifiées.

Valeur : 5 à 32768

Valeur par défaut : 30

Multiplicateur du délai de maintien

Si le nud voisin ne reçoit pas de paquet LLDP de le BMC dans le délai spécifié (intervalle de transmission × multiplicateur du délai de maintien), le nud voisin effacera automatiquement les paquets précédemment réservés.

Valeur : 2 à 10

Valeur par défaut : 4
