LDAP

Présentation de la fonction

La page LDAP vous permet de voir et de configurer les informations utilisateur LDAP (Lightweight Directory Access Protocol).

L'BMC fournit une fonction d'accès pour les utilisateurs LDAP. Un utilisateur LDAP peut se connecter à l'interface utilisateur Web BMC ou utiliser un outil SSH pour se connecter à l'interface CLI d'BMC. L'utilisation d'un compte d'utilisateur de domaine pour accéder à l'BMC améliore la sécurité du système.

Sur le serveur LDAP, DisplayName et CN (common name) doivent être identiques.

L'BMC prend en charge un maximum de six serveurs de domaine.

Lors de la connexion à l'interface utilisateur Web BMC, le serveur de domaine peut être spécifié manuellement ou faire l'objet d'une recherche automatique. Lors de la connexion à l'interface CLI BMC, le serveur de domaine peut faire l'objet d'une recherche automatique.

L'BMC prend en charge Windows Active Directory (AD), Linux OpenLDAP, et FreeIPA.

Tableau 1 LDAP

Paramètre

Description

LDAP

La fonction LDAP permet aux utilisateurs du domaine d'accéder à l'BMC.

Il est désactivé par défaut.

REMARQUE:

Lorsqu'un compte LDAP est utilisé pour se connecter à BMC après l'activation de la fonction LDAP, le serveur d'authentification garantit les politiques de sécurité (vérification de mot de passe, durée de validité du mot de passe, durée de vie minimum du mot de passe, période inactive, anciens mots de passe interdits et blocage en cas de l'échec de connexion) liées au compte.

Contrôleur 1

L'BMC prend en charge un maximum de six contrôleurs de domaine (serveurs). Lorsqu'un utilisateur tente de se connecter à l'interface utilisateur Web BMC via LDAP, il peut sélectionner le contrôleur de domaine ou Correspondance automatique.

Les contrôleurs 1 à 6 ont les mêmes paramètres.

REMARQUE:

Les paramètres marqués d'un astérisque (*) sont obligatoires.

Paramètres de base

Adresse du serveur LDAP

Adresse IP du serveur LDAP.

Format : une adresse IPv4 ou IPv6 ou un nom de domaine

Une fois la vérification du certificat activée, définissez ce paramètre sur le nom de domaine complet du serveur LDAP (nom d'hôte, nom de domaine) et configurez les informations d'adresse DNS sur la page Réseau.

REMARQUE:
Le nom de domaine doit répondre aux exigences suivantes :
  • Une chaîne de 0 à 255 caractères.
  • Seulement les chiffres, les lettres, les traits d'union (-) et les points (.) sont autorisés.
  • Ne peut pas commencer par un trait d'union ou un point, ni se terminer par un trait d'union.
  • Un maximum de 63 caractères sont autorisé entre deux points.

Port LDAPS

Numéro de port pour le service LDAP.

Valeur : un entier compris entre 1 et 65535

Valeur par défaut : 636

REMARQUE:

L'BMC ne prend en charge que le service LDAP utilisant une transmission en cryptage. Vous devez effectuer la configuration correspondante sur le serveur LDAP.

Domaine

Domaine utilisateur auquel appartient un utilisateur LDAP défini dans le contrôleur de domaine.

Le nom de domaine doit répondre aux exigences suivantes :
  • Une chaîne de 0 à 255 caractères.
  • Seulement les chiffres, les lettres, les traits d'union (-) et les points (.) sont autorisés.
  • Ne peut pas commencer par un trait d'union ou un point, ni se terminer par un trait d'union.
  • Un maximum de 63 caractères sont autorisé entre deux points.

Lier le nom distinctif

Nom distinctif (DN) d'un utilisateur proxy LDAP.

Par exemple, CN=nom d'utilisateur,OU=société,DC=domaine,DC=com, qui doit être identique au nom distinctif défini sur le serveur LDAP.

Plage de valeurs : une chaîne de 255 octets (64 à 255 caractères). La longueur spécifique varie avec le nombre d'octets de chaque caractère.

Lier le mot de passe

Mot de passe d'authentification pour l'utilisateur proxy LDAP.

Valeur : une chaîne de 0 à 20 caractères, y compris des chiffres, des lettres et des caractères spéciaux

Dossier utilisateur

Répertoire sur le serveur LDAP de l'utilisateur LDAP qui peut se connecter à l'BMC.

Format : «CN=xxx» ou «OU=xxx».

Lorsqu'il y a plusieurs niveaux de nœuds, le nœud de niveau supérieur suit le nœud de niveau inférieur avec une virgule entre les deux.

Par exemple, si l'utilisateur infotest est dans \testusers\part1 sur le serveur LDAP, saisissez OU=part1,OU=testusers.

REMARQUE:

Pour plus de détails sur la différence entre CN et OU, voir la description du protocole LDAP.

Dans Windows AD, par exemple, c'est le cas :

  • CN si Type est Container.
  • OU si Type est Organizational Unit.

Plage de valeurs : une chaîne de 255 octets (64 à 255 caractères). La longueur spécifique varie avec le nombre d'octets de chaque caractère.

Vérification du certificat LDAP

Vérification du certificat du serveur LDAP, qui peut être activée ou désactivée.

Activer la vérification de certificat pour des raisons de sécurité.

Une fois la vérification du certificat activée, vous devez importer le certificat de l'autorité de certification LDAP, installer les émetteurs des certificats AD, DNS et de l'autorité de certification sur le serveur LDAP, puis importer le certificat de l'autorité de certification sur le serveur LDAP et BMC.

Valeur par défaut : Désactivé

Niveau de vérification du certificat

Niveau de vérification du certificat LDAP.

  • Demand Refuser l'accès à l'BMC si le certificat client est incorrect ou si aucun certificat n'est disponible. Pour des raisons de sécurité, utilisez l'option par défaut (Demand).
  • Allow Autoriser l'accès à l'BMC même si le certificat client est incorrect ou si aucun certificat n'est disponible.

Valeur par défaut : Demand

Certificat LDAP

Le certificat doit être au format.cer,.pem,.cert ou.crt.

REMARQUE:
  • Le système prend plus de temps pour télécharger les fichiers de certificat dont la taille dépasse 100 Mo. Actualisez la page pour le dernier statut.
  • La chaîne de certificats d'un maximum de 10 niveaux est prise en charge.

Rôle

Rôle attribué à un groupe LDAP.

  • Administrateur : Les utilisateurs auxquels le rôle Administrateur est attribué peuvent effectuer toutes les opérations.
  • Opérateur : Les utilisateurs auxquels le rôle Opérateur est attribué peuvent effectuer la gestion de base, le contrôle à distance, le support à distance, le contrôle d'alimentation, l'interrogation des informations et configurer leurs propres données.
  • Utilisateur courant : Les utilisateurs auxquels le rôle Utilisateur courant est attribué peuvent demander des informations et configurer leurs propres données.
  • Rôle personnalisé : Les utilisateurs auxquels le rôle Personnalisé 1 au rôle Personnalisé 4 est attribué peuvent effectuer les opérations spécifiées.
REMARQUE:

L'autorisation utilisateur par défaut pour un nouveau groupe d'utilisateurs LDAP est Utilisateur courant.

Règles de connexion

Règles de connexion qui s'appliquent au groupe LDAP.

Interfaces de connexion

Interfaces via lesquelles les membres du groupe LDAP peuvent se connecter à l'BMC.

Valeur :
  • SSH : Interfaces par lesquelles les membres du groupe LDAP peuvent se connecter à BMC.
  • Web : L'utilisateur peut utiliser un navigateur Web pour se connecter à l'interface utilisateur Web BMC.
  • Redfish : L'utilisateur peut utiliser un outil Redfish pour se connecter à l'BMC.
REMARQUE:

Un nouveau groupe d'utilisateurs LDAP prend en charge toutes les interfaces de connexion.

Importer un certificat LDAP

Cliquez sur Ajouter le fichier en regard de Certificat LDAP et sélectionnez le certificat LDAP à importer.

Pour des raisons de sécurité, mettez périodiquement à jour le certificat.

Si « Fichier téléchargé avec succès » s'affiche, vous pouvez consulter les détails du certificat téléchargé dans la zone d'informations du certificat. Pour plus de détails sur les paramètres, voir Tableau 2.

Upload d'une CRL

Le fichier de révocation de certificat est codé en Base64 et au format *.crl. Il ne peut pas dépasser 100 Ko.

  1. Obtenez un fichier de révocation de certificat auprès de l'autorité de certification (CA).
  2. Dans la zone Informations sur le certificat du serveur, cliquez sur Charger à côté de CRL.
  3. Sélectionnez le fichier de révocation de certificat.
  4. Saisissez le mot de passe de l'utilisateur actuellement connecté et cliquez sur Confirmer.

Supprimer un groupe LDAP

  1. Dans la zone Groupe LDAP, cliquez sur Supprimer pour supprimer le groupe LDAP.

    Une boîte de dialogue s'affiche et vous invite à saisir le mot de passe utilisateur actuel.

  2. Entrez le mot de passe de l'utilisateur actuel.
  3. Cliquez sur OK.

    Le message « Opération réussie » s'affiche.

Modifier un groupe LDAP

  1. Dans la zone Groupe LDAP, cliquez sur Modifier pour que le groupe LDAP soit édité.
  2. Entrez le mot de passe utilisateur actuel et modifiez les paramètres du groupe LDAP. Pour plus de détails sur les paramètres, voir Tableau 3.
  3. Cliquez sur Enregistrer.

    Une fois les informations du groupe LDAP modifiées ou supprimées, les utilisateurs qui se sont connectés au KVM ne seront pas automatiquement déconnectés du système. Pour vous déconnecter des comptes utilisateurs KVM, allez à la page Utilisateurs en ligne et déconnectez-vous des comptes utilisateurs.
