Kerberos

Описание функции

На странице Kerberos можно просмотреть и настроить основные параметры Kerberos, а также информацию о группе пользователей.

BMC предоставляет интерфейс доступа для пользователей Kerberos. Использование учетной записи пользователя Kerberos для входа в систему повышает уровень безопасности системы. Вы можете войти в веб-интерфейс BMC как пользователь Kerberos.

Табл. 1 Kerberos

Параметр

Описание

Kerberos

Включение или отключение Kerberos.

По умолчанию данная функция отключена.

ПРИМЕЧАНИЕ:

Политики безопасности (проверка сложности пароля, срок действия пароля, минимальный срок действия пароля, предыдущие пароли запрещены, неактивный временной интервал и политика блокировки пользователя), настроенные на сервере аутентификации, применяются к пользователям Kerberos, пытающимся войти в систему BMC.

Основные настройки

ПРИМЕЧАНИЕ:

Параметры со знаком звездочки (*) являются обязательными.

Область

Область Kerberos.

Значение: строка длиной до 255 символов

Значение может содержать цифры, прописные буквы и специальные символы (включая пробелы).

Адрес сервера Kerberos

Адрес сервера Kerberos.

Если функция Kerberos включена, введите полное доменное имя (Имя хоста.Имя домена) сервера Kerberos, а затем настройте DNS на странице Настройки BMC > Настройки сети.

Формат: адрес IPv4 или IPv6 или доменное имя

Доменное имя должно соответствовать определенным требованиям:
  • Оно должно включать от 0 до 255 символов.
  • Разрешается содержать только цифры, буквы, дефисы (-) и точки (.).
  • Не может начинаться с дефиса или точки или заканчиваться дефисом.
  • Между любыми двумя точками должно быть не более 63 символов.

Порт Kerberos

Номер порта службы Kerberos.

Диапазон значений: от 1 до 65535

Значение по умолчанию: 88

Таблица ключей

Таблица ключей Kerberos, которую необходимо загрузить.

Таблица ключей должна быть в формате .keytab.

По умолчанию она оставлена пустой.

ПРИМЕЧАНИЕ:
  • Размер таблицы ключей не может быть 0 КБ или больше 1 МБ.
  • В целях безопасности периодически обновляйте ключ.

Группа пользователей Kerberos

Отображается информация обо всех группах пользователей Kerberos.

BMC поддерживает максимум пять групп пользователей Kerberos.

Имя группы

Имя группы, к которой принадлежит пользователь Kerberos.

Диапазон значений: строка из 255 байтов (длиной от 64 до 255 символов). Конкретная длина зависит от количества байтов каждого символа.

SID

Идентификатор безопасности, используемый для авторизации Kerberos и группы пользователей. Например, S-1-5-21-310440588-250036847-580389505-500.

Диапазон значений: строка из 255 байтов (длиной от 64 до 255 символов) без пробелов. Конкретная длина зависит от количества байтов каждого символа.

Роль

Роль группы пользователей Kerberos.

  • Администратор: пользователи, которым назначена роль Администратор, могут выполнять все операции.
  • Оператор: пользователи с ролью Оператор выполняют базовые операции, удаленный контроль, доступ к удаленной среде, управление питанием, запрос информации и конфигурируют свои собственные данные.
  • Стандартный пользователь: пользователи с ролью Стандартный пользователь запрашивают информацию и конфигурируют свои собственные данные.
  • Ограниченная роль: пользователи, которым назначены роли от Ограниченная роль 1 до Ограниченная роль 4, могут выполнять только заданные операции.
ПРИМЕЧАНИЕ:

Стандартный пользователь — это разрешение пользователя по умолчанию для новой группы пользователей Kerberos.

Интерфейсы входа

Интерфейсы, через которые участники группы Kerberos смогут войти в BMC.

  • Web

    По умолчанию новые группы пользователей Kerberos могут войти в систему через WebUI.

  • Redfish

    По умолчанию новые группы пользователей Kerberos могут войти в систему через Redfish API.

Папка группы

Имя папки пользователя на сервере Kerberos.

Формат: CN=xxx или OU=xxx

При наличии нескольких уровней узлов, узел верхнего уровня следует за узлом нижнего уровня с запятой между ними.

Например, если группа пользователей grouptest находится в \testgroups\part1 на сервере Kerberos, введите OU=part1,OU=testgroups.

ПРИМЕЧАНИЕ:

Более подробная информация о различиях между CN и OU приведена в описании протокола Kerberos.

Например, в Windows AD:

  • Если параметром Type является Container, узел — CN.
  • Если параметром Type является Organizational Unit, узел — OU.

Диапазон значений: строка из 255 байтов (длиной от 64 до 255 символов). Конкретная длина зависит от количества байтов каждого символа.

Правила входа

Правила входа в систему, применимые к группе Kerberos.

Изменить

Отображение региона для настройки существующей группы Kerberos.

Изменение группы Kerberos.

Активация Kerberos и конфигурирование сервера домена

  1. Установите для Kerberos.
  2. Настройте сервер домена. Подробная информация о параметрах приведена в Табл. 1.
  3. Нажмите Сохранить.

Импорт таблицы ключей

Таблица ключей должна быть в формате .keytab. Размер таблицы ключей не может быть 0 КБ или больше 1 МБ.

  1. Нажмите Выбрать файл и выберите таблицу ключей для импорта.
  2. Нажмите Открыть.

    После загрузки таблицы ключей на экране появится сообщение «Файл успешно загружен».

  3. Введите имя текущего пользователя.

    Пользователю Kerberos, который входит в систему с помощью единого входа, не нужно вводить пароль.

  4. Нажмите Сохранить.

Удаление группы пользователей Kerberos

  1. В поле Группа пользователей Kerberos найдите группу пользователей Kerberos, которую необходимо удалить, и нажмите Удалить.

    Отображается диалоговое окно Подтвердить с просьбой ввода текущего пароля пользователя.

    Пользователю Kerberos, который входит в систему с помощью единого входа, не нужно вводить пароль.

  2. Введите текущий пароль пользователя.
  3. Нажмите Да.

  1. В поле Группа пользователей Kerberos найдите группу пользователей Kerberos, которую необходимо изменить, и нажмите Изменить.

    На экране появится страница Изменение группы пользователей.

  2. Измените параметры. Подробная информация о параметрах приведена в Табл. 2.
  3. Нажмите Сохранить.
