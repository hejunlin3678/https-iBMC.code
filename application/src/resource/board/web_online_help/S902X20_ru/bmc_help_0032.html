Web-службы

Описание функций

На странице Web-службы доступны следующие операции:

  • Просмотр и настройка основных параметров веб-служб.
  • Просмотр используемых сертификатов SSL.
  • Настройка и импорт сертификатов SSL.

Сертификат SSL необходим для установления безопасного канала SSL посредством HTTPS между веб-браузером клиента и веб-сервером, по которому будут передаваться зашифрованные данные. SSL обеспечивает защиту передаваемой информации от раскрытия и проверку подлинности веб-сайта в процессе доступа. Серверы поддерживают замену сертификатов SSL. В целях обеспечения безопасности рекомендуется заменить исходный сертификат и ключи на собственный сертификат и пару ключей (открытый и приватный), после чего незамедлительно обновить сертификат.

  • Сертификат SSL может быть как отдельным сертификатом, так и цепочкой сертификатов, состоящей из 10 уровней.
  • Файл сертификата может быть в формате .crt, .cer, .pem, .pfx или .p12. Файл сертификата в формате .crt, .cer или .pem не может превышать 1 МБ. Файл сертификата в формате .pfx или .p12 не может превышать 100 КБ.
  • MD5 и SHA1 — слабые алгоритмы подписи, которые создают риски в области безопасности. BMC не поддерживает импорт сертификатов MD5 и SHA1.

Основные настройки

HTTP

Поддержка интернет-браузинга и преобразования страниц HTTP. Для обеспечения связи между браузером и BMC служба HTTP включена по умолчанию. После установления соединения используется безопасный протокол HTTPS.

Значение по умолчанию — 80.

ПРИМЕЧАНИЕ:

Если служба HTTP отключена, то после ввода в адресной строке браузера http://IP-адрес сетевого порта управления BMC не произойдет автоматическое переключение на службу HTTPS.

HTTPS

Поддержка интернет-браузинга и преобразования страниц посредством протокола HTTPS или Redfish.

Значение по умолчанию — 443.

ПРИМЕЧАНИЕ:

Если служба HTTP отключена, пользователь не сможет войти в WebUI BMC через браузер.

Порт

Номер порта, используемого для службы.

Диапазон значений: от 1 до 65535

Срок тайм-аута (мин)

Максимальное время ожидания (в минутах), по истечении которого произойдет выход пользователя из WebUI BMC.

Диапазон значений: от 5 до 480

Значение по умолчанию: 5

Режим сеанса
Режим входа пользователя в BMC WebUI с помощью аккаунта.
  • Общий: каждый аккаунт может использоваться для входа в WebUI BMC с четырех клиентов одновременно.
  • Приватный: каждый аккаунт может использоваться для входа в WebUI BMC только с одного клиента.

Значение по умолчанию — Приватный.

Сертификат SSL

Издатель

Информация об издателе сертификата SSL, включая:

  • CN: имя пользователя.
  • OU: подразделение пользователя.
  • O: компания или организация пользователя.
  • L: город пользователя.
  • S: район или регион пользователя.
  • C: страна пользователя.

Получатель

Информация о пользователе (текущем BMC) сертификата SSL. Получатель и Издатель имеют одинаковые обозначения полей.

ПРИМЕЧАНИЕ:

В качестве CN необходимо указывать полное доменное имя сервера (FQDN) в формате Имя хоста.Имя домена.

Срок действия

Срок действия сертификата SSL.

Серийный номер

Серийный номер сертификата SSL, который используется для идентификации и переноса сертификата.

Настройка информации и импорт сертификата SSL

Данная операция выполняется в случае необходимости подачи заявки на получение сертификата SSL.

  1. В разделе Сертификат SSL нажмите Настроить.
  2. Выберите Генерировать CSR, введите данные запроса на подпись сертификата (CSR, Certificate Signing Request) и нажмите Создать.
  3. Отправьте экспортированный файл CSR в центр сертификации SSL и подайте заявку на получение сертификата SSL.

    Получив официальный сертификат SSL, сохраните его на клиенте.

  4. Выберите Импорт сертификата сервера в диалоговом окне Настройка сертификата.
  5. Выберите официальный сертификат SSL на локальном клиенте.

    • Файл сертификата может быть в формате .crt, .cer, .pem, .pfx или .p12. Файл сертификата в формате .crt, .cer или .pem не может превышать 1 МБ. Файл сертификата в формате .pfx или .p12 не может превышать 100 КБ.
    • MD5 и SHA1 — слабые алгоритмы подписи, которые создают риски в области безопасности. BMC не поддерживает импорт сертификатов MD5 и SHA1.

  6. Нажмите Да.
  7. Введите пароль в поле Пароль сертификата.
  8. Нажмите Да.

    Сертификат вступает в силу сразу после его импорта.

    Файл CSR связан с сертификатом сервера, выданным организацией CA. Не генерируйте новый файл CSR перед импортом сертификата сервера. В противном случае новый файл CSR заменит исходный файл CSR, который нельзя будет восстановить. Новый файл CSR используется для подачи заявки на получение нового сертификата сервера от организации CA.

  9. Повторите вход в WebUI BMC.

Импорт сертификата SSL

  • Данная операция выполняется только при наличии сертификата SSL на клиенте.
  • В целях обеспечения безопасности рекомендуется использовать надежный алгоритм шифрования, например RSA2048, для настраиваемого сертификата SSL.
  1. На странице в разделе Сертификат SSL нажмите Настроить.

    Откроется страница, на которой можно настроить данные о сертификате SSL.

  2. Выберите Импорт сертификата сервера.
  3. Выберите файл сертификата SSL для импорта.

    • Файл сертификата может быть в формате .crt, .cer, .pem, .pfx или .p12. Файл сертификата в формате .crt, .cer или .pem не может превышать 1 МБ. Файл сертификата в формате .pfx или .p12 не может превышать 100 КБ.
    • MD5 и SHA1 — слабые алгоритмы подписи, которые создают риски в области безопасности. BMC не поддерживает импорт сертификатов MD5 и SHA1.

  4. Нажмите Да.
  5. Введите пароль в поле Пароль сертификата.
  6. Нажмите Да.

    Сертификат вступает в силу сразу после его импорта.

    Если размер выгружаемого файла превышает 100 МБ, на экране появится сообщение об ошибке запроса на отображение страницы. Для решения проблемы обновите страницу.

  7. Повторите вход в WebUI BMC.
