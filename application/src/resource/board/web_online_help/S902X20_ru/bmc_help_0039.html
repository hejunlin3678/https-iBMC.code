Часовой пояс

Часовой пояс BMC.

Настройки параметров Регион, Часовой пояс и DSTосуществляются в зависимости от фактических требований.

Значения по умолчанию: Другие, UTC, Включен

ПРИМЕЧАНИЕ:
  • При выборе Автоматически получать данные NTP через DHCPv4 значение часового пояса будет получено автоматически.
  • Если DST включен, в часовых поясах, в которых применяется переход на летнее время (DST), BMC автоматически скорректирует время на один час в момент перехода на летнее время и после завершения периода летнего времени.
  • При выполнении синхронизации времени в ОС запустите команду hwclock --utc -w. Благодаря этой команде согласованность между временем ОС и временем BMC обеспечена.

NTP

NTP

NTP позволяет серверу BMC синхронизировать время с сервером NTP.

По умолчанию данная функция отключена.

Автоматически получать адрес IP
  • Функция автоматического назначения адреса IPv4 серверу NTP.
  • Функция автоматического назначения адреса IPv6 серверу NTP.
  • Настройка IP-адресов серверов NTP вручную.

    Значение по умолчанию — Настроить IP-адрес вручную.

УВЕДОМЛЕНИЕ:
  • При автоматическом получении IP-адреса сетевого порта управления BMC информация о NTP может быть получена автоматически или вручную. Рекомендуется выбрать автоматическое получение.
  • При выборе ручной настройки IP-адреса сетевого порта управления BMC информация о NTP также вводится вручную.
  • Сервер 1 до Сервер 3
  • От Основной сервер 1 до Основной сервер 3 или От Альтернативный сервер 1 до Альтернативный сервер 3

Адреса серверов NTP.

Значение: адрес IPv4 или IPv6 или имя домена
ПРИМЕЧАНИЕ:
Имя домена должно соответствовать следующим требованиям:
  • Cтрока длиной до 67 символов.
  • Имя домена может состоять только из цифр, букв, дефисов (-) и точек (.).
  • Имя домена не может начинаться с дефиса (-) или заканчиваться им.
  • Имя домена может состоять из не более 63 символов между любыми двумя точками.
Предоставляется одно из следующих решений.
  • Сервер 1 до Сервер 3: Доступны три сервера NTP, которые работают одновременно.
  • От Основной сервер 1 до Основной сервер 3 или От Альтернативный сервер 1 до Альтернативный сервер 3: Доступны три группы серверов NTP. В каждой группе левый сервер является основным, а правый — альтернативным. Выбор серверов производится согласно следующим правилам:
    • Выбирается по одному серверу из каждой группы.
    • Если оба адреса в группе недействительны, группа серверов будет отброшена.
    • Если в группе есть только один действительный адрес, выбирается сервер с действительным адресом.
    • Если оба адреса в группе действительны, выбирается сервер с адресом IPv6. Если оба сервера имеют адреса IPv4 или IPv6, выбирается основной сервер (левый).
ПРИМЕЧАНИЕ:

Переключение между активным NTP-сервером и резервным NTP-сервером зависит от интервала синхронизации между BMC и NTP-сервером (Минимальный интервал опроса ≤ интервала синхронизации ≤ Максимальный интервал опроса). При невозможности синхронизации BMC с активным сервером несколько раз резервный сервер становится NTP-сервером.

Минимальный интервал опроса

Минимальный интервал BMC, в течение которого система выполняет синхронизацию времени с сервером NTP, т.е. минимальный интервал опроса пакетов NTP.

Например, если данный параметр установлен в значение 6, интервал опроса равен 2 в степени 6, т.е. 1 минута и 4 секунды.

Значения: от 3 до 17

Значение по умолчанию — 6 (1 мин. 4 сек.).

Максимальный интервал опроса

Максимальный интервал BMC, в течение которого система выполняет синхронизацию времени с сервером NTP, т.е. максимальный интервал опроса пакетов NTP.

Например, если данный параметр установлен в значение 6, интервал опроса равен 2, т.е. 1 минута и 4 секунды.

Значения: от 3 до 17

Значение по умолчанию — 10 (17 мин. 4 сек.).

Аутентификация сервера

Включение или отключение BMC аутентификации сервера для связи с сервером NTP.

По умолчанию данная функция отключена.

Групповой ключ NTP

Секретный ключ, загружаемый в BMC, для подтверждения подлинности данных после включения функции Аутентификация сервера.

ПРИМЕЧАНИЕ:
  • Для генерирования секретных ключей можно установить генератор ключей (например, ntp-keygen). Формат файла секретного ключа: .keys. Размер не может превышать 2 МБ.
  • Только файлы ключей, сгенерированные алгоритмом SHA256 или SHA512 могут быть выгружены.
  • В целях обеспечения безопасности необходимо периодически обновлять ключ.

Настройка часового пояса

  1. Выберите значение из выпадающих списков для параметров Регион и Часовой пояс.

    Более подробная информация о параметрах приводится в Табл. 1.

  2. Нажмите Сохранить.

    При успешно выполненной настройке на экране появится сообщение «Операция выполнена успешно».

    • Для проверки соответствия системного времени и времени BMC выполните команду hwclock --utc -w.
    • После изменения часового пояса необходимо перезапустить BMC для того, чтобы время системы вступило в силу для загрузки ЦПУ, использования памяти, использования полосы пропускания сетевого порта, использования диска, журнала энергопотребления, температуры поступающего воздуха, отображаемых в веб-интерфейсе BMC.

Настройка информации о NTP

  1. В области Настройки NTP задайте параметры на основании сервисных требований.

    Более подробная информация о параметрах приводится в Табл. 1.

  2. Нажмите Сохранить.

    При успешно выполненной настройке на экране появится сообщение «Операция выполнена успешно».
