LDAP

Описание функций

На странице LDAP можно посмотреть информацию о пользователях LDAP и настроить соответствующие параметры.

BMC поддерживает доступ пользователей LDAP. Пользователям LDAP предоставляется возможность войти в WebUI BMC или воспользоваться CLI BMC посредством инструмента SSH. Вход в BMCпод аккаунтом пользователя домена повышает уровень системной безопасности.

На сервере LDAP параметры DisplayName и CN должны иметь одинаковые значения.

BMC поддерживает максимум шесть серверов доменов.

При входе в WebUI BMC поддерживается настройка сервера домена вручную или автоматически. При входе в CLI BMC поиск сервера домена осуществляется автоматически.

BMC поддерживает активный AD Windows и OpenLDAP Linux.

Описание параметров

Табл. 1 LDAP

Параметр

Описание

LDAP

Функция LDAP обеспечивает пользователям домена доступ к BMC.

По умолчанию данная функция отключена.

ПРИМЕЧАНИЕ:

К пользователям LDAP, пытающимся войти в BMC, применяются политики безопасности (проверка сложности пароля, срок действия пароля, минимальный срок действия пароля, запрещенные предыдущие пароли и ограничение времени ожидания, а также политика блокировки аккаунтов пользователей), настроенные на сервере аутентификации.

Контроллер 1

BMC поддерживает максимум шесть контроллеров доменов (серверов). При входе в WebUI BMC посредством LDAP пользователь может выбрать контроллер домена или Автоподбор.

Контроллеры 1–6 имеют одинаковые параметры.

ПРИМЕЧАНИЕ:

Параметры, отмеченные звездочкой (*), являются обязательными для настройки.

Основные настройки

Адрес сервера LDAP

IP-адрес сервера LDAP.

Возможные значения: адреса IPv4 или IPv6, а также имена доменов.

После включения проверки сертификата установите данному параметру значение FQDN сервера LDAP (Имя хоста.Имя домена) и сконфигурируйте адрес DNS на странице Сеть.

ПРИМЕЧАНИЕ:
Имя домена должно соответствовать следующим требованиям:
  • Cтрока длиной до 255 символов.
  • Имя домена может состоять только из цифр, букв, дефисов (-) и точек (.).
  • Имя домена не может начинаться с дефиса (-) или точки (.), либо заканчиваться дефисом (-).
  • Имя домена может состоять из не более 63 символов между любыми двумя точками.

Порт LDAPS

Номер порта для службы LDAP.

Значение: целое число в диапазоне от 1 до 65535

Значение по умолчанию: 636

ПРИМЕЧАНИЕ:

По умолчанию включена функция шифрования передаваемых данных. На сервере LDAP необходимо выполнить соответствующие настройки.

Домен

Домен пользователей, к которому относится пользователь LDAP, определенный в контроллере домена.

Имя домена должно соответствовать следующим требованиям:
  • Cтрока длиной до 255 символов.
  • Имя домена может состоять только из цифр, букв, дефисов (-) и точек (.).
  • Имя домена не может начинаться с дефиса (-) или точки (.), либо заканчиваться дефисом (-).
  • Имя домена может состоять из не более 63 символов между любыми двумя точками.

DN

Уникальное имя (DistinГрафический интерфейс пользователя (GUI)shed name, DN) пользователя прокси LDAP.

Например, CN=username,OU=company,DC=domain,DC=com, которые должны соответствовать DN, заданному на сервере LDAP.

Диапазон значений: строка длиной 255 байт (от 64 до 255 символов). Длина зависит от количества байтов каждого символа.

Пароль

Пароль для аутентификации пользователя прокси LDAP.

Значение: строка длиной от 1 до 20 символов, состоящая из букв, цифр и специальных символов.

Папка пользователя

Каталог на сервере LDAP пользователя LDAP, который имеет право осуществлять вход в BMC.

Формат: "CN=xxx" или "OU=xxx"

При наличии нескольких уровней узел верхнего уровня указывается после узла нижнего уровня через запятую.

Например, если пользователь infotest находится в \testusers\part1 на сервере LDAP, введите OU=part1,OU=testusers.

ПРИМЕЧАНИЕ:

Более подробная информация об отличиях между коротким названием (CN) и подразделением (OU) приводится в описании протокола LDAP.

Например, в активном каталоге Windows:

  • CN, если Тип — Контейнер.
  • OU, если Тип — Организационная единица.

Диапазон значений: строка длиной 255 байт (от 64 до 255 символов). Длина зависит от количества байтов каждого символа.

Подтверждение сертификата LDAP

Включение и отключение проверки сертификата сервера LDAP.

В целях обеспечения безопасности рекомендуется включить проверку сертификата.

После включения проверки сертификата необходимо импортировать сертификат CA LDAP, настроить AD, DNS и изготовителя сертификата CA на сервере LDAP, а затем импортировать сертификат CA на сервер LDAP и в BMC.

Значение по умолчанию: Отключить

Уровень подтверждения сертификата

Уровень подтверждения сертификата LDAP.

  • Demand: Запрет доступа к BMC, если сертификат клиента является недействительным или недоступен. В целях обеспечения безопасности рекомендуется использовать опцию по умолчанию (Demand).
  • Allow: Предоставление доступа к BMC, даже если сертификат клиента является недействительным или недоступен.

Значение по умолчанию: Demand

Сертификат LDAP

Сертификат LDAP может иметь расширение .cer, .pem, .cert или .crt.

ПРИМЕЧАНИЕ:
  • Системе требуется больше времени для загрузки файлов сертификатов, размер которых превышает 100 МБ. Обновите страницу для получения последнего статуса.
  • Поддерживается цепочка сертификатов, количество уровней которой не превышает 10.

Роль

Роль, назначенная группе LDAP.

  • Администратор: Пользователи с полномочиями Администратор могут выполнять любые операции.
  • Оператор: Пользователям с полномочиями Оператор доступны базовые операции управления, дистанционное управление, дистанционная загрузка, управление энергопотреблением, запрос информации и настройка собственных данных.
  • Стандартный пользователь: Пользователи с полномочиями Стандартный пользователь могут запрашивать информацию и настраивать собственные данные.
  • Настроить: Пользователи с полномочиями Ограниченная роль 1–4 могут выполнять только определенные операции.
ПРИМЕЧАНИЕ:

Стандартный пользователь — это разрешение пользователя по умолчанию для новой группы пользователей LDAP.

Правила входа

Правила входа в систему, которые применяются к группе LDAP.

Интерфейсы входа

Интерфейсы, с помощью которых пользователи группы LDAP осуществляют вход в BMC.

Значения:
  • SSH: для входа в CLI BMC используется инструмент SSH (например, PuTTY).
  • Web: для входа в WebUI BMC используется веб-браузер.
  • Redfish: для входа в BMC используется инструмент Redfish.
ПРИМЕЧАНИЕ:

Новая группа пользователей LDAP поддерживает все интерфейсы входа в систему.

Включение LDAP и настройка контроллеров LDAP

  1. Установите переключатель LDAP в положение .
  2. Настройте параметры контроллеров LDAP. Более подробная информация о параметрах приводится в Табл. 1.

В целях обеспечения безопасности необходимо периодически обновлять сертификат.

Если на экране появилось сообщение «Файл успешно выгружен», то в области данных о сертификате будет отображаться подробная информация о нем. Более подробная информация о параметрах приводится в Табл. 2.

Табл. 2 Данные о сертификате

Параметр

Описание

Издатель

Организация, выпустившая сертификат LDAP. Издатель и Получатель имеют одинаковые параметры.

Получатель

Информация о пользователе (текущем сервере) сертификата LDAP, включая:

  • CN: имя пользователя.
  • OU: подразделение пользователя.
  • O: компания, которой принадлежит пользователь.
  • L: город пользователя.
  • S: район или регион пользователя.
  • C: страна пользователя.

Срок действия

Срок действия сертификата LDAP.

Серийный номер

Серийный номер сертификата LDAP, который используется для идентификации и переноса сертификата.

Списки отозванных сертификатов
Статус отзыва сертификата LDAP.
  • Настроено: файл CRL был выгружен; при установке соединения TLS будет выполнена проверка сертификата на предмет отзыва.
  • Не настроено: файл CRL не выгружен.

Срок действия CRL

Срок действия списка отозванных сертификатов LDAP.

ПРИМЕЧАНИЕ:

Если срок действия CRL истекает, функция аутентификации LDAP будет недоступна.

Выгрузка CRL

Файл отзыва сертификатов имеет кодировку Base64 и формат *.crl. Размер файла не может превышать 100 КБ.

  1. Получите файл CRL в центре сертификации (CA).
  2. В области Данные о сертификате сервера нажмите Выгрузить рядом с CRL.
  3. Выберите файл CRL.
  4. Введите пароль текущего пользователя и нажмите Сохранить.

Удаление группы LDAP

  1. В области группы LDAP нажмите Удалить для удаления группы LDAP.

    Откроется диалоговое окно с просьбой ввести текущий пароль пользователя.

  2. Введите текущий пароль пользователя.
  3. Нажмите Да.

    Появится сообщение «Операция выполнена успешно».

Редактирование данных группы LDAP

  1. В области группы LDAP нажмите Изменить для изменения параметров группы LDAP.
  2. Введите текущий пароль пользователя и измените параметры группы LDAP. Более подробная информация о параметрах приводится в Табл. 3.
  3. Нажмите Сохранить.

    После изменения информации о группе LDAP или удаления группы LDAP пользователи, вошедшие в KVM, не будут автоматически выходить из системы. Для того, чтобы произвести выход данных пользователей из системы KVM, перейдите на страницу Пользователи в сети и выполните соответствующие операции.
