Безопасность

Описание функций

На странице Безопасность доступны следующие операции:

  • Просмотр и настройка правил повышения уровня безопасности для пользователей BMC.
  • Просмотр и настройка прав доступа для локальных пользователей BMC.

Описание параметров

Табл. 1 Безопасность

Параметр

Описание

Режим блокировки системы

Режим блокировки системы BMC, который можно включить или отключить.

Данная функция отключена по умолчанию.

Если эта функция включена, любые попытки изменить следующие конфигурации системы будут заблокированы:

  • Импорт конфигурации
  • Управление BMA
  • Настройка опции загрузки в BIOS
  • Обновление встроенного ПО: перезагрузка, переключение образа и обновление
  • Общие настройки: управление хранилищем, конфигурация сети, информация о продукте, управление SmartProvisioning, порог мониторинга, язык, часовой пояс и NTP
  • Регулирование мощности: настройка мощности, повторный сбор и очистка статистики мощности и интеллектуальная регулировка скорости вентилятора
  • Базовые настройки пользователя: управление лицензиями и настройки SNMP
  • Базовые настройки веб-сервиса и функции VNC
  • Базовые настройки виртуальной консоли и виртуального носителя
  • Базовые настройки безопасности (кроме режима блокировки системы)
  • Базовые настройки отладки и диагностики: системные журналы, запись скриншотов и видео, а также отчет о тревогах
ПРИМЕЧАНИЕ:

Эта функция доступна только при применении лицензии продвинутого уровня. Только администратор имеет право настраивать эту функцию.

Управление пользователями ОС

Функция управления пользователями служебной системы.

При отключении данной функции команды IPMI, отправленные сервисной системой для управления пользователями, такие как добавление или удаление пользователей, установка разрешений и паролей, становятся недействительными.

Данная функция включена по умолчанию.

  • Когда функция включена, служебная система осуществляет управление пользователями BMC.
  • Если функция отключена, служебная система не может управлять пользователями BMC.

В целях обеспечения безопасности рекомендуется отключить данную функцию.

Проверка пароля

Функция проверки пароля определяет, соответствует ли пароль установленным требованиям и является ли он достаточно сложным. Данная функция включена по умолчанию.

Функция проверки пароля включена по умолчанию. В данную проверку входят:
  • Проверка сложности паролей локальных пользователей, имен сообществ Trap, имен сообществ SNMPv1/v2c, паролей шифрования SNMPv3 и паролей VNC.
  • Проверка минимальной длины паролей локальных пользователей и паролей шифрования SNMPv3.
ПРИМЕЧАНИЕ:
  • В целях обеспечения безопасности рекомендуется включить проверку сложности пароля.
  • Если включена функция проверки сложности пароля, пароль не может совпадать со слабыми паролями, содержащимися в словаре паролей. (Для экспорта слабых паролей из словаря паролей выполните команду ipmcset -t user -d weakpwddic -v export.)

Аутентификация пароля SSH

Функция аутентификации пароля SSH предоставляет пользователям возможность входить в BMC по протоколу SSH с помощью пароля или открытого ключа.

  • Отключить: вход пользователей выполняется по протоколу SSH с помощью только открытых ключей.
  • Включить: вход пользователей выполняется по протоколу SSH с помощью паролей или открытых ключей.

Данная функция включена по умолчанию.

Предотвращение повторной привязки DNS

Позволяет пользователям включать или отключать функцию предотвращения повторной привязки DNS.

После включения данной функции вы сможете получить доступ к BMC, используя только имя хоста, FQDN или IP-адрес, настроенный на BMC.

Данная функция отключена по умолчанию.

Версия TLS

Настройка версии протокола TLS, который используется для обеспечения безопасности и целостности данных во время обмена данными между двумя приложениями.

Для установления безопасного соединения между веб-браузером и веб-сервером рекомендуется включить TLS.

  • TLS 1.2 или более поздние версии: поддерживается TLS 1.2 или TLS 1.3.
  • Только TLS 1.3: поддерживается только TLS 1.3.

Значение по умолчанию — TLS 1.2 или более поздние версии.

ПРИМЕЧАНИЕ:
  • При включении только TLS 1.3 для получения доступа к встроенной удаленной консоли Java необходимо использовать AdoptOpenJDK 11.
  • Эта опция действительна только для служб Веб и Redfish BMC.

Срок действия пароля (дн.)

Период времени в днях, в течение которого пользователь может использовать пароль.

Диапазон значений: от 0 до 365

0 означает, что пароль имеет неограниченный срок действия.

Значение по умолчанию: 0

ПРИМЕЧАНИЕ:

В целях обеспечения безопасности рекомендуется настроить подходящий срок действия, по истечении которого пользователь будет периодически менять пароль.

Минимальная длина пароля

Минимальная длина пароля локального пользователя или пароля шифрования SNMPv3.

Этот параметр доступен только при включении функции проверки пароля.

Диапазон значений: от 8 до 20

Значение по умолчанию: 8

Минимальный срок действия пароля (дн.)

Минимальный период в днях, в течение которого должен использоваться пароль. В течение данного периода сменить пароль невозможно.

Диапазон значений: от 0 до 365

0 означает, что минимальный срок действия пароля не ограничен.

Значение по умолчанию: 0

ПРИМЕЧАНИЕ:

Значение минимального срока действия пароля должно быть, по крайней мере, на 10 дней меньше срока действия пароля.

  • Если параметр Срок действия пароля (дн.) имеет значение 10 или меньше, то параметр Минимальный срок действия пароля (дн.) может иметь только значение 0.
  • Если параметр Минимальный срок действия пароля (дн.) имеет значение 354 или больше, то параметр Срок действия пароля (дн.) может иметь только значение 0.

Максимальный период неактивности (дн.)

Максимальный период неактивности пользователя в днях, по истечении которого учетная запись пользователя будет отключена.

Значения:

  • 0
  • От 30 до 365

0 означает, что неиспользуемые учетные записи отключаться не будут.

Значение по умолчанию: 0

Пользователь для экстренного входа

Имя пользователя для входа в BMC в экстренных случаях.

Данный пользователь не имеет ограничений на вход в систему или использование интерфейсов входа, и у пароля такого пользователя нет срока действия.

Значение по умолчанию — NULL.

ПРИМЕЧАНИЕ:
  • Пользователем для экстренного входа в систему может быть только администратор.
  • Настройки учетной записи пользователя для экстренного входа доступны только администратору.

Запрет использования предыдущих паролей

Количество предыдущих паролей, которые не могут использоваться в качестве нового пароля.

Диапазон значений: от 0 до 5

0 означает, что все предыдущие пароли можно использовать повторно.

Значение по умолчанию: 5

Политика блокировки пользователей

Максимально допустимое количество попыток входа в систему при неправильно введенном пароле и срок блокировки пользователя.

  • Максимально допустимое количество попыток входа в систему при неправильно введенном пароле – это целое число в диапазоне от 1 до 6 или значение Не ограничено (блокировка пользователя отключена). По умолчанию установлено значение 5.
  • Срок блокировки пользователя (в минутах) – это целое число в диапазоне от 1 до 30. По умолчанию установлено значение 5.

После блокировки учетной записи пользователь может повторно войти в систему только после истечения срока блокировки.

ПРИМЕЧАНИЕ:
  • В целях обеспечения безопасности функцию блокировки пользователей рекомендуется включить.
  • Для разблокировки пользователя в экстренной ситуации выполните команду unlock в CLI. Более подробная информация приводится в Руководстве пользователя BMC сервера.

Уведомление об истечении срока действия сертификата (дн.)

За несколько дней до истечения срока действия сертификатов BMC пользователи получают уведомление. Например, если установлено значение 7, сигнал тревоги будет выдан, когда срок действия сертификата будет меньше или равен 7 дням.

Диапазон значений: от 7 до 180

Значение по умолчанию: 90
Табл. 2 Правила входа

Параметр

Описание

Имя

Имя правила.

Время

Период времени, в течение которого пользователям разрешен вход в систему.

Значение может быть в одном из следующих форматов:

  • YYYY-MM-DD:

    Например: с 2013-08-30 до 2013-12-30

  • HH:MM:

    Например: с 08:30 до 20:30

  • YYYY-MM-DD HH:MM:

    Например: с 2013-08-30 08:30 до 2013-12-30 20:30

ПРИМЕЧАНИЕ:
  • Должна быть указана дата от 1970 до 2050 г.г.
  • Время начала и время окончания действия правил входа должны быть указаны в одном формате.

IP

IP-адрес или диапазон IP-адресов, для которых разрешен вход в систему. Значение может быть в одном из следующих форматов:

  • IPv4 (xxx.xxx.xxx.xxx): IP-адрес.
  • IPv4/маска подсети (xxx.xxx.xxx.xxx/маска): сегмент IP-адресов.

MAC

MAC-адрес или диапазон MAC-адресов, для которых разрешен вход в систему. Значение может быть в одном из следующих форматов:

  • xx:xx:xx:xx:xx:xx: MAC-адрес.
  • xx:xx:xx: сегмент MAC-адресов.

Статус

Статус правила входа.

По умолчанию данная функция отключена.

Операция

Нажмите Изменить. Поля Время, IP, MAC и Статус в разделе Правила входа доступны для редактирования.

Настройка правил входа в систему

BMC поддерживает максимум три правила входа. Пользователи, которые соответствуют любому из трех правил, могут осуществлять вход в BMC.

Правила входа действуют для локальных пользователей, групп LDAP, служб SNMPv3 или интерфейсов CLP (ssh), KVM_VMM, RMCP и интерфейсов Redfish только в том случае, если они отвечают следующим двум условиям:

  • Правила сконфигурированы и активированы на вкладке Правила входа.
  • Правила входа были выбраны в области конфигурирования.
  • Если правило входа является пустым, а статус правила включен и сохранен, то нет никаких ограничений входа.
  • Если текстовое поле правила входа является пустым, то отсутствует ограничение правила входа.
  1. На вкладке Правила входа нажмите Изменить в столбце Операция.
  2. Установите для правила статус .
  3. Задайте правила входа. Более подробная информация о параметрах приводится в Табл. 2.
  4. Нажмите Сохранить.

Настройка пользовательских ролей

Разрешения на выполнение операций для ролей по умолчанию (Администратор, Оператор и Обычный пользователь) не могут быть изменены, но администратор может установить разрешения на выполнение операций для пользовательских ролей.

  1. В списке функций выберите модули для пользовательских ролей.

    В таблице Табл. 3 приведено описание разных разрешений.

  2. Нажмите Сохранить.

    Откроется диалоговое окно, введите пароль текущего пользователя и нажмите Да.

Настройка сообщения системы безопасности при входе

  1. На вкладке Сообщение системы безопасности при входе установите параметр Сообщение системы безопасности в положение .
  2. Введите текст сообщения в соответствующее поле.
