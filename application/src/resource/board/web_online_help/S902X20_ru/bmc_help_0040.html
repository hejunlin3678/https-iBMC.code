Обновление встроенного ПО

Описание функций

На странице Обновление встроенного ПО доступны следующие операции:

  • Просмотр данных о версии встроенного ПО сервера.
  • Перезагрузка BMC.
  • Переключение сервисов на доступный образ BMC.
  • Обновление встроенного ПО сервера по внеполосному каналу.
  • Обновление встроенного ПО сервера по внутриполосному каналу.
BMC имеет следующие образы раздела:
  • Образ основного раздела: образ раздела BMC, который работает в настоящее время.
  • Образ резервного раздела: резервное копирование образа основного раздела. При неисправности образа основного раздела образ резервного раздела переходит в рабочее состояние, а исходные образы основного раздела переходит в состояние резервного копирования и синхронизируют версию образа с текущим образом основного раздела. При обновлении BMC одновременно обновляются образы основного и резервного раздела.
  • Образ доступного раздела: зарезервированный образ раздела BMC. Можно нажать кнопку Переключить на доступный образ, чтобы образ доступного раздела переходил в образ основного раздела. При этом исходный образ основного раздела переходит в состояние резервного копирования и синхронизирует данные с текущим образом основного раздела, а исходный образ резервного раздела автоматически переходит в доступное состояние.
  • В процессе обновления не отключайте питание сервера и не перезагружайте BMC.
  • После завершения обновления встроенного ПО выполните перезагрузку BMC, чтобы новая версия вступила в силу. При этом выполнять перезагрузку сервера не нужно. Таким образом, процедура обновления встроенного ПО не оказывает влияния на службы, работающие на сервере.
  • После обновления встроенного ПО блока питания выполнять перезагрузку сервера не требуется.
  • После обновления встроенного ПО BMC или контроллера SD-карты, BMC автоматически перезапускается, чтобы обновление вступило в силу.
  • Перед обновлением BIOS или CPLD остановите работу всех служб на сервере. Таким образом, при перезагрузке сервера не произойдет прерывание обслуживания.
  • Если обновление BIOS или CPLD производится во время включения системы, обновление BIOS вступит в силу после выключения и повторного включения или перезагрузки системы, а обновление CPLD вступит в силу после выключения системы.
  • Если обновление BIOS или CPLD производится во время выключения системы, обновление вступит в силу после включения системы.
  • Перед обновлением встроенного ПО по внутриполосному каналу, остановите службы, работающие на сервере, чтобы предотвратить прерывание служб во время автоматического перезапуска сервера.

Просмотр версий встроенного ПО

  1. В строке меню выберите Настройки BMC.
  2. В дереве навигации выберите Обновление встроенного ПО.

    В правой части экрана отображается страница Информация о версии, на которой указаны имя и версия встроенного ПО сервера.

Обновление встроенного ПО (Внеполосный канал)

BMC используется в качестве примера в следующих операциях, которые также применимы к обновлению другого встроеного ПО.

  1. Выберите Обновление встроенного ПО > Внеполосный канал.
  2. Нажмите Выбор файла и выберите файл обновлений.

    • Формат файла обновления — .hpm. Размер не может превышать 90 МБ.
    • Имя файла обновления допускает цифры, английские буквы, пробел, скобки (), точки (.), нижние подчеркивания (_) и дефисы (-).

  3. Нажмите Обновить.

    На экране появится сообщение:

    Выполнить эту операцию?

  4. Нажмите Да.

    Начнется процесс обновления.

    После успешного обновления произойдет автоматическая перезагрузка BMC и на экране отобразится страница входа в систему.

    Перезагрузка BMC занимает несколько минут.

Обновление встроенного ПО (Внутриполосный канал)

Следующие операции применимы к встроенному ПО (например, встроенное ПО сетевой платы, платы RAID-контроллера и жесткого диска), которое поддерживает обновление с помощью Smart Provisioning. Загрузите пакет обновления встроенного ПО и файл подписи перед обновлением.

  1. Выберите Обновление встроенного ПО > Внутриполосный канал.
  2. Нажмите Выбрать файл рядом с полями Файл обновления и Файл подписи, чтобы выбрать и выгрузить файл обновления и файл подписи.

    • Формат файла обновления — .zip, размер файла не может превышать 499 МБ.
    • Формат файла подписи — .asc, размер файла не может превышать 90 МБ.

  3. Нажмите Выгрузить.

    На экране появится сообщение:

    Перейти на страницу управления Smart Provisioning (SP) и настроить ОС для загрузки с SP в следующий раз?
    • Если вам нужно выгрузить другие пакеты обновления встроенного ПО, нажмите кнопку Отмена и повторите 2–3, чтобы выгрузить пакет обновления встроенного ПО и файл подписи.
    • Если вам нужно удалить выгруженные пакеты обновления встроенного ПО, нажмите Отмена, а затем нажмите Очистить.

  4. Нажмите Да.

    На экране откроется страница Управление Smart Provisioning.

  5. Включите функцию Загрузка ОС с SP и нажмите кнопку Подтвердить в появившемся диалоговом окне.
  6. Перезагрузите сервер и обновите встроенное ПО на Smart Provisioning.

Переключение образов BMC

Переключение образов для встроенного ПО BMC рекомендуется выполнять только при необходимости. Во время обновления переключение образов не является обязательной процедурой.

  1. На странице Информация о версии нажмите Переключить на доступный образ.

    На экране появится сообщение:

    Выполнить переключение?

  2. Нажмите Да.

    Если успешного переключения произойдет автоматическая перезагрузка BMC и на экране отобразится страница входа в систему.

    Перезагрузка BMC занимает несколько минут.

Перезагрузка BMC

Перезагрузка BMC выполняется при необходимости. Данная операция не является обязательной в процессе обновления.

  1. На странице Информация о версии нажмите Перезагрузка BMC.

    На экране появится сообщение:

    Выполнить переключение?

  2. Нажмите Да.

    После перезагрузки BMC на экране отобразится страница входа.

    Перезагрузка BMC занимает несколько минут.
