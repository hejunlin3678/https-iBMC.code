Обновление сертификата SSL

Описание параметров

Табл. 1 Обновление сертификатa SSL

Параметр

Описание

Обновление сертификата SSL

ПРИМЕЧАНИЕ:
  • Сертификат SSL обновлен только в том случае, если отключена системная блокировка, пользователь имеет разрешение на настройку безопасности и настроен сервер CA.
  • Для обновления SSL-сертификата необходимо импортировать корневой сертификат сервера и сертификат клиента. В противном случае аутентификация не будет выполнена.

Автоматическое обновление

Включение или отключение функции автоматического обновления.

После включения Автоматическое обновление, сертификат SSL автоматически обновляется с интервалом в один час.

Данная функция отключена по умолчанию.

Обновление сертификата

Чтобы обновить сертификат SSL, нажмите кнопку Обновление сертификата.

Конфигурация сервера CA

Адрес сервера

IP-адрес сервера CA.

IPv4-адрес, IPv6-адрес или доменное имя с максимальной длиной 67 символов.

Порт

Номер порта сервера CA.

Диапазон значений: 1-65535

Путь

Путь к серверу CA.

Строка от 1 до 128 символов. Начинается с косой черты (/) и состоит из цифр (0-9), букв (a-z, A-Z), подчеркивания (_), и косой черты (/).

Субъект сертификата

Страна (C)

Страна, которой принадлежит сертификат.

Двухбуквенный код страны. Допускаются все английские буквы.

Провинция (S)

Штат или провинция, которым принадлежит сертификат.

Строка от 0 до 128 символов. Допускаются цифры (0-9), буквы (a-z, A-Z), дефисы (-), знаки подчеркивания (_), периоды (.) и пробелы.

Город (L)

Город или местоположение, которому принадлежит сертификат.

Строка от 0 до 128 символов. Допускаются цифры (0-9), буквы (a-z, A-Z), дефисы (-), знаки подчеркивания (_), периоды (.) и пробелы.

Компания (O)

Компания или организация, которой принадлежит сертификат.

Строка от 0 до 64 символов. Допускаются цифры (0-9), буквы (a-z, A-Z), дефисы (-), знаки подчеркивания (_), периоды (.) и пробелы.

Отдел (OU)

Организационное подразделение или отдел, которому принадлежит сертификат.

Строка от 0 до 64 символов. Допускаются цифры (0-9), буквы (a-z, A-Z), дефисы (-), знаки подчеркивания (_), периоды (.) и пробелы.

Общее название (CN)[c(1]

Общее название сертификата.

Строка от 1 до 64 символов. Допускаются цифры (0-9), буквы (a-z, A-Z), дефисы (-), знаки подчеркивания (_), периоды (.) и пробелы.

Внутреннее название (IN)

Внутреннее название сертификата.

Строка от 0 до 64 символов. Допускаются цифры (0-9), буквы (a-z, A-Z), дефисы (-), знаки подчеркивания (_), периоды (.) и пробелы.

Электронная почта (E)

Адрес электронной почты сертификата.

Строка от 0 до 255 символов в соответствии с соглашением об адресах электронной почты.

Корневой сертификат сервера

Корневой сертификат сервера, полученный от официального центра сертификации. Сертификат, используемый для выполнения аутентификации TLS на пакетах сертификатов сервера, отправленных с сервера ЦС до установления соединения.

ПРИМЕЧАНИЕ:
  • Поддерживаются корневые сертификаты сервера в формате .crt, .cer, или .pem. Код сертификата в формате *.cer - Base64. Максимальный размер сертификата составляет 100 КБ. Дубликаты сертификатов не допускаются.
  • В целях безопасности периодически обновляйте сертификат.

Сертификат клиента

Сертификат клиента, полученный от официального центра сертификации.

ПРИМЕЧАНИЕ:
  • Поддерживаются сертификаты клиента в формате *.p12 или *.pfx. Максимальный размер сертификата составляет 100 КБ.
  • В целях безопасности периодически обновляйте сертификат.

Данные о сертификате

Отображение информации о выгруженном сертификате, в том числе:

  • Издатель
  • Получитель
  • Срок действия
  • Серийный номер
  • Список отозванных сертификатов с одним из следующих статусов:
    • Настроено: файл отозванных сертификатов был выгружен, и проверка отозванных сертификатов будет выполнена при установке соединения TLS.
    • Не настроено: файл отозванных сертификатов не выгружен.
ПРИМЕЧАНИЕ:
  • Файл CRL должен быть в формате *.crl. Имя файла не может быть пустым. Максимальный размер файла CRL составляет 100 КБ.
  • Если срок действия CRL истек, соответствующая функция аутентификации будет недоступна.
  • Сертификат клиента не поддерживает CRL.

Выгрузка сертификатa

  1. Нажмите Выгрузить рядом с параметром Корневой сертификат сервера или Сертификат клиента.

    Появится диалоговое окно для выбора локального файла или выгрузки сертификата клиента.

    • Поддерживаются корневые сертификаты сервера в формате .cer, .crt, или .pem. Код сертификата в формате *.cer - Base64. Максимальный размер сертификата составляет 100 КБ. Дубликаты сертификатов не допускаются.
    • Поддерживаются сертификаты клиента в формате *.p12 или *.pfx. Максимальный размер сертификата составляет 100 КБ.

  2. Выберите файл в правильном формате.
  3. Нажмите OK.

Конфигурирование CRL

Файл CRL должен быть в формате *.crl. Имя файла не может быть пустым. Максимальный размер файла CRL составляет 100 КБ.

  1. Получите файл CRL из центра сертификации.
  2. В области Данные о сертификате нажмите Выгрузить рядом с параметром Списки отозванных сертификатов.
  3. Выберите файл CRL.
  4. Введите пароль текущего пользователя и нажмите Сохранить.

Удаление CRL

Удаление CRL может привести к рискам безопасности при использовании сертификатов с истекшим сроком действия.

  1. Найдите целевой сертификат и нажмите Удалить CRL в столбце CRL.

    Откроется диалоговое окно подтверждения.

  2. Нажмите Да.
