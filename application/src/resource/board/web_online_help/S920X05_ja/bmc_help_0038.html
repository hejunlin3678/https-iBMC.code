ネットワーク構成

機能説明

ネットワーク画面で、BMC管理ネットワークポートのネットワーク構成を表示および設定できます。

  • ホスト名
  • ネットワークポート
  • ネットワークプロトコル
  • DNS情報
  • VLAN
  • LLDP

管理ネットワークポートのIPアドレスを変更すると、ネットワークが切断されます。必要な場合にのみ、管理ネットワークポートのIPアドレスを変更してください。

パラメータの説明

表 1 ネットワーク構成

カテゴリ

説明

BMCホスト名

BMCホスト名。

値：1～64文字。

値には、英字、数字、およびハイフン(-)を含めることができますが、ハイフンで開始または終了することはできません。

モード

サーバー管理ネットワークポート、つまりBMCネットワークポートのタイプ。

デフォルト値：固定

固定

このオプションを選択した場合、次のポートからBMCネットワークポートを指定する必要があります。

  • 専用ポート：専用BMCポート（つまり、サーバーの管理ネットワークポート）。
  • PCIeポート：PCIeカードのポート（つまり、NC-SIをサポートし、NC-SIケーブル経由で接続されているPCIeカード）。
  • OCPポート：OCPカードのポート。

デフォルト値は[専用ポート]です。

自動

このオプションを選択した場合、BMCはポートの状態に基づいてBMCポートを自動的に選択します。

自動選択の対象となるポートを選択する必要があります。複数のポートが使用可能な場合、BMCは次の優先度に応じてポートを選択します。

  • 専用ポート>PCIeポート（ポート1～2またはポート1～4）
  • 専用ポート>OCPポート（ポート1～2またはポート1～4）
NOTE:

BMC管理ネットワークポートに指定されている場合、ネットワークポートの右にが表示されます。

NCSIモード

モードが[自動]に設定されているときのNCSIネットワークポートの切り替えモードです。

デフォルト値は[手動切替]です。

自動フェイルオーバー

ポートの接続状態に基づいて、優先度の高い外部ネットワークポートに自動的に切り替わります。自動選択ルールに基づき、BMCは次の順で拡張ポートを選択します。

ポート1 > ポート2またはポート1 > ポート2 > ポート3 > ポート4。

手動切替

手動で優先順位の高い外部ネットワークポートに切り替えます。

BMCネットワークポート
  • モードが固定の場合、管理ネットワークポートを指定します。
  • モードが自動の場合、自動ネゴシエーション用のネットワークポートを選択します。

ネットワークプロトコル

有効にできるIPバージョン：

  • IPv4：IPv4のみをサポートします。このオプションを選択すると、IPv4アドレスのみを設定できます。
  • IPv6：IPv6のみをサポートします。このオプションを選択すると、IPv6アドレスのみを設定できます。
  • IPv4/IPv6：IPv4とIPv6をサポートします。このオプションを選択すると、IPv4アドレスとIPv6アドレスを設定できます。

デフォルト値：IPv4/IPv6

IPv4
  • 自動IPアドレス取得：このオプションをクリックすると、IPv4アドレスがBMCネットワークポートに自動的に割り当てられます。
  • IPアドレスの手動設定：このオプションをクリックすると、BMCポートのIPv4アドレスを手動で設定できます。

IPv4アドレス情報には、IPアドレス、サブネットマスク、ゲートウェイ、およびMACアドレスが含まれています。

デフォルト値は[IPアドレスの手動設定]です。

NOTE:
  • MACアドレス：ネットワークインターフェースカード（NIC）の物理アドレスを指定します。
  • デフォルトゲートウェイを使用しない場合、ゲートウェイを静的IPアドレスとフローティングIPアドレスのネットワークセグメント内の任意のIPアドレスに設定します。

IPv6
  • 自動IPアドレス取得：このオプションをクリックすると、IPv6アドレスがBMCネットワークポートに自動的に割り当てられます。
  • IPアドレスの手動設定：このオプションをクリックすると、BMCネットワークポートのIPv6アドレスを手動で設定できます。

IPv6アドレス情報には、IPアドレス、プレフィクス、ゲートウェイ、リンクローカルアドレス、およびIPアドレス2が含まれています。

デフォルト値は[IPアドレスの手動設定]です。

NOTE:
  • ローカルリンクはローカルリンク通信に使用されます。
  • IPアドレス2は、Stateless Address Autoconfiguration（SLAAC）を使用する場合、最大15個のIPv6アドレスをサポートします。
  • デフォルトゲートウェイを使用しない場合、ゲートウェイを静的IPアドレスとフローティングIPアドレスのネットワークセグメント内の任意のIPアドレスに設定します。

DNS
  • DNS IPv4アドレス自動取得：このオプションをクリックすると、IPv4アドレスがDNSサーバーに自動的に割り当てられます。
  • DNS IPv6アドレス自動取得：このオプションをクリックすると、IPv6アドレスがDNSサーバーに自動的に割り当てられます。
  • IPアドレスの手動設定：このオプションをクリックすると、DNS情報を手動で設定できます。DNSアドレス情報には、ドメイン、優先サーバー、および代替サーバーが含まれています。

デフォルト値は[IPアドレスの手動設定]です。

注記:
  • BMC管理ネットワークポートのIPアドレスが自動的に取得されている場合は、DNS情報を自動または手動で取得することができます。自動取得をお勧めします。
  • BMC管理ネットワークポートのIPアドレスを手動で設定する場合、DNS情報も手動で設定する必要があります。

ドメイン：サーバーのドメイン名。

ドメイン名は次の要件を満たす必要があります。
  • 0～67文字が含まれます。

  • 数字、英字、ハイフン（-）、およびピリオド（.）のみが使用できます。

  • ドメイン名の最初にハイフンまたはピリオドを使ったり、ドメイン名の最後にハイフンを使ったりすることはできません。

  • 2つのピリオドの間に最大63文字が使用できます。

優先サーバー：優先DNSサーバーのIPアドレス。

値：IPv4、IPv6アドレスまたは空白
  • 代替サーバー1：最初の代替DNSサーバーのIPアドレス。
  • 代替サーバー2：2番目の代替DNSサーバーのIPアドレス。

値：IPv4、IPv6アドレスまたは空白

VLAN

NC-SIポートのVLANを設定します。

デフォルト値：[無効]

NOTE:
  • サービスネットワークと管理ネットワーク間のアイソレーションを実装するために、VLANを有効にし、VLAN IDを設定することをお勧めします。
  • BMC管理ネットワークポートとして、専用ポートが選択されている場合、VLAN設定は無効になります。専用ポート以外のポートがBMC管理ネットワークポートとして選択されている場合、VLAN設定は有効になります。

VLAN ID

NC-SIポートが所属するVLAN。

値の範囲：1~4094の整数

NOTE:

VLAN IDを保存した後、数秒後に設定が有効になります。

LLDP

NOTE:

専用ネットワークポートがBMCの管理ネットワークポートとして使用されている場合にのみ、LLDP機能をサポートします。

LLDP

LLDPを有効にすると、BMCは標準パケットを介して直接接続されたデバイスに自身のMACアドレスを送信できます。これにより、ネットワーク管理システムはリンク通信のステータスを照会および判断できます。

デフォルト値:無効

稼働モード

動作モードは送信でなければなりません。

BMCはLLDPパケットの送信のみをサポートし、LLDPパケットを受信できません。

送信遅延(s)

現在の動作モードでは、BMCのローカル配置が変更されると（主には管理ネットワークポートのスイッチングまたはそのネットワークケーブルの挿抜）、通知用のLLDPパケットは隣接デバイスに送信します。

送信遅延（単位：秒）は、ローカル配置の頻繁な変更に起因するLLDPパケットの大量送信を防ぐために使用されます。遅延期間中にBMCローカル配置の変更が検出されると、システムはそのカウンターをリセットします。次のLLDPパケットは、遅延期間が期限切れになったときにのみ送信されます。

値の範囲:1～8192

デフォルト値:2

送信間隔(s)

ローカル配置情報が変更されていない場合、LLDPパケットは一定の間隔（単位：秒）で隣接デバイスに送信されます。

値の範囲:5～32768

デフォルト値:30

ホールドタイム乗数

指定した期間内（送信間隔×ホールドタイム乗数）にBMCからLLDPパケットを受信していない場合、隣接ノードは保留されたパケットを自動的にクリアします。

値の範囲:2～10

デフォルト値:4
