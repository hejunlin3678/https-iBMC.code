ファームウェア更新

機能説明

[ファームウェア更新]画面で、次の操作を実行することができます。

  • サーバーのファームウェアバージョン情報を表示します。
  • BMCを再起動します。
  • サービスを利用可能なBMCイメージに切り替えます。
  • アウトオブバンドチャネルを介してサーバーのファームウェアをアップグレードします。
  • インバンドチャネルを介してサーバーのファームウェアをアップグレードします。
BMC には次のパーティションイメージがあります。
  • プライマリパーティションイメージ：BMCパーティションイメージが動作しています。
  • セカンダリパーティションイメージ：プライマリパーティションイメージのバックアップです。プライマリパーティションイメージに障害が発生した場合、セカンダリパーティションイメージがプライマリ状態に変更し、元のプライマリパーティションイメージがセカンダリ状態に変更して、イメージバージョンがプライマリパーティションイメージと同期されます。BMCを更新する場合、プライマリとセカンダリのパーティションイメージを同時に更新する必要があります。
  • 利用可能なパーティションイメージ：保持されているBMCのパーティションイメージです。[利用可能なイメージへの切り替え]ボタンをクリックし、利用可能なパーティションイメージをプライマリーパーティションイメージとして作成することができます。一方、元のプライマリパーティションイメージはセカンダリ状態に変更し、データが現在のプライマリパーティションイメージと同期されます。元のセカンダリイメージは自動的に利用可能な状態に変更します。
  • アップグレード処理中にサーバーの電源を切断したり、BMCを再起動したりしないでください。
  • 新しいバージョンを有効にするために、BMCファームウェアの更新後にBMCを再起動してください。サーバー上で実行しているサービスに影響を与えないため、サーバーを再起動する必要はありません。
  • PSUファームウェアを更新した後、サーバーを再起動する必要はありません。
  • BMCまたはSDカードコントローラのファームウェアを更新すると、BMCが自動的に再起動し、更新が有効になります。
  • BIOSまたはCPLDを更新する前に、サーバーで実行しているサービスを停止してください。これにより、サーバーの再起動時のサービス中断が防止されます。
  • システムの稼働中にBIOSまたはCPLDをアップグレードする場合、BIOSのアップグレードを有効にするには、システムの電源サイクルまたは再起動を実行する必要があります。CPLDのアップグレードを有効にするには、システムの電源オフを実行する必要があります。
  • システムが稼働していない状態でBIOSまたはCPLDをアップグレードする場合、BIOSまたはCPLDのアップグレードを有効にするには、システムの電源オンを実行する必要があります。
  • インバンドチャネルを介してファームウェアをアップグレードする前に、サーバーの自動再起動中にサービスが中断されないように、サーバー上で実行されているサービスを停止します。

ファームウェアバージョンの表示

  1. メニューバーでBMC設定を選択します。
  2. ナビゲーションツリーでファームウェア更新を選択します。

    右側に[バージョン情報]画面が表示され、、サーバーのファームウェア名とバージョン情報が表示されます。

ファームウェアの更新（アウトオブバンドチャネル）

以下の操作はBMCを例として、他のファームウェアのアップグレードにも適用されます。

  1. [ファームウェアアップデート] > [アウトオブバンドチャネル]を選択します。
  2. ファイルの選択をクリックして更新ファイルを選択します。

    • アップグレードファイルの形式は.hpmです。そのサイズは90MBを超えることができません。
    • アップグレードファイル名には、数字、英字、スペース、括弧（）、ドット（.）、アンダースコア（_）、ハイフン（-）が使用できます。

  3. 更新をクリックします。

    次のメッセージが表示されます。

    この操作を実行しますか？

  4. はいをクリックします。

    更新が開始されます。

    更新が成功すると、BMCが自動的に再起動し、ログイン画面が表示されます。

    BMCが再起動するまで数分待ちます。

ファームウェアの更新（インバンドチャネル）

以下の操作は、Smart Provisioningでアップグレード可能なファームウェア（NIC、RAIDコントローラーカード、HDDのファームウェアなど）に適用されます。アップグレードする前に、ファームウェアの更新パッケージおよび署名ファイルをダウンロードする必要があります。

  1. [ファームウェアアップデート] > [インバンドチャネル]を選択します。
  2. [更新ファイル]と[署名ファイル]の横にある[ファイルの選択]をクリックし、更新ファイルおよび署名ファイルを選択してアップロードします。

    • 更新ファイルは.zip形式であり、499 MBを超えることができません。
    • 署名ファイルは.asc(pgp)と.p7s(cms)形式であり、90 MBを超えることができません。

  3. [アップロード]をクリックします。

    次のメッセージが表示されます。

    スマートプロビジョニング(SP)管理画面に移動し、次回OSがSPから起動するように設定しますか？
    • 他のファームウェアの更新パッケージをアップロードする必要がある場合は、[キャンセル]をクリックし、2~3を繰り返して、ファームウェアの更新パッケージおよび署名ファイルをアップロードします。
    • アップロードされているファームウェアの更新パッケージを削除する場合は、[キャンセル]をクリックし、[クリア]をクリックします。

  4. [OK]をクリックします。

    [Smart Provisioning管理]画面が表示されます。

  5. [SPからOSを起動]を有効にして、表示されたダイアログボックスで[確認]をクリックします。
  6. サーバーを再起動して、Smart Provisioningでファームウェアをアップグレードします。

BMCイメージの切り替え

必要に応じてBMCファームウェアのイメージを切り替えてください。この操作は更新時に必須ではありません。

  1. バージョン情報画面から利用可能なイメージへの切り替えをクリックします。

    次のメッセージが表示されます。

    BMC利用可能なパーティションイメージに切り替えますか？

  2. はいをクリックします。

    切り替えが成功すると、BMCが自動的に再起動し、ログイン画面が表示されます。

    BMCが再起動するまで数分待ちます。

BMCの再起動

必要に応じてBMCを再起動してください。この操作は更新時に必須ではありません。

  1. バージョン情報で、BMCの再起動をクリックします。

    次のメッセージが表示されます。

    BMCを再起動しますか？

  2. はいをクリックします。

    BMCが再起動し、ログイン画面が表示されます。

    BMCが再起動するまで数分待ちます。
