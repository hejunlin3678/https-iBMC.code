SNMP

機能説明

簡易ネットワーク管理プロトコル（Simple Network Management Protocol、SNMP）は、一連のネットワーク管理標準で構成されます。その中には、アプリケーション層プロトコル、データベースモデル、およびリソースオブジェクトのグループが含まれます。このプロトコルは、ネットワーク管理システムがネットワークに接続されているデバイスを検出するのに役立ちます。

SNMP画面では、SNMPサービスの関連パラメータを照会・構成できます。

BMCは以下のSNMPバージョンに対応します。

  • [SNMPv1:]SNMPの最初の公式バージョンであり、RFC1157で定義されます。SNMPv1を使用した場合、セキュリティリスクが発生する可能性があります。セキュリティ上の理由から、SNMPv3を使用することをお勧めします。
  • [SNMPv2]:RFC1901で定義された実験的なプロトコルであり、コミュニティに基づいた管理アーキテクチャを採用します。SNMPv2を使用した場合、セキュリティリスクが発生する可能性があります。セキュリティ上の理由から、SNMPv3を使用することをお勧めします。
  • [SNMPv3]:SNMPの3回目の公式バージョンです。以前のバージョンに基づいてセキュリティとリモート構成を強化しました。

読み込み専用コミュニティ

読み込み用のコミュニティ名。

値は以下のとおりです。
  • パスワードチェック機能が無効になっている場合、コミュニティ名は次の要件を満たす必要があります。
    • ロングパスワード機能が有効になっている場合、コミュニティ名は16～32桁の文字列である必要があります。スペースは使用できません。
    • ロングパスワード機能が無効になっている場合、コミュニティ名は1～32桁の文字列である必要があります。スペースは使用できません。
  • パスワードチェック機能が有効になっている場合、コミュニティ名は次の要件を満たす必要があります。
    • 長さ:
      • ロングパスワード機能が有効になっている場合、コミュニティ名は16～32桁の文字列である必要があります。
      • ロングパスワード機能が無効になっている場合、コミュニティ名は8～32桁の文字列である必要があります。スペースは使用できません。
    • `~!@#$%^&*()-_=+\|[{}];:'",<.>/?のうち、少なくとも1つ含む必要があります。

    • 大文字（A～Z）、小文字（a～z）、数字（0～9）のうち、2種類を含む必要があります。
    • スペースを使用できません。
  • 弱いパスワードチェック機能が有効になっている場合、パスワードは弱いパスワード辞書に含まれているパスワードと同じにはできません（ipmcset -t user -d weakpwddic -v export <filepath | file_URL>コマンドを使用することで、弱いパスワード辞書から弱いパスワードをエクスポートできます）。
