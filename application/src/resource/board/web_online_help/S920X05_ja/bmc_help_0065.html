BMA管理

機能説明

ベースボード管理エージェント(BMA)は、帯域内管理エージェントソフトウェアです。

BMA管理の画面で、リモートコンソールを通じてサーバーのオペレーティングシステムにBMAをインストールすることができます。

BMAをインストールした後、BMAの基本情報（バージョン番号、稼働状態、ドライバーバージョンなど）が画面に表示されます。

利用可能なBMAバージョン

現在のオペレーティングシステムにインストール可能なBMAのバージョン。

例えば、Linux 1.10は、Linux用のBMAのバージョンが1.10であることを示します。

インストーラーの状態

サーバーのオペレーティングシステムとインストールされるBMA間の接続状態。

  • [準備ができていません]: サーバーのオペレーティングシステムに接続されていないか、接続に失敗したことを示します。
  • [準備ができました]: サーバーのオペレーティングシステムに正しく接続されていることを示します。

BMAの状態

BMAバージョン

サーバーのオペレーティングシステムにインストールされているBMAのバージョン。

--は、BMAがインストールされていないか、稼働していないことを示します。

BMA稼働状態

BMAソフトウェアの稼働状態。

  • --は、BMAがインストールされていないか、稼働していないことを示します。
  • Runningは、BMAがインストール済みで、正常に稼働していることを示します。
  • Unhealthyは、BMAがインストール済みで、正常に実行できないことを示します。

BMAドライバーバージョン

BMAのドライバーのバージョン。

--は、BMAがインストールされていないか、稼働していないことを示します。

BMAのインストール

  • Linuxシステムのクライアントでのみ、BMAがインストールできます。
  • BMAドライブをマウントするには、リモートメディアの使用権限が必要です。
  • リモートコンソールを起動するには、リモート制御機能の使用権限が必要です。
  • BMAをインストールするには、リモートメディアとリモート制御機能の使用権限が必要です。
  1. BMAのインストールをクリックします。

    BMAは、サーバーのオペレーティングシステムへの接続を確立します。接続が正常に確立されると、[説明]ページが表示されます。

  2. [リモートコンソールを開く]をクリックします。

    リモートコンソールが起動します。

  3. リモートコンソールで、管理者としてサーバーのオペレーティングシステムにログインし、オペレーティングシステムのユーザー名とパスワードを入力します。
  4. サーバーのオペレーティングシステムのデバイスリストで、BMAドライバーディスクを特定します。BMAが表示されていない場合は、BMAドライバーディスクをマウントして、BMA に移動します。BMAドライバーディスクのマウント方法については、5に移動します。BMAドライバーディスクのマウント方法については、BMAドライバーディスクのマウントを参照してください。
  5. LinuxオペレーティングシステムでREADME.TXTファイルを開き、Supported Operating Systems in this ReleaseにリストされているLinuxオペレーティングシステムのバージョン情報を表示します。

    1. オペレーティングシステム画面の空白エリアで右クリックします。
    2. 表示されるメニューからOpen Terminalを選択すると、サーバーのオペレーティングシステムのCLIが表示されます。
    3. cd Linuxコマンドとlsコマンドを順次に実行して、LinuxのREADME.TXTファイルを確認します。
      [root@localhost ~]# cd Linux
[root@localhost Linux]# ls
app config drivers install.sh README.TXT script
    4. cat README.TXTコマンドを実行します。
      [root@localhost ~]# cat README.TXT
BMA on-board installation package
Version 2.1.3.020

********************************************************************************
Installation
********************************************************************************

* On the Linux operating systems, execute "sh install.sh -s" from the "Linux" directory to install BMA silently and execute "sh install.sh -u" to upgrade BMA.

For more information on installation instructions, includingsilent installation options, see the "BMA 2.0 User Guide".

********************************************************************************

********************************************************************************

* EulerOS 2.0 SP8
* CentOS 7.6 on aarch64

      • 現在インストレールされているLinuxのバージョンがSupported Operating Systems in this Releaseにリストされているものと一致する場合、6に進みます。
      • 現在インストレールされているLinuxのバージョンがSupported Operating Systems in this Releaseにリストされているものと一致しない場合、README.txtに表示されている任意のLinuxバージョンにアップデートしてから、6に進みます。

  6. BMAをインストールします。

    BMAのインストール方法の詳細については、最新バージョンの『BMA User Guide』を参照してください。BMAのインストールファイルのパスは、使用するオペレーティングシステムによって異なります。

BMAドライバーディスクのマウント

  1. オペレーティングシステムの空白の部分を右クリックします。
  2. 表示されたメニューからOpen Terminalを選択します。サーバーのオペレーティングシステムのCLIが表示されます。
  3. lsscsiコマンドを実行して、BMAドライバーディスクのプロパティを表示します。

    [root@localhost ~]# lsscsi
[0:0:0:0]    disk    ATA      ST4000NM0033-9ZM SN06  -
[0:0:1:0]    disk    ATA      ST4000NC001-1FS1 CN02  -
[0:0:2:0]    disk    ATA      WDC WD6000F9PZ-3 0R01  /dev/sdc
[0:0:3:0]    disk    ATA      WDC WD6000F9PZ-3 0R01  /dev/sdd
[0:0:4:0]    disk    HGST     HUS726060AL4210  A523  -
[0:0:5:0]    disk    HGST     HUS726060AL4210  A7MH  -
[0:0:6:0]    disk    ATA      SSDSC2BB016T7H   0121  /dev/sde
[0:0:7:0]    disk    HGST     HUS726060AL4210  A523  /dev/sdf
[0:1:0:0]    disk    LSI      Logical Volume   3000  /dev/sdb
[0:1:1:0]    disk    LSI      Logical Volume   3000  /dev/sda
[19:0:0:0]   disk    SERVER   BMA USB Device   225  /dev/sdv

    コマンド出力では、BMA USB DeviceはBMAドライバーディスクのノード名、/dev/sdvはBMAドライバーディスクに割り当てられたドライブレターを示します。

  4. mount /dev/sdv /home/fileコマンドを実行して、BMA ドライバーディスクを/home/fileディレクトリにマウントします。

    この例では、/home/fileはマウントされたファイルを格納するためのディレクトリです。実際の要件に応じてディレクトリを作成してください。このディレクトリでは、フォルダー名に、数字、英文字、アンダースコア（_）、ハイフン（-）、およびドット（.）を入力できます。

    [root@localhost ~]# mount /dev/sdv /home/file
#

  5. ls /home/fileコマンドを実行して、BMAドライバーが正常にマウントされているかを確認します。

    [root@localhost ~]# ls /home/file
Linux

    コマンド出力に、Linuxが表示されている場合は、BMAドライバーディスクが正常にマウントされていることを示します。
