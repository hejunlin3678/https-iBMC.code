Локальные пользователи

Описание функций

На странице Локальные пользователи доступны различные операции по управлению пользователями BMC.

BMC поддерживает до 16 пользователей. В число доступных операций входит просмотр информации о пользователях, добавление пользователей, редактирование данных и удаление пользователей.

Табл. 1 Локальные пользователи

Параметр

Описание

Добавить

Добавление локального пользователя.

ID пользователя

Уникальный идентификатор пользователя в системе BMC.

Имя пользователя

Имя пользователя для входа в BMC.

Роль
Полномочия, выделенные пользователю. Роль определяет операции, которые может выполнять пользователь.
  • Администратор: пользователь, который может выполнять любые операции.

    Права пользователя Администратор изменить нельзя.

  • Оператор: пользователь, который может выполнять базовые операции управления, операции по управлению KVM, VMM и энергопотреблением, а также осуществлять запрос информации и настраивать собственные пароли.

    Права пользователя Оператор изменить нельзя.

  • Стандартный пользователь: пользователь, который может осуществлять запрос информации и настраивать собственные пароли.

    Права пользователя Стандартный пользователь изменить нельзя.

  • Ограниченная роль 1–4: пользователь, который может выполнять только определенные операции.
  • Без доступа: Пользователи с ролью Без доступа не могут выполнять операции.

Интерфейсы входа

Интерфейсы, с помощью которых пользователь осуществляет вход в BMC.

Значения:

  • SNMP: для входа в BMC используется инструмент SNMP (например, браузер MIB).
  • SSH: для входа в CLI BMC используется инструмент SSH (например, PuTTY).
  • IPMI: для входа в CLI BMC используется инструмент IPMI (например, IPMItool).
  • Local: для входа в CLI BMC используется последовательный порт сервера.
  • SFTP: для входа в файловую систему BMC используется инструмент SFTP (например, Xftp).
  • Web: для входа в WebUI BMC используется веб-браузер.
  • Redfish: для входа в BMC используется инструмент Redfish.

Операция
  • Изменить – редактирование данных локального пользователя.
  • Отключить – отключение учетной записи пользователя.
  • Включить – включение учетной записи пользователя.
  • Удалить – удаление пользователя.
    ПРИМЕЧАНИЕ:
    • Операция удаления применима ко всем локальным пользователям, включая администраторов, операторов, стандартных пользователей и пользователей с ограниченной ролью.
    • Если в системе только один администратор, его нельзя отключить или удалить.
    • Если на странице Пользователи и безопасность > Безопасность включить функцию Управление пользователями ОС, то добавлять пользователей BMC можно будет путем отправки стандартных команд IPMI из ОС.

Добавление пользователей

BMC поддерживает добавление до 15 пользователей.

  1. Нажмите Добавление пользователя.

    Откроется страница для добавления пользователя.

  2. Настройте параметры пользователя. Более подробная информация о параметрах приводится в Табл. 3.

    • Пользователь с ID 1 является резервным пользователем, который определен в спецификациях IPMI. Данный пользователь не обладает правами на вход в BMC.
    • Пользователь с ID 2 является пользователем по умолчанию.
      Табл. 3 Добавление пользователя

      Параметр

      Описание

      ID нового пользователя

      Идентификатор добавляемого пользователя.

      Диапазон значений: от 3 до 17

      Имя пользователя

      Имя добавляемого пользователя.

      Значение: строка длиной от 1 до 16 символов.

      Имя пользователя должно соответствовать следующим требованиям:

      • Имя пользователя может содержать буквы, цифры и специальные символы (за исключением :<>&,'"/\%).
      • Имя пользователя не должен содержать пробелы или начинаться с #, +или –.
      • Имя пользователя не может содержать только одну точку (.) или две последовательные точки (..).

      Пароль

      Пароль для входа в BMC.

      В целях обеспечения безопасности рекомендуется включить проверку сложности пароля и периодически менять пароль.

      ПРИМЕЧАНИЕ:
      • Включать или отключать функцию проверки сложности пароля могут только администраторы.
      • Отключение функции проверки сложности пароля может представлять угрозу безопасности. Если возможно, включите эту функцию.

      Значения:

      • Если проверка сложности пароля отключена, то пароль является обязательным для настройки параметром и должен представлять собой строку, не превышающую 20 символов, состоящую из цифр, букв и специальных символов. Если пароль содержит менее восьми символов, пользователь не может использовать интерфейс SNMPv3.
      • Если проверка сложности пароля включена, то должны выполняться следующие условия:
        • Пароль должен содержать от 8 до 20 символов.
        • Пароль должен содержать, как минимум, один пробел или один из следующих специальных символов:

          `~!@#$%^&*()-_=+\|[{}];:'",<.>/?

        • Пароль должен включать символы, как минимум, двух видов:
          • Прописные буквы: от A до Z
          • Строчные буквы: от a до z
          • Цифры: от 0 до 9
        • Пароль не должен совпадать с именем пользователя в прямом и обратном порядке расположения символов.
        • Если включена проверка слабого пароля, то пароль не может совпадать с паролями, занесенными в справочник слабых паролей (для экспорта слабых паролей из справочника используется команда ipmcset -t user -d weakpwddic -v export).
          ПРИМЕЧАНИЕ:

          В целях безопасности не используйте пароли, состоящие только из повторяющихся строк, такие как aa, abababab или abcdabcd.

    Пароль шифрования SNMPv3

    В целях обеспечения безопасности связи пароль шифрования задается для пользователя, который использует SNMP. Пароль шифрования должен соответствовать правилам, применяемым к паролям локальных пользователей.

    Значение по умолчанию: совпадает с паролем для входа пользователя.

    ПРИМЕЧАНИЕ:
    • Если этот параметр не задан, пароль шифрования SNMPv3 будет синхронизирован с паролем пользователя для входа. В целях обеспечения безопасности рекомендуется установить пароль шифрования SNMPv3. Если задан пароль шифрования SNMPv3, этот пароль не будет синхронизирован с паролем пользователя для входа.
    • В целях безопасности не используйте пароли, состоящие только из повторяющихся строк, такие как aa, abababab или abcdabcd.

Удаление пользователя

  1. В списке локальных пользователей выберите пользователя, которого хотите удалить и нажмите Удалить.

    Откроется диалоговое окно с просьбой ввести текущий пароль пользователя.

  2. Введите текущий пароль пользователя и нажмите Да.

    Пользователь будет удален из списка пользователей.

Импорт открытого ключа SSH

  • После того, как на клиенте был сгенерирован закрытый ключ, необходимо импортировать соответствующий открытый ключ в BMC для обеспечения безопасного доступа пользователей SSH к BMC.
  • У каждого пользователя имеется только один открытый ключ. Старый открытый ключ будет заменен импортированным новым.
  • Поддерживаются открытые ключи RSA или DSA в формате RFC 4716 или OpenSSH. Длина ключа RSA составляет 2048 или 4096 битов, а длина ключа DSA — 2048 битов.
  • Формат файла открытого ключа: .pub. Размер не может превышать 2 КБ.
  1. Найдите пользователя, для которого необходимо импортировать открытый ключ SSH, и щелкните слева от имени пользователя.
  2. Нажмите Выгрузить рядом с пунктом Открытый ключ SSH.
  3. Выберите режим импорта.

    Выберите Файл открытого ключа или Текст открытого ключа.

  4. Нажмите Выбрать файл, чтобы выбрать открытый ключ.
  5. Введите пароль текущего пользователя.
  6. Нажмите Сохранить.
