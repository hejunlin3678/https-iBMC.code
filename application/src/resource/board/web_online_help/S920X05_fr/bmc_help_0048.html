Échec d'ouvrir la console distante intégrée Java

Symptôme

Symptôme

Causes possibles

La console distante intégrée Java ne peut pas être ouverte.
  • Le JRE n'est pas installé correctement.
  • La version JRE n'est pas compatible avec le BMC.

Solution

  1. Vérifiez si le JRE est correctement installé.

    Le BMC prend en charge le JRE AdoptOpenJDK 8 et le JRE AdoptOpenJDK 11.

    • Si la version JRE est correcte, contactez le support technique.
    • Si la version JRE est incorrecte, allez à 2.

  2. Téléchargez le package binaire JRE correspondant au système d'exploitation du client sur le site officiel AdoptOpenJDK.
  3. Installez le JRE AdoptOpenJDK.

    • Configurez manuellement les variables d'environnement JAVA_HOME et PATH après avoir décompressé le paquet.
    • Téléchargez IcedTea Web à partir du site officiel AdoptOpenJDK, décompressez-le et configurez le dossier bin sur la variable d'environnement PATH.

  4. Connectez-vous à l'interface Web de le BMC , dans la zone Console virtuel de la page Accueil, choisissez Démarrer > Console à distance avec Java intégré (privée) ou Console à distance avec Java intégré (partagée).

    Au cours de ce processus, le fichier .jnlp est automatiquement téléchargé.

  5. Ouvrez le fichier .jnlp.

    Le fichier .jnlp est sensible au facteur temps, S'il n'est plus valide, téléchargez-le à nouveau en vous référant au point 4.

    • Si le client utilise l'interface de ligne de commande (CLI) Linux ou Windows, passez au répertoire où se trouve .jnlp et exécutez javaws kvm.jnlp.
    • Si le client utilise une interface utilisateur graphique, localisez le fichier .jnlp, cliquez avec le bouton droit de la souris sur le fichier et choisissez javaws. Si javaws n'est pas disponible dans le menu raccourci, allez dans le dossier bin du répertoire d'installation d'IcedTea Web.
