« La configuration du certificat de confiance a expiré » affiché lors de l'ouverture de la console distante intégrée HTML5

Symptôme

Symptôme

Causes possibles

Lorsqu'un utilisateur tente d'ouvrir la console à distance intégrée HTML5, le système affiche « Échec de l'ouverture de la définition de KVM car la configuration du certificat de confiance a expiré »

Avant qu'une connexion soit établie entre le client KVM (console distante intégrée HTML5) et le serveur, le certificat SSL doit être vérifié. Si la vérification échoue, la connexion ne peut pas être établie.

Solution

  1. Sur l'interface utilisateur Web BMC, choisissez Services > Services Web. Dans la zone Informations sur le certificat du serveur, vérifiez si le certificat du serveur a expiré.

    • Si oui, allez à 2.
    • Si non, allez à 3.

  2. Faites générer un nouveau certificat et remplacez le certificat original.
  3. Redémarrez l'BMC
  4. Vérifiez si la console à distance intégrée HTML5 peut être ouverte.

    • Si oui, aucune autre mesure n'est requise.
    • Si non, allez à 5.

  5. Contactez le support technique.
