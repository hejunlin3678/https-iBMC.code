# 1. 批量导入标签 说明

- 批量导入标签： 主要用于批量将设备信息导入系统中。

# 2. 文件清单

- README_ZH.md
 
- README_EN.md

- RFIDImportTempFile.csv

# 3. 导入步骤

- 步骤1. 使用EXCEL打开RFIDImportTempFile.csv。

- 步骤2. 根据README指导，将所需内容按规则填入RFIDImportTempFile.csv 相应位置中。

- 步骤3. 填写完成后保存，会弹出“如果另存为CSV（逗号分隔），您的工作薄中的部分功能可能丢失。是否继续使用此格式” ，选择是（Y）。

- 步骤4. 单击界面的 “批量导入标签” 按钮，选择步骤3中填完的RFIDImportTempFile.csv文件进行上传。

         导入结果会在右上方提示，结果分为：

         - i   : 若显示 “导入成功”，则表示操作成功。
 
         - ii  : 若显示 “导入位置Unknown的信息失败。” ，则表示Upos字段填写有误或未填。
 
         - iii : 若显示 “导入位置XX的信息失败。”，则表示某U位设备信息填写有误。
  
# 4. RFIDImportTempFile.csv字段详解

- 1）. 标记为*的字段为必填字段。

- 2）. 各字段定义及描述如下：

  - *Upos ：设备在机柜中所在U位号，正整数类型，不超过机柜U位高度，且该U位对应位置需要有已被识别的标签。
  
  - *Model ：描述设备的型号，字符串类型，不超过48个字符，如 "2288 V5"。
 
  - *DeviceType ：描述设备的类型，字符串类型，不超过48个字符，如 "Server"。

  - *Manufacturer ：描述设备的制造商，字符串类型，不超过48个字符，如 "Huawei"。
 
  - *SerialNumber ：描述设备的序列号，字符串类型，不超过48个字符，如 "8101250GTU6TJ9022810"。
 
  - PartNumber ：描述设备的部件号，字符串类型，不超过48个字符，如 "3A"。
 
  - *UHeight ：描述设备占用U位高度，正整数类型，单位为U，最小值为1，最大不超过机柜U位高度，如 "2"。
 
  - LifeCycleYear ：描述设备的规划生命周期，正整数类型，单位为年，如 "10"。
 
  - CheckInTime ：描述设备的首次上架时间，格式为YYYY/MM/DD，如 "2019/10/01"。
 
  - WeightKg ：描述设备的额定重量，正整数类型，最小值为1，单位为千克（Kg），如 "1000"。
 
  - RatedPowerWatts ：描述设备的额定功率，正整数类型，最小值为1，单位为瓦特（W），如 "4000"。
 
  - AssetOwner ：描述设备的资产所有人，字符串类型，不超过48个字符，如 "Huawei"。
  
- 3）. 字符串类型字段可包含数字、字母以及以下特殊字符：
  - _ `~!@#$%^&*()-=+{}|;,'.:"?/<>\-
  
# 5. 注意事项

- 用户在修改RFIDImportTempFile.csv文件时，不可删除、修改第一行即模板文件中的参数字段。
